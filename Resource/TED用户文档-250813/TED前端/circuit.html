电路环境与配置 (Circuit)

1. 概述

TED 不仅提供了用于描述电路元件（Instance）、模块（Module）、几何图形（Box, NBox 等）和节点（Node）的类，还维护了一个全局的电路上下文 (Circuit Context)。这个上下文负责管理整个设计会话的状态，包括：

  • 当前活动的模块: 在哪个模块的定义或测试平台中执行代码。
  • PDK (Process Design Kit) 信息: 加载和访问当前工艺库的规则、层级、模型等。
  • 全局配置与环境变量: 控制 TED 行为的设置，如使用的仿真器、是否检查 DRC、是否启用缓存等。
  • 模块缓存: 存储已构建的模块定义，避免重复计算。
  • 仿真与优化状态: 管理仿真任务、变量、Datasweep、优化结果等。
  • 报告生成: 汇总仿真、验证和布局结果。

本篇文档主要介绍如何访问和修改全局电路上下文、管理环境变量以及理解报告生成机制。

2. 全局电路上下文 (Circuit)

TED 使用一个全局的单例 Circuit 对象来管理整个设计环境。用户通常通过 get_circuit() 函数来访问这个对象。

from tedcore.kernel import get_circuit, get_pdk_name, get_simulator

# 获取当前的全局 Circuit 对象
circuit_context = get_circuit()

# 通过 Circuit 对象访问全局信息
current_pdk = circuit_context.get_pdk() # 获取当前加载的 PDK 对象
print(f"当前 PDK: {current_pdk.name}") # 或使用 get_pdk_name()
print(f"当前仿真器设置: {get_simulator()}") # 获取配置的仿真器

# Circuit 对象在首次调用 get_circuit() 时自动初始化
# 可以通过 circuit.reset() 清空状态 (通常在测试或新会话开始时使用)
# circuit_context.reset()

Circuit 对象内部维护了许多状态，包括：

  • 模块栈 (module_stack)：跟踪当前正在定义的模块。
  • 模块缓存 (modules)：存储已构建的 Module 对象，键为模块的哈希名。
  • PDK 对象 (_pdk)：包含当前工艺库信息。
  • 配置对象 (_config)：存储全局设置。
  • 变量字典 (variables, tb_variables)：存储优化或扫描用的变量。
  • 仿真相关数据 (datasweep, opctrl, lsf, sim_acc, watch_signals)。
  • 报告字典 (report)。

3. 环境管理 (setenv, getenv)

TED 提供了类似上下文管理器的方式来临时修改全局设置，这对于在特定代码块内改变行为非常有用。

3.1 setenv 上下文管理器

使用 with setenv(**kwargs): 可以临时覆盖一个或多个全局配置项的值。退出 with 代码块后，这些配置项会自动恢复到之前的值。

from pyted import setenv, getenv, module, testbench
from tedcore.kernel import get_simulator

print(f"默认仿真器: {get_simulator()}") # 假设默认为 'spectre'
print(f"默认强制格点对齐: {getenv('force_on_grid')}") # 假设默认为 False

with setenv(simulator="hspice", force_on_grid=True, skip_layout=True):
    # 在这个代码块内，设置被临时修改
    print(f"  -> 块内仿真器: {get_simulator()}")       # 输出: hspice
    print(f"  -> 块内强制格点: {getenv('force_on_grid')}") # 输出: True
    print(f"  -> 块内跳过布局: {getenv('skip_layout')}")  # 输出: True

    @module
    def TempModule():
        # 此模块构建时会使用 HSpice 设置，并跳过布局相关操作
        print(f"    构建 TempModule 时仿真器: {get_simulator()}")
        pass
    TempModule()

# 退出 with 代码块后，设置恢复
print(f"恢复后仿真器: {get_simulator()}")       # 输出: spectre
print(f"恢复后强制格点: {getenv('force_on_grid')}") # 输出: False
print(f"恢复后跳过布局: {getenv('skip_layout')}")  # 输出: False

# 常用 setenv 参数示例:
# with setenv(skip_layout=True): ...     # 跳过所有布局操作
# with setenv(drc_checking=False): ...  # 临时禁用 DRC 检查
# with setenv(force_rebuild=True): ...  # 强制重新构建模块，不使用缓存
# with setenv(simulator="hspice"): ...  # 切换仿真器

3.2 getenv 函数

使用 getenv(key) 获取当前生效的全局配置项的值。

from pyted import getenv

simulator = getenv("simulator")
is_drc_enabled = getenv("drc_checking")
current_corners = getenv("corners") # 获取当前设置的工艺角

print(f"当前仿真器: {simulator}")
print(f"DRC 检查已启用: {is_drc_enabled}")
print(f"工艺角: {current_corners}")

4. 全局配置 (Config)

全局配置由 Circuit 对象内部的 Config 对象管理。可以通过 get_circuit().update_config(**kwargs) 来永久修改当前会话的配置（与 setenv 的临时修改相对）。

from tedcore.kernel import get_circuit

circuit = get_circuit()

# 获取当前配置值
print(f"当前 drc_g1 配置: {circuit.get_config('drc_g1')}") # 假设默认为 True

# 永久修改配置项 (在下次 reset 前一直有效)
circuit.update_config(drc_g1=False, via_min_number=1)

print(f"修改后 drc_g1 配置: {circuit.get_config('drc_g1')}") # 输出: False
print(f"修改后 via_min_number 配置: {circuit.get_config('via_min_number')}") # 输出: 1

# 注意：PDK 加载时会设置许多初始配置。
# 环境变量（如 TED_SIMULATOR=hspice）通常会覆盖初始配置。

5. PDK 与 Layer 访问

全局上下文提供了访问当前 PDK 和 Layer 信息的便捷方式。

from tedcore.kernel import get_pdk, get_layer, get_layer_from_number, lmin
from tedcore.kernel import M1, VIA1 # 假设 M1, VIA1 是预定义 Layer

# 获取 PDK 对象
pdk = get_pdk()
print(f"PDK 名称: {pdk.name}")
print(f"PDK 最小沟道长度 (lmin): {lmin()} nm") # lmin() 是快捷方式

# 获取 Layer 对象
metal1_pin = get_layer("M1", purpose="pin")
via1_drawing = get_layer("VIA1") # 默认 purpose='drawing'
layer_68_20 = get_layer_from_number(68, 20) # 通过 GDS Layer/Datatype

print(f"M1 Pin: {metal1_pin}")
print(f"VIA1 Drawing: {via1_drawing}")
print(f"GDS 68/20: {layer_68_20}")

6. 自动命名 (auto_rename)

在 @module 或 @testbench 函数内部调用 auto_rename()，可以将 Python 代码中定义的变量名应用到对应的 Node 或 Instance 上，极大地提高了生成网表的可读性。

from pyted import module, Node, auto_rename
from ted_device import Inv

@module
def AutoNamingDemo():
    # 初始节点和实例名称是自动生成的 (如 n0_, Inv_0 等)
    input_signal = Node()
    intermediate_signal = Node()
    output_signal = Node()
    inverter_stage1 = Inv()
    inverter_stage2 = Inv()

    input_signal >> inverter_stage1 >> intermediate_signal
    intermediate_signal >> inverter_stage2 >> output_signal

    # 调用 auto_rename 后：
    # 节点在网表中会变成 input_signal, intermediate_signal, output_signal
    # 实例在网表中会变成 inverter_stage1, inverter_stage2
    auto_rename()

    # 后续布局或操作将使用这些新名称
    place(inverter_stage1, inverter_stage2)

# --- 控制重名检查 ---
from tedcore.kernel import skip_duplicate_name_check, is_duplicate_name_check_enabled

print(f"默认是否检查重名? {is_duplicate_name_check_enabled()}") # 通常 True

with skip_duplicate_name_check():
    # 在此块内，即使两个实例变量同名，也不会报错（但可能导致网表问题）
    inv_a = Inv()
    inv_a = Inv() # 不会报错，但第二个 Inv 实例无法通过 'inv_a' 引用
    print(f"块内是否检查重名? {is_duplicate_name_check_enabled()}") # False

print(f"恢复后是否检查重名? {is_duplicate_name_check_enabled()}") # True

8. 报告生成 (Report, SubReport)

TED 可以生成 Markdown 格式的电路报告，汇总设计和仿真信息。通常由测试框架或顶层脚本调用。

from tedcore.kernel.circuit_report import Report, SubReport
# 假设 circuit_module 是顶层 Testbench Module 对象
# 假设 dc_tester, tran_tester 是运行了仿真的 TesterBase 对象

# 1. 创建主报告
report = Report(module=circuit_module, name="FullChipAnalysis")

# 2. 添加子报告 (来自各个 Tester)
# (假设 Tester 对象有 sim_table, waveform_lib 等属性)
dc_sub_report = SubReport(tester=dc_tester, title="DC Analysis")
tran_sub_report = SubReport(tester=tran_tester, title="Transient Analysis")

report.append(dc_sub_report)
report.append(tran_sub_report)

# 3. (可选) 添加 PVT 或优化结果表格
# report.get_pvt_report()
# report.get_variable_opt_report()

# 4. (可选) 添加版图截图
# report.get_layout_report() # 需要 Tester 或 Module 中设置了 ly_fig

# 5. 生成并输出报告
report.creat_full_report() # 组合所有部分
report.output() # 保存到 .md 文件
print(f"报告已生成: {report.report_path}")

9. 常用工具函数 (common.py)

common.py 提供了一些全局可用的便捷函数。

from pyted import debug, skip_unless, PDK_NAMES, T65, S180BCD

# 1. 调试打印 (带上下文信息)
debug("当前变量值:", some_variable, color="yellow")

# 2. 条件跳过测试 (用于 Pytest)
@skip_unless(TED_PDK=[T65, S180BCD], simulator="spectre")
def test_only_on_specific_pdk_and_sim():
    # 这个测试只会在 PDK 是 T65 或 S180BCD，且仿真器是 spectre 时运行
    pass

# 3. 单位转换
nm_value = 150
um_value = convert_nm_size_to_other(nm_value, unit="um") # 150nm -> 0.15um
print(f"{nm_value} nm is {um_value} um")

# 4. 对齐到格点
value = 157
grid_aligned_value = ensure_grid(value, func="round") # 四舍五入到最近的格点
print(f"{value} 对齐到格点后: {grid_aligned_value}") # 取决于 PDK grid

# 5. 添加 Label 到当前模块
add_label("My Important Node", layer=M3, x=1000, y=2000)
