模块 (Module)

1. 概述

在 TED 框架中，Module（模块）是电路设计的基本单元定义，它类似于面向对象编程中的类或硬件描述语言（如 Verilog）中的 module。正如 "TED 前端基础文档" 所述，Module 定义了电路的结构、端口、参数以及可选的物理布局信息。通过实例化 Module，可以创建在电路中实际使用的 Instance 对象。

本篇文档详细介绍了 Module 类的功能、其构建过程、状态查询、层次结构管理、物理验证接口、数据导出方法以及相关的辅助类和函数。

通常，用户通过定义一个带有 @module (或 @moduledef) 装饰器的 Python 函数，或继承 EnhancedInstance 的类来创建 Module 定义。调用被 @module 装饰的函数（例如 MyModule(param=value)) 会创建一个该模块的实例 (Instance)。要获取模块本身的定义 (Module)，通常在该实例上调用 .module() 方法，例如 MyModule(param=value).module()。

from pyted import module, Instance, Input, Output, Vdd, Vss # PMOS, NMOS assumed from context
# from ted_device import PMOS, NMOS # More explicit if they are from a library

# 假设 PMOS, NMOS 来自 ted_device 或类似库
@module
def PMOS(m=1): # Simplified PMOS for example
    pass

@module
def NMOS(m=1): # Simplified NMOS for example
    pass

# 使用 @module 装饰器定义一个 Module 的 "蓝图"
@module
def Inverter(x: Input, z: Output, m: int = 1):
    Vdd() >> PMOS(m=m) % x >> z >> NMOS(m=m) % x >> Vss()
    # 在实际模块中，通常会有 place() 调用
    # current_module().place(...) or place(...)

# 'Inverter' 本身是一个模块构建器/工厂
print(f"Type of Inverter: {type(Inverter)}")
# Output: Type of Inverter: <class 'function'> (or a specific TED builder type)

# 1. 创建模块实例
# 调用 Inverter(...) 会执行模块构建逻辑（如果需要），并返回一个 Instance
inv_inst_default: Instance = Inverter()
inv_inst_m2: Instance = Inverter(m=2)

print(f"Type of inv_inst_default: {type(inv_inst_default)}") # <class 'tedcore.kernel.src.instance.instance.Instance'>
print(f"Instance inv_inst_m2 parameters: {inv_inst_m2.get_all_parameters(skip_hashcode=True)}")

# 2. 获取模块定义 (Module object)
#    - Inverter().module() 获取默认参数 (m=1) 的模块定义
#    - Inverter(m=2).module() 获取 m=2 版本的模块定义
inverter_def_default: Module = inv_inst_default.module() # or Inverter().module()
inverter_def_m2: Module = inv_inst_m2.module()     # or Inverter(m=2).module()

print(f"Type of inverter_def_default: {type(inverter_def_default)}") # <class 'tedcore.kernel.src.moduledef.mod.Module'>
print(f"Module definition for m=1, name_hash: {inverter_def_default.name_hash()}")
print(f"Module definition for m=2, name_hash: {inverter_def_m2.name_hash()}")
assert inverter_def_m2.get_parameter('m') == 2

模块的定义是唯一的，由其来源、名称和参数共同决定。具有相同定义的模块在 TED 中通常只构建一次，并被缓存。

2. Module 核心类 (mod.py)

Module 类是 TED 中用于表示电路模块定义的 Python 封装。它提供了丰富的 API 来管理和查询模块的各种信息，并作为创建 Instance 的蓝图。

2.1 模块定义与构建

  • 创建方式:

    • 主要通过 @module 或 @moduledef 装饰器修饰一个 Python 函数。
    • 也可以通过继承 EnhancedInstance 类并实现其 core() 方法来定义模块。

  • 构建触发: 当调用被 @module 装饰的函数时（如 MyModule(param=value)），TED 会启动模块的构建流程。

  • 内部表示: 每个 Python Module 对象内部关联一个底层的 Rust 实现 (rsmodule)，负责存储模块的电路网表、参数、布局（可选）等核心数据。

  • 构建流程与缓存 (_build.py, frontend2.md): TED 的模块构建包含缓存机制以提高效率。对于具有相同模块来源（路径）、相同模块名和相同参数值的模块调用，TED 通常只执行一次完整的构建过程。

    1. 预构建 (Pre-build):

      • 解析模块函数的签名，区分端口（无默认值参数）和参数（有默认值参数）。
      • 根据模块路径、模块名和调用时传入的所有参数值计算唯一的哈希标识符 name_hash。
        • 重要: 模块参数应使用 Python 中可哈希 (hashable) 的类型（如 int, float, str, bool, tuple）以确保缓存的可靠性。
      • 使用 name_hash 在全局模块库中查询缓存。
      • 缓存命中: 若找到，则直接使用缓存的模块定义创建实例，跳过后续构建阶段。
      • 缓存未命中: 若未找到，则：
        • 创建一个新的 Module 对象。
        • 将新模块对象压入当前的模块调用栈 (get_circuit().module_stack)。在构建期间，可通过 current_module() 访问此对象。
        • 根据函数签名创建模块的端口节点。

    2. 构建 (Build):

      • 执行用户代码: 执行用户定义的模块函数体。
      • 自动归属: 函数体内创建的所有节点 (Node()) 和实例（如子模块实例）会自动添加到 current_module() 代表的模块中。

    3. 后构建 (Post-build):

      • 执行 current_module().auto_rename()，根据变量名为节点和实例命名（详见 node.md 和 frontend2.md）。
      • 计算模块尺寸（若有布局信息）。
      • 构建依赖的子模块列表。
      • 将构建好的模块对象以 name_hash 为键注册到全局模块库。
      • 将当前模块对象从模块调用栈中弹出。
    • ModuleBuilder: 内部类，负责协调模块的构建流程。
    • build(): 核心函数，处理参数、检查缓存、调用用户定义的函数/方法来填充模块内容、运行回调、设置端口、序列化（如果启用缓存）等。

  • 唯一性与 name_hash:

    • name_hash() 或 get_name_hash(): 获取模块的唯一哈希名称，该名称综合了模块路径、用户定义名及其实际参数值，是模块缓存的键。
    • get_key(): 获取模块的内部哈希键。
    • 序列化/反序列化: 支持将构建好的模块保存到本地文件或远程服务器 (serialize, deserialize, remote)，以便后续快速加载 (load_module)。

  • 导入 (_import.py, _build.py):

    • ted_import(): 从序列化文件或其他格式（如 GDS+SPICE）导入模块定义。
    • import_netlist(): (方法) 尝试通过导入外部网表文件来定义模块内容。

示例：获取特定参数的模块定义

# test_module.py: test_module_definition_and_instantiation
@module
def MyParamModule(x: Input, z: Output, m: int = 1, w: float = 0.1):
    # Module content
    pass

# 获取 m=2, w=0.2 版本的模块定义
param_module_def = MyParamModule(m=2, w=0.2).module()
assert param_module_def.get_parameter('m') == 2
assert param_module_def.get_parameter('w') == 0.2

# 默认参数的模块定义
default_param_module_def = MyParamModule().module()
assert default_param_module_def.get_parameter('m') == 1
assert default_param_module_def.get_parameter('w') == 0.1

2.2 属性与状态查询

Module 类提供了多种方法来获取其定义信息和状态。

基本信息:

  • name() 或 get_name(): 获取模块的用户定义名称（通常是Python函数名）。
  • name_hash() 或 get_name_hash(): 获取模块的唯一哈希名称（包含参数信息和路径等，用于缓存和唯一识别）。
  • get_key(): 获取模块的内部哈希键，通常与 name_hash 相关。
  • has_parameter(name): 检查模块定义是否包含名为 name 的参数。
    • 示例: assert inv_module_s1_def.has_parameter('strength')
  • get_parameter(name): 获取模块定义中名为 name 的参数的当前值。此值是用于构建或标识此特定 Module 对象版本的值。
    • 示例: strength_val = inv_module_s1_def.get_parameter('strength')
  • get_all_parameters(original_type=True, skip_hashcode=True): 获取模块定义的所有参数及其值的字典。
    • original_type: 为 True 时，参数值保持其原始Python类型。
    • skip_hashcode: 为 True 时，不包含内部用于哈希的 hashcode 参数。
  • get_model(): 获取模块关联的 SPICE 模型名称（如果通过 current_module().set_model() 设置）。
  • has_model(): 判断是否设置了 SPICE 模型。
  • get_category(): 获取模块的分类（用户可通过 current_module().set_category() 自定义）。
  • get_type(): 获取模块的类型（如 "layout", "sim", "lvs", "Other", 或通过 current_module().set_type() 设置的类型如 "Mos"）。
  • get_module_path(): 获取定义该模块的 Python 文件路径。
  • get_cellname(): 获取导出 GDS 时指定的 Cell 名称（通过 current_module().set_cellname() 设置）。
  • get_gds_struct_name(): 获取导出 GDS 时实际使用的结构名称（通常是哈希名称 name_hash）。

端口与节点信息: (关于节点的详细信息，请参阅 node.md)

  • get_input(), get_output(), get_ctrl(): 获取模块定义的输入、输出、控制端口的 CircuitNode 对象。这些返回的 CircuitNode 可能是代表多个端口（或一个多位端口）的节点束 (Bundle)。如果是一个束，单个端口节点通过索引访问（如 ports_bundle[0])。
  • get_supplies(), get_vdd(), get_vss(): 获取模块定义的电源/地端口节点 (CircuitNode)。
  • terminals(): 获取模块所有普通（非电源）端口节点的 CircuitNode 束。
  • all_terminals(): 获取模块所有端口节点（包括电源）的 CircuitNode 束。
  • terminal(name): 根据名称获取特定端口节点。
  • all_nodes(): 获取模块内部定义的所有节点（包括端口和内部节点）的 CircuitNode 束。
  • all_internal_nodes(): 获取模块内部定义的所有非端口节点的 CircuitNode 束。
  • get_node_dict(): 获取模块内所有节点（名称到 CircuitNode 的映射）。
  • node_count(): 获取模块内的节点总数。
  • exist_node(node_or_name): 检查模块内是否存在指定名称（字符串）或 CircuitNode 对象的节点。
    • 示例: assert bus_driver_module.exist_node("internal_signal"), assert bus_driver_module.exist_node(ctrl_node_obj)

示例：访问端口束

# test_module.py: test_port_and_node_access
@module
def MyBusDriver(data_in: Input(4), data_out: Output(4), enable: Ctrl):
    internal_node = Node(name="internal_signal") # Explicitly name for exist_node test
    enable >> internal_node
    # ... (connections)
    place() # Assume some placement

bus_driver_module_def = MyBusDriver().module()

input_bundle = bus_driver_module_def.get_input()
# input_bundle is a CircuitNode representing all input ports.
# If 'data_in' is the only Input defined (even if multi-bit), it's input_bundle[0].
assert input_bundle[0].name() == "data_in"
assert len(input_bundle[0]) == 4 # 'data_in' is 4 bits wide

all_defined_terminals = bus_driver_module_def.terminals().flatten() # Get a flat list of all terminal Nodes
print(f"Terminal names: {[n.name() for n in all_defined_terminals]}")

布局与几何信息:

  • size() 或 get_size(): 获取模块的物理尺寸（宽度, 高度）。
  • get_area(): 获取模块的面积。
  • has_size(): 判断模块是否已计算尺寸。
  • box (属性): 获取模块的边界框 Box (左下角为 (0,0) 相对于模块自身原点)。
  • get_pr_boundary(): 获取模块的 Place & Route 边界 (Box)。
  • has_pr_boundary(): 判断是否已设置 P&R 边界。
  • get_port_rects(): 获取模块定义的端口矩形字典 (Dict[str, List[Instance]])。端口的物理实现（矩形）本身被视为 Instance。

状态检查 (_module_sort.py):

提供一系列 is_* 方法判断模块类型或状态：

  • is_base_geometry(), is_rect(), is_rrect(), is_polygon(), is_via_array(): 判断是否为基础几何图形模块。
  • is_empty(): 判断是否为空模块。根据测试用例，此方法可能特指名为 "Empty" 且无内容的模块。
    • 示例: assert Empty().module().is_empty() (True for @module def Empty(): pass)
    • 示例: assert EmptyMod().module().is_empty() (False for @module def EmptyMod(): pass)
  • is_placed(): 布局是否已完成。
  • is_stdcell(): 是否为标准单元（通常从库导入）。
  • is_builtin(), is_builtin_lvs(), is_builtin_sim(): 是否为 TED 内建模块（用于 LVS 或仿真）。
  • has_builtin_lvs(), has_builtin_sim(): 模块内部是否（递归地）包含 LVS/Sim 内建模块。
  • is_testbench(): 是否为测试平台模块。
    • 示例: assert current_module().is_testbench() (within a @testbench function)
  • is_mos(), is_nmos(), is_pmos(), is_mos_series(): 判断是否为 MOS 器件或其系列。通常对基础器件模块（如 NMOS().module()）调用。
    • 示例: nmos_device_module = NMOS().module(); assert nmos_device_module.is_nmos()

2.3 模块操作

参数与配置 (在构建期间通过 current_module() 调用，或部分在构建后对 Module 对象调用):

  • set_param(**kwargs): (通常由框架在模块构建时根据函数参数隐式调用) 在构建期间设置模块参数。
  • current_module().set_model(model_name): 在模块构建函数内部，设置模块的 SPICE 模型。直接在已构建的 Module 对象上调用 set_model 可能会失败或行为未定义。
  • current_module().set_name(name): 在模块构建函数内部，设置模块的用户定义名称（通常会覆盖基于函数名的默认名称）。
    • 示例 (类模块中): current_module().set_name("MyNamedClassModule")
  • rename(new_name): 在模块构建完成后，重命名该 Module 定义的用户名称。慎用，可能影响依赖此名称的缓存或查找。
    • 示例: module_def.rename("NewConfigurableModule")
  • current_module().set_category(category): 在模块构建函数内部，设置模块分类。
  • current_module().set_type(type_name): 在模块构建函数内部，设置模块类型 (如 "Mos", "Resistor", "Capacitor", "Other")。
  • current_module().set_cellname(cellname): 在模块构建函数内部，设置导出 GDS 时的 Cell 名称。
  • set_builtin_lvs(), set_builtin_sim(), set_builtin_lvs_and_sim(): 将模块标记为 LVS/Sim 内建模块。这些通常在 Module 对象上调用。
    • 示例: empty_mod_def.set_builtin_lvs()
  • set_export_supplies(value: bool): 控制导出网表时是否包含 VDD/VSS 端口。
  • set_skip_autovia(): 禁止在该模块内部自动生成 VIA。
  • set_flatten_tag(tag): 设置模块在特定 flatten 目的下的行为标签。
    • 示例: current_module().set_flatten_tag("unit_level")

层次化与实例化:

  • instantiate(**kwargs): 基于此 Module 定义创建一个新的 Instance 对象。可以传入参数以覆盖模块定义中的默认参数值，从而创建特定变体的实例。
  • instances() 或 get_instances(func=None): 获取模块内部定义的所有直接子实例列表 (List[Instance])。
    • 可选地通过 func (一个接收 Instance 并返回布尔值的函数) 进行过滤。
    • 重要: 返回的列表中可能不仅包含用户在模块函数中显式创建的子模块实例或几何图形实例，还可能包含由子实例端口自动生成的端口矩形实例，这取决于子实例的定义和布局方式。
    • 示例: subblock_instances = top_def.get_instances(func=lambda inst: inst.module().name() == "SubBlock")
  • get_devices(lvs=None, sim=None): 获取模块内部的所有非基础几何图形的子实例（即器件或子模块）。
    • 可根据 lvs (布尔值) 或 sim (布尔值) 过滤是否为 LVS/Sim 内建类型的器件。
    • 测试用例说明: "Non-geometry sub-instances (i.e., devices or sub-modules)"。
    • 示例: devices = top_def.get_devices()
  • get_children(): 获取模块内部所有子实例的 InstList。行为与 get_instances() 类似，但返回类型不同。
  • get_child(name): 根据名称获取特定的子实例 (Instance)。
    • 示例: s1_retrieved = top_def.get_child("S1_inst")
  • flatten(purpose="netlist", units=None): 根据指定目的将模块内部的层次结构打平，返回一个新的 Module 对象，其中包含打平后的内容。
    • purpose="routing": 打平成 GDS 基础图形（矩形、VIA 阵列、多边形）。
    • purpose="rect": 打平成 GDS 基础矩形。
    • purpose="netlist": 打平成适合 LVS 的网表层级。
    • purpose="netlist_sim": 打平成适合仿真的网表层级。
    • purpose="unit", units=["tag1", "tag2"]: 打平到指定 flatten_tag 的子模块层级，带有这些标签的子模块不会被进一步打平。
    • 示例: flat_routing_module = outer_module_def.flatten(purpose="routing")
    • 示例: flat_unit_module = outer_module_def.flatten(purpose="unit", units=["unit_level"])

节点与连接操作:

  • node_rename(old_name_or_node, new_name_or_node): 在模块内部重命名节点。通常由 CircuitNode.rename() 内部调用 (见 node.md)。
  • push_input(node), push_output(node): (内部方法) 在构建期间将节点添加到模块的输入/输出端口列表。
  • connect_instance(left, right), connect_instances(src): (用于内部或高级操作) 声明两个或多个子实例之间存在连接（通常由 >> 等操作符自动处理）。
  • disconnect_instance(left, right), disconnect_instances(src): (用于内部或高级操作) 声明两个或多个子实例之间断开连接。

布局与物理操作:

  • auto_set_port(): (_module_port.py) 根据内部矩形的优先级、层级等自动选择并设置模块的端口矩形。
  • disable_auto_port(): 禁用 auto_set_port 的自动行为。
  • merge_rects(layer, spacing): 合并模块内指定层级满足条件的矩形（通常在打平后调用）。
  • set_pr_boundary(box: Box): 设置模块的 Place & Route 边界 (Box)。
    • 示例: mod_def.set_pr_boundary(custom_box)
  • expand_pr_boundary(x, y): 扩展模块的 P&R 边界。
  • auto_set_pr_boundary(): 自动将 P&R 边界设置为模块的实际包围盒 (box)。
    • 示例: mod_def.auto_set_pr_boundary()

包含外部文件 (通常在 Testbench 或仿真相关的模块内，通过 current_module() 调用):

  • current_module().include(incfile: str): 在导出 SPICE 网表时包含指定的 .inc 文件。
  • current_module().ahdl_include(incfile: str): 在导出 Spectre 网表时包含指定的 Verilog-A (.va) 文件。
    • 示例: current_module().include("my_spice_models.inc")

2.6 仿真集成

仿真通常在 @testbench 装饰的模块中进行配置和启动。实际的仿真命令 (如 tran(), dc(), ac()) 是全局函数，它们在当前测试平台模块的上下文中执行。Module 类提供以下方法（通常由 current_module() 在测试平台内部调用）来配置仿真：

  • watch(*args, **kwargs): 声明在仿真中需要观察/保存波形的节点或信号。
  • get_sim_current(*Node): 声明需要获取仿真中通过模块特定端口的电流。
  • assert_voltage_greater_than(Node, Value): 添加仿真断言，检查节点电压。
  • sim(simtype, args, options, ...): (较底层接口) 运行仿真。内部根据全局设置调用具体的仿真器接口。通常用户会使用更高级的 tran(), dc() 等函数。

  • show_route_report(): 显示布线的评估报告。

4. 辅助类

FakeModule (fake_module.py)

  • FakeModule: 与 FakeInstance 类似，当模块的详细定义不需要时（例如，在 skip_layout 模式下引用一个纯布局模块），会使用 FakeModule 作为占位符。它提供了一个极简的 API 以确保代码流程的兼容性，但本身不包含实际的电路或布局信息。

EnhancedInstance (emulated.py)

TED 提供了一种使用类而不是函数来定义模块的方式，通过继承 EnhancedInstance (或其基类 EmulatedInstance)。用户在继承类中实现 core() 方法来描述电路和布局逻辑。这对于组织复杂模块的代码、状态和辅助方法非常有用。实例化这个类时，会自动调用其 core() 方法来构建模块。

示例：使用 EnhancedInstance 定义模块

from pyted import EnhancedInstance, Input, Output, Rect, M1, Vdd, Vss, row, place, current_module

class MyClassModule(EnhancedInstance):
    def core(self, length: int = 100, width: int = 50): # 参数定义
        # self.inp, self.outp, self.ctrl 等是 EnhancedInstance 可能提供的预定义端口
        # 或者用户可以自行通过 Input, Output, Ctrl 在 core 方法参数中定义
        # (以下示例假设没有预定义端口，直接使用Vdd/Vss)
        
        current_module().set_name("MyNamedClassModuleFromCore") # 设置模块名

        self.r1 = Rect(M1, length, width, connectTo=Vdd())
        self.r2 = Rect(M1, length, width, connectTo=Vss()).move_by(length + 10, 0)
        
        # 假设 EnhancedInstance 自动处理了 place() 或需要显式调用
        # place(row(self.r1, self.r2, spacing=10)) # 示例布局

# 实例化类模块
class_inst = MyClassModule(length=150, width=75)
# 获取模块定义
class_module_def = class_inst.module()

assert isinstance(class_module_def, Module)
assert class_module_def.name() == "MyNamedClassModuleFromCore"
assert class_module_def.get_parameter("length") == 150
# assert class_module_def.is_placed() # Depends on place() call in core

5. 核心辅助函数

  • current_module() 或 get_current_module(): 返回当前正在构建或操作的 Module 上下文。这是在模块定义函数 (@module def ...) 或 EnhancedInstance.core() 方法内部访问和配置当前模块对象的标准方式。
    • 例如，在模块函数内调用 current_module().set_category("Analog")。
  • get_priority_from_config() 等: 从配置中获取不同类型布局元素（默认、总线、保护环）的优先级。

1. 定义与实例化模块

模块通过 @module 装饰器定义。调用装饰函数创建实例，再通过 .module() 获取模块定义。

# 1. 使用 @module 定义一个简单的反相器模块
@module
def MyInverter(inp: Input, outp: Output, strength: int = 1):
    inv_inst = Inv(m=strength) # 假设 Inv 是一个已定义的模块
    inp >> inv_inst >> outp
    place(inv_inst) # 示例布局

# MyInverter 现在是一个模块工厂

# 2. 获取特定参数的 Module 定义对象
inv_module_def_s1 = MyInverter(strength=1).module()
inv_module_def_s4 = MyInverter(strength=4).module()

print(f"Strength=1 的模块哈希名: {inv_module_def_s1.name_hash()}")
print(f"Strength=4 的模块哈希名: {inv_module_def_s4.name_hash()}") # 哈希值应不同

# 3. 实例化模块以创建 Instance
#    可以直接调用模块工厂函数并传递参数
@Testbench # Testbench is an alias for @testbench
def Top():
    a, z1, z4 = Node(3)
    inst_s1: Instance = MyInverter(strength=1)
    inst_s4: Instance = MyInverter(strength=4)
    # 或者使用获取到的特定 Module 定义对象进行实例化
    # inst_s1_alt: Instance = inv_module_def_s1.instantiate()

    a >> inst_s1 >> z1
    a >> inst_s4 >> z4

3. 层次结构与打平 (Flatten)

Module 可以包含子 Instance，形成层次结构。flatten 方法用于按需展平这个结构。

@module
def InnerBlock(a: Input): # Added Input for connection
    Rect(M1, 100, 100, a)
    place()
    current_module().set_flatten_tag("unit_level")

@module
def OuterBlock(c: Input): # Added Input for connection
    inner1 = InnerBlock(c).rename("inner1")
    inner2 = InnerBlock(c).rename("inner2")
    r_outer = Rect(M2, 500, 100, c).rename("r_outer")
    row(inner1, inner2, r_outer, spacing=100)
    place()

# 获取 OuterBlock 的 Module 定义对象
outer_module_def = OuterBlock().module()

# 1. 获取子实例 (不打平)
#    根据 test_flatten_operation, 如果子模块(InnerBlock)有端口且place()了，
#    其端口矩形也会被视为 OuterBlock 定义内的实例。
#    InnerBlock(a) has port 'a'. If 'Rect' inside InnerBlock is its port rect, it might be counted.
#    Let's assume get_instances() primarily returns explicitly defined instances here for simplicity of example output,
#    but acknowledge the test case's findings for more complex scenarios.

8. 使用 current_module()

在 @module 函数或 EnhancedInstance.core() 内部，使用 current_module() 获取当前正在构建的 Module 对象以进行配置。

@module
def ModuleUsingCurrent(param_a: int):
    cm = current_module()
    print(f"当前模块用户定义名: {cm.name()}") # ModuleUsingCurrent
    print(f"当前模块哈希名: {cm.name_hash()}") # Includes param_a value

    cm.set_category("DigitalBlock")
    cm.set_type("Combinational")
    cm.set_cellname(f"LogicUnit_p{param_a}")

    # ... 模块内容 ...
    r = Rect(M1, 100, 100, Vss())
    place(r)

# 获取模块定义并检查设置
mod_def_p10 = ModuleUsingCurrent(param_a=10).module()
assert mod_def_p10.get_category() == "DigitalBlock"
assert mod_def_p10.get_cellname() == "LogicUnit_p10"

7. 总结

TED 的 Module 是电路结构和行为的封装单元。它不仅定义了电路的连接和参数，还集成了布局信息、物理验证接口、多种格式的导出能力以及与仿真流程的紧密结合。通过强大的构建、缓存和解析机制，Module 为描述、管理和复用复杂的电路设计提供了坚实的基础。理解 Module 的生命周期、属性、方法以及相关的构建 (current_module()) 和获取 (MyMod().module()) 方式，是掌握 TED 设计流程的关键。
