实例 (Instance)

1. 概述

在 TED 框架中，Instance（实例）是电路设计中的核心对象之一。正如 "TED 前端基础文档" 中所述，Module（模块）定义了电路结构（如同类），而 Instance 则是 Module 被具体调用时生成的对象，代表了电路中一个实际存在的、具有特定连接和位置的组件或子电路。

本篇文档详细介绍了 Instance 类的功能、常用方法以及与之相关的辅助类和函数，涵盖了实例的创建、属性访问、连接、变换、布局操作、物理验证、数据导出等多个方面。

代码中的 Instance 对象通常是通过调用一个用 @module 或 @moduledef 修饰的函数或类来创建的。

from pyted import module, Instance, Input, Output, NMOS, PMOS, Node

@module
def Inv(x: Input, z: Output):
    Vdd() >> PMOS() % x >> z >> NMOS() % x >> Vss()

# 创建 Inv 模块的实例
x, z = Node(2)
inv_inst: Instance = Inv() # inv_inst 就是一个 Instance 对象
x >> inv_inst >> z

2. Instance 核心类 (instance.py)

Instance 类是 TED 中用于表示电路实例的 Python 封装。它提供了丰富的 API 来操作和查询实例的各种信息。

2.1 初始化与基本概念

  • 创建实例: 通常通过调用模块函数或使用 module.instantiate() 方法创建。
  • 内部表示: 每个 Python Instance 对象内部关联一个底层的 Rust 实现 (rsinst)，负责核心的电路数据管理和计算。wrap_instance 函数用于封装 Rust 对象。
  • 唯一性: 在一个模块内部，实例的名称通常需要是唯一的。TED 提供了自动重命名机制（auto_inst_rename）来处理命名冲突。
  • 生命周期: 实例可以通过 kill() 方法从其父模块中移除，通过 alive() 方法检查其是否存在。

2.2 属性与状态查询

Instance 类提供了大量方法来获取实例的状态和属性，其中很多 getter 方法实现在 _getsetter.py 文件中。

几何与位置信息:

  • box (属性): 获取实例在父模块坐标系下的边界框 (Box 对象)。
  • get_x1(), get_y1(): 获取边界框左下角坐标。
  • get_x2(), get_y2(): 获取边界框右上角坐标。
  • get_w(), get_h(): 获取实例的宽度和高度。
  • get_position(): 获取左下角坐标 (x1, y1)。
  • get_size(): 获取尺寸 (width, height)。
  • get_rotation_angle(): 获取实例的旋转角度 (0, 90, 180, 270)。
  • is_flipx(), is_flipy(), get_reflected(): 判断实例是否进行了翻转。
  • get_pr_boundary(): 获取实例的 Place & Route 边界 (Box)。

模块与连接信息:

  • module() 或 get_module(): 获取该实例所属的 Module 对象。
  • get_module_name(): 获取所属模块的名称。
  • name() 或 get_name(): 获取实例自身的名称。
  • get_input(), get_output(), get_ctrl(): 获取实例的输入、输出、控制节点 (CircuitNode)。
  • terminals(): 获取实例的所有普通（非电源）端口节点。
  • all_terminals(): 获取实例的所有端口节点（包括电源）。
  • get_terminal(node_name): 根据模块 PIN 名称获取实例对应的节点 (_node.py)。
  • get_terminal_dict(): 获取模块 PIN 名称到实例节点的映射字典 (_node.py)。
  • vdd(), vss(): 获取实例连接的电源和地节点。
  • get_supplies(): 获取实例的电源/地节点对。

参数信息:

  • get_parameter(name): 获取指定名称的参数值（自动转换类型）。
  • get_parameter_str(name): 获取指定名称的参数值（字符串形式）。
  • has_parameter(name): 判断实例是否有指定名称的参数。
  • get_all_parameters(): 获取实例的所有参数字典。

布局与物理信息:

  • get_layer(): 获取实例的层级（仅对图形实例有效）。
  • get_port_rects(): 获取实例所有端口的端口矩形字典 (Dict[str, InstList])。
  • get_port_rect_list(): 获取打平的所有端口矩形列表 (List[Instance])。
  • port(key) 或 __getitem__(key): 获取指定名称的端口矩形 (InstList)。
  • has_port(name): 判断实例是否包含指定名称的端口。
  • get_flatten_nboxes(): 获取实例打平后的所有 NBox (带层级和节点的 Box) 列表。
  • get_layer_nboxes(): 获取按层级分组的 NBox 字典 (Dict[Layer, List[NBox]])。
  • get_rects_by_layer(layer): 获取实例内指定层级的所有矩形 (List[NRect])。
    • get_bounding_nbox_by_layer(layer): 获取实例在指定层级的整体外接 NBox。
  • get_h_min_spacing() / get_v_min_spacing(): (实例方法，_getsetter.py) 获取实例内部相对于其边界的最小水平/垂直间距要求。
  • get_h_min_spacing(inst1, inst2) / get_v_min_spacing(inst1, inst2): (独立函数，instance.py 末尾) 计算两个实例间的最小水平/垂直间距要求。
  • get_priority(): 获取实例的布局优先级。
  • get_group(): 获取实例所属的分组名。
  • has_current(name), get_current(name): 查询端口或自身（若是矩形）是否设置/获取了电流值。

类型检查 (_sort.py):

提供一系列 is_* 方法判断实例类型：

  • is_shape(): 是否为矩形或多边形。
  • is_base_geometry(): 是否为基础几何图形 (Rect, RRect, Via)。
  • is_rectangle(), is_rect(), is_rrect(): 是否为不同类型的矩形。
  • is_polygon(): 是否为多边形。
  • is_via_array(): 是否为 VIA 阵列。
  • is_port_rect(): 是否为端口矩形。
  • is_mos(), is_nmos(), is_pmos(), is_mos_series(): 是否为 MOS 管或其系列。
  • is_renamed(): 是否被重命名过。
  • is_skip_pv(): 是否跳过物理验证。
  • is_hide_layout(): 在版图中是否隐藏。

2.3 实例操作

连接:

  • input(*nodes), output(*nodes), ctrl(*nodes): 手动设置实例的输入、输出、控制节点。
  • replace_ctrl(*nodes): 替换实例的控制节点。
  • __rshift__ (>>), __rrshift__ (>>), __lshift__ (<<), __rlshift__ (<<): 实现 >> 和 << 连接操作符，连接实例的输出到输入。
  • __mod__ (%): 实现 % 连接操作符，连接输出到控制端口。
  • __invert__ (~): 获取实例的输出节点（若只有一个输出）。

变换:

  • move_to(x, y), move_by(dx, dy): 移动实例。
  • move_center_to(cx, cy), move_cx_to(cx), move_cy_to(cy): 按中心点移动。
  • move_x1_to(x), move_y1_to(y), move_x2_to(x), move_y2_to(y): 按边界移动。
  • rotate(angle, center=None): 旋转实例。
  • flipx(), flipy(): 翻转实例。
  • correct_pos(func='round'): 将实例位置对齐到工艺格点。
  • r0(), r90(), r180(), r270(), my(), mx(), etc.: (来自 common_method) 提供快捷的旋转和翻转组合操作。

电源设置:

  • set_vdd(*nodes), set_vss(*nodes): 重新设置实例的 VDD/VSS 连接。

参数与配置:

  • rename(new_name): 重命名实例。
  • set_paramstr(param_str): 设置实例的参数字符串。
  • set_priority(priority): 设置布局优先级。
  • set_group(group): 设置实例所属的分组。
  • add_tag(tag), remove_tag(tag), add_tags(tags): 添加或移除标签。
  • set_current(current, name): 设置端口或自身的电流值。
  • set_route_width(width): 设置布线宽度（可能影响自动布线）。
  • set_cellname(cellname): 设置导出 GDS 时的 Cell 名称。

布局与层次化操作:

  • flatten(items, category='tag'): 根据标签或名称打平实例的子层级，返回可重新布局的子实例列表。
  • clone(): 创建一个参数相同但连接独立的新实例。
  • kill(): 从父模块中删除此实例。
  • clear_via(): 清除实例内部所有的 VIA 矩形，返回非 VIA 的部分，用于处理 VIA 重叠。
  • clear_layout(): 清除实例内部的所有布局信息（几何图形），只保留电路连接信息，返回 LVS/Sim 类型的子实例。
  • split(): (可能与 cplace 相关) 将实例拆分为可并行布局的部分。
  • to_subclass(): 尝试将通用 Instance 对象转换为更具体的子类（如 NMOS, NRect），如果可识别的话。

布局辅助 (_layout.py):

这些方法通常用于在实例周围或内部添加辅助布局元素。

  • add_vbus(), add_hbus(): 在实例旁边添加垂直/水平总线，并返回包含实例和总线的 InstList。
  • get_vbus(), get_hbus(): 生成与实例尺寸匹配的垂直/水平总线，但不组合。
  • add_label(): 在实例中心添加文本标签。
  • cover(): 在实例的边界框上覆盖指定层级的矩形。
  • cover_port(): 在指定的端口矩形上覆盖指定层级的矩形。
  • fuse(): 将实例内指定层级的相邻或重叠矩形融合成更大的矩形。
  • add_tap(): 在实例旁边添加 TAP (衬底接触)。
  • hide_layout(), show_layout(): 控制实例在最终版图中的可见性。
  • merge_rects(): 合并实例内指定层级、满足特定条件（如对齐、间距小）的矩形。

2.4 物理验证与导出 (_export.py, instance.py)

  • 物理验证 (PV):
    • lvs(...): 执行 Layout Versus Schematic (LVS) 检查。
    • drc(...): 执行 Design Rule Check (DRC)。
    • pex(...): 执行 Parasitic Extraction (PEX)。
    • svs(...): 执行 Source Versus Source (SVS) 网表比对。
    • svs_from_virtuoso(...): 与 Virtuoso 导出的网表进行 SVS 比对。
    • skip_pv(): 标记实例（通常是金属图形）在 PV 检查中应被忽略。
    • get_drc_report(): 获取DRC报告路径。
  • 导出:
    • to_gds(filepath=None, flatten=False): 导出 GDSII 版图文件。
    • to_hspice(): 获取实例的 HSpice 网表描述行。
    • to_verilog(): 获取实例的 Verilog 网表描述行。
    • export_hspice(), export_spectre(), export_verilog(), export_lef(), export_def(): 导出完整的网表或抽象文件（调用模块的导出方法）。
    • to_subckt(): 获取实例对应模块的 SPICE 子电路定义。
    • to_png(): 导出实例版图的 PNG 图片。
    • to_systemc(): 导出 SystemC 描述。
    • to_graph(): 将实例的内部结构转换为图形表示（用于分析）。
    • to_viewer_html() / to_virtuoso_viwer_json() / export_serialize_context_json(): 导出用于前端可视化的数据。

2.5 仿真与调试

  • show(label=None): 在连接的可视化前端显示实例的版图。
  • debug(): 生成表示该实例如何连接和参数化的代码字符串。
  • get_sim_current(*Node): (仿真相关) 声明需要获取仿真中通过指定节点的电流。
  • assert_voltage_greater_than(Node, Value): (仿真相关) 添加仿真断言，检查节点电压。
  • get_call_path(): 获取创建该实例的代码调用栈信息。

3. 辅助类

3.1 DelayedInstance (delayed.py)

  • 用途: 作为一个临时的占位符实例。允许先定义连接关系 (>>, %)、设置电源 (set_vdd, set_vss)、甚至进行变换 (rotate, flipx)，然后在需要时才通过 initialize() 方法实际创建真正的 Instance。
  • 场景:
    • 当模块的选择或参数依赖于后续计算时。
    • 在复杂的布局算法中，先确定拓扑连接，后实例化具体器件。
  • API: 提供了与 Instance 类似的连接、电源设置、变换接口，以及 set_module() 和 set_params() 来指定最终要实例化的模块和参数。alive() 和 kill() 控制其是否最终被实例化。

3.2 FakeInstance (fake_instance.py)

  • 用途: 当设置了 setenv(skip_layout=True) 时，所有与布局相关的操作（如 Rect, row, col, guardring 等）会返回 FakeInstance 而不是真正的 Instance 或 InstList。
  • 行为: FakeInstance 提供了与 Instance 和 InstList 兼容的 API 接口（大部分方法是空操作或返回默认值），目的是让代码在跳过布局时也能正常执行，不会因为缺少布局对象而报错。它不包含任何实际的电路或版图信息。
  • 场景: 主要用于只需要生成网表或进行仿真的流程，可以跳过耗时的布局计算。

3.3 EmulatedInstance & EnhancedInstance (emulated.py)

  • 用途: 这两个类（EnhancedInstance 继承自 EmulatedInstance）提供了一种使用 类 而不是函数来定义 TED 模块的方式。用户可以继承这些类，并在 core() 方法中编写模块的电路描述和布局逻辑。
  • 优点: 对于复杂的模块，使用类可以更好地组织代码、状态和辅助方法。
  • 机制: 内部通过 moduledef 装饰器将 core() 方法包装成一个标准的 TED 模块。实例化这个类时，会自动调用 core() 方法来构建内部的 Instance。
  • EnhancedInstance: 可能增加了对不同模式（layout, sim, lvs）的支持或其它增强功能。

4. 独立辅助函数

  • get_h_min_spacing(inst1, inst2) / get_v_min_spacing(inst1, inst2): 计算两个给定实例之间的最小水平/垂直间距要求，考虑它们内部所有层级的 DRC 规则。这对于手动精确放置实例非常有用。

    # 3. 翻转 (Flip)

    nmos_inst.flipy() # 沿 Y 轴（中心垂直线）左右翻转
    print(f"左右翻转? {nmos_inst.is_flipy()}") # True
    nmos_inst.flipx() # 沿 X 轴（中心水平线）上下翻转
    print(f"上下翻转? {nmos_inst.is_flipx()}") # True

    # 快捷变换 (来自 common_method)
    nmos_inst.r90() # 旋转 90 度
    nmos_inst.my()  # 左右翻转 (Mirror Y)

6. 总结

TED 的 Instance 是连接电路逻辑描述、物理布局和各种设计流程（验证、导出、仿真）的关键。它提供了极为丰富的功能集，让用户能够以编程方式精确地控制和查询电路中的每一个组件。理解 Instance 的 API 以及 DelayedInstance、FakeInstance 等辅助类的作用，对于高效使用 TED 进行电路设计至关重要。
