ModuleBase 类及其功能拓展

ModuleBase 是 TED 框架中一个核心的基类，它及其子类提供了两种强大的工作模式：普通模式 和 工厂/修饰器模式。这两种模式被统一在一个类中实现，旨在最大程度地简化应用层代码，提供灵活且高效的模块定义和管理方式。

设计理念

ModuleBase 的设计核心在于简洁性和可扩展性。通过将普通模式和工厂模式集成到同一个类中，用户可以根据不同的需求选择最适合的模块定义方式，而无需引入额外的类或 API。同时，通过 Mixin 机制，ModuleBase 及其子类可以方便地扩展功能，实现模块的定制化，满足各种复杂的电路设计需求。

  • 普通模式 Module（ModuleBase的子类）和Instance之间的关系类似编程中的类和对象的关系。Module定义模块的功能，Instance是Module的实例化后的对象。一个Module可以实例化出任意多个Instance，这些Instance可以通过inst.module()函数访问到其所属的Module。TED提供的Module具备大部分通用的功能，如仿真，导出，物理验证等。在某些场合，也可以直接使用ModuleBase定义功能更单一的模块。

  • 工厂模式/修饰器模式 ModuleBase的子类同时也可以作为修饰器来定义Module，类似@module修饰器。通过继承ModuleBase创建不同的子类，实现各自的成员函数，可以定义出来不同的“模块工厂”，实现更专业的功能。

TED框架确保这些新的“模块工厂”，所生产出来的新的Module，和其实例化出来的Instance，具备前面的Module-Instance之间的关系。

工厂、模块和实例之间的关系是：工厂生成模块，模块生成实例。

ModuleBase 基类：模块功能的基石

ModuleBase 是所有模块的基石，它包含了模块最基本且必备的功能和 API。这些功能通过集成一系列 Mixin 类实现，确保了模块具备核心的操作能力。

核心功能 (通过 Mixin 集成)

ModuleBase 集成了以下 Mixin 类，提供了丰富的基础功能：

  • GetSetterMixin: 提供属性的获取和设置方法。
  • ImportMixin: 处理模块的导入功能。
  • PortMixin: 管理模块的端口定义和操作。
  • SortMixin: 提供模块的类型判断功能。
  • GeometryMixin: 处理模块的几何信息和尺寸。
  • LayoutMixin: 管理模块的布局相关功能。
  • ParameterMixin: 处理模块的参数。
  • PVMixin: 提供物理验证相关功能。
  • InstanceMixin: 管理模块的实例。

示例代码:

class ModuleBase(
    GetSetterMixin,
    ImportMixin,
    PortMixin,
    SortMixin,
    GeometryMixin,
    LayoutMixin,
    ParameterMixin,
    PVMixin,
    InstanceMixin,
):
    """
    ModuleBase 是所有模块的基类，集成了多种基础功能Mixin。
    """
    pass

扩展 ModuleBase：定制你的模块

除了 ModuleBase 提供的基础功能，你还可以通过继承 ModuleBase 或结合其他 Mixin 类来创建更专业的模块。

Module 类：功能增强的模块

Module 类是 ModuleBase 的一个子类，它通过集成额外的 Mixin 扩展了 ModuleBase 的功能，使其具备电路仿真、物理验证和导出等高级能力。

扩展功能 (通过 Mixin 实现)

Module 类在 ModuleBase 的基础上，进一步集成了以下 Mixin：

  • SimMixin: 提供电路仿真相关功能，例如导出仿真网表 (e.g., Spice netlist)。
  • CalibreMixin: 提供物理验证 (PV) 相关功能，例如运行 Calibre DRC/LVS 检查。
  • ExportMixin: 提供模块的导出功能，例如导出为不同格式的文件。

示例代码:

class ModulePlugins(SimMixin, CalibreMixin, ExportMixin):
    """
    ModulePlugins 包含了 ModuleBase 之外的扩展功能。
    """
    pass

class Module(ModuleBase, ModulePlugins):
    """
    Module 类继承了 ModuleBase 和 ModulePlugins，提供了更全面的功能。
    """
    pass

使用方法:

当你需要一个具备仿真、物理验证和导出能力的模块时，可以直接使用 Module 类。

用例:

from pyted import *

@skip_unless(TED_PDK=[T65])
def test_module_extended_features():
    @Module()
    def Dut2():
        pass

    # Dut2 继承了 ModuleBase 的基础功能
    Dut2().get_name() # 获取模块名称

    # Dut2 继承了 Module 的扩展功能，例如 to_hspice_subckt()
    Dut2().module().to_hspice_subckt() # 导出 HSPICE 子电路网表
    print("Dut2 模块已成功导出 HSPICE 子电路网表。")

    # 验证一个只继承 ModuleBase 的模块不具备这些扩展功能
    @ModuleBase()
    def Dut():
        pass

    Dut().get_name()
    try:
        Dut().to_hspice_subckt()
        assert "不该有to_spice_subckt"
    except Exception as e:
        print(f"验证成功：{e}") # Output: 'Dut'对象没有'to_hspice_subckt'属性

在上面的例子中，Dut2 由于使用 @Module() 修饰，因此具备 to_hspice_subckt 方法，而 Dut 由于使用 @ModuleBase() 修饰，则不具备该方法。

工厂模式/修饰器模式：定义和创建模块

ModuleBase 类实现了 __call__ 函数，这使得它能够作为 Python 的修饰器（decorator）使用，从而以一种声明式的方式定义模块。这种用法可以看作是工厂模式或修饰器模式的体现。

设计理念

通过 __call__ 函数，ModuleBase 及其子类可以直接用于修饰函数，将函数转换为一个可实例化的模块定义。这种设计简化了模块的创建流程，并使其更符合 Python 的习惯用法。

使用方法

你可以使用 @ModuleBase() 或其子类（如 @Module()、自定义的 @Analog() 等）作为修饰器来定义模块。

注意区分:

  • @module (小写，不加括号)：这是 PyTed 中传统的定义模块的修饰器。
  • @Module() (大写，需要加括号)：这是 ModuleBase 类及其子类作为修饰器使用的方式。

对于简单的模块定义，建议优先使用传统的 @module 方式，因为它更简洁。然而，当你需要利用 ModuleBase 或其子类的 __init__ 方法来传递参数，或自定义模块创建逻辑时，@Module() 方式则显得更为强大。

API 详情

当 ModuleBase 或其子类作为修饰器使用时，被修饰的函数将成为一个模块的构建蓝图。__call__ 方法会在内部处理这个函数，并返回一个可实例化的模块对象。

用例：自定义模块类型和行为

通过工厂模式，你可以创建具备特定属性和行为的自定义模块类型。

1. 创建自定义模块类型并扩展功能:

你可以继承 Module 类来创建新的模块类型，并添加自己特有的方法。这些方法在修饰器模式下创建的模块实例中是可用的。

from pyted import *

@skip_unless(TED_PDK=[T65])
def test_custom_module_type():
    class Analog(Module): # 继承 Module，因此拥有 Module 的所有功能
        def special_analog_func(self):
            """模拟一个模拟电路特有的功能。"""
            print("call: special_analog_func")
            return self

    @Analog() # 使用自定义的 Analog 模块作为修饰器
    def Inv(A: "Input", ZN: "Output"):
        """一个简单的反相器模块。"""
        pass

    inv_instance = Inv() # 实例化 Inv 模块
    inv_module = inv_instance.module() # 获取 Inv 模块的 Module 对象

    # Inv 模块拥有 Analog 类特有的方法
    inv_module.special_analog_func() # 输出: call: special_analog_func

    # Inv 模块也拥有 Module 类的功能，例如导出 HSPICE 网表
    inv_module.to_hspice_subckt()
    print("Inv 模块已成功导出 HSPICE 子电路网表。")

    # debug(get_circuit().modules) # 查看当前电路中注册的模块
    # assert len(get_circuit().modules) == 1 # 验证只存在一个模块（Inv）

这个例子展示了如何定义一个 Analog 模块类，并通过 @Analog() 修饰器来创建具有 special_analog_func 方法的 Inv 模块。

2. 在模块定义时传递参数 (通过 __init__)

你可以为自定义的 Module 子类定义 __init__ 方法，并在修饰器使用时传递参数。这些参数将用于初始化模块的实例。

from pyted import *

@skip_unless(TED_PDK=[T65])
def test_module_with_init_params():
    class Digital(Module):
        def __init__(
            self,
            top_layer: "Layer" = M4, # 默认顶层为 M4
            util: float = 0.5,
            ratio: float = 0.5,
            from_server: bool = True,
        ):
            super().__init__() # 调用父类 Module 的构造函数
            self.top_layer = top_layer
            self.util = util
            self.ratio = ratio
            self.from_server = from_server
            print(f"Digital 模块初始化: top_layer={self.top_layer}, util={self.util}")

        def post_build(self):
            """模块构建后的回调函数，可用于触发自动化布局等。"""
            print(f"post_build for Digital module: {self.get_name()}")
            # 这里可以集成外部工具，例如 ICC 布局布线
            # from ted_interface.icc.icc_pr import AutoLayoutICC
            # AutoLayoutICC(
            #     module=self,
            #     util=self.util,
            #     ratio=self.ratio,
            #     top_layer=self.top_layer,
            #     from_server=self.from_server,
            # ).build()
            place() # 进行布局

    @Digital(top_layer=M4, util=0.7) # 在修饰器中传递参数
    def DEC4TO16(A: Input(4), AB: Input(4), Z: Output(16), ZB: Output(16)):
        """一个4转16译码器模块。"""
        from pyted.modules import Nand2, Inv

        # 模块内部的逻辑定义
        (
            (
                ((A[3] * 8, AB[3] * 8), (A[2] * 4, AB[2] * 4) * 2)
                >> Nand2() * 16
                >> Inv() * 16,
                ((A[1] * 2, AB[1] * 2) * 4, (A[0], AB[0]) * 8)
                >> Nand2() * 16
                >> Inv() * 16,
            )
            >> Nand2() * 16
            >> ZB
            >> Inv() * 16
            >> Z
        )

    dec_instance = DEC4TO16()
    dec_module = dec_instance.module()

    # 验证参数是否正确设置
    assert dec_module.top_layer == M4
    assert dec_module.util == 0.7
    print(f"DEC4TO16 模块的 top_layer: {dec_module.top_layer}, util: {dec_module.util}")

    dec_module.lvs() # 执行物理验证（LVS）
    print("DEC4TO16 模块已执行 LVS 检查。")

此示例展示了如何通过 @Digital(top_layer=M4, util=0.7) 将参数传递给 Digital 类的 __init__ 方法，从而定制 DEC4TO16 模块的布局行为。post_build 方法则在模块构建完成后被调用，可以用于触发自动化布局等。

3. 定义 Chiplet 模块并导出 SystemC 代码

你可以定义专门用于 Chiplet 设计的模块，并集成 SystemC 导出功能。

from pyted import *

class Chiplet(Module):
    # __init__中设置的属性，会在所有用这个module创建的module中都生效
    def __init__(self, sc_module=None):
        super().__init__()
        self.sc_module = sc_module

    def to_systemc(self):
        """
        导出 SystemC 代码。
        这里的实现可能需要处理模板，例如将模块的参数和结构映射到 SystemC 代码。
        """
        if self.sc_module:
            print(f"Generating SystemC code for {self.sc_module}")
        else:
            print("SystemC module name not specified for this Chiplet.")
        # 这里是 SystemC 代码生成的实际逻辑
        return self

def test_chiplet_design():
    # 定义 SRAM Chiplet 模块
    @Chiplet(sc_module="path/to/somefile.c::SimpleBus0x1747907098_2")
    def SRAM(
        addr: Input(64),
        data: Input(64),
        clk: Ctrl,
        we: Ctrl,
        re: Ctrl,
        param=123 # Instance 独有的参数用这个方式设置
    ):
        # SRAM 模块个性化的参数（所有 instance 共享）
        current_module().sram_param1 = 456
        # 添加一个矩形描述 Chiplet 的尺寸
        Rect(TEXT, 1000, 1000)
        place()

    # 定义 CPU Chiplet 模块
    @Chiplet(sc_module="path/to/somefile.c::CPU_2")
    def CPU():
        # 添加一个矩形描述 Chiplet 的尺寸
        Rect(TEXT, 2000, 1000)
        place()

    # 定义顶层模块 TOP，包含多个 Chiplet
    @Chiplet(sc_module="path/to/somefile.c::TOP")
    def TOP():
        # 实例化子模块并进行布局
        sram = SRAM().move_to(1000, 0) # 移动 SRAM 模块到指定位置
        cpu = CPU().move_to(3000, 0) # 移动 CPU 模块到指定位置
        # 可以使用 row/col/lay 进行布局，或者用 move_to 等语法
        # row(sram, cpu, spacing=1000)
        place()

    # 导出 SystemC 代码并验证
    sram_instance = SRAM()
    sram_module = sram_instance.module()
    print(f"SRAM SystemC Module: {sram_module.sc_module}")
    sram_module.to_systemc() # 调用 SystemC 导出功能
    print(f"SRAM 模块尺寸: {sram_module.size()}")
    assert sram_module.sc_module == "path/to/somefile.c::SimpleBus0x1747907098_2"
    assert sram_module.size() == (1000, 1000)

    top_instance = TOP()
    top_module = top_instance.module()
    print(f"TOP 模块尺寸: {top_module.size()}")
    assert top_module.size() == (4000, 1000)

在这个例子中，Chiplet 类被定义为可以包含 sc_module 属性，并在 to_systemc 方法中模拟 SystemC 代码生成。通过 @Chiplet() 修饰器，你可以为 SRAM、CPU 和 TOP 等模块指定各自的 sc_module 名称，并在实例化后调用 to_systemc() 方法来导出对应的 SystemC 代码。同时，模块的尺寸信息也可以通过 Rect 和 place 方法进行定义和获取。

两种工作模式的统一

ModuleBase 及其子类之所以在同一个类中同时实现普通模式和工厂/修饰器模式，是为了简化应用层代码。用户无需记忆两个不同的类或 API 来实现相似的功能，而是可以根据具体的使用场景选择最自然的方式。

  • 普通模式：当你需要一个 Module 实例来访问其固有的 API 时 (例如 inst.module().get_name())。
  • 工厂/修饰器模式：当你需要定义一个新的模块类型，并且可能需要定制其行为或传递初始化参数时 (例如 @Module() 或 @Chiplet(sc_module=...))。

这种统一的设计使得 TED 框架更加内聚且易于使用，为用户提供了极大的灵活性。
