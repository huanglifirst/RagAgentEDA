实例列表 (InstList)

1. 概述

在 TED 中，除了单个的 Instance 对象外，经常需要处理一组相关的实例。InstList 类就是为此设计的容器，它代表一个实例的有序集合。InstList 不仅提供了管理这个集合的基本列表操作，还允许对集合中的所有实例进行集体操作，例如统一移动、旋转、设置电源连接、执行布线、物理验证等。

InstList 对象通常通过以下方式产生：

  1. 使用布局函数（如 row, col, lay, guardring, add_vbus 等）组织多个 Instance 或 InstList。
  2. 对一个 Instance 使用乘法操作符 * m 或 * (start, end) 来创建阵列。
  3. 显式地使用 InstList([...]) 构造函数。

理解 InstList 对于进行结构化布局、执行自动布线和批量处理实例至关重要。

2. InstList 核心类 (instlist.py)

InstList 封装了一个实例集合，并提供了丰富的 API 进行管理和操作。

2.1 初始化与内部结构

  • 创建:
    • InstList(items=None, name=None, schematic_sequence=None, ...): 构造函数。
      • items: 包含 Instance 或嵌套 InstList 的列表/元组。
      • name: 为该 InstList 指定一个名称。
      • schematic_sequence: 一个元组 (start, end) 或字典，用于处理由 * 操作符创建的、具有特定索引顺序的实例集合。
  • 内部表示:
    • _instances: 一个 Python 列表，存储展开后的所有 Instance 对象。
    • rs_instlist: 对应的底层 Rust rsted.InstList 对象，用于高效执行集体操作。
    • children: 存储传入 items 的原始结构（可能包含嵌套 InstList），主要用于 _flatten.py 中的特殊打平逻辑。
    • schematic_sequence: 一个字典，映射用户指定的索引（如 insts[5] 中的 5）到内部 _instances 列表的实际索引。
    • from_schematic: 布尔值，指示该 InstList 是否由带特定序列的 * 操作符创建。

2.2 基本列表操作

InstList 提供了类似 Python 列表的接口：

  • __len__(): 返回集合中的实例数量。
  • __getitem__(index):
    • 如果 index 是整数：返回指定索引的 Instance。支持 Python 负数索引。如果 InstList 具有 schematic_sequence，则 index 是用户指定的索引（如 insts[start] 到 insts[end]）。
    • 如果 index 是切片 (slice): 返回一个新的 InstList，包含切片选中的实例子集。
  • __iter__(), __next__(): 支持迭代，可以 for inst in instlist: 遍历其中的 Instance。迭代顺序遵循 schematic_sequence（如果存在）。
  • append(inst): 向列表末尾添加一个 Instance。
  • extend(insts): 将另一个 InstList 或 Instance 列表中的所有实例添加到当前列表末尾。
  • first(): 获取 schematic_sequence 中索引最小的实例。
  • mid(): 获取列表（按内部顺序）中间的实例。
  • get(index): 等同于 __getitem__(index)。
  • get_single_item(index): 根据 schematic_sequence 映射获取内部列表中的实例。

2.3 名称与层次结构

  • name(): (_getsetter.py) 获取 InstList 的名称。
  • rename(name=None): (_getsetter.py) 重命名 InstList 及其包含的所有实例。如果 InstList 具有 schematic_sequence，子实例会被命名为 {name}_{index}。
  • get_child(name): (_getsetter.py) 在 InstList 包含的实例中查找指定名称的子实例。

2.4 集体几何与变换

这些方法将操作应用于列表中的所有实例。

  • box (属性): (_getsetter.py, instlist.py) 计算并返回包含所有实例的最小外包矩形 (Box)。
  • get_w(), get_h(), get_size(), get_position(), get_x1(), get_y1(), get_x2(), get_y2(): (_getsetter.py) 获取整个列表的外包框的几何属性。
  • move_to(x, y), move_by(dx, dy): 移动整个实例列表。
  • move_center_to(cx, cy), move_cx_to(cx), move_cy_to(cy): 按中心点移动整个列表。
  • move_x1_to(x), move_y1_to(y), move_x2_to(x), move_y2_to(y): 按边界移动整个列表。
  • rotate(angle, center=None): 旋转整个列表（相对于列表自身的原点或指定中心）。
  • flipx(), flipy(): 翻转整个列表（相对于列表自身的中心轴）。
  • get_rotation_angle(): 获取列表的整体旋转角度。
  • is_flipx(), is_flipy(): (_sort.py) 判断列表是否整体进行了翻转。
  • correct_pos(func='round'): (instlist.py) 对列表中的每个实例调用 correct_pos，使其位置对齐到格点。
  • get_pr_boundary(), set_pr_boundary(box), expand_pr_boundary(x, y): (_getsetter.py) 获取、设置或扩展整个列表的 Place & Route 边界。

2.5 集体连接与设置

  • get_input(), get_output(): (_getsetter.py) 获取列表中所有实例对应端口节点的组合 (CircuitNode 束)。例如，如果列表包含 3 个 Inv 实例，get_input() 会返回一个包含 3 个节点的 CircuitNode 束，每个节点对应一个 Inv 的输入。
  • get_nodes(): (_getsetter.py) 获取列表中所有实例的所有非电源终端节点，并返回一个名称到 CircuitNode 的映射字典。
  • set_vdd(*nodes), set_vss(*nodes): (instlist.py) 将列表中所有实例的 VDD/VSS 连接到指定的节点。
  • set_port(): (_getsetter.py) 将列表中所有（通常是顶层金属）矩形实例标记为端口形状，并关联到它们连接的节点。
  • add_label(label, layer): (instlist.py) 在列表中心添加文本标签。
  • add_tag(tag): (instlist.py) 给列表中的所有实例添加标签。
  • set_route_group(group): (_getsetter.py) 为列表内所有实例设置布线组。
  • set_intra_group_routing(group): (_getsetter.py) 为列表内所有实例设置组内布线标志。
  • reconnect(*node_maps): (instlist.py) 对列表内所有 RRect 实例，根据提供的映射表 (old_node, new_node) 重新连接其节点。

2.6 布局信息与查询

  • get_node_rects(nets, exclude_nets, layer, all): (_getsetter.py) 获取列表中与指定节点（nets）相关的矩形实例（NRect），按节点名称分组返回字典。可以按层级 (layer) 或是否已连接 (all=False) 进行过滤。
  • get_layer_nboxes(): (_getsetter.py) 获取列表中所有实例在不同层级上的 NBox，按层级分组返回字典。
  • get_bounding_nbox_by_layer(layer): (_getsetter.py) 获取列表在指定层级上的最小外包 NBox。
  • get_flatten_nboxes(): (_getsetter.py) 获取列表中所有实例打平后的 NBox 列表。
  • get_nboxes_by_tag(tag): (_getsetter.py) 获取列表中带有指定标签的所有 NBox。
  • get_h_projection(), get_v_projection(): (_getsetter.py) 获取列表在水平或垂直方向上的投影区间 (List[Interval])。
  • get_route_cost(): (_getsetter.py) 估算列表内实例间连接的布线成本（基于曼哈顿距离等）。
  • get_net_line_length(): (_getsetter.py) 获取每个节点在列表内部连接的估计线长（曼哈顿距离）。
  • is_row(), is_col(), is_lay(): (_sort.py) 判断该 InstList 是通过哪种布局函数（row, col, lay）创建的。

2.7 层次化与打平

  • flatten(purpose="guardring", **params): (_flatten.py) 特殊打平方法。根据 purpose（如 "guardring"）递归地打平 InstList 的内部结构。打平过程会根据 IngoreSwitcher 和 StopDescentSwither 类中的规则，决定哪些类型的子项应该被忽略，哪些应该停止向下递归并直接返回。这通常用于提取特定类型的实例（如 MOS 管、保护环）以进行后续处理。
  • to_instance(kill_instance=True): (instlist.py) 重要方法。将 InstList 及其包含的所有实例合并成一个新的单一 Instance。这个新的 Instance 内部会包含一个自动生成的 Module，其内容就是原 InstList 的布局结果。
    • kill_instance=True: 合并后，原 InstList 中的实例会被标记为 "killed"（不再存在于父模块中）。
    • kill_instance=False: 原 InstList 中的实例仍然保留，用户需要手动 kill() 这个返回的单一 Instance（通常在布局操作完成之后），否则会导致重复。

2.8 自动布线 (_route.py)

InstList 是执行自动布线的主要对象。

  • route(...): 核心布线方法。尝试在 InstList 包含的实例（及其端口矩形）之间自动连接指定的网络。
    • 主要参数:
      • nets/exclude_nets: 指定需要布线/排除布线的网络。
      • width/node_widths: 全局/特定网络的布线宽度。
      • spacing/node_spacings: 全局/特定网络的布线间距。
      • top_layer: 允许布线的最高层级。
      • vlayer/hlayer: 指定垂直/水平布线使用的层级列表。
      • via: VIA 尺寸（以能容纳的最小 VIA 数量计）。
      • preferred_direction: 是否允许路由器优先选择层级的默认布线方向。
      • strictly_preferred_direction: 是否严格遵守层级的首选方向。
      • allow_poly_route/ignore_poly_route: 是否允许/忽略 Poly 层的布线。
      • expand_x/expand_y: 扩展布线区域。
      • disable_priority/disable_rip_up: 控制布线算法行为。
      • pattern_route_config/shielding_config: 传入更详细的模式布线或屏蔽线配置对象。
      • auto_fusion: 布线后是否自动合并生成的矩形。
      • intra_group_routing: 是否只在指定组内进行布线。
      • method: (未使用，保留参数)
    • 内部实现: 调用底层的 Rust 布线引擎执行 detailed routing。
    • 结果: 在当前模块中添加布线产生的金属和 VIA 矩形 Instance。
  • channel_route(...): (特定场景) 可能用于通道式布线。
  • pattern_route(...): (特定场景) 强制使用模式布线。
  • smt_route(...): (特定场景) 可能调用 SMT（Steiner Minimum Tree）布线算法。
  • smartbus(...): 高级总线生成与优化。自动为指定网络生成总线（水平和垂直），并使用力导引或迭代优化算法确定总线位置，以最小化整体线长或满足其他约束。

2.9 物理验证与导出

  • lvs(...), svs(...), svs_from_virtuoso(...), drc(...): 对 InstList 整体执行 LVS, SVS, DRC 检查（通常先调用 to_instance() 转换为单一实例）。
  • to_gds(*nodes, filepath, flatten): 导出 InstList 的 GDSII 文件。可选只导出连接到特定 nodes 的几何图形。
  • to_graph(draw=False): 将 InstList 内部的连接关系转换为 NetworkX 图。

2.10 其他

  • kill(): 杀掉（删除）InstList 中的所有实例。
  • alive(): (_sort.py) 判断 InstList 是否存活（其 Rust 对象是否存在）。
  • get_survivors(): (_getsetter.py) 返回列表中所有仍然存活的实例组成的新 InstList。
  • clear_via(): (instlist.py) 递归地清除列表内所有实例中的 VIA，返回不含 VIA 的几何图形组成的 InstList。

3. 辅助类

3.1 EmulatedInstList (emulated_instlist.py)

  • 用途: 允许用户像定义 EmulatedInstance 一样，通过继承此类并实现 layout() 方法来定义一个 InstList 的生成逻辑。
  • 行为: layout() 方法会在首次需要几何信息（如调用 move_by, box 等）时被延迟调用 (lazily called)。这允许更灵活地组织和复用产生 InstList 的代码。

3.2 FakeInstList (fake_instlist.py)

  • 用途: 在 setenv(skip_layout=True) 等场景下，作为 InstList 的占位符。
  • 行为: 提供与 InstList 兼容的最小 API 接口，但所有操作都是空操作或返回默认值（如空列表、FakeNode）。确保在跳过布局计算时代码能继续执行而不出错。

好的，下面将根据 instlist 目录下提供的测试用例代码，并结合其他相关功能的测试，为之前编写的 InstList 文档构造一些具体的功能示例。

4. TED 实例列表 (InstList) 功能示例

本节提供基于 InstList 类常用功能的代码示例，旨在阐明如何使用 InstList 来管理和操作实例集合。

1. 创建 InstList

InstList 可以通过多种方式创建：

from pyted import InstList, Instance, Node, Input, Output, module
from ted_device import Inv # 假设有 Inv 模块
from layout import row, col # 假设有布局函数

# 假设 inv1, inv2, inv3 是已创建的 Instance 对象
inv1, inv2, inv3 = Inv().rename("i1"), Inv().rename("i2"), Inv().rename("i3")

# 1. 显式创建
explicit_list = InstList([inv1, inv2, inv3], name="explicit_inverters")
print(f"显式创建: {explicit_list}, 长度: {len(explicit_list)}")

# 2. 使用乘法操作符 * m (创建阵列，默认索引 0 到 m-1)
inv_array_default = Inv() * 3 # 创建 3 个 Inv 实例
inv_array_default.rename("arr_def") # 实例会被命名为 arr_def_0, arr_def_1, arr_def_2
print(f"乘法创建 (默认索引): {inv_array_default}, 长度: {len(inv_array_default)}")
print(f"  -> 第一个实例名: {inv_array_default[0].name()}")

# 3. 使用乘法操作符 * (start, end) (创建阵列，指定索引范围)
#    (改编自 test_instlist.py::test01)
#    创建 2 个 Inv 实例，用户指定的索引为 2 和 1 (从 2 到 1)
inv_array_seq = Inv() * (2, 1)
inv_array_seq.rename("arr_seq") # 实例会被命名为 arr_seq_2, arr_seq_1
print(f"乘法创建 (指定序列): {inv_array_seq}, 长度: {len(inv_array_seq)}")
print(f"  -> 索引 2 的实例名: {inv_array_seq[2].name()}") # 使用用户索引访问
print(f"  -> 索引 1 的实例名: {inv_array_seq[1].name()}")

# 4. 通过布局函数创建 (如 row, col, lay 等)
#    row() 返回一个包含 inv1 和 inv2 的 InstList
row_list = row(inv1, inv2, spacing=100, name="inverter_row")
print(f"布局函数创建: {row_list}, 类型: {type(row_list)}")

2. 访问元素与迭代

可以像操作 Python 列表一样访问 InstList 中的元素。

# 使用上面的 inv_array_seq (索引为 2, 1)
inv_array_seq = Inv() * (2, 1)
inv_array_seq.rename("arr_seq")

# 1. 按索引访问 (使用用户指定的 Schematic Sequence 索引)
inst_at_index_2 = inv_array_seq[2]
print(f"访问索引 2: {inst_at_index_2.name()}") # arr_seq_2

inst_at_index_1 = inv_array_seq[1]
print(f"访问索引 1: {inst_at_index_1.name()}") # arr_seq_1

# 使用负数 Python 索引 (相对于内部列表顺序)
# 对于 (2, 1)，内部顺序可能是 [arr_seq_2, arr_seq_1]
# 所以 -1 访问的是 arr_seq_1
last_internal = inv_array_seq[-1]
print(f"内部最后一个 (索引1): {last_internal.name()}") # arr_seq_1

# 2. 迭代 (按 Schematic Sequence 顺序)
print("迭代 arr_seq:")
for inst in inv_array_seq:
    # 迭代顺序会是 arr_seq_2, arr_seq_1
    print(f"  - {inst.name()}")

3. 集体几何变换

可以对整个 InstList 应用移动、旋转、翻转等操作。

# 创建一个包含3个实例的行
row_list = row(Inv(), Inv(), Inv(), spacing=100, name="transform_row")
print(f"初始 Box: {row_list.box}")

# 1. 集体移动
row_list.move_by(1000, 500)
print(f"移动后 Box: {row_list.box}")

# 2. 集体旋转 (围绕列表的中心点)
cx, cy = row_list.box.center_coord()
row_list.rotate(90, center=(cx, cy))
print(f"旋转后 Box: {row_list.box}") # 注意 W/H 会交换
print(f"旋转后角度: {row_list.get_rotation_angle()}") # 90

# 3. 集体翻转 (围绕列表的中心轴)
row_list.flipy() # 左右翻转
print(f"左右翻转? {row_list.is_flipy()}") # True

4. 集体连接与设置

可以将节点束连接到 InstList，或统一设置列表内实例的属性。

# (改编自 test_instlist.py::test03)
inputs = Node(2, name="a")    # a<1:0>
outputs = Node(2, name="b")   # b<1:0>
control = Node(name="ctrl")

# 创建包含 2 个 MySimpleModule 实例的列表 (索引 0, 1)
# 假设 MySimpleModule 有 inp: Input, outp: Output, ctl: Ctrl
dut_array = MySimpleModule() * 2
dut_array.rename("dut") # dut_0, dut_1

# 1. 集体连接
# a<0> -> dut_0.inp, a<1> -> dut_1.inp
# ctrl -> dut_0.ctl, ctrl -> dut_1.ctl
# dut_0.outp -> b<0>, dut_1.outp -> b<1>
inputs >> dut_array % control >> outputs

# 2. 获取集体端口
combined_inputs: CircuitNode = dut_array.get_input()
print(f"集体输入节点: {combined_inputs}") # Bundle of 2 nodes
print(f"输入是否连接到 a? {combined_inputs == inputs}") # True

combined_outputs: CircuitNode = dut_array.get_output()
print(f"集体输出节点: {combined_outputs}") # Bundle of 2 nodes

# 3. 集体设置电源
vdda_net = Node("VDDA")
dut_array.set_vdd(vdda_net) # 将 dut_0 和 dut_1 的 VDD 都连接到 VDDA

5. 转换为单一实例 (to_instance)

可以将一个布局好的 InstList 封装成一个新的、单一的 Instance，便于在更高层次复用。

# 创建一个行布局 (返回 InstList)
inv_row = row(Inv(), Inv(), spacing=50, name="two_invs")

# 将这个行布局转换为一个名为 "TwoInvRow" 的新模块的实例
# 默认情况下，原始的两个 Inv 实例会被 kill
compound_inst: Instance = inv_row.to_instance().rename("compound_two_invs")

print(f"新实例名称: {compound_inst.name()}")
print(f"新实例的模块哈希名: {compound_inst.module().name_hash()}")

# 现在可以像操作普通实例一样操作 compound_inst
compound_inst.move_by(100, 100)
place(compound_inst)

6. 在 InstList 上执行布线

route() 方法通常应用于包含已放置实例的 InstList。

7. 特殊打平 (flatten)

InstList 的 flatten 方法用于按特定目的提取内容。

# 假设 guardring_layout 是一个包含 MOS Instance 和 GuardRing Instance 的 InstList
# guardring_layout = guardring(mos_inst, ...)

# 目的：为 guardring 相关操作打平
# IngoreSwitcher.guardring 会忽略基础几何图形
# StopDescentSwither.guardring 会在 MOS, GuardRing, Tap 等层级停止递归
flattened_for_gr = guardring_layout.flatten(purpose="guardring")

print("Flattened for guardring:")
for item in flattened_for_gr:
    # 通常会得到 MOS Instance, GuardRing Instance 等，基础几何图形被过滤
    print(f" - {item}")

这些示例展示了 InstList 的灵活性和强大功能，特别是在处理实例集合的布局、连接和布线方面。

5. 总结

InstList 是 TED 中用于管理和操作实例集合的核心类。它不仅提供了方便的列表操作接口，更重要的是实现了对集合内所有实例的集体几何变换、连接设置、布线、验证和导出等功能。通过 to_instance() 方法，可以将一个布局好的 InstList 封装成一个可复用的单一 Instance。InstList 也是执行各种自动布线算法的主要入口点。理解 InstList 的集体操作特性、schematic_sequence 的概念以及 route() 和 to_instance() 的用法，对于使用 TED 进行高效的层次化和自动化布局布线至关重要。
