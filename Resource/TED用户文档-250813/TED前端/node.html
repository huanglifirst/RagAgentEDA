节点 (Node)

1. 概述

在 TED 框架中，Node（节点）或其封装类 CircuitNode 代表了电路中的电气连接点或网络（Net）。正如 "TED 前端基础文档" 所述，节点是连接 Instance（实例）各个端口（输入、输出、控制）的基本单位。理解和正确使用节点是构建电路拓扑结构的基础。

本篇文档详细介绍了如何创建和操作 TED 节点，包括单节点和节点束（Bus）的处理、命名、连接、索引、切片以及与模块和实例的交互。

2. 创建节点

创建节点的主要方式是使用 Node() 工厂函数。

from pyted import Node, CircuitNode

# 创建单个节点，自动命名（如 "n0_"）
n1 = Node()

# 创建单个节点，并指定名称
clk = Node(name="clk")

# 创建一个包含 8 个节点的节点束 (Bus)，索引范围 [0, 7]，基础名称自动生成
bus_auto = Node(8) # 等价于 Node(0, 7)

# 创建一个包含 8 个节点的节点束，索引范围 [0, 7]，并指定基础名称 "data"
# 内部节点名如 "data<0>_", "data<1>_", ... "data<7>_"
data_bus = Node(8, name="data") # 等价于 Node(0, 7, name="data")

# 创建一个索引范围为 [7, 0] 的 8 位节点束（常用于 MSB 在左的表示）
addr_bus = Node(7, 0, name="addr")

# 创建一个索引范围为 [15, 8] 的 8 位节点束
high_byte = Node(15, 8, name="byte")

# Node() 函数返回的是 CircuitNode 类的实例
print(type(clk)) # <class 'tedcore.kernel.src.node.node.CircuitNode'>

注意:

  • 不带参数的 Node() 创建单个节点。
  • Node(size) 创建一个 size 位，索引从 0 到 size-1 的节点束。
  • Node(start, stop) 创建一个索引从 start 到 stop（包含两端）的节点束。start 可以大于 stop。
  • name 参数指定节点或节点束的基础名称。未指定时会自动生成内部名称（通常以 _ 结尾，如 n0_ 或 data<0>_）。不以下划线结尾的名称被视为用户显式命名。
  • 节点名称不能以数字开头（除非是从 LVS 网表等外部源导入的特殊情况）。

3. CircuitNode 核心类 (node.py)

CircuitNode 类是 TED 中节点的主要 Python 接口，它封装了底层的 Rust rsted.Node 对象。一个 CircuitNode 实例可以代表一个单节点 (Single) 或一个节点束 (Bundle)。

3.1 基本属性与状态查询

  • name(escape=False, square=False, underline=False) 或 get_name(): 获取节点（或节点束的基础）名称。
    • escape: 是否转义 Verilog 等不兼容的尖括号 < > 为 \< \>。
    • square: 是否将 < > 替换为 [ ]。
    • underline: 是否将 < > 替换为 _ _。
  • len() 或 __len__(): 获取节点束包含的节点数量（单节点长度为 1）。
  • is_single(): 判断是否为单节点。
  • is_bundle(): 判断是否为节点束。
  • is_named(): 判断节点是否被用户显式命名过（名称不以下划线结尾）。
  • is_port(): 判断该节点是否为当前模块的一个端口 (PIN)。
  • get_voltage(): 获取节点的电压值（如果已通过仿真或 set_voltage 设置）。

3.2 操作节点束 (Bundle)

节点束提供了类似 Python 列表的操作方式：

  • 索引访问: bundle[index] 或 bundle.get(index)
    • 获取节点束中指定索引的单个 CircuitNode。
    • 索引遵循 Node() 创建时指定的范围（如 Node(7, 0) 的有效索引是 7 到 0）。
    • 也支持 Python 负数索引（如 bundle[-1]）。
  • 切片: bundle[start:stop:step]
    • 获取节点束的一个子集，返回一个新的 CircuitNode 束。
    • 支持 Python 标准切片语法（包括负数索引和步长）。
  • 自定义范围切片: bundle(start, stop, step=None)
    • 与 __getitem__ 切片类似，但使用函数调用语法。
    • start 和 stop 定义了包含两端的索引范围。
  • 迭代: for node in bundle:
    • 可以遍历节点束中的每一个单节点 (CircuitNode)。迭代顺序遵循索引的定义顺序（例如，Node(7, 0) 会先迭代索引 7，然后 6，依此类推）。
  • 展平: bundle.flatten()
    • 将节点束（无论多少层嵌套）彻底打平成一个包含所有单节点的 Python 列表 List[CircuitNode]。
  • 分块: bundle.chunks(chunk_size)
    • 将节点束按指定大小 chunk_size 分割成多个子束，返回 List[CircuitNode]。
  • 重塑: bundle.reshape_for_instlist(instlist)
    • 这是一个特殊操作，用于将节点束的内部顺序重新排列，以匹配 InstList（实例列表）的连接需求。当一个多比特的节点束连接到一个由 * m 产生的 InstList 时，需要调用此方法来确保正确的逐位连接。

3.3 节点操作

  • rename(name): 重命名节点或节点束的基础名称。这会与当前模块的 node_rename 方法交互，确保在模块命名空间内唯一。
    data = Node(4, name="data_old")
data.rename("data_new")
print(data[0].name()) # 输出类似 "data_new<0>_"
  • set_voltage(voltage): 设置节点的电压值（主要用于仿真设置或分析）。
  • set_named(named=True): 标记节点是否为用户显式命名。

3.4 连接性

  • to(other): 将当前节点连接到 other（可以是另一个 CircuitNode 或 Instance）的输入端。
    node_a = Node(name="a")
node_b = Node(name="b")
node_a.to(node_b) # 等价于 node_a >> node_b
  • 连接操作符: CircuitNode 支持 >>, <<, % 等连接操作符，用于连接到其他节点或实例。
    from ted_device import Inv
inp = Node(name="in")
out = Node(name="out")
inv = Inv()
inp >> inv >> out # 使用 >> 连接
  • __eq__(other): 判断两个 CircuitNode 是否代表同一个电气网络（即它们是否连接在一起）。
    node_a = Node()
node_b = Node()
node_c = node_a # node_c 指向 node_a
node_a >> node_b
print(node_a == node_b) # True (已连接)
print(node_a == node_c) # True (同一对象)

3.5 端口关联

  • port(inst): 将一个或多个 Instance（通常是版图中的矩形或多边形）标记为代表当前 CircuitNode 的端口形状。这对于版图生成和 LEF/DEF 导出至关重要。
    from pyted import Rect, M1
clk_pin = Node(name="clk")
# 创建一个 M1 矩形，并将其标记为 clk_pin 的端口形状
clk_rect = Rect(M1, 100, 100, connectTo=clk_pin)
clk_pin.port(clk_rect)
# 或者直接在 Rect 创建时调用 set_port()
clk_rect.set_port() # 内部会调用 clk_pin.port(clk_rect)

3.6 静态与类方法

  • CircuitNode.empty(): 创建一个特殊的空节点。
  • CircuitNode.new_bundle_from(name=None, nodes=[...]): 从一个现有的 CircuitNode 列表创建一个新的节点束。
  • CircuitNode.to_nodes(nodes): 将包含 str 或 CircuitNode 的列表转换为 List[CircuitNode]。
  • CircuitNode.convert_to_rs_node(node): 将 str 或 CircuitNode 转换为底层的 Rust rsted.Node 对象。
  • CircuitNode.try_escape_name(name, escape=False): 尝试将 TED 内部命名（如 a_1_）转换回带尖括号的形式（a<1>），或进行转义。

4. 特殊节点 (Vdd, Vss)

如 "TED 前端基础文档" 所述，Vdd() 和 Vss() 是两个特殊的函数，用于获取当前 Module 上下文中的电源和地节点 (CircuitNode)。

from pyted import Vdd, Vss, Node

# 获取当前模块的 VDD 和 VSS 节点
vdd_node = Vdd()
vss_node = Vss()

print(vdd_node.name()) # 通常是 "_vdd_" 或用户通过 setenv 指定的名称
print(vss_node.name()) # 通常是 "_vss_" 或用户通过 setenv 指定的名称

# 可以像普通节点一样使用它们进行连接
gnd = Node(name="gnd")
gnd >> Vss() # 将 gnd 连接到当前模块的 VSS
  • 顶层模块 (@testbench) 的 Vdd() 和 Vss() 默认连接到全局电源地（通常名为 VDD, VSS 或 vdd!, gnd! 等，取决于 PDK 配置）。
  • 子模块中的 Vdd() 和 Vss() 默认连接到其实例化时所在的父模块的 Vdd() 和 Vss()。可以使用 Instance.set_vdd() 和 Instance.set_vss() 来改变实例的电源连接。

5. FakeNode 辅助类 (fake_node.py)

  • 用途: 当设置了 setenv(skip_layout=True) 或在某些不需要节点详细信息的场景下，FakeNode 作为 CircuitNode 的占位符。
  • 行为: 它提供了与 CircuitNode 兼容的极简 API（如 __len__, __getitem__, __iter__ 返回空或 None），以避免在跳过布局或相关计算时因缺少节点对象而引发错误。它不代表任何实际的电气连接。

6. TED 节点 (Node / CircuitNode) 功能示例

本节提供基于 Node() 函数和 CircuitNode 类常用功能的代码示例，旨在阐明如何在 TED 中创建、操作和连接节点。

1. 创建节点与节点束

from pyted import Node, CircuitNode, Vdd, Vss # 导入 Node 工厂函数和 CircuitNode 类

# --- 创建单个节点 ---
# 自动命名 (如 n0_, n1_, ...)
auto_named_node = Node()
print(f"自动命名节点: {auto_named_node}")

# 用户指定名称
clk = Node(name="clk")
reset_n = Node(name="reset_n")
print(f"用户命名节点: {clk}, {reset_n}")

# --- 创建节点束 (Bus) ---
# 创建一个 8 位总线，索引 0 到 7，自动基础名称
bus_auto_base = Node(8) # 等价于 Node(0, 7)
print(f"8位总线 (自动基名): {bus_auto_base}, 长度: {len(bus_auto_base)}")
print(f"  -> 第一个元素: {bus_auto_base[0].name()}") # 例如 n2<0>_

# 创建一个 4 位总线，索引 3 到 0 (MSB 在前)，指定基础名称 "addr"
addr_bus = Node(3, 0, name="addr")
print(f"4位地址总线 (MSB在前): {addr_bus}, 长度: {len(addr_bus)}")
print(f"  -> addr<3>: {addr_bus[3].name()}") # addr<3>_
print(f"  -> addr<0>: {addr_bus[0].name()}") # addr<0>_

# 创建一个索引范围 [15, 8] 的总线
high_byte = Node(15, 8, name="data_high")
print(f"高位字节总线: {high_byte}, 长度: {len(high_byte)}") # 长度为 8
print(f"  -> data_high<15>: {high_byte[15].name()}")

2. 访问节点束元素（索引与切片）

# 创建一个 8 位数据总线 data<7:0>
data_bus = Node(7, 0, name="data")

# 1. 索引访问单个位
lsb = data_bus[0]      # 获取 data<0>
msb = data_bus[7]      # 获取 data<7>
bit4 = data_bus[4]     # 获取 data<4>
# 使用负数索引
second_lsb = data_bus[-2] # 获取 data<1> (倒数第二个)

print(f"LSB: {lsb}")
print(f"MSB: {msb}")
print(f"Bit 4: {bit4}")
print(f"Second LSB: {second_lsb}")

# 2. 切片获取子总线 (返回新的 CircuitNode 束)
# 使用 Python 标准切片 [start:stop:step] (stop不包含)
low_nibble = data_bus[3:0:-1] # 获取 data<3:0>，注意反向索引顺序
# 或者更直观地写成：
# low_nibble = data_bus[0:4] # 获取 data<0> 到 data<3> (顺序 0, 1, 2, 3)
# 但由于原始定义是 7 到 0，按原始索引切片更符合习惯：
low_nibble_rev = data_bus[3::-1] # data<3>, data<2>, data<1>, data<0>

middle_bits = data_bus[5:1]    # 获取 data<5> 到 data<2> (顺序 5, 4, 3, 2)
even_bits = data_bus[6:0:2]    # 获取 data<6>, data<4>, data<2>, data<0>

print(f"Low Nibble (3 downto 0): {low_nibble_rev}")
print(f"Middle Bits (5 downto 2): {middle_bits}")
print(f"Even Bits (6 downto 0, step 2): {even_bits}")
print(f"Slice length: {len(middle_bits)}") # Output: 4

# 3. 使用调用语法切片 (start, stop, step=None) (start, stop 均包含)
high_nibble_call = data_bus(7, 4) # 获取 data<7> 到 data<4>
print(f"High Nibble (call syntax): {high_nibble_call}")

3. 迭代与重命名

control_bus = Node(2, 0, name="ctrl") # ctrl<2>, ctrl<1>, ctrl<0>

# 1. 迭代总线中的每个节点
print("Iterating and renaming:")
for i, node_bit in enumerate(control_bus):
    new_name = f"control_signal_{i}"
    node_bit.rename(new_name) # 重命名单个位
    print(f"  Original Index (relative): {i}, New Name: {node_bit.name()}")

# 2. 重命名整个总线的基础名称
control_bus.rename("config")
print(f"Renamed bus element example: {control_bus[1].name()}") # config_signal_1 (之前已重命名位)

# 3. 单独重命名某个位
control_bus[0].rename("config_enable")
print(f"Final names: {[n.name() for n in control_bus]}")
# Output: ['control_signal_0', 'control_signal_1', 'config_enable'] (迭代顺序 2, 1, 0)

4. 连接节点

节点是电路连接的核心。

from ted_device import Inv, Mux2 # 假设有这些模块

# 创建节点
a = Node(name="a")
b = Node(name="b")
c = Node(name="c")
sel = Node(name="sel")
z = Node(name="z")

# 1. 节点直接连接
a >> b # a 的输出连接到 b 的输入 (逻辑上等同于同一个网络)

# 2. 节点连接到实例
inv1 = Inv()
b >> inv1 >> z # b 连接到反相器输入，反相器输出连接到 z

# 3. 节点连接到控制端口
mux = Mux2()
# 假设 Mux2 输入顺序是 in0, in1，输出是 out，控制是 sel
(a, c) >> mux % sel >> z # a, c 连接到数据输入，sel 连接到控制输入，mux 输出到 z

# 4. 使用 .to() 方法连接 (等效于 >>)
a.to(b) # a 连接到 b

7. 实用方法

bus = Node(7, 0, name="mybus") # 8 位总线

# 1. 展平 (Flatten)
flat_nodes: List[CircuitNode] = bus.flatten()

# 2. 分块 (Chunks)

chunks = bus.chunks(chunk_size=2)
print(f"Number of chunks: {len(chunks)}") # 4
print(f"First chunk (bits 7, 6): {chunks[0]}") # CircuitNode(mybus<7>, mybus<6>)
print(f"Second chunk (bits 5, 4): {chunks[1]}")# CircuitNode(mybus<5>, mybus<4>)

8. 节点比较

使用 == 可以判断两个 CircuitNode 对象是否代表同一个电气网络。

a = Node(name="a")
b = Node(name="b")
c = Node(name="c")
a_copy = a # Python 变量指向同一个对象

print(f"a == b? {a == b}") # False (未连接)
print(f"a == a_copy? {a == a_copy}") # True (同一对象)

# 连接 a 和 b
a >> b

print(f"a == b (after connect)? {a == b}") # True (现在是同一个网络)
print(f"a == c? {a == c}") # False

这些示例涵盖了 Node 和 CircuitNode 的主要功能。正确理解和运用这些功能是使用 TED 构建和连接电路的基础。

7. 总结

Node 和 CircuitNode 是 TED 中描述电路连接的基础。无论是单个信号线还是多位总线，都可以通过 Node() 函数方便地创建和命名。CircuitNode 类提供了丰富的 API 用于索引、切片、迭代、连接和管理这些节点，是构建和操作电路拓扑的核心工具。理解单节点与节点束的区别以及它们与 Instance 和 Module 的交互方式，对于使用 TED 进行电路设计至关重要。
