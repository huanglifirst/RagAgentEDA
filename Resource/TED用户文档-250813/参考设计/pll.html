PLL设计

本教程将通过分析一个PLL（锁相环）电路的设计代码，详细介绍如何使用TED框架进行模块化电路设计。我们将遵循从底到顶的顺序，解释各个子模块的实现，并最终构建完整的PLL系统。

1. 模块层次化结构

在深入代码之前，我们先了解一下PLL主要模块的构建关系。一个典型的PLL通常包含以下核心模块，它们之间存在层次化的调用关系：

2. 底层/通用单元详解

这些是构成更复杂模块的基础单元，通常是标准单元或常用的小电路。

2.1. 反相器 (INVD*)

反相器是最基本的逻辑门。代码中定义了不同驱动能力的反相器（如 INVD0, INVD1, INVD2, INVD4, INVD8），数字后缀通常代表其相对驱动强度。

  • 代码示例 (circuits/pll/PLL_jjq/src/generator/BIAS_gmid.py):
    @module
def INVD0(I: Input, ZN: Input):

    ZN >> NMOS(l=6e-08, w=1.95e-07, m=1, model="n12_lp") % (I) >> Vss()  # MU1_M_u2
    ZN >> PMOS(l=6e-08, w=2.6e-07, m=1, model="p12_lp") % (I) >> Vdd()  # MU1_M_u3
  • 解释:
    • @module 装饰器：将一个Python函数定义为一个电路模块。
    • I: Input, ZN: Input: 定义模块的输入端口 I 和输出端口 ZN。TED使用类型提示来区分端口方向。
    • NMOS(...), PMOS(...): 实例化NMOS和PMOS晶体管。参数 l, w, m 分别代表晶体管的沟道长度、宽度和倍数（multiplicity）。model 指定了PDK中定义的晶体管模型。
    • >>: 用于连接电路节点。例如 ZN >> NMOS(...) 表示输出端口ZN连接到NMOS的漏极（Drain）。
    • %: 用于连接晶体管的栅极（Gate）。例如 NMOS(...) % (I) 表示输入端口I连接到NMOS的栅极。
    • Vss(), Vdd(): 代表电路的全局地和电源节点。

2.2. 时钟缓冲器 (CKND*)

用于缓冲时钟信号，提高驱动能力。

  • 代码示例 (circuits/pll/PLL_jjq/src/generator/BIAS_gmid.py):
    @module
def CKND2(I: Input, ZN: Input):

    (
        ZN * 2
        >> PMOS(l=6e-08, w=5.2e-07, m=1, model="p12_lp") * 2 % (I * 2)
        >> Vdd() * 2
    )  # complete_parallel size: 2
    (
        ZN * 2
        >> NMOS(l=6e-08, w=3.1e-07, m=1, model="n12_lp") * 2 % (I * 2)
        >> Vss() * 2
    )  # complete_parallel size: 2
  • 解释:
    • ZN * 2, I * 2, Vdd() * 2, Vss() * 2: TED允许使用乘法操作符来表示并联的多个相同单元或连接。这里表示 CKND2 内部由两个并联的反相器构成，以提供2倍驱动能力。
    • PMOS(...) * 2, NMOS(...) * 2: 同样表示并联实例化两个相同的晶体管。

2.3. 与非门 (ND*D*)

实现与非逻辑功能。数字后缀代表输入端口数和驱动强度。

  • 代码示例 (circuits/pll/PLL_jjq/src/generator/BIAS_gmid.py):
    @module
def ND2D2(A1: Input, A2: Input, ZN: Input):
    XU3_0_net6, XU3_1_net6 = Node(2)
    (
        ZN >> NMOS(l=6e-08, w=3.9e-07, m=1, model="n12_lp") % (A1) >> XU3_0_net6
    )  # MU3_0_M_u3
    (
        XU3_0_net6 >> NMOS(l=6e-08, w=3.9e-07, m=1, model="n12_lp") % (A2) >> Vss()
    )  # MU3_0_M_u4
    (
        ZN >> NMOS(l=6e-08, w=3.9e-07, m=1, model="n12_lp") % (A1) >> XU3_1_net6
    )  # MU3_1_M_u3
    (
        XU3_1_net6 >> NMOS(l=6e-08, w=3.9e-07, m=1, model="n12_lp") % (A2) >> Vss()
    )  # MU3_1_M_u4
    ZN >> PMOS(l=6e-08, w=5.2e-07, m=1, model="p12_lp") % (A2) >> Vdd()  # MU3_0_M_u2
    ZN >> PMOS(l=6e-08, w=5.2e-07, m=1, model="p12_lp") % (A1) >> Vdd()  # MU3_0_M_u1
    ZN >> PMOS(l=6e-08, w=5.2e-07, m=1, model="p12_lp") % (A2) >> Vdd()  # MU3_1_M_u2
    ZN >> PMOS(l=6e-08, w=5.2e-07, m=1, model="p12_lp") % (A1) >> Vdd()  # MU3_1_M_u1
  • 解释:
    • Node(2): 创建两个内部节点。
    • 该模块实现了两路并联的二输入与非门（每个输入驱动一个PMOS和一个NMOS，两个NMOS串联，两个PMOS并联）。

2.4. 触发器 (mipi_sdfr_HRMS_SPLL, SPLL_DLATCH*, SPLL_LOCK_DET_DFF)

用于存储状态或分频。代码中使用了多种类型的触发器，包括主从触发器 (mipi_sdfr_HRMS_SPLL) 和锁存器 (SPLL_DLATCH*)。

  • 代码示例 (circuits/pll/PLL_jjq/src/generator/TEST_CK.py):
    @module
def mipi_sdfr_HRMS_SPLL(
    d: Input, rb: Input, dvss: Input, dvdd: Input, q: Input, ck: Input, qb: Input
):
    # ... 内部晶体管和电容连接 ...
    (IQN, q) >> CKND2()
    (IQN, q) >> CKND2()
    (ck, C1) >> CKND2()
    (ck, C1) >> CKND2()
    (q, qb) >> CKND2()
    (q, qb) >> CKND2()
    (C1, C2) >> CKND2()
    (C1, C2) >> CKND2()
  • 解释:
    • mipi_sdfr_HRMS_SPLL 是一个带有复位 (rb) 功能的主从触发器。
    • 内部包含了复杂的晶体管级实现，并使用了时钟缓冲器 (CKND2) 来驱动内部节点和输出。
    • dvss, dvdd: 可能是局部的地和电源，用于特定部分的供电。
    • C(...): 实例化电容，用于模拟寄生电容或实现特定功能。

2.5. 电阻/电容

代码中也定义了电阻 (res_*) 和电容 (mom*, n50var) 单元。

  • 代码示例 (circuits/pll/PLL_jjq/tests/test_CP_OP2_generator.py):
    @module
def res_R1(MINUS: Input, PLUS: Input, segL=6e-06, segW=2e-06, model="res_pposab"):
    (
        MINUS
        >> (r1 := RpolySingleFromVirtuoso3T(l=segL, w=segW, model=model, contact=2))
        >> PLUS
    )
    place(via_min_number=1)
    route()
  • 代码示例 (circuits/pll/PLL_jjq/src/generator/LPF1.py):
    @module
def C_n50var_mom14_3t(
    a: Input,
    b: Output,
):
    # ... (根据c2值计算momcap和moscap的尺寸) ...
    (a >> (C1_1 := n50var(l=c1_size[0], w=c1_size[1], connectGates="Top")) >> b)
    (b >> (C1_2 := mom14_3t(finger_l=c0_size[0], nf=c0_size[1])) >> a)
    # ... (版图处理) ...
  • 解释:
    • RpolySingleFromVirtuoso3T: 可能是从Virtuoso导入的多晶硅电阻单元。
    • mom14_3t, n50var: 分别代表MOM (Metal-Oxide-Metal) 电容和基于NMOS的MOS电容。
    • lookup_*_size: 代码中使用了查找表函数（例如 lookup_resistance_size, lookup_momcap_size, lookup_moscap_size）来根据目标值（电阻或电容）反查所需的器件尺寸（L, W, nf 等），这是电路生成中的常见技巧。
    • place() 和 route(): 表明这些模块也包含了版图布局和布线信息。

3. 子模块详解

3.1. BIAS (偏置电路)

BIAS 模块负责产生电路所需的各种偏置电流。

  • 代码路径: circuits/pll/PLL_jjq/src/generator/BIAS_gmid.py
  • 代码片段 (BIAS module):
    @module
def BIAS(
    ib2p6u: Input,
    pd_bias: Input,
    pd_5v: Input, # 可能是外部5V电源域的使能信号
    ib5u_cp_op1: Output,
    ib10u_cp: Output,
    ib10u_cp_op2: Output,
    ib10u_test_buf: Output,
    ib20u_vco: Output,
):
    # ... (内部节点定义) ...
    (pd_bias, pdb) >> (inv1 := INVD0())
    (pdb, pda) >> (inv2 := INVD0())
    # ... (gmid 设计计算 W1, W2) ...
    net78 >> (nm8 := NMOS(l=L2 * u, w=W2 * u, m=1)) % Vss() >> Vss()
    # ... (大量 NMOS 和 PMOS 电流镜结构) ...
    pcm1 = CurrentMirror(...) # 实例化电流镜基类
    pcm1.input(net023, VB=net01)
    pcm1.output(ib10u_cp, m=[4, 4])
    pcm1.output(ib5u_cp_op1, m=[2, 2])
    pcm1.output(ib10u_cp_op2, m=[4, 4])
    pcm1.output(ib20u_vco, m=[8, 8])
    pcm1.output(ib10u_test_buf, m=[4, 4])
    # ... (版图布局与布线) ...
    place(via_min_number=1)
    route(pattern_max_width=300, top_layer="M3", via=2)
  • 解释:
    • 输入 ib2p6u 是一个参考电流 (2.6uA)。
    • pd_bias, pd_5v 是使能或掉电信号。
    • 内部使用了大量的NMOS和PMOS晶体管构成了复杂的电流镜结构，将参考电流 ib2p6u 镜像并按比例放大，产生多个不同的偏置电流输出。
    • 设计技巧:
      • Gm/Id 设计: 代码注释和变量命名（如 gmid1, W1, L1）表明可能采用了 Gm/Id 设计方法来确定晶体管尺寸，以达到特定的性能指标（如增益、带宽、功耗）。Gm/Id 设计法允许在不同的偏置电流和晶体管尺寸下，维持相似的性能特性。
      • Cascode 电流镜: 为了提高输出电阻和精度，实际电路中可能使用了Cascode结构的电流镜（代码中未完全展示细节，但复杂连接暗示了这一点）。
      • CurrentMirror 类: 可能使用了TED提供的 CurrentMirror 基类来简化多路输出电流镜的构建和版图生成。

3.6. 分频器 (DIV)

PLL中包含多种分频器：

  • SPLL_PRE_DIV: 对输入的参考时钟 CKIN 进行预分频（/1, /2, /4, /8），由 PRE_DIV_SEL 控制。
  • SPLL_LOOP_DIV: 环路内的反馈分频器，对VCO输出进行分频（8-31），分频比由 LOOP_DIV_CTRL 和 LOOP_DIV_SEL 控制。
  • SPLL_POST_DIV: 对VCO输出或环路分频器输出进行后分频（/1, /2, /4, /8, /16），产生最终的系统时钟 CKO_SYS，由 POST_X1_CTRAL 和 POST_DIV_SEL_SYS 控制。

这些分频器通常由级联的触发器（如 mipi_sdfr_HRMS_SPLL）和逻辑门（如 SPLL_POST_DIV_2_3_B_ls_2, SPLL_POST_DIV_2_3_B_2 用于实现 /2 和 /3 分频）以及多路选择器（如 SPLL_MX4）构成。

    • circuits/pll/PLL_jjq/src/generator/PRE_DIV.py
    • circuits/pll/PLL_jjq/src/generator/LOOP_DIV.py
    • circuits/pll/PLL_jjq/src/generator/POST_DIV.py
  • 设计技巧:
    • 可编程分频: 通过控制信号选择不同的分频比，使PLL能够工作在不同的频率。
    • 高速触发器: 分频器（尤其是预分频和环路分频）需要在VCO的高频下工作，因此使用了高速触发器 (mipi_sdfr_HRMS_SPLL)。
    • 同步/异步复位: 触发器带有复位端 (RST, RSTB)，用于PLL的初始化或锁定过程。

5. 测试与性能分析框架

TED提供了一套测试框架，用于仿真和验证电路性能。

5.1. Tester 类

  • 每个子模块（如CP, PFD, VCO, DIV）通常都有一个对应的 Tester 类（例如 CPTester, VCOTester），继承自 TesterBase。

  • Tester 类负责：

    • 初始化: 设置仿真所需的参数（如电源电压 vdd, 输入频率 f_ckin, 目标电流 i_ideal, 控制电压 vctrl 等）。
    • 构建测试平台 (testbench 方法):
      • 实例化待测模块 (self.dut())。
      • 连接电源 (VSource)、时钟源 (Clock)、电流源 (ISource)、探针 (IProbe) 和负载（如电容 C）。
      • 设置仿真命令（如 tran, stb, pss, pnoise, dcsweep）及其参数。
      • 关键: 使用 >> 和 % 连接测试平台中的器件和DUT。
    • 运行仿真: 调用仿真器执行。
    • 关联性能分析类 (performance 方法): 返回一个对应的 Performance 类实例。

  • 代码示例 (circuits/pll/PLL_jjq/tests/testers/CP_testers/charge_tester.py):

    class ChargeTester(TesterBase):

5.2. Performance 类

  • 每个 Tester 类通常对应一个 Performance 类（例如 CPChargePerformance, VCOpnPerformance），继承自 Performance。

  • Performance 类负责：

    • 提取性能指标 (performance 方法): 从仿真结果 (self.raw) 中计算或提取所需的性能指标（如增益 gain, 相位裕度 pm, 带宽 gbw, 相位噪声 phase_noise, 分频比 divider, 功耗 pw 等）。
    • 数据后处理: 可能包含复杂的信号处理（如查找过零点、计算FFT、积分等）。TED的 T 模块（可能是 torch 的封装）提供了这些功能。
    • 绘图 (plot 方法): 使用 watch 函数指定需要绘制的波形，并调用 self.raw.show() 生成波形图。

  • 代码示例 (circuits/pll/PLL_jjq/tests/performance/CP_performance/op1_stb.py):

5.3. 顶层 Tester (CPTester, VCOTester, PFD_CP_Tester 等)

  • 这些类通常继承自 TesterBase，用于组织对一个完整子模块（如CP、VCO）的多种测试。

  • 在 __init__ 或 build 方法中，它们会实例化多个具体的 Tester（如 ChargeTester, DischargeTester, PhaseNoiseTester）。

  • 并行仿真: 使用 get_circuit().parallel.start() 和 get_circuit().parallel.stop() 来并行运行多个独立的测试，加快仿真速度。

  • 结果汇总与报告: 从各个子 Tester 的结果中提取关键性能指标，并使用 PrettyTable 生成格式化的性能汇总表。

  • 绘图: 调用各个子 Tester 的 plot 方法生成所有相关的波形图。

6. Generator 类

代码中广泛使用了继承自 Generator (或特定子类如 OpampGenerator, VcoGenerator) 的类来定义电路模块。这使得模块具有更强的复用性和自动化流程能力。

  • 特点:

    • core 方法: 定义电路的核心连接（晶体管级网表）。
    • wrapper 方法: 定义模块的标准测试接口或更高层级的连接方式。
    • test 方法: 提供了一个标准的测试入口，通常会调用对应的顶层 Tester 类。
    • report 方法: 用于生成模块的性能报告。
    • 版图流程集成: Generator 类通常包含 drc(), lvs(), pex() 等方法，用于执行设计规则检查、版图网表对比和寄生参数提取。
    • 前后仿流程: test 方法通常接受 stage 参数 ('pre' 或 'post') 来区分前仿（基于原理图）和后仿（基于提取的网表）。setenv(stage=...) 用于设置仿真阶段。

7. 关键设计技巧总结

  • 模块化与层次化: 将复杂电路分解为可管理、可复用的子模块。
  • 参数化设计: 使用 Variable 或模块参数使设计可配置、可优化。
  • Gm/Id 设计法: 一种系统性的模拟电路设计方法，用于平衡性能指标。
  • 查找表 (LUT): 用于根据性能目标反查器件尺寸，加速设计迭代。
  • 自动化测试框架: 使用 TesterBase 和 Performance 类实现标准化的仿真验证流程。
  • 并行仿真: 利用 get_circuit().parallel 加快多项测试的速度。
  • 版图自动化: 使用 place, route, row, col, guardring, smartbus 等API生成版图。
  • 前后仿流程: 通过 stage 控制和 pex() 方法支持从原理图到版图的完整验证流程。
  • Generator 类: 将电路定义、测试、报告和版图流程封装在一起，实现高度自动化。

QA (常见问题解答)

Q1: TED框架是什么？看起来像Python？ A1: 从代码结构看，TED很可能是一个基于Python的电路设计自动化（EDA）框架。它允许设计者使用Python语法来描述电路连接（网表）、定义模块、设置仿真、提取性能并生成版图。它似乎封装了底层的PDK信息、仿真器接口（如Spectre）和版图工具接口。

Q2: @module 装饰器有什么作用？ A2: @module 装饰器用于将一个Python函数声明为一个电路模块。TED框架会解析这个函数的定义，包括其输入输出端口（通过类型提示 Input/Output）和内部的器件实例化及连接，从而在内部构建出电路的表示。

Q3: >> 和 % 操作符是做什么的？ A3: 这看起来是TED框架自定义的操作符，用于简化电路连接的描述。 * >> 可能表示信号流向或节点连接，通常用于连接器件的漏极/源极或模块端口。例如 ZN >> NMOS(...) 或 Clock(...) >> up。 * % 可能专门用于连接晶体管的栅极。例如 NMOS(...) % (I)。

Q4: 为什么有很多 *_layout.py 和 *_generator.py 文件？ A4: 这体现了设计流程的分离： * *_layout.py 文件通常包含模块的 手动或半自动版图实现。它们可能直接调用底层的版图单元（如 layout_mos, row, col）进行布局布线。 * *_generator.py 文件通常包含模块的 生成器 (Generator) 类 定义。这些类封装了电路的原理图 (core 方法)、测试接口 (wrapper 方法)、标准测试流程 (test 方法) 以及可能的版图生成逻辑（可能调用 *_layout.py 中的模块或直接生成）。这种方式更利于自动化和参数化设计。Generator 类通常也集成了运行DRC/LVS/PEX的功能。

Q5: Gm/Id设计方法在代码中是如何体现的？ A5: 虽然代码中没有直接展示完整的Gm/Id计算流程，但可以从以下几点推断其应用： * 查找表 (LUT): BIAS_gmid.py 和 VCO_gmid.py 中使用了 LUT_DB().lookup_Vgs(...) 和 LUT_DB().lookupfz(...)。这表明设计者预先通过仿真生成了晶体管特性（如 GMOVERID, ID/W）随偏置电压变化的查找表。设计时，根据目标 GMOVERID（对应特定的性能，如增益带宽积）和偏置电流 I，可以反查出所需的 VGS，进而查找 ID/W（电流密度），最终计算出所需的晶体管宽度 W = I / (ID/W)。 * 参数化: 核心模块（如VCO Ring, Bias电流镜）的尺寸（L, W）被设计为可配置的，这使得可以通过调整Gm/Id相关参数来优化电路。

Q6: 如何进行仿真和性能提取？ A6: 通过 Tester 和 Performance 类： * 仿真设置: 在 Tester 的 testbench 方法中，使用 VSource, Clock, IProbe, C 等设置激励和负载，并调用仿真命令如 tran(), stb(), pss(), pnoise(), dcsweep()。 * 性能提取: 在 Performance 的 performance 方法中，通过 self.raw.get_signal("节点/器件名") 获取仿真波形数据，然后使用 T 模块（可能是 torch 封装）或 numpy, scipy 进行计算（如 T.abs(), T.db20(), T.phase(), np.diff(), simpson()）来得到性能指标。对于特定仿真（如stb），也可以直接从 self.measure 字典获取由仿真器内部测量函数计算好的结果。

Q7: 前后仿流程是如何管理的？ A7: * stage 参数: 许多 test 方法接受 stage 参数（如 'pre' 或 'post'）。 * setenv(stage=...): 通过上下文管理器设置当前仿真阶段。 * pex() 方法: Generator 类通常有 pex() 方法，用于调用寄生参数提取工具（如StarRC），生成后仿所需的网表（通常是 *.pex.netlist）。 * 条件加载: 仿真流程会根据 stage 的值来决定是包含原理图网表 (stage='pre') 还是包含提取后的网表 (stage='post') 进行仿真。后仿会考虑版图引入的寄生电阻和电容。代码中 circuits/pll/PLL_jjq/tests/test_pll_layout/ 目录下的测试文件大量使用了 with setenv(stage="post"): 和 module.pex(...) 来执行后仿真。load_lpe_params=True 可能用于加载LPE（Layout Parasitic Extraction）相关的配置文件或模型。

Q8: 这套代码看起来很复杂，如何开始学习和使用？ A8: 1. 理解基础: 先学习 INVD0, ND2D2 等简单通用单元的写法，掌握 @module, Input/Output, 器件实例化 (NMOS, PMOS, Subckt) 和连接 (>>, %) 的基本语法。 2. 学习子模块: 选择一个相对简单的子模块，如 PFD 或 LPF，理解其电路原理和对应的TED代码实现。 3. 掌握测试框架: 学习如何编写 Tester 和 Performance 类，设置 testbench，运行仿真并提取结果。可以从简单的测试（如 charge_tester.py）入手。 4. 理解Generator: 学习 Generator 类的结构，特别是 core 和 wrapper 的区别，以及 test, report, drc/lvs/pex 方法的用法。 5. 从顶向下分析: 理解 SPLL_TOP_CORE 如何实例化和连接各个子模块。 6. 实践: 尝试修改现有模块的参数或连接，观察仿真结果的变化。然后尝试基于现有单元构建一个新的简单模块。
