SAR-ADC设计

本教程将引导您了解如何使用 TED 框架来设计一个逐次逼近型 ADC (SAR ADC)。我们将按照从底到顶的顺序，逐步解释各个模块的实现，并结合代码进行说明，指出关键的设计技巧和 TED 框架的使用方法。

1. 基础模拟模块

电路设计通常从基础单元开始。在这个 SAR ADC 设计中，基础模拟模块包括开关、比较器和电容阵列的核心单元。

1.1 开关 (Switch)

SAR ADC 需要开关来对输入信号进行采样，并在 DAC 中切换电容。

  • 文件: tedhub/circuit/saradc/src/Diff_Sw.py

  • 功能: 该文件定义了差分开关模块。TED 允许我们方便地定义和选择不同的开关实现。

    • CMOS_SWitch: 使用 ted_device 中的 NMOS 和 PMOS 构建了一个基本的 CMOS 传输门。参数 w 用于控制晶体管宽度。
    • Diff_Sw1: 使用 circuits.ancells 中的 Passgate （可能是预定义的传输门版图单元）构建差分开关。它还展示了如何通过 palign 和 nalign 控制版图细节。
    • Diff_Sw2: 使用上面定义的 CMOS_SWitch 构建差分开关。
    • Boostrap: 使用了 tedhub.circuit.bootstrapped_switch 中的 bit12_sw6，这是一个更高级的自举开关，通常用于高性能采样。
    • Diff_Sw (函数): 这是一个工厂函数，根据传入的 model 参数 ("1", "2", 或 "3") 返回不同实现的差分开关模块。这是 TED 中实现设计灵活性的常用技巧。

  • 代码示例 (Diff_Sw1):

    # tedhub/circuit/saradc/src/Diff_Sw.py
@module
def Diff_Sw1(
    VIP: Input, VIN: Input, VOP: Output, VON: Output, CLK: Ctrl, CLKB: Ctrl
):
    from circuits.ancells import Passgate # 导入预定义的 Passgate 单元

    # 实例化 Passgate 并连接
    (
        VIP
        >> (
            sw1 := Passgate( # 使用 Passgate 作为开关
                m=6, lp=0.06, ln=0.06, wp=w, wn=w, palign=BOTTOM, nalign=TOP
            )
            % (CLK, CLKB) # 使用 % 连接控制信号
        )
        >> VOP # 使用 >> 连接数据信号
    )
    (
        VIN
        >> (
            sw2 := Passgate(
                m=6, lp=0.06, ln=0.06, wp=w, wn=w, palign=BOTTOM, nalign=TOP
            )
            % (CLK, CLKB)
        )
        >> VON
    )
    # 版图布局提示
    row(sw1, sw2) # 将两个开关水平排列
    place() # 放置单元
    route() # 进行布线

  • TED 设计技巧:

    • 使用 @module 装饰器定义可重用模块。
    • 使用 Input, Output, Ctrl 定义端口类型。
    • 通过 >> 连接数据路径，通过 % 连接控制信号。
    • 通过参数 (w, model) 实现模块的灵活性和可配置性。
    • 使用 row, col, place, route 等进行版图布局和布线描述。
    • 导入并使用预定义的库单元 (circuits.ancells.Passgate, tedhub.circuit.bootstrapped_switch.bit12_sw6)。

1.2 比较器 (Comparator)

比较器是 SAR ADC 的核心，用于比较输入信号和 DAC 输出。

  • 文件:

    • tedhub/circuit/saradc/src/Cmp.py (基于晶体管的原理图级 Double-Tail 比较器)
    • tedhub/circuit/saradc/src/saradc_cmp.py (考虑版图的 StrongArm 结构比较器)

  • 功能:

    • DT_CMP: 实现了一个双尾比较器 (Double-Tail Comparator) 的原理图。它直接使用 NMOS 和 PMOS 晶体管以及标准单元 (Buff, Inv) 搭建。这是快速进行功能验证的设计方式。
    • SAR_CMP2_AMP, SAR_CMP2_LATCH: 将 StrongArm 比较器分解为预放大级和锁存级，便于单独设计和优化。它们使用了 PMOS_ABBA_CI_VDD, NMOS_ABBA_CO 等可能是预定义的、具有特定版图结构（如 ABBA 共质心）的单元，以提高匹配性能。还使用了 layout_mos, guardring 等版图生成相关的函数。
    • SAR_CMP2: 整合了 SAR_CMP2_AMP 和 SAR_CMP2_LATCH，并添加了时钟反相器和输出缓冲器。它还通过 standard_cell, col, add_vbus 等定义了模块的整体版图结构。

  • 代码示例 (SAR_CMP2):

    # tedhub/circuit/saradc/src/saradc_cmp.py
@module
def SAR_CMP2(
    CLK: Ctrl,
    VIP: Input,
    VIN: Input,
    CMPO: Output,
    CMPOB: Output,
):
    # 生成内部时钟信号
    PH1BX = ~(inv1 := Inv(CLK, size=2))
    PH1X = ~(buff1 := Buff(CLK, size=2))
    Q, QB = Node(2) # 定义内部节点
    # 输出缓冲
    Q >> (buff2 := Buff(size=1)) >> (buff3 := Buff(size=4)) >> CMPO
    QB >> (buff4 := Buff(size=1)) >> (buff5 := Buff(size=4)) >> CMPOB
    VOP = Node()
    VON = Node()
    # 串联实例化放大级和锁存级
    Serial(
        (VIP, VIN),
        amp := SAR_CMP2_AMP() % PH1X, # 实例化放大级
        (VOP, VON),
        latch := SAR_CMP2_LATCH() % PH1BX, # 实例化锁存级
        (Q, QB),
    )
    # --- 版图定义 ---
    # 将标准单元组合
    std = standard_cell(((inv1, buff1, buff2, buff3, buff4, buff5),))
    # 添加 M2 水平总线
    std = std.add_hbus(PH1X, PH1BX, layer=M2, vias=1, location=BOTTOM)
    # 垂直排列标准单元、锁存器、放大器
    final = col(std, latch, amp, align=CENTER, spacing=M3.get_min_spacing() + 100)
    # 添加电源和时钟垂直总线
    final = final.add_vbus(Vdd(), layer=M1, vias=3, location=LEFT)
    final = final.add_vbus(Vss(), layer=M1, vias=3, location=RIGHT)
    final = final.add_vbus(PH1X, PH1BX, layer=M1, vias=1, location=LEFT, spacing=500)
    place() # 放置
    route() # 布线

  • TED 设计技巧:

    • 模块化设计：将复杂比较器分解为放大和锁存阶段 (SAR_CMP2_AMP, SAR_CMP2_LATCH)。
    • 版图感知设计：使用 layout_mos, guardring, standard_cell, col, row, add_vbus, add_hbus 等直接在代码中描述版图结构，有利于性能优化和匹配。
    • 使用 Serial 简化串行连接描述。
    • 使用 ~ 操作符获取反相信号或模块输出。
    • 使用 := 在实例化时同时赋值给变量。

1.3 电容单元 (Capacitor Unit)

  • 文件:

    • tedhub/circuit/saradc/src/CapUnit.py (原理图级)
    • ted_device.Capacitor (基础库)
    • ted_device.SARCapGrid (版图级阵列)

  • 功能: 定义 SAR ADC 中的单位电容。

    • CapUnit: 一个简单的原理图级模块，内部只有一个 ted_device.Capacitor。
    • 实际版图设计中 (saradc_cap.py) 使用了 ted_device.SARCapGrid，这可能是一个专门用于生成电容阵列版图的类。

  • 代码示例 (CapUnit):

    # tedhub/circuit/saradc/src/CapUnit.py
@module
def CapUnit(CTop: Input, CBottom: Output):
    # 实例化一个 30fF 的电容
    (CTop, CBottom) >> Capacitor(c=30e-15)

2. 电容数模转换器 (Capacitor DAC)

电容 DAC 是 SAR ADC 的另一个关键部分，它根据数字码产生模拟电压。

  • 文件:

    • tedhub/circuit/saradc/src/CapArray.py (原理图级阵列)
    • tedhub/circuit/saradc/src/CapTop.py (原理图级带开关的 DAC)
    • tedhub/circuit/saradc/src/saradc_cap.py (版图级阵列和 DAC)

  • 功能:

    • CapArray: 定义了电容阵列的原理图。它通过循环实例化 CapUnit 来构建不同权重的电容分支。model 参数用于选择是否包含用于匹配的虚拟接地电容。采用了 Vcm-Based 开关策略，终端电容 (Cterminal) 接到 VCMI。
    • CapBottom: 定义了电容阵列下极板的开关逻辑（原理图级），包括连接到 VCMI 的开关 (Passgate) 和连接到 VREFP/VREFN 的开关 (PN pair)。
    • CapTop: 整合了 CapArray 和 CapBottom，形成完整的原理图级电容 DAC。
    • SAR_CAP_ARRAY: 使用 ted_device.SARCapGrid 生成电容阵列的版图。它接受一个列表定义各电容分支的权重 [1, *[2**k for k in range(N - 1)], 2**(N - 2)]。最后一个元素 2**(N-2) 可能对应虚拟电容。
    • SAR_CAP_TOP_func: 定义了版图级电容 DAC 的核心逻辑和连接关系，包括实例化 PassGate 和 PNPair 作为开关，并连接到 SAR_CAP_ARRAY 实例。
    • SAR_CAP_TOP: 将 SAR_CAP_TOP_func 返回的各个版图单元（开关、电容阵列、总线）组合成最终的版图。使用了 row, col, rotate, move_by, hbus, vbus 等进行复杂的版图布局。

  • 代码示例 (CapArray1):

    # tedhub/circuit/saradc/src/CapArray.py
@module
def CapArray1(
    CSP: Input(N - 1), CSN: Input(N - 1), VCMI: Input, VP: Output, VN: Output
):
    # --- CapArray2 的内容 (省略) ---
    # (CSP, CSN, VCMI) >> CapArray2() >> (VP, VN) # 假设 CapArray2 在内部定义或导入

    # --- 接地虚拟电容 ---
    tem1, tem2 = Node(2)
    # VP 通过 CapUnit 连接到 tem1
    VP * (2 ** (N - 2)) >> CapUnit() * (2 ** (N - 2)) >> tem1 * (2 ** (N - 2))
    # tem1 通过 3fF 电容连接到 Vss
    (tem1 * (2 ** (N - 2)), Vss() * (2 ** (N - 2))) >> Capacitor(c=3e-15) * (
        2 ** (N - 2)
    )
    # VN 部分类似
    VN * (2 ** (N - 2)) >> CapUnit() * (2 ** (N - 2)) >> tem2 * (2 ** (N - 2))
    (tem2 * (2 ** (N - 2)), Vss() * (2 ** (N - 2))) >> Capacitor(c=3e-15) * (
        2 ** (N - 2)
    )

  • TED 设计技巧:

    • 参数化设计：DAC 的位数 N 作为参数传入模块。
    • 原理图与版图分离：先用 CapArray/CapTop 进行原理图设计和仿真，再用 SAR_CAP_ARRAY/SAR_CAP_TOP 进行版图设计。
    • 版图生成：大量使用 row, col, rotate, hbus, vbus 等进行手动或半自动的版图布局布线。
    • 循环和列表推导：用于生成重复性的结构，如电容分支和开关 (for i in range(...), *[...])。
    • 使用 * 操作符进行多重实例化和连接。

3. 数字控制逻辑

数字逻辑负责产生 SAR 转换所需的时序和控制信号。

  • 文件:

    • tedhub/circuit/saradc/src/Digital.py
    • tedhub/circuit/saradc/src/saradc_digital.py (可能包含更具体的版图实现)

  • 功能:

    • SAR_CKDIV: N 倍分频器，产生比较器时钟 CLK_CMP 和低速的采样结束信号 CLK_DIV。使用 D 触发器 (DFNCN) 和逻辑门 (And2, Buff) 实现。
    • SAR_CTRL: 单比特的 SAR 控制逻辑单元。接收时钟、比较器输出、上一位的输出 (DIN) 和复位信号，产生当前位的数字输出 (DOUT)、DAC 控制信号 (Q, QB, SWN, SWNB, SWP, SWPB)。同样使用 DFNCN, Buff, Nor2, Inv 等标准单元实现。
    • ClockTree, ClockTree_N, ClockTree_P: 用于缓冲和分配采样时钟 (CLK_SAMP, CLKB_SAMP)。_N 和 _P 版本可能是为差分信号路径设计的特定缓冲树，使用了标准单元 CKB4 并进行了版图布局。
    • Ideal_CMP_DIV: 为多核仿真 (tds) 生成理想的时钟信号，避免时序问题。
    • SARADC_DIGITAL_TOP: 顶层数字控制器。它实例化 SAR_CKDIV (或 Ideal_CMP_DIV) 和 N 个 SAR_CTRL 单元，并将它们连接起来形成完整的 SAR 逻辑。它还实例化 ClockTree 来生成最终的采样时钟。此模块可能使用 ted_interface.icc 调用布局布线工具自动生成标准单元的版图，并定义了端口位置约束。

  • 代码示例 (SARADC_DIGITAL_TOP):

    # tedhub/circuit/saradc/src/Digital.py
def SARADC_DIGITAL_TOP(N=8, Fclk=5e6, model="Single"):
    @module
    def _SARADC_DIGITAL_TOP(...): # 输入输出端口省略
        RSTB_INT, CLK_DIV, net05 = Node(3)

        # --- 生成时钟 (根据仿真模式选择) ---
        # (实际代码中使用了 Ideal_CMP_DIV)
        U = Ideal_CMP_DIV(Bit=N, Fclk=Fclk)
        U() >> (CLK_DIV, CLK_CMP)

        # --- 生成内部复位信号 RSTB_INT ---
        CLK_DIV >> Inv(size=1) >> net05
        (net05, RSTB) >> And2(size=1) >> Buff(size=2) >> Buff(size=8) >> RSTB_INT

        # --- 实例化 N 个 SAR_CTRL 单元 ---
        th = ~TieHigh() # 获取高电平 TieHigh() 的反相，即低电平？存疑，可能是 TieLow
        # MSB 控制单元 (DIN 接地/高?)
        (
            (CLK_CMP, CLK_DIV, CMPO, th, RSTB_INT)
            >> SAR_CTRL()
            >> (DOUT[N - 1], Q[N - 1], ..., SWPB[N - 1]) # 输出省略
        )
        # 其他位的控制单元 (DIN 接上一位的 Q)
        for i in range(N - 2, -1, -1):
            (
                (CLK_CMP, CLK_DIV, CMPO, Q[i + 1], RSTB_INT) # DIN = Q[i+1]
                >> SAR_CTRL()
                >> (DOUT[i], Q[i], ..., SWPB[i]) # 输出省略
            )

        # --- 实例化时钟树 ---
        CLK_DIV >> ClockTree() >> (CKOUT, CLKB_SAMP, CLK_SAMP)

        # --- 版图集成 (如果使用 saradc_digital.py) ---
        # icc( ... ) # 调用 ICC 进行布局布线
    return _SARADC_DIGITAL_TOP

  • TED 设计技巧:

    • 标准单元的使用：大量复用 ted_device 或 circuits.mstdcells 中定义的标准逻辑门 (Inv, Buff, And2, Nor2, DFNCN)。
    • 层次化设计：将顶层逻辑分解为时钟分频、控制单元和时钟树。
    • 参数化：数字逻辑的位数 N 作为参数。
    • 自动化布局布线：通过 ted_interface.icc (如果可用) 集成布局布线工具，实现标准单元的自动化版图生成。需要定义端口约束。
    • 仿真模式选择：使用 model 参数和 Ideal_CMP_DIV 来适应不同的仿真需求 (单核 vs 多核)。

4. 顶层 SAR ADC 集成

最后一步是将所有的模拟和数字模块组装起来。

  • 文件:

    • tedhub/circuit/saradc/src/SarAdc.py (原理图级顶层)
    • tedhub/circuit/saradc/src/saradc.py (版图级顶层)

  • 功能:

    • _SarTop: 定义了 SAR ADC 的顶层原理图连接关系。它实例化了数字顶层 (SARADC_DIGITAL_TOP)、时钟树 (ClockTree_N, ClockTree_P)、差分输入开关 (Diff_Sw)、电容 DAC (CapTop) 和比较器 (DT_CMP)，并将它们连接起来。
    • SARADC: 定义了 SAR ADC 的顶层版图集成。它实例化的是版图级的模块，如 SARADC_DIGITAL_TOP (带 icc 的版本)、ClockTree_N, ClockTree_P, PassGate (作为输入开关)、SAR_CAP_TOP_func (获取 DAC 版图组件)、SAR_CMP2。它还负责复杂的手动版图布局，包括放置各个模块、添加电源/地网络 (Vdd, Vss)、添加解耦电容 (Decap)、创建和连接各种金属层总线 (hbus, vbus)。它还提供了 place_type 参数来选择手动布局 (manual) 或尝试自动布局 (greedy)。

  • 代码示例 (SARADC 简化版图部分):

5. 仿真与验证

TED 提供了丰富的测试和验证功能。

  • 文件: tedhub/circuit/saradc/tests/ 目录下的所有 test_*.py 文件。

  • 功能: 这些文件展示了如何为每个模块和整个 ADC 编写测试平台 (testbench)。

    • 模块功能仿真: test_cmp.py, test_Diff_Sw.py, test_CapTop.py, test_SAR_CKDIV.py, test_sar_dig.py 等文件通过设置不同的输入激励 (VSource, Clock, Vsin)，运行瞬态仿真 (tran)，并观察输出波形来验证各个模块的基本功能和性能（如比较器延迟、失调，开关的 ENOB 等）。
    • 顶层 ADC 仿真: test_Sar_FFT.py, test_saradc.py 设置了完整的 ADC 仿真环境。它们使用 Vsin 产生输入信号，使用 Clock 产生时钟和复位信号，实例化顶层 ADC (SarTop 或 SARADC)，并连接一个理想 DAC (Ideal_Dac) 将数字输出转换回模拟域。然后运行瞬态仿真 (tran 或多核 tds)。
    • 性能分析: Adc_FFT 函数 (来自 tedhub.circuit.bootstrapped_switch.src.model_functions.FFT) 用于对仿真结果进行 FFT 分析，计算 SNR, SINAD, SFDR, ENOB 等 ADC 关键指标。CMP_Tester 类包含了自动测试比较器各项指标（功耗、延迟、失调、回踢噪声）的逻辑。
    • 版图验证: test_sar_cap.py, test_sar_dig.py, test_sar_cmp.py, test_saradc_lvs.py 等文件调用 .lvs() 和 .pex() 方法进行版图验证 (Layout Versus Schematic) 和寄生参数提取。test_PostSim.py 和 test_saradc.py 中的 stage="post" 展示了如何运行后仿真 (基于提取的寄生参数)。
    • 参数扫描/多角仿真: 测试文件通过定义不同的测试函数 (test1, test2, ...) 或在测试函数内部改变参数 (如 test_cmp.py 中的 tb(Pnum=..., Nnum=...)) 来测试不同条件下模块的行为。结合 skip_unless 可以根据 PDK 或测试级别选择性地运行测试。

  • 代码示例 (FFT 测试):

6. 总结

这个 SAR ADC 的 TED 实现展示了该框架的强大功能：

  1. 模块化和层次化: 将复杂系统分解为可管理、可重用的模块。
  2. 参数化设计: 通过参数轻松调整电路规格（如 ADC 位数 N）。
  3. 原理图与版图结合: 支持从纯原理图设计到考虑版图细节的设计流程。
  4. **版图生成与控制:**提供了描述和生成版图的工具 (row, col, hbus, vbus, guardring, layout_mos)，并能集成外部 P&R 工具 (icc)。
  5. 强大的仿真与验证: 内建仿真命令 (tran, tds) 和验证命令 (lvs, pex)，易于编写测试平台和进行性能分析。
  6. 灵活性: 可以方便地替换模块实现（如不同的开关、比较器），并使用理想模型辅助设计。

要使用 TED 开发类似的电路模块，您可以遵循以下步骤：

  1. 定义模块接口: 使用 @module 和端口类型 (Input, Output, Ctrl) 定义模块的功能和连接。
  2. 实现模块功能: 使用 TED 提供的器件 (NMOS, PMOS, Capacitor 等) 或标准单元 (Inv, Buff 等)，或者实例化其他已定义的子模块来构建电路。
  3. (可选) 添加版图信息: 使用 row, col, place, route, hbus, vbus 等描述版图结构。
  4. 编写测试平台: 使用 @testbench 设置仿真激励，运行仿真 (tran, tds)，并分析结果或进行版图验证 (lvs, pex)。
  5. 迭代优化: 根据仿真和验证结果，修改模块设计或版图实现。

Q&A

Q1: TED 框架适合什么类型的电路设计？

A1: 从示例代码来看，TED 特别适合模拟和混合信号电路设计，尤其是那些需要精细版图控制和验证的设计，如 ADC, DAC, PLL, LDO 等。它也支持数字标准单元的设计和集成。

Q2: 如何在 TED 中使用特定的 PDK 工艺库？

A2: 通常在运行测试或设计时，会通过环境变量或配置文件指定 PDK。代码中可以看到 @skip_unless(TED_PDK=[T65, T40]) 这样的装饰器，表明测试用例依赖于特定的 PDK (T65 或 T40)。基础器件如 NMOS, PMOS 的行为和版图规则会根据选定的 PDK 变化。您可能需要查阅 TED 的文档了解如何配置和切换 PDK。

Q3: src/Digital.py 和 src/saradc_digital.py 有什么区别？

A3: 两个文件都定义了 SAR ADC 的数字逻辑。Digital.py 似乎更侧重于原理图级的实现和理想化的时钟生成 (Ideal_CMP_DIV)，而 saradc_digital.py 包含了更多版图相关的细节，比如使用了 ted_interface.icc 进行自动布局布线，并定义了更具体的端口约束。在顶层模块 src/saradc.py (版图级) 中，实例化的是 SARADC_DIGITAL_TOP，它可能来自 saradc_digital.py 或经过 icc 处理后的版本。

Q4: 代码中 ~TieHigh() 是什么意思？

A4: TieHigh() 可能是 TED 中表示连接到电源 Vdd 的特殊对象。~ 操作符通常表示反相。所以 ~TieHigh() 很可能表示连接到地 Vss (逻辑低电平)。在 SAR_CTRL 模块中，MSB 的 DIN 输入连接到 ~TieHigh()，意味着最高有效位的输入固定为低电平。(需要查阅 TED 文档确认 TieHigh 和 ~ 的确切含义)

Q5: 如何学习 TED 框架？

A5:

  1. 官方文档: (如果提供) 阅读 TED 框架的官方文档和教程。
  2. 示例代码: 仔细研究像这个 SAR ADC 这样的示例项目，理解其结构、API 用法和设计流程。
  3. 基础模块: 从简单的模块（如反相器、与门、基本开关）开始，尝试编写原理图、版图和测试平台。
  4. 逐步构建: 在掌握基础模块后，尝试构建更复杂的电路，如比较器、DAC 等。
  5. 实践: 动手修改示例代码，尝试不同的参数和版图布局，观察对仿真结果和版图的影响。
