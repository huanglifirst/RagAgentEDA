自动布线器 (Router)

1. 概述

TED 提供了强大的自动布线功能，旨在根据电路网表连接，在已放置的实例（Instance）或几何图形（NBox）之间自动生成金属走线和通孔（VIA）。这是实现自动化版图设计的关键步骤之一，极大地减少了手动绘制连线的工作量。

布线功能主要通过 InstList 对象的 route 方法（或其他专用布线函数）调用。布线器会考虑设计规则（DRC）、用户指定的层级、线宽、间距以及各种优化选项来生成布线路径。

本篇文档将详细介绍 TED 布线器的核心功能、配置参数、专用布线函数以及相关的辅助工具。

2. 核心布线函数 (route)

最常用的布线入口是 InstList 对象的 route 方法，或者在模块级别调用 route()（将对当前模块内的所有可布线对象进行布线）。

from pyted import InstList, route # route 可以作为独立函数调用，作用于当前模块

# 假设 placed_insts 是一个包含已放置实例的 InstList 对象
# 例如：placed_insts = row(inst1, inst2, spacing=500)

# 对 placed_insts 中的实例进行布线
# routed_layout 现在包含原始实例以及新生成的布线形状
routed_layout: InstList = placed_insts.route(
    nets=["net_a", "net_b"], # 只布线这两个网络
    top_layer=M5,          # 最高允许布线到 M5
    width=150,             # 默认线宽 150nm
    node_widths=[("net_a", 200)], # net_a 线宽指定为 200nm
    via=2                  # VIA 尺寸至少能容纳 2x2 的最小 VIA 阵列
)

# 或者在模块最后调用，对模块内所有未布线连接进行处理
@module
def MyBlock():
    # ... 定义并放置实例 ...
    place(...)
    # 对模块内所有连接进行布线
    route(top_layer=M6)

2.1 主要参数详解

route 函数接受大量参数来控制布线行为：

目标选择 (Target Selection):

  • nets: List[Union[CircuitNode, str]] - 指定只对列表中的网络进行布线。如果为 None（默认），则尝试布线所有需要连接的网络。
  • exclude_nets: List[Union[CircuitNode, str]] - 指定不进行布线的网络列表。
  • insts: List[Instance] - 指定只在列表中的这些实例之间（或其端口）进行布线。

层级与材料 (Layer Control):

  • top_layer: Union[Layer, str] - 允许布线使用的最高金属层级。默认为 PDK 中定义的最高可用金属层。
  • vlayer: List[Union[Layer, str]] - 显式指定允许用于垂直走线的金属层列表。
  • hlayer: List[Union[Layer, str]] - 显式指定允许用于水平走线的金属层列表。 (注意：如果指定了 vlayer 或 hlayer，布线器将严格只使用这些层进行对应方向的布线)
  • allow_poly_route: bool - 是否允许在 Poly 层上布线（默认为 False）。
  • ignore_poly_route: bool - 是否在分析连接性时忽略 Poly 层（默认为 False）。

线宽与间距 (Width & Spacing):

  • width: int - 全局默认布线线宽 (单位: nm)。默认为 0，表示使用层级的最小线宽。
  • node_widths: List[Tuple[Union[CircuitNode, str], int]] - 为特定网络指定线宽。例如 [("clk", 300), (data_bus[0], 200)]。
  • spacing: int - 全局默认布线间距 (单位: nm)。默认为 0，表示使用层级的最小间距。
  • node_spacings: List[Tuple[Union[CircuitNode, str], int]] - 为特定网络指定与其他网络的间距。
  • min_spacing_of_route: int - 强制所有布线路径之间的最小间距，即使 DRC 允许更小。

通孔控制 (Via Control):

  • via: int - 指定 VIA 的尺寸，通常表示能容纳的最小 VIA 的数量（例如 via=1 表示单个最小 VIA，via=2 表示 2x2 VIA 阵列或能容纳它的尺寸）。

算法与优化 (Algorithm Tuning):

  • preferred_direction: bool - 是否允许布线器优先使用层级的“首选”布线方向（通常水平或垂直）。默认为 True。
  • strictly_preferred_direction: bool - 是否严格遵守首选方向，不允许在非首选方向上走短线。默认为 False。
  • diagonal_route: bool - 是否允许生成 45 度斜角布线。默认为 False。
  • disable_priority: bool - 是否忽略 NBox/Instance 的 priority 属性进行布线。默认为 False。
  • disable_rip_up: bool - 是否禁用布线器的 "rip-up and reroute"（撕线重布）功能。默认为 False (即允许撕线重布)。
  • stair_optimize: bool - 是否启用阶梯状布线路径的优化，使其更平直。默认为 True。
  • double_via_optimize: bool - 是否尝试优化并插入双通孔（如果规则允许）。默认为 True。
  • auto_fusion: bool - 布线完成后是否自动融合相邻的同网络、同层级金属矩形。默认为 False。
  • max_distance: int - 限制单条布线路径的最大曼哈顿距离 (nm)。超过此距离的连接可能不会被布线。

模式布线控制 (Pattern Routing Control):(模式布线通常用于总线或重复性结构)

  • disable_pattern_route: bool - 禁用模式布线策略。默认为 False。
  • only_pattern_route: bool - 强制只使用模式布线。默认为 False。
  • route_type: List[str] - 指定允许的模式布线类型（如 "AUTO", "BUS", "STAR" 等）。默认为 ["AUTO"]。
  • pattern_min_width, pattern_max_width: int - 模式布线适用的线宽范围。
  • max_npref_len: int - 模式布线中，在非首选方向上允许的最大走线长度。
  • pattern_config: PatternRouteConfig - 传入一个更详细的模式布线配置对象。

高级选项:

  • layer_extension: bool - 是否允许布线稍微超出目标端口矩形的边界以改善连接性。默认为 False。
  • expand_x, expand_y: int - 向外扩展布线区域边界 (nm)。
  • show_mst: bool - (调试用) 是否显示布线器内部生成的最小生成树。
  • show_error_line: bool - (调试用) 是否显示布线失败的连接线。
  • intra_group_routing: Union[bool, List[str]] - 如果为 True，则只在实例组内部进行布线；如果为字符串列表，则只在这些指定的组内布线。默认为 False。
  • virtual_connect: List[Union[CircuitNode, str]] - 在本次布线中将列表中的节点视为已连接，即使没有物理路径。
  • shielding_config: RouteShieldingConfig - 传入屏蔽线配置对象。
  • version: Literal[V1, V2] - 选择使用的布线器版本（V2 可能通过 iroute 实现）。
  • decompose_multi_networks: bool - 重要，如果为 True，route 函数不执行布线，而是返回一个列表，其中每个元素是一个元组 (NRect, NRect)，代表属于同一网络但初始不连通（需要布线连接）的两个矩形对。这对于分析连通性或实现自定义连接策略很有用。

2.2 返回值

  • 通常情况下 (decompose_multi_networks=False)，route 方法会修改当前的 Module，添加布线所需的 Instance (Rect 和 ViaArray)，并返回调用该方法的原始 InstList 对象（或一个新的包含结果的 InstList）。
  • 当 decompose_multi_networks=True 时，返回 List[Tuple[NRect, NRect]]。

3. 其他布线函数

除了通用的 route 函数，可能还存在一些特定场景的布线函数（具体可用性需查阅 TED API）：

  • channel_route(...): 可能用于经典的通道布线场景，参数可能包含 channel_data 来描述通道信息。
  • pattern_route(...): 强制使用模式布线策略。
  • smt_route(...): 可能基于 Steiner Minimum Tree 算法进行布线。
  • iroute(...): 可能是 route(..., version=V2) 的直接入口。

4. 布线成本估算 (get_routing_cost)

在执行实际布线之前，可以使用 get_routing_cost 函数来估算布线的复杂度或线长。

from algo.router.route_cost import get_routing_cost
from pyted import InstList

# 假设 placed_layout 是一个包含已放置实例的 InstList
# ...

# 估算布线成本 (参数与 route 函数类似)
is_routable, estimated_cost = get_routing_cost(
    instlist=placed_layout,
    nets=["net_a"],
    top_layer=M3
)

if is_routable:
    print(f"网络 'net_a' 预计可布线，估算成本: {estimated_cost}")
    # 可以根据成本决定是否继续布线或调整布局
    placed_layout.route(nets=["net_a"], top_layer=M3)
else:
    print("网络 'net_a' 预计难以布线。")
  • 参数: 与 route 函数类似，用于指定估算范围和条件。
  • 返回值: Tuple[bool, int] - 第一个元素表示根据估算是否认为可布线，第二个元素是估算的成本值（例如总线长）。

5. 示例说明

示例 1：基本点对点布线

@module
def PointToPoint():
    n = Node()
    # 创建两个 M1 矩形，相距一段距离
    r1 = Rect(M1, 500, 500, connectTo=n)
    r2 = Rect(M1, 500, 500, connectTo=n).move_by(3000, 2000)

    placed = place(r1, r2) # 放置矩形

    # 使用默认设置在 r1 和 r2 之间布线网络 n
    routed = placed.route()

    place(routed) # 放置最终结果

@testbench
def Top1():
    PointToPoint().to_gds("point_to_point.gds")

示例 2：多层布线与线宽指定

@module
def MultiLayerWidth():
    clk = Node("clk")
    data = Node("data")

    r_clk_m1 = Rect(M1, 200, 800, clk)
    r_data_m1 = Rect(M1, 500, 200, data).move_by(0, 1500)

    r_clk_m4 = Rect(M4, 200, 800, clk).move_by(4000, 0)
    r_data_m4 = Rect(M4, 500, 200, data).move_by(4000, 1500)

    placed = place(r_clk_m1, r_data_m1, r_clk_m4, r_data_m4)

    # 布线，允许最高到 M4，clk 线宽 300nm，data 线宽 150nm
    routed = placed.route(
        top_layer=M4,
        node_widths=[(clk, 300), (data, 150)],
        via=2 # 使用 2x2 VIA
    )
    place(routed)

@testbench
def Top2():
    MultiLayerWidth().to_gds("multi_layer_width.gds")

示例 4：分解多网络连接点对

@module
def DecomposeDemo():
    n1, n2 = Node(2)
    # 同一个网络 n1 的两个不连通部分
    r1 = Rect(M1, 500, 500, n1)
    r2 = Rect(M1, 500, 500, n1).move_by(2000, 0)
    # 网络 n2 的一个部分
    r3 = Rect(M2, 500, 500, n2).move_by(1000, 1000)

    placed = place(r1, r2, r3)

    # 不执行布线，而是获取需要连接的点对
    connection_pairs = placed.route(decompose_multi_networks=True)

    print("需要布线的连接点对:")
    for rect_a, rect_b in connection_pairs:
        # rect_a 和 rect_b 是 NRect 对象

6. 详细使用样例

下面是一个更完整的例子，结合了布局、多个网络、不同层级和布线参数：

from pyted import *
from ted_device import Nand2, Inv # 假设有标准单元

@module
def FullRouteExample():
    # 定义节点
    a, b, clk, reset = Node(name="a"), Node(name="b"), Node(name="clk"), Node(name="reset")
    n1, n2, z = Node(3, name="n") # n<1>, n<2>, n<3> (使用自动生成的名称)

    # 实例化标准单元
    nand_inst = Nand2().rename("U1_NAND")
    inv_inst1 = Inv().rename("U2_INV")
    inv_inst2 = Inv(size=2).rename("U3_INV_large") # 使用更大尺寸的反相器

    # 电路连接
    (a, b) >> nand_inst >> n1
    clk >> inv_inst1 >> n2
    n1 >> inv_inst2 >> z
    # reset (假设连接到未显示的部分)

    # 布局 (使用 standard_cell 布局，假设返回 InstList)
    # 将实例放置在网格中，并添加电源轨
    placed_cells: InstList = standard_cell(
        [
            [nand_inst, inv_inst1], # 第一行
            [None, inv_inst2]       # 第二行，留空位
        ],
        vdd_rail_top=True,
        vss_rail_bottom=True
    )

    # 执行布线
    # - 布线 a, b, n1, n2, z, clk (排除 reset)
    # - 最高到 M4
    # - clk 信号使用 200nm 线宽，其他默认 (假设 M1/M2/M3 最小线宽)
    # - 使用 2x2 VIA
    # - 启用阶梯优化和双通孔优化
    # - 允许在 Poly 上布线 (如果 PDK 支持且需要)
    routed_chip: InstList = placed_cells.route(
        exclude_nets=[reset],
        top_layer=M4,
        node_widths=[(clk, 200)],
        via=2,
        allow_poly_route=False, # 通常不推荐在标准单元内部进行 Poly 布线
        stair_optimize=True,
        double_via_optimize=True,
        auto_fusion=True # 布线后合并相邻金属块
    )

    # 最终放置 (包含单元和布线)
    place(routed_chip)

    # (可选) 导出或验证
    routed_chip.to_gds("full_route_example.gds")
    # routed_chip.to_instance().lvs() # 需要先转为 Instance 再验证

@testbench
def TopFullRoute():
