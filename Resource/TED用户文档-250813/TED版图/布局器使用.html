布局器使用

SMTPlacer：

使用方式：

from algo import SMTPlacer
inputs = []
constraints = []
SMTPlacer(
    instances=inputs,
    spacing=1000,
    constraints=constraints,
).place()

布局完成时会输出所有实例坐标和success信息提示

出现unsat说明无法满足所有约束条件，请检查约束条件列表

出现unknown说明时限内未找到可行解，需要适当增大timeout，也可能是不存在满足所有条件的解法

参数：

def __init__(
    self,
    instances=[],
    width: Optional[int] = None,
    height: Optional[int] = None,
    spacing: int = 0,
    grid_point: int = 5,
    enable_rotation=[],
    flexible=[],
    min_density: float = 0,
    area_cost_rate: float = 1,
    line_cost_rate: float = 0,
    offset_cost_rate: float = 0,
    line_weight={},
    line_sequence=[],
    constraints=[],
    timeout: int = 300,
    show_msg: bool = False,
    show_res: bool = True,
    iter_method: Literal["auto", "binary"] = "auto",
    iter_times: int = 10,
    iter_precision: int = 10,
    layout_count: int = 1,
):
  • instances：待布局的实例列表
  • width、height：最大宽度和高度限制(单位为nm)，留空表示需要自动优化
  • spacing：实例之间的最小间距(单位为nm)
  • grid_point：工艺的格点大小(单位为nm)，与使用工艺相关，除工艺要求外不建议更改
  • enable_rotation：开启旋转的实例列表
  • flexible：弹性布局的实例列表
  • min_density：整个布局的最小密度，[0, 1]之间的浮点数，设置为大于0时开启，可以防止一些“由于以优化半周长代替面积作为目标，导致面积并非最优”的情况，但开启时可能会减慢运行速度
  • area_cost_rate：面积优化比率，[0, 1]之间的浮点数(保留三位小数)，越大表示越偏向优化面积，设置为0时关闭(不建议)
  • line_cost_rate：线长优化比率，[0, 1]之间的浮点数(保留三位小数)，越大表示越偏向优化线长，设置为0时关闭
  • offset_cost_rate：偏移量优化比率，[0, 1]之间的浮点数(保留三位小数)，越大表示越偏向所有实例对于其参考坐标的总偏移量(见下文identify_pos)，设置为0时关闭
  • line_weight：连线权重，以{node: weight}的字典形式输入，其中node为连接到的电学节点，weight为[0, 100]之间的整数，表示权重(倍数)，当实例较多时可以将大部分节点权重设置为0以加快速度
  • line_sequence：连线顺序，以[[node1, node2], [node3, node4], ...]的列表形式输入，按照输入的电学节点的顺序自动进行增量布局(目前存在与增量布局相同的问题，可能导致效果较差)
  • constraints：约束条件列表，详见后文
  • timeout：时间限制(单位为秒)，一般设置1~10分钟，运行速度会影响布局质量，时间越长则布局效果大概率更好
  • show_msg：展示约束计算的详细信息，供调试使用
  • show_res：展示布局结果坐标
  • iter_method："binary"使用外部二分迭代的方法，此时timeout会分给每次迭代，具体每次迭代的时间限制为timeout*2/iter_times
  • iter_times：迭代次数，当iter_method设为"binary"时启用
  • iter_precision：迭代的目标精度(单位为nm)，达到目标精度时停止，当iter_method设为"binary"时启用
  • layout_count：输出布局结果的数量(准备实现)

API：

@classmethod
def identify_pos_relationship(
    cls,
    insts: List[Instance],
    mode: Literal["RELA", "ADJA", "MIX"] = "MIX",
    max_range: int = 0,
):
    ...
    return constraints

对于已经摆放好的布局，识别实例间位置关系

横坐标上具有重叠部分的两个实例，会识别它们之间的上下关系

纵坐标上具有重叠部分的两个实例，会识别它们之间的左右关系

  • insts：摆放好的实例列表
  • mode：识别模式。"RELA"为仅识别相对位置关系，在实例位置发生重叠时会根据重叠率判断是上下关系还是左右关系。"ADJA"为仅为根据max_range识别相邻关系。"MIX"为根据max_range判断使用何种方式。
  • max_range：判定相邻的最大距离，在大于时识别相对位置关系，小于等于这个距离则不保留位置，而是转化为相邻的关系
  • 返回自动识别到的约束条件列表，可以与原有的列表进行合并
constraints = []
constraints += SMTPlacer.identify_pos_relationship(inputs)
SMTPlacer(
    instances=inputs,
    spacing=1000,
    constraints=constraints,
).place()

可供在人工布局后自动缩放间距，或者在原有布局的一些位置改变间距重新摆放，以保证距离原位置偏移量最小(在布局器参数offset_cost_rate中设置)

@classmethod
def identify_pos(cls, insts=[], offset=0):
    ...
    return constraints

对于已经摆放好的布局，将实例目前摆放的位置坐标作为参考

目前存在的问题：继承的摆放位置位于原点(0, 0)处，坐标不能为负导致新的实例无法添加到左方和下方，这个问题未来会解决

  • insts：摆放好的实例列表
  • offset：在重新布局时允许实例新坐标距离原坐标的最大偏移量，0代表不允许发生任何偏移，-1代表不会限制最大偏移量
  • 返回自动识别到的约束条件列表，可以与原有的列表进行合并

可以进行增量式布局，以下示例中第一次先对inputs列表中的前10个实例进行布局，第二次在保持这些实例位置不变的情况下添加剩余的实例

constraints = []
SMTPlacer(
    instances=inputs[:10],
    spacing=1000,
    constraints=constraints,
).place()

constraints = []
constraints += SMTPlacer.identify_pos(inputs[:10])
SMTPlacer(
    instances=inputs,
    spacing=1000,
    constraints=constraints,
).place()

或者对已经摆放好的布局进行细微调整，如改变一部分间距以留出布线通道，使得重新布局时的位置不会偏离原有位置太远

约束条件：

人工或自动添加一系列约束条件，会在满足所有给定条件的情况下进行布局

注意不同约束条件会叠加影响，可以使用多种类型的约束条件共同作用来达到想要的效果

用法如下：

inputs=[r1, r2, r3, r4, r5, r6, r7]
constraints = [
    ["RELA", r1, r2, "TOP", "LEFT"],
    ["ADJA", [r1, r2]],
    ["POS", r1, "LEFT"],
    ["POS", r1, "TOP"],
    ["POS", r2, "LEFT"],
    ["POS", r2, "BOTTOM"],
]
SMTPlacer(
    instances=inputs,
    spacing=1000,
    constraints=constraints,
).place()

这个示例中把实例r1放在版图的左上角，实例r2放在版图的左下角，并且让r1与r2相邻摆放，由此限制整个版图的height为r1和r2的height之和

下面列出了所用函数的参数类型和默认值，请按照(除self以外的)顺序，在列表中的相应位置填入对应参数，一些需要留空的位置可以填None跳过

例如想要r1和r2在任意位置上相邻摆放，需要将它们之间的min_spacing和max_spacing设置为0，可以在参数pos和align的位置都填None

["RELA", r1, r2, None, None, 0, 0]

"POS"：

def add_pos(self, inst: Instance, pos: str):

位置约束

  • pos：实例在版图上摆放的位置，可以为单个字符串(如"LEFT")，或者以列表的形式同时满足一系列需求(如("LEFT", "BOTTOM"))
  • pos可选项："LEFT", "CENTER", "RIGHT", "TOP", "MIDDLE", "BOTTOM", 或者"ANY_EGDE"(任意边界), "ANY_CORNER"(任意角落)

"REF"：

def add_pos_ref(self, inst: Instance, coordinate: Tuple[int, int], offset=0):

位置坐标参考

  • coordinate：坐标(以二元组的形式表示)，如(0, 0)
  • offset：在重新布局时允许实例新坐标距离原坐标的最大偏移量

"ALIG"：

def add_align(self, inst1: Instance, inst2: Instance, align):

对齐约束

  • align：对齐的位置
  • align可选项："LEFT", "CENTER", "RIGHT", "TOP", "MIDDLE", "BOTTOM", 或者使用形如("LEFT", "BOTTOM")的二元组，表示“左侧或者底部”对齐

"RELA"：

def add_pos_relationship(
    self,
    inst1: Instance,
    inst2: Instance,
    pos: Optional[str] = None,
    align: Optional[str] = None,
    min_spacing=0,
    max_spacing: Optional[int] = None,
):

相对位置关系约束

  • pos：实例间摆放的相对位置关系，可以为None
  • align：对齐的位置，可以为None
  • pos可选项："LEFT", "RIGHT", "TOP", "BOTTOM"
  • pos为"LEFT", "RIGHT"时，align需为"TOP", "MIDDLE", "BOTTOM"
  • pos为"TOP", "BOTTOM"时，align需为"LEFT", "CENTER", "RIGHT"
  • min_spacing：最小间距
  • max_spacing：最大间距，不会小于布局器要求的间距，若小于(如设置为0)时会自动调整

"ADJA"：

def add_adjacent(
    self, insts: List[Instance], spacing=0, align1="BOTTOM", align2="LEFT"
):

相邻约束

  • spacing：列表中实例之间的间距
  • align1、align2：对齐的位置为align1或align2

"ROW"：

def add_row(self, insts: List[Instance], align="BOTTOM", spacing=0):

在自动布局时排成一行摆放，与TED中的row()相似

"COL"：

def add_col(self, insts: List[Instance], align="LEFT", spacing=0):

在自动布局时排成一列摆放，与TED中的col()相似

正常情况下对于输入的实例只会计算其外接矩形的长和宽

使用add_row()与add_col()组合，可以在布局器中排成“L”型等形状

"SYMM"：

def add_symm(
    self,
    insts=[],
    direction: Literal["HORIZONTAL", "VERTICAL"] = "VERTICAL",
    axis="ANY",
    align="CENTER",
    spacing=0,
):

对称摆放

整个insts列表中的实例使用同一个对称轴，其中的每个子列表表示这些实例放在同一行/列上

["SYMM", [[r1, r2], r3, [r4, r5, r6]], "HORIZONTAL"]

表示对称轴在水平方向上，实例r1和r2在同一列，实例r4、r5、r6在同一列

  • direction：对称轴方向，水平或垂直
  • axis：对称轴
  • direction可选项："HORIZONTAL", "VERTICAL"
  • direction为"VERTICAL"时，align需为"TOP", "MIDDLE", "BOTTOM"， axis需为"CENTER", "ANY"
  • direction为"HORIZONTAL"时，align需为"LEFT", "CENTER", "RIGHT"， axis需为"MIDDLE", "ANY"
  • 此外，axis还可以设置为具体数字，表示对称轴固定在对应位置上

"OBS"：

def add_obstacle(self, size: Optional[Tuple[int, int]], pos):

添加障碍物，在版图中提供空位

  • size：为None则大小可以改变，此时会尽可能增大其大小以使其他位置的摆放更密集
  • pos：在版图上摆放的位置，与上文POS中相同
