几何形状 (Geometry)

1. 概述

几何图形是 TED (乃至所有 IC 设计工具) 中物理版图的基础。TED 提供了一系列类和函数来创建、操作和查询各种几何形状，从基本的点、线、矩形到更复杂的、带有电气连接信息的多边形和 VIA 结构。这些几何对象是进行布局设计、DRC（设计规则检查）、LVS（版图与原理图比对）以及最终生成 GDSII 文件的核心。

本篇文档详细介绍了 TED 中主要的几何类，包括：

  • 基础几何体: Box, Point, Line, Vector
  • 带层级/节点信息的几何体: NBox, NPolygon, NRect, Label
  • 几何体创建函数: Rect, Polygon, ViaArray, MultiRects, MultiMetals
  • 组合结构: VArea, VAreaList
  • 容器类: NBoxList
  • 辅助类: GeometryObserver

2. 基础几何图元

这些类提供了基本的几何形状定义和操作，不直接关联特定的工艺层级或电气节点。

2.1 Box 类 (box.py)

Box 是表示轴对齐矩形的基础类，由其宽度 (w)、高度 (h) 以及左下角坐标 (x, y) 定义。

核心功能:

  • 创建:
    • Box(w, h, x, y): 通过宽度、高度和左下角坐标创建。
    • Box.new(x1, y1, x2, y2): (类方法) 通过左下角和右上角坐标创建。
    • Box.new_by_coords(x1, y1, x2, y2): (静态方法) 同上。
    • Box.new_by_points(points): (静态方法) 通过四个顶点（或五个闭合点）创建。
    • Box.new_by_boxes(boxes): (静态方法) 计算多个 Box 的最小外包矩形 (Bounding Box)。
    • Box.new_by_insts(insts): (静态方法) 计算多个 Instance 的最小外包矩形。
  • 属性访问:
    • w, h, x, y: 获取或设置宽度、高度、左下角 x, y 坐标。
    • x1, y1, x2, y2: 获取或设置左下角和右上角坐标。
    • cx, cy: 获取中心点坐标。
    • area: 获取面积。
    • size(): 返回 (w, h) 元组。
    • get_coord(): 返回 (x1, y1, x2, y2) 元组。
    • left_bottom_coord(), right_top_coord(), left_top_coord(), right_bottom_coord(), center_coord(): 获取特定点的坐标。
    • get_points(): 返回构成矩形的五个顶点列表（用于闭合）。
  • 变换:
    • move_to(x, y), move_by(dx, dy): 移动 Box。
    • move_center_to(cx, cy), move_cx_to(cx), move_cy_to(cy): 按中心点移动。
    • move_x1_to(x), move_y1_to(y), move_x2_to(x), move_y2_to(y): 按边界坐标移动。
    • rotate(angle, center=(0,0)): 旋转 Box（角度需为 90 的倍数）。
    • flipx(y), flipy(x): 沿水平线 y 或垂直线 x 翻转。
    • expand(size), expand_h(size), expand_v(size): 向外扩展 Box。
    • set_x1_to(val), set_x2_to(val), set_y1_to(val), set_y2_to(val): 调整某一边界的位置（会改变尺寸）。
  • 几何查询:
    • is_intersecting(other): 是否与另一个 Box 相交。
    • is_tangent(other): 是否与另一个 Box 相切。
    • is_tangent_or_intersecting(other): 是否相切或相交。
    • is_covered(other): 是否被另一个 Box 完全覆盖。
    • is_point_in(x, y): 指定点是否在 Box 内部或边界上。
    • intersection(other): 计算与另一个 Box 的交集（返回新的 Box 或 None）。
    • union(other): 与另一个 Box 合并（修改自身）。
    • get_parallel_size(other, orientation): 计算与另一个 Box 在指定方向上的平行边重叠长度。
    • get_h_spacing(other), get_v_spacing(other): 计算与另一个 Box 的水平或垂直间距。
    • get_h_position(target), get_v_position(target): 判断自身相对于 target Box 的水平或垂直位置（LEFT, RIGHT, TOP, BOTTOM, OVERLAP）。
  • 几何运算:
    • merge(box_list): (静态方法) 将列表中的 Box 进行合并（仅合并满足特定条件的相邻或重叠 Box），返回合并后的 List[Box]。使用 RTree 进行高效计算。
    • sub_horizontal(boxes), sub_vertical(boxes): (实验性) 计算水平或垂直方向上的差集，返回 List[Box]。
  • 静态布局助手:
    • Box.row(*boxes, spacing, align): 将多个 Box 水平排列。
    • Box.col(*boxes, spacing, align): 将多个 Box 垂直排列。
    • Box.lay(*boxes, x_align, y_align): 将多个 Box 在原地根据对齐方式重叠放置。
    • Box.row_by_target(...), Box.col_by_target(...), Box.lay_by_target(...): 相对于目标 Box 进行排列或放置。
  • 转换:
    • to_nbox(layer, node): 将 Box 转换为 NBox 对象。
    • multi_rects(*layers, node, priority): 基于此 Box 创建多层 NRect 实例。

2.2 Point 类 (point.py)

表示二维笛卡尔坐标系中的一个点。

  • 创建: Point(x, y)。
  • 属性: x, y。
  • 操作: copy(), move_dy(dx, dy), bbox_list() (创建一个以此点为中心的小 BBox)。
  • 比较: 支持 <, >, == 比较（基于 x 优先，然后 y）。

2.3 Line 类 (line.py)

表示一个水平或垂直的线段集合（内部可能包含多个不连续的线段）。主要用于内部几何运算。

  • 创建: Line(a, b, orientation)，a 和 b 定义线段端点在一个轴上的坐标。
  • 操作: __sub__(other) (计算线段差集), to_boxes(prep_line) (与垂直线段组合生成 Box 列表)。

2.4 Vector 类 (vector.py)

表示一个从 start 点到 end 点的向量（目前仅支持水平或垂直）。

  • 创建: Vector(start_point, end_point)。
  • 属性: p1, p2, x1, y1, x2, y2, min_x, max_x, min_y, max_y。
  • 操作: copy(), get_length(), contain_length(), fix_point(), get_direction(), has_intersect(), overlap().
  • 用途: 可能用于更底层的布局或布线算法，用户通常不直接操作。

3. 带层级/节点信息的几何体

这些类在基础几何形状上增加了工艺层级 (Layer) 和电气连接 (CircuitNode) 的信息，是版图设计中的主要操作对象。

3.1 NBox 类 (nbox.py)

NBox (Node-Box) 是 TED 布局算法中非常核心的一个类。它代表一个带有层级和节点信息的矩形 (Box + Layer + Node)。它不是一个 Instance，而是一个更底层的、用于几何计算和表示的数据结构。

核心功能:

  • 创建:
    • NBox(layer, w, h, connectTo, ...): 直接创建。
    • NBox.new_by_box(box, layer, node, ...): (静态方法) 从 Box 创建。
    • NBox.new_by_coords(layer, x1, y1, x2, y2, node): (静态方法) 从坐标创建。
    • NBox.new_by_rs_nbox(rs_nbox): (静态方法) 从 Rust 对象创建。
  • 属性: box (Box), layer (Layer), node (CircuitNode), name, priority, tags.
  • 变换: 提供了与 Box 类似的移动 (move_to, move_by, ...), 旋转 (rotate), 翻转 (flipx, flipy), 扩展 (expand) 方法。
  • 几何查询: 提供了与 Box 类似的查询方法 (is_intersecting, is_tangent, is_covered, get_parallel_size, etc.)，但会检查层级是否相同。
  • DRC 感知分离:
    • separate_h(pv, interval): 在给定的水平区间内，通过移动自身来解决与 PhysicalVerification 对象 (pv) 中其他 NBox 的 DRC 冲突（间距不足或重叠），找到离原始位置最近的合法位置。
    • separate_v(pv, interval): 类似地在垂直方向上解决 DRC 冲突。
    • get_left/right/top/bottom_min_spacing(nboxes): 计算自身到给定 nboxes 列表在特定方向上的最小 DRC 间距要求。
  • 转换:
    • to_rect(check_width, ensure_rect): 将 NBox 转换为 NRect (一个 Instance)。如果层级是 VIA 层且 ensure_rect=False，则会尝试创建 ViaArray 实例。
    • to_npolygon(): 将 NBox 转换为等效的 NPolygon。
    • to_code(): 生成创建此 NBox 的 Python 代码字符串。
  • 静态布局助手:
    • NBox.row(*nboxes, spacing, align): 水平排列 NBox 或 NBoxList。
    • NBox.col(*nboxes, spacing, align): 垂直排列。
    • NBox.lay(*nboxes, x_align, y_align): 重叠放置。
    • NBox.layc(*nboxes): 居中重叠放置。
    • NBox.row/col/lay/layc_by_target(...): 相对于目标 NBox 进行排列或放置。
  • 连接与合并:
    • NBox.connect_by_npolygon(b1, b2, width): (静态方法) 创建一个连接两个 NBox 中心的 NPolygon（如果可能，创建斜角连接）。
    • NBox.merge(nbox_list): (静态方法) 合并 NBox 列表（按层级和节点）。
    • NBox.merge_by_layer(nbox_list): (静态方法) 按层级合并。
    • NBox.auto_connect(nboxes): (静态方法) 基于几何接触自动推断并设置 NBox 列表的节点连接。
  • 观察者模式 (GeometryObserver): NBox 继承自 GeometryObserver，可以订阅其他 NBox 或 Instance 的变换通知，并在自身发生变换时通知其订阅者。这对于维持布局中对象间的相对关系非常重要。

3.2 NPolygon 类 (npolygon.py)

NPolygon 代表一个带有层级和节点信息的多边形 (List[Point] + Layer + Node)。

核心功能:

  • 创建:
    • NPolygon(ly, points, node, ...): 通过顶点列表、层级和节点创建。
    • NPolygon.new_by_rs_polygon(rs_polygon): (静态方法) 从 Rust 对象创建。
    • NPolygon.new_by_nboxes(nboxes): (静态方法) 通过合并（通常是相邻或重叠的）NBox 列表来创建 List[NPolygon]。
  • 属性: points, layer, node, name, tags, box (获取外包矩形)。
  • 变换: move_to, move_by, rotate, mirror_x, mirror_y.
  • 几何查询: is_intersecting, is_point_in, has_slant_line (是否包含非直角边)。
  • 转换:
    • to_polygon(): 将 NPolygon 转换为 Polygon 实例 (Instance)。
    • to_nbox(): 如果多边形恰好是矩形，则转换为 NBox。
    • to_nboxes(orientation): 将直角多边形分解为 NBox 列表（按水平或垂直切割）。
  • 标记: add_tag, has_tag, get_tags, skip_pv.
  • 可视化: draw(), NPolygon.draw_from_list(...).

3.3 NRect 类 (rect.py)

NRect 是 Instance 的一个子类，专门用于表示矩形实例。它是由 Rect() 函数创建的。

  • 继承: 拥有 Instance 的所有方法和属性。
  • 特有属性:
    • layer (Layer): 获取矩形的层级。
    • node (CircuitNode): 获取矩形连接的节点（仅对可布线层级 RRect 有效，通过 ctrl() 获取）。
  • 转换: to_nbox(): 转换为 NBox。
  • 查询: 提供了 is_tangent, is_intersecting 等几何查询（委托给内部的 NBox）。
  • DRC 分离: separate_h, separate_v (静态方法 separate_rects_h/v)，同样委托给 NBox 实现。

3.4 Label 类 (label.py)

表示版图中的文本标签。

  • 创建: Label(content, layer, x, y, node)。
  • 属性: content, layer, x, y, node。
  • 转换:
    • to_real(): 将此 Label 对象实际添加到当前模块的版图中。
    • to_drawing_label(): 尝试获取对应绘图层 (如 M1-drawing) 的 Label 对象。
    • from_gdstk_label(): 从 gdstk 库的 Label 对象创建。

4. 几何体创建函数

这些函数提供了创建常用几何 Instance 的便捷方式。

4.1 Rect() (rect.py)

创建 NRect 实例的主要函数。

  • 参数: layer, w, h, connectTo (连接的节点), skip_via (是否跳过自动打孔), priority (布线优先级), ensure_min_width, check_width, tags。
  • 行为: 根据层级是普通层还是可布线层创建不同的内部 Rust 对象 (Rect 或 RRect)，并进行 DRC 最小宽度检查（如果启用）。

4.2 Polygon() (polygon.py)

创建多边形 Instance。

  • 参数: layer, points (顶点列表), connectTo。
  • 行为: 创建一个代表多边形的 Instance。

4.3 ViaArray() (via_array.py)

创建 VIA 阵列 Instance。

  • 参数: layer (必须是 VIA 层), w, h (阵列区域的宽高), connectTo, check_width。
  • 行为: 创建一个在指定区域内自动填充 VIA 孔的 Instance。

4.4 MultiRects() 和 MultiMetals() (rect.py)

用于快速创建多层金属/VIA 堆叠结构。

  • MultiRects(layers, w, h, connectTo, ...): 在指定的多个层级上创建相同尺寸的矩形，并可选连接到同一节点。
  • MultiMetals(bottom, top, l, w, connectTo, ...): 创建从 bottom 金属层到 top 金属层的所有金属和 VIA 层的矩形堆叠。内部调用 MultiRects。

5. 组合几何结构

这些类表示由多个基本几何体组成的常见版图结构。

5.1 VArea 类 (varea.py)

VArea (Via Area) 代表一个标准的 VIA 连接结构，通常由三层构成：底层金属、VIA 切割层、顶层金属。

  • 结构: 内部包含三个 NBox 对象：bottom_metal, via, top_metal。
  • 创建:
    • VArea(top_metal, via, bottom_metal): 直接用三个 NBox 创建。
    • VArea.new_by_rs_varea(rs_varea): (静态方法) 从 Rust 对象创建。
    • VArea.custom_via(...): (静态方法) 核心创建方法。根据指定的 VIA 层、上下金属层、目标宽度或 VIA 数量、缩进选项等参数，自动计算并生成一个或多个 VArea 对象列表，满足 DRC 要求。
  • 属性: node, box (整体外包框), top_metal, via, bottom_metal。
  • 操作: move_to, move_by, add_tag, join_in_virtuoso_msg.
  • 转换: to_rects() 或 to_real_shape(): 将 VArea 包含的三个 NBox 转换为 NRect 实例列表。

5.2 VAreaList 类 (varea_list.py)

一个简单的列表容器，用于管理多个 VArea 对象。

  • 行为: 提供列表的基本操作（索引 [], len, append, extend）以及对列表内所有 VArea 应用相同操作的方法（如 add_tag, join_in_virtuoso_msg, to_rects）。

6. 容器类

6.1 NBoxList 类 (nbox_list.py)

用于管理 NBox 对象的列表容器，提供了强大的批处理和布局功能。

  • 创建: NBoxList(items) 或 NBoxList.new(*items)。
  • 列表操作: __len__, __getitem__, append, extend, sort.
  • 集体变换: move_to, move_by, move_x1_to, 等，将操作应用于列表中的所有 NBox。
  • 属性: box (获取整个列表的外包矩形)。
  • 核心方法:
    • flatten(): 将可能嵌套的 NBoxList 展平成只包含 NBox 的一级列表。
    • reset_box(): 重新计算外包矩形。
    • merge(): 合并列表中可合并的 NBox（按层级和节点）。
    • merge_by_layer(): 按层级合并 NBox。
    • auto_connect(): 基于几何接触自动推断并设置 NBox 节点的连接关系。
    • to_rects(...): 将列表中的所有 NBox 转换为 NRect 实例列表。
    • get_bounding_box(), get_bounding_nbox(): 获取外包框 (Box 或 NBox)。
    • draw_from_list(...): 可视化列表中的 NBox。
    • get_h_projections(), get_v_projections(): 获取水平或垂直投影区间。
    • 静态布局助手: row(), col(), lay(), layc(), row/col/lay/layc_by_target(): 对列表中的 NBox 应用排列或放置操作，返回 NBoxList 自身（链式调用）。
    • auto_via(): 重要，在列表中的 NBox 之间自动检测需要 VIA 的地方，并生成相应的 VAreaList。

7. 观察者模式 (geometry_observer.py)

  • GeometryObserver: 一个基类，实现了观察者（发布/订阅）模式。允许一个几何对象（如 NBox）订阅另一个对象的特定变化（如尺寸、移动、旋转、翻转）。当被观察对象发生变化时，会调用订阅者注册的回调函数。
  • 用途: 主要用于布局算法中，当一个对象移动或改变时，需要自动更新与之相关的其他对象的位置或形状，以维持相对关系或满足约束。NBox 类继承了此类。

8. 总结

TED 的 geometry 模块提供了从基础到高级的几何对象和操作。Box 是核心的纯几何矩形表示，而 NBox 则增加了层级和节点信息，成为布局算法和 DRC 处理的关键。NRect 是矩形的 Instance 表现形式。NPolygon 和 Polygon 处理多边形。VArea 和 ViaArray 封装了 VIA 结构。NBoxList 和 VAreaList 提供了方便的容器和批处理操作，尤其是 NBoxList 的 auto_via 和布局方法非常实用。理解这些类的关系和功能对于进行有效的版图生成至关重要。
