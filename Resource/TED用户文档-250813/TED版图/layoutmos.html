MOS版图API

摘要

本文档详细介绍了 pattern_route 相关模块和 LayoutMos 类及其相关函数的功能和使用方法。主要内容包括：

  1. layout_mos 函数：用于对 MOS 管集合进行自动化布局，支持多种配置选项，如打孔数量、供电连接、Dummy管添加、分块布局等。
  2. LayoutMos 类：layout_mos 函数返回的对象类型，提供了对布局后 MOS 管阵列进行操作和查询的方法，例如获取指定位置的 MOS 管、连接不同的 LayoutMos 对象等。
  3. VerticalizeMos 类：用于将 MOS 管列表进行纵向排列并添加保护环。
  4. auto_layout_stdcell 函数：结合 layout_mos 和模拟退火算法，实现标准单元的自动化布局，优化 MOS 管排列顺序。
  5. Pattern Routing (route 函数相关)：展示了 route 函数在模式化布线（Pattern Routing）方面的应用，包括跨层连接、直线/L型/Z型/U型布线、桥接、屏蔽线等多种布线模式。

文档结合了代码中的测试用例，提供了丰富的示例，并补充了完整使用样例和 Q&A，帮助用户更好地理解和使用相关 API。

1. MOS 管自动布局 (layout_mos 函数与 LayoutMos 类)

1.1 layout_mos 函数

layout_mos 函数是进行 MOS 管自动化布局的核心函数。它接收一个 MOS 管列表以及多种配置参数，生成一个包含布局信息的 LayoutMos 对象。

函数签名:

def layout_mos(
    mosfets: List[Mos] = [],
    gate: int = 1,
    ds: int = 1,
    supply: int = 0,
    tap: int = 0,
    vsupply_spacing: Optional[int] = None,
    aa_spacing: Optional[int] = None,
    connect_single_ds: bool = False,
    connect_supply_ds: bool = True,
    ds_layer: Optional[Layer] = None,
    single_gate: bool = False,
    part_max_m: Optional[int] = None,
    part_align: int = LEFT,
    part_spacing: Optional[int] = None,
    gate_top_layer: Optional[Layer] = None,
    gate_m1_shrink_vias: int = 0,
    gate_spacing: Optional[int] = None,
    gate_m1_align: int = CENTER,
    s_shaped: bool = True,
    keepout: Optional[Union[int, Tuple[int, int, int, int]]] = None,
    connect_g_by_po: bool = True,
    add_m2: bool = False,
    outer_dummy_l: Optional[float] = None,
    outer_dummy_m: int = 0,
    inner_dummy_l: Optional[float] = None,
    inner_dummy_m: int = 0,
    poly_dummy: bool = False,
    ds_width: Optional[int] = None,
    share_ds: bool = True,
    mos_align: int = BOTTOM,
    grouped_mos_with_spacing: Tuple[List[List[Mos]], List[int]] = None,
    ds_spacing: Optional[int] = None,
    bus_ds: bool = False,
) -> LayoutMos

主要参数说明:

  • mosfets: 需要布局的 MOS 管列表。列表中的 MOS 管必须具有相同的 model 和 w (宽度)。
  • gate: 栅极(G)连接矩形的高度（以 via 数量表示）。
  • ds: 漏极(D)和源极(S)连接矩形的高度（以 via 数量表示）。
  • supply: VDD/VSS 供电矩形的高度（以 via 数量表示）。
  • tap: Tap 矩形的高度（以 via 数量表示）。若指定，supply 参数失效。
  • vsupply_spacing: 供电/Tap 矩形到 MOS 管的间距。None 表示自动计算最小 DRC 间距。
  • aa_spacing: 有源区 (AA) 间距。当无法源漏共用时使用。
  • ds_layer: D/S 连接层。None 表示 M2。
  • single_gate: 是否单独连接每个栅极。
  • part_max_m: MOS 管分组布局时，每组（每行/每列）的最大 MOS 管数量。None 表示不分组。
  • part_align: 分组对齐方式 (LEFT, CENTER, RIGHT)。仅当 part_max_m 不为 None 时生效。
  • part_spacing: 分组间距。None 表示自动计算最小 DRC 间距。
  • s_shaped: 是否蛇形排列（奇数行翻转）。仅当 part_max_m 不为 None 时生效。
  • keepout: 布局边缘到 MOS 管的保护距离。None 表示自动计算。可以是一个整数（四边相同）或一个四元组 (left, top, right, bottom)。
  • connect_g_by_po: 栅极连接是否通过 PO 层。
  • gate_top_layer: 栅极连接的最高金属层。None 为 M1。
  • gate_m1_shrink_vias: M1 栅极连接矩形相对 PO 的收缩量（以 via 数表示）。
  • add_m2: 是否添加 M2 覆盖 D/S。
  • outer_dummy_m: 两侧 Dummy 管数量。
  • inner_dummy_m: 内部 Dummy 管数量（用于连接不同 MOS 组）。
  • poly_dummy: 是否使用 Poly Dummy。False 使用 MOS Dummy。
  • ds_width: 扩展 D/S 矩形的宽度。None 不扩展。
  • share_ds: 是否允许源漏共享。
  • mos_align: MOS 管垂直对齐方式 (TOP, MIDDLE, BOTTOM)。
  • grouped_mos_with_spacing: 手动指定 MOS 分组和间距，格式为 (分组列表, 间距列表)。如果指定，会覆盖自动分组逻辑。
  • ds_spacing: M2 源漏矩形之间的间距。
  • bus_ds: 是否将 D/S 连接成总线形式。

示例 (来自 test_layout_mos.py:test02):

# 对 PMOS 列表进行布局，指定 gate 和 tap via 数量，设置 keepout
g1, g2 = Node(2)
m1 = Mos(l=1e-6, w=5e-6, model="pch") % g1
m2 = Mos(l=1e-6, w=5e-6, model="pch") % g2
m3 = Mos(l=1e-6, w=5e-6, model="pch") % g1
m4 = Mos(l=1e-6, w=5e-6, model="pch") % g2
m5 = Mos(l=1e-6, w=5e-6, model="pch") % g1
Vdd() >> m1 >> m2 >> m3 >> m4 >> m5 >> Vdd()
layout_mos(
    mosfets=[m1, m2, m3, m4, m5],
    gate=1,       # 栅极连接用1个 via 高度
    tap=1,        # Tap 用1个 via 高度
    keepout=500,  # 四周保护距离 500
    supply=0,     # supply 参数因 tap > 0 而失效
    connect_g_by_po=False, # 栅极不通过 PO 连接
)
place(via_min_number=1)

示例 (来自 test_layout_mos.py:test04):

# 使用 M1 进行栅极连接 (gate=2)，添加 Tap (tap=3)
layout_mos(
    mosfets=[m1, m2, m3, m4, m5],
    gate=2,       # 栅极连接用 M1
    tap=3,        # Tap 用3个 via 高度
    keepout=400,
    supply=0,
    connect_g_by_po=False,
)
place(via_min_number=1)

示例 (折叠布局，来自 test_layout_mos.py:test11):

# 将一个 m=8 的 PMOS 分成两行 (part_max_m=4) 进行布局
V3, Vo = Node(2)
Vdd() >> (pm11 := PMOS(l=1e-6, w=2e-6, m=8)) % V3 >> Vo
mosfets = layout_mos(mosfets=pm11, part_max_m=4) # 每行最多4个 MOS
place(via_min_number=1)

1.2 LayoutMos 类方法

layout_mos 函数返回 LayoutMos 对象，该对象继承自 InstList，并提供了一些额外方法。

  • get_mos(rows: int, cols: int) -> Mos: 获取布局后位于 rows 行 cols 列的 MOS 管实例。行列索引从 0 开始，支持负数索引。

    • 示例 (来自 test_layout_mos.py:test11):
      # 获取布局结果 mosfets
mosfets = layout_mos(mosfets=pm11, part_max_m=4)
# 获取第 0 行，第 0 列的 MOS 管 (左上角)
top_left_mos = mosfets.get_mos(0, 0)
# 获取最后一行，最后一列的 MOS 管 (右下角)
bottom_right_mos = mosfets.get_mos(-1, -1)

  • connect(other: LayoutMos, g_via: int = 1) -> List[NRect]: 连接当前 LayoutMos 对象和位于其下方的另一个 LayoutMos 对象的 D/S/G。要求两个对象的 gate 参数 > 0，single_gate 参数相同，且未旋转。

    • 示例 (来自 test_layout_mos.py:test13):
      # 创建两个 LayoutMos 对象 p1, p2
p1 = layout_mos(...)
p2 = layout_mos(...)
# 将 p1 和翻转后的 p2 垂直排列
col(p1, p2.flipx(), spacing=1000)
# 连接 p1 和 p2 的 D/S/G
p1.connect(p2, g_via=2) # 使用 2 via 宽度的 M2 连接栅极

  • get_left_and_right_mos() -> Tuple[Mos, Mos]: 返回布局中最左侧和最右侧的 MOS 管实例。

  • get_left_mos() -> Mos: 返回布局中最左侧的 MOS 管实例。

  • get_right_mos() -> Mos: 返回布局中最右侧的 MOS 管实例。

  • row(*items: LayoutMos, align: Optional[str] = None, mode="overlap") -> InstList: (静态方法) 将多个 LayoutMos 对象排成一行。

    • mode="overlap": MOS 管源漏共享。
    • mode="touch": 实例边缘零间距接触。

2. 纵向 MOS 管布局 (VerticalizeMos 类)

此类用于将 MOS 管列表（可以包含 None 作为占位符）进行纵向排列，并自动添加保护环。

类初始化:

VerticalizeMos(mosfets: List[Mos], vias: int = 1)
  • mosfets: MOS 管列表，可以包含 None。列表中的 MOS 管必须 L/W/Model/Gate Node 相同。
  • vias: 顶部和底部用于连接栅极的 PO/M1 矩形的高度（以 via 数表示）。

示例 (来自 test_verticalize_mos.py:test01):

@module
def Dut(a):
    m1 = NMOS(m=1, w=1e-6, l=1e-6) % a
    m2 = NMOS(m=2, w=1e-6, l=1e-6) % a
    m3 = NMOS(m=1, w=1e-6, l=1e-6) % a
    # 创建 VerticalizeMos 实例，m2 和 m3 之间有一个空位
    mos = VerticalizeMos([m1, m2, None, m3])
    mos.rotate(90) # 旋转90度
    guardring(mos, Vss()) # 添加保护环
    place()
    route()

3. 标准单元自动布局 (auto_layout_stdcell 函数)

此函数结合了 layout_mos 和模拟退火算法，用于自动优化标准单元（包含 PMOS 和 NMOS）的 MOS 管排列顺序，并完成布局和保护环添加。

函数签名:

def auto_layout_stdcell(
    pmos_list: List[Mos],
    nmos_list: List[Mos],
    palign=BOTTOM,
    nalign=TOP,
    pheight=10e3,
    nheight=7e3,
    spacing=2e3,
    top_metalp=[None, M1, None, TAP],
    top_metaln=[None, TAP, None, M1],
    gate=2,
    gate_m1_shrink_vias=0,
    gate_spacing=530,
    ds=1,
    ds_layer=None,
    initial_temps=[500, 1000, 1500, 2000],
    cooling_rates=[0.991, 0.992, 0.995, 0.997],
    final_temp=0.01,
)

主要参数说明:

  • pmos_list: PMOS 列表。
  • nmos_list: NMOS 列表。
  • palign, nalign: PMOS 和 NMOS 在保护环内的垂直对齐方式。
  • pheight, nheight: PMOS 和 NMOS 区域的高度（用于 TEXT 层）。
  • spacing: PMOS 和 NMOS 保护环之间的间距。
  • top_metalp, top_metaln: PMOS 和 NMOS 保护环的顶层金属设置。
  • gate, gate_m1_shrink_vias, gate_spacing, ds, ds_layer: 传递给 layout_mos 的参数。
  • initial_temps, cooling_rates, final_temp: 模拟退火算法的参数。函数会尝试不同的初始温度和降温速率组合，选择能量最优（通常代表线长最短）的结果。

辅助函数:

  • add_pmos(d, g, s, l, w, m, model): 创建 m 个 PMOS 实例，用于 m > 1 的情况，会自动处理翻转以优化布局。
  • add_nmos(d, g, s, l, w, m, model): 创建 m 个 NMOS 实例，类似 add_pmos。

示例 (来自 test_auto_layout_stdcell.py:Nand4):

@module
def Nand4(...):
    # ... (定义 pmos 和 nmos 连接) ...
    pmos_list = [p1, p2, p3, p4]
    nmos_list = [n1, n2, n3, n4]
    if use_auto_layout:
        auto_layout_stdcell(
            pmos_list,
            nmos_list,
            palign,
            nalign,
            pheight,
            nheight,

4. Pattern Routing 示例

以下示例展示了 route 函数结合 PatternRouteConfig 实现的各种模式化布线策略。模式化布线通常用于在特定场景下（如同层连接、跨层连接）生成更规整、符合设计规则的布线路径。

核心参数:

  • route(..., pattern_config=PatternRouteConfig(...))
  • PatternRouteConfig(patterns_to_use=[Pattern.XXX, ...], ...): 指定要使用的布线模式。

常用模式 (Pattern):

  • StraightMid, StraightSide: 直线连接（中心对齐或边缘对齐）。
  • SpecialL, UpperL, LowerL: L 型连接。
  • HorizontalZ, VerticalZ: Z 型连接。
  • UpperU, LowerU, LeftU, RightU: U 型连接。
  • AcrossLayers: 跨层连接（自动选择 M1/M2/M3 等）。
  • AcrossLayersL: 跨层 L 型连接。
  • AcrossLayersP: 跨层连接（优先平行）。
  • AcrossLayersSE: 跨层连接（带 Source/Target 扩展）。
  • BridgeUp, BridgeDown: 桥接（向上或向下层跳线）。
  • Shielding: 屏蔽线模式。
  • PlaceVia: 在重叠区域放置 Via。

示例 (跨层连接 M2-M3, 来自 test_via_patterns.py:test1):

from pyted import *
from ted_device import NMOS, PMOS
from algo.router.pattern_route.auto_layout_stdcell import auto_layout_stdcell

@module
def Nand4Auto(A1, A2, A3, A4, ZN: Output, wp=2, wn=1, l=0.6, m=1):
    # 定义 PMOS 和 NMOS (注意：这里简化了 P 和 N 函数的定义)
    def P(w, l=l, mp=m): return PMOS(l=l * 1e-6, w=w * 1e-6, m=mp)
    def N(w, l=l, mn=m): return NMOS(l=l * 1e-6, w=w * 1e-6, m=mn)

    # 电路连接 (与 test_auto_layout_stdcell.py 中 Nand4 相同)
    (ZN >> (p1 := P(wp)) % A1 >> Vdd() >> (p2 := P(wp)) % A2 >> ZN >>
     (p3 := P(wp)) % A3 >> Vdd() >> (p4 := P(wp)) % A4 >> ZN)
    (Vss() >> (n1 := N(wn)) % A1 >> (n2 := N(wn)) % A2 >>
     (n3 := N(wn)) % A3 >> (n4 := N(wn)) % A4 >> ZN)

    pmos_list = [p1, p2, p3, p4]
    nmos_list = [n1, n2, n3, n4]

    # 使用 auto_layout_stdcell 进行自动布局
    auto_layout_stdcell(
        pmos_list=pmos_list,
        nmos_list=nmos_list,
        palign=BOTTOM,
        nalign=TOP,
        pheight=10e3,
        nheight=7e3,
        spacing=2e3,
        top_metalp=[None, M1, None, TAP], # P 保护环顶层金属
        top_metaln=[None, TAP, None, M1], # N 保护环顶层金属
        gate=2, # G 连接使用 M1
        gate_m1_shrink_vias=0,
        gate_spacing=530,
        ds_layer=M1 # D/S 连接使用 M1
    )

    # 布线
    route(nets=[ZN])
    route() # 布线其余网络

6. Q&A

Q1: layout_mos 和 auto_layout_stdcell 有什么区别？

A1:

  • layout_mos 主要负责对单一类型（P 或 N）的 MOS 管列表进行布局，可以控制 Dummy 管、分块、连接方式等细节。它本身不进行 MOS 管的排序优化。
  • auto_layout_stdcell 是更高层次的函数，它内部调用 layout_mos。它接收 PMOS 和 NMOS 列表，利用模拟退火算法优化两组 MOS 管的排列顺序，以获得更好的布局效果（通常是更短的连线），然后完成标准单元的整体布局（包括 PMOS、NMOS 和保护环）。

Q2: 什么时候应该使用 layout_mos 的 part_max_m 参数进行折叠布局？

A2: 当 MOS 管数量很多，导致单行布局过长时，可以使用 part_max_m 将 MOS 管分成多行（或多列，如果旋转后）进行折叠布局，以获得更方形的版图，节省面积。

Q3: layout_mos 中的 keepout 参数设为 None 和设为 0 有什么区别？

A3:

  • keepout=None: 程序会自动计算一个合适的 keepout 值，这个值通常能保证当两个相同的 LayoutMos 实例以零间距拼接时，它们内部的 MOS 管之间的距离满足最小 DRC 要求。
  • keepout=0: 明确指定 keepout 为 0，生成的 LayoutMos 实例的边界将紧贴最外层的 MOS 管（或 Dummy 管）。这在需要手动精确控制与其他单元间距时可能有用，但直接拼接可能会违反 DRC。

Q4: 如何实现不同金属层之间的 Pattern Routing 连接？

A4: 在 route 函数中设置 via 参数（允许的最大 via 层数，如 via=2 表示允许使用 V1 和 V2），并设置 pattern_config=PatternRouteConfig(patterns_to_use=[Pattern.AcrossLayers, ...])。AcrossLayers 是最常用的跨层连接模式。也可以使用 AcrossLayersL, AcrossLayersP, AcrossLayersSE 等特定模式。

Q5: 模拟退火布局 (auto_layout_stdcell) 的效果不好怎么办？

A5: 可以尝试调整模拟退火的参数：

  • 增加 initial_temps 中的初始温度值或尝试更多温度点。
  • 减慢 cooling_rates 中的降温速率（更接近 1 的值）或尝试更多速率。
  • 降低 final_temp。
  • 如果效果仍然不理想，可能需要手动调整 MOS 管顺序或使用 grouped_mos_with_spacing 参数强制指定分组和间距进行 layout_mos 布局。

Q6: layout_fin.py 和 layout_mos.py 的关系是什么？

A6: layout_fin.py 似乎是针对 FinFET 工艺的 layout_mos 实现。它导入并使用了 layout_mos.py 中的 LayoutMos 类，并对其进行了特定于 FinFET 的调整（例如，处理鳍片相关的布局逻辑，如 _row_mos 中对 PO 边距的处理）。当 PDK 为 FinFET 工艺（如 S12, T12）时，应使用 layout_fin.py 中的 layout_mos。
