层 (Layer)

layer 模块包是 tedcore 或 pyted 框架中用于处理集成电路版图工艺层级 (Layout Layers) 的核心组件。它封装了与特定工艺层级相关的各种信息，包括其标识（名称、层号、数据类型）、设计规则（最小宽度、间距、面积、包围规则等）、以及层级之间的关系（如上下层金属/通孔、对应的障碍物层/Pin层等）。

该包主要提供了以下功能：

  1. Layer 类: 表示单个工艺层级及其属性和规则。
  2. LayerList 类: 管理 Layer 对象的有序集合，并提供聚合操作。
  3. ViaConfig 类: 用于配置通孔阵列并计算其所需尺寸。
  4. 预定义层级常量: 在 layers.py 中定义了常用层级的便捷访问变量 (如 M1, VIA1, PO 等)。
  5. 辅助函数: 提供获取特定层级集合（如所有布线层、所有通孔层）的工具函数。

通过该模块，开发者可以在版图设计代码中以面向对象的方式方便地引用层级、查询设计规则、导航层级堆栈，从而提高代码的可读性、可维护性和对不同工艺的适应性。

II. 核心类 (Core Classes)

A. Layer 类 (layer.py)

Layer 类是表示单个版图工艺层级的核心数据结构。它封装了来自底层 PDK (Process Design Kit) 数据（可能通过 rsted 库访问）的层级信息和相关设计规则。

1. 初始化 (__init__)

class Layer:
    def __init__(self, ly, is_original_name=False):
        # ly: 底层库 (如 rsted) 代表层级的对象
        # is_original_name: 布尔值，指示 ly 是否基于 PDK 原始名称获取
        self.ly = ly
        self.is_original_name = is_original_name
  • 每个 Layer 对象都包装了一个底层的 ly 对象，该对象持有实际的工艺数据。
  • is_original_name 标志用于区分是通过 TED 映射名还是 PDK 原始名获取的层级，可能影响某些方法的行为（如 reload）。

2. 特殊方法 (Special Methods)

  • __str__(): 返回层级的字符串表示。
  • __eq__, __ne__, __gt__, __ge__, __lt__, __le__: 基于层号 (layer) 和数据类型 (datatype) 实现层级对象的比较运算，允许排序和判断相等性。
  • __hash__(): 基于层号和数据类型计算哈希值，使 Layer 对象可用作字典键或集合元素。
  • __rshift__(other) -> LayerList: 重载 >> 运算符。layer1 >> layer2 返回一个 LayerList，包含从 layer1 到 layer2 之间的所有金属布线层（闭区间）。自动处理顺序。

3. 设计规则查询 (DRC Rule Queries)

提供一系列方法来获取与该层级相关的设计规则：

  • get_min_width(current=None) -> int: 获取最小宽度。可选 current 参数考虑电流密度影响 (EM)。
  • get_min_diagonal_width() -> int: 获取 45 度方向最小宽度。
  • get_max_width() -> int: 获取最大宽度。
  • get_min_width_by_current(current) -> int: 根据电流计算最小宽度。
  • get_width(via=1, level=ALL) -> int: 计算承载 via 个通孔（考虑 level 指定的金属层包围）所需的尺寸。
  • get_min_spacing(other=None) -> int: 获取最小间距（层内或与其他层 other 之间）。
  • get_min_area() -> int: 获取最小面积。
  • get_via_max_spacing() -> int: 获取通孔最大间距（仅 Via 层）。
  • get_max_en(other=None) -> int: 获取最大包围 (Enclosure)。
  • get_min_en() -> int: 获取最小包围。
  • get_upper/lower_metal_max/min_en() -> int: 获取相对于上/下金属层的最小/最大包围（主要用于 Via 层）。
  • check_spacingtable(paral, width1, width2, length1=None, length2=None) -> int: 检查基于平行长度和宽度的复杂间距规则。
  • get_min_width_from_area(value) -> int: 根据给定边长 value 计算满足最小面积所需的另一边最小长度。
  • get_max_current_density() -> float: 获取最大允许电流密度。
  • has_min_width(), has_max_width(), etc.: 判断该层级是否定义了相应的规则。
  • get_whole_en() -> int: 获取 "whole enclosure" 规则。

4. 层级属性与关系 (Layer Properties and Relationships)

  • layer() -> int, datatype() -> int: 获取层号和数据类型。
  • get_name() -> str, get_full_name() -> str: 获取 TED 名称和完整名称 (名称-用途)。
  • get_original_name() -> str, get_original_purpose() -> str, get_original_full_name() -> str: 获取 PDK 原始名称和用途。
  • get_stdcell_name() -> str: 获取标准单元库中对应的层名。
  • is_drawing() -> bool: 判断用途是否为 "drawing"。
  • can_via() -> bool: 判断是否可在该层上打孔（通常是金属层）。
  • is_via_layer(), is_route_layer(), is_route_aux_layer(), is_pin_layer(), is_obs_layer(): 判断层级类型。
  • is_thick_metal() -> bool: 判断是否为厚金属层。
  • is_ted_defined_layer() -> bool: 判断是否为 TED 自定义层级。
  • lower_metal(), lower_via(), upper_metal(), upper_via() -> Layer: 获取相邻的层级。
  • has_lower_metal(), has_lower_via(), etc.: 判断相邻层级是否存在。
  • lower_metal_obs(), upper_metal_obs() -> Layer: 获取相邻的障碍物层（仅 OBS 层）。
  • metal() -> Layer: 获取对应的金属层 (从 OBS/PIN 层)。
  • obs() -> Layer: 获取对应的障碍物层 (从 Metal/PIN 层)。
  • pin() -> Layer: 获取对应的 Pin 层 (从 Metal/OBS 层)。
  • get_upper() -> Layer: 获取层级在典型连接堆栈中的上一层 (Metal->Via, Via->Metal, AA->CO)。
  • get_index_from_route_layer(), get_index_from_via_layer() -> int: 获取在全局布线/通孔层列表中的索引。

5. 实用与辅助方法 (Utilities and Helpers)

  • export_lef_info() -> str: 导出 LEF 格式信息。
  • get_count(limit, width=None, spacing=None) -> int: 计算给定 limit 长度内可放置的最小矩形数量。
  • get_bboxs_consider_en(via=1, metal_layer=None) -> List[Box]: 计算承载 via 个通孔（考虑不同排列方式和指定 metal_layer 包围）所需的最小包围盒列表 (Box 对象)。
  • reload(): 重新从底层 (rsted) 加载层级数据，更新内部状态。

6. 静态方法 (Static Methods)

  • get_layer_from_number(layer, datatype) -> Layer: 根据层号和数据类型获取 Layer。
  • from_str(s) -> Layer: 从字符串表示解析 Layer。
  • has_layer(name) -> bool: 检查是否存在指定名称的层级。
  • get_top_metal(), get_bottom_metal(), get_top_via(), get_bottom_via() -> Layer: 获取顶/底层金属/通孔。
  • is_ted_declared_layer(layer_name) -> bool: 检查层名是否在 TED 配置中声明。
  • get_layers(bottom, top, reverse=False) -> LayerList: 获取 bottom 到 top 的金属布线层列表。
  • get_connect_layers(bottom, top, reverse=False) -> LayerList: 获取 bottom 到 top 的完整连接路径（含 Via）层级列表。
  • get_all_layers(is_original=False, func=None) -> LayerList: 获取所有层级（TED 或原始 PDK），可选择过滤。

7. YAML 配置交互 (YAML Configuration Interaction)

  • get_layeryaml(): 读取关联的层级 YAML 配置文件。
  • update_yaml(key, value): 更新关联 YAML 文件中的值。
  • update_min_spacing_to(other, spacing): 更新到 other 层的最小间距规则。

B. LayerList 类 (layer_list.py)

LayerList 是一个继承自列表（或行为类似列表）的容器类，专门用于存储 Layer 对象的有序序列。

1. 初始化与访问

  • __init__(*layers): 使用任意数量的 Layer 对象初始化列表。内部维护列表和字典（用于快速查找索引），并验证无重复项。
  • __getitem__(item): 支持整数索引和切片，切片端点可以是整数或 Layer 对象。
  • __len__(): 返回列表长度。
  • __eq__(other): 比较列表内容是否相等。
  • __str__(): 返回列表的字符串表示。
  • append(layer), extend(layers): 添加单个或多个 Layer 对象。

2. 导航

  • get_next(layer): 获取指定 layer 的下一个元素。
  • get_prev(layer): 获取指定 layer 的上一个元素。

3. 聚合规则计算 (Aggregate Rule Calculation)

计算列表中所有层级的规则聚合值（通常是最大值，表示最严格的约束）：

  • get_width(via=1) -> int: 最大 get_width() 值。
  • get_min_width(current=None) -> int: 最大最小宽度。
  • get_min_spacing() -> int: 最大最小间距。
  • get_min_en() -> int: （仅 Via 层）最大最小包围。
  • get_max_en() -> int: （仅 Via 层）最大最大包围。

4. 列表转换 (List Transformation)

  • get_via_layers(lower_via=False, upper_via=True) -> LayerList: 获取列表中金属层之间的通孔层。
  • obs() -> LayerList: 返回包含列表中每个金属层对应障碍物层的新 LayerList。

C. ViaConfig 类 (via_config.py)

ViaConfig 类用于方便地配置和计算通孔阵列所需的尺寸。

1. 初始化 (__init__)

class ViaConfig:
    def __init__(self, via_layer=None, num=1, level=ALL):
        # via_layer: 通孔层 Layer 对象 (默认 VIA1)
        # num: 通孔数量 (默认 1)
        # level: 计算尺寸时考虑的金属层范围 (ALL, UPPER, LOWER)
        # ...

2. 属性 (Properties)

  • via_layer -> Layer: 获取通孔层。
  • num -> int: 获取通孔数量。
  • level -> str: 获取金属层考虑范围。

3. 核心方法 (eval)

  • eval() -> int: 计算并返回基于当前配置所需的尺寸（调用 self.via_layer.get_width()）。结果会被缓存。

4. 工厂方法 (from_value)

  • from_value(*args) -> ViaConfig: 静态方法，可以从 int, Layer, str 参数组合创建 ViaConfig 实例。

5. 特殊方法 (Special Methods)

  • __repr__(): 提供详细的字符串表示。
  • __call__(): 允许像函数一样调用对象，返回 eval() 结果。
  • __int__(): 允许转换为整数。
  • 算术和比较运算符 (__add__, __sub__, __mul__, __truediv__, __floordiv__, __ge__, __gt__, etc.): 允许 ViaConfig 对象直接参与运算和比较，运算时自动调用 eval() 获取尺寸值。

III. 层级定义与辅助函数 (layers.py)

此文件主要负责定义可直接在代码中使用的层级常量，并提供获取特定层级组的便捷函数。

1. 层级常量定义

通过调用 get_layer("层级名称")（来自 tedcore.kernel.src.circuit）为常见的工艺层级创建 Python 变量。

  • 基础层级: TAP, AA, OD, PO, POLY, GT (包含别名)。
  • 金属层: M1 到 M9。
  • 通孔层: CO, VIA1 到 VIA8。
  • 注入/阱/其他: PP, RH, RPDMY, RPO, NP, PW, NW, DNW, TEXT。
  • 障碍物层: PO_OBS (M0_OBS), M1_OBS 到 M9_OBS, CO_OBS (V0_OBS), V1_OBS 到 V9_OBS。
  • 注释掉的层级: 如 AP, VIA9, Pin 层 (M1-PIN 等) 表明当前配置下未使用或未完全支持。

2. 辅助函数

  • get_route_layers() -> List[Layer]: 返回所有金属布线层 (GT, M1, M2, ...)。
  • get_via_layers() -> List[Layer]: 返回所有通孔层 (CO, VIA1, VIA2, ...)。
  • get_mix_layers() -> List[Layer]: 返回交错的金属与通孔层列表 (GT, CO, M1, VIA1, ...)。
  • get_route_obs_layers() -> LayerList: 返回所有布线层对应的障碍物层列表。
  • get_all_layers() -> List[Layer]: 返回当前工艺定义的所有层级对象列表。
  • reload_layers(): 重新加载本文件中定义的所有 Layer 常量，使它们获取最新的底层数据。

VI. 依赖关系 (Dependencies)

layer 模块包主要依赖于：

  • tedcore.kernel.src.circuit: 提供 get_layer 和 get_circuit 函数来访问当前加载的电路/PDK 信息。
  • rsted: （推测）底层的 C++ 或 Python 库，用于实际加载和处理 PDK 数据，提供 Layer 类所包装的 ly 对象以及全局层级信息。
  • ted_utils: 提供辅助常量 (ALL, UPPER, LOWER), 工具函数 (debug, ensure_int_multiple, CEIL, factor_combinations) 和可能的 LayerYaml 类用于 YAML 配置交互。
  • pyted: （在测试代码中）测试框架和版图生成 DSL。
  • typing: 用于 Python 类型提示。
