版图基础

写带版图信息的模块

对于 block 级别的模块，具有版图信息的子电路实例化，并通过布局API完成摆放后，当前模块就具有版图信息。

对于最小单元的模块，如PCell，需要通过实例化具体的几何图形，并通过布局API完成摆放来完整模块的版图信息。

基础几何形状

在版图设计中最基础的单元是 Pcell，TED 可以搭建 Pcell。设计 Pcell 版图需要将基础几何形状：矩形、多边形等进行组合。

矩形

TED中最常用的基础几何形状是矩形，下面的例子描述了一个M1层的矩形，其长度（x方向的尺度）和宽度（y方向的尺度）分别为l和w，其左下角放置在(x, y)位置，该矩形电学上连接到节点n。

r=Rect(M1, l, w).move_to(x, y) % n
# 或者
r=Rect(M1, l, w, connectTo = n).move_to(x, y)

在TED中，根据Rect所在的图层不同，有两种类型，一种用于描述电学连接的矩形。所有的金属层，poly层，ct，via都属于RRect。其它的层，如NP，PP, AA，NW，TEXT等图层的矩形本身没有电学节点的概念，因此在TED中被认为是普通的矩形而不能进行电学连接。需要注意的是，衬底和N-阱严格来讲也有电学连接的意义，但是是比较特殊的一种电学连接，在TED中暂不进行考虑，而认为这些层的矩形不参与电学连接。

需要注意的是，在TED的实现中，这里一个矩形Rect实际上就是TED中的一个特殊的Module，而一个矩形的实例也是一个TED中标准的实例（Instance）。因此Module和Instance中具备的所有成员函数在Rect和它的实例r中都会继承。此外，从上面的代码可以看出来，每个Rect具有一个端口，这个端口属于控制端口，因此可以也应该用%来描述该矩形对应的电学节点。

多边形

TED 中较为少见的基础几何形状是多边形，应该尽可能少用。下面的例子描述了一个M1层的凸形多边形，其点集应该是(0, 0), (3, 0), (3, 1), (2, 1), (2, 2), (1, 2), (1, 1), (0, 1)，该多边形电学上连接到节点n。

points = [(0, 0), (3, 0), (3, 1), (2, 1), (2, 2), (1, 2), (1, 1), (0, 1)]
polygon = Polygon(M1, points, connectTo=a)

基本布局语法

基础布局主要有三种横向布局、纵向布局和重叠布局。

  • row: 水平摆放
  • col: 竖直摆放
  • lay: 重叠摆放

row

row 将器件以 spacing 间距摆成一行。其器件参数主要是矩形或者子电路实例。另外还有两个可选参数

  • 参数spacing是用来控制水平摆放器件之间的间距，默认值是0；
  • 参数align是用来控制Y轴方向，有三个选项，分别是：
    • BOTTOM: 向下对齐，是默认值
    • MIDDLE: 居中对齐
    • TOP: 向上对齐
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
row(r1, r2, spacing=1, align=BOTTOM)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
row(r1, r2, spacing=1, align=MIDDLE)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
row(r1, r2, spacing=1, align=TOP)

col

col和row的使用方式类似，不同之处在于col是控制的竖直走向的器件摆放而已。略有区别的是参数align是用来控制X轴方向，有三个选项

  • LEFT：向左对齐，是默认值
  • CENTER：居中对齐
  • RIGHT：向右对齐
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
col(r1, r2, spacing=1, align=LEFT)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
col(r1, r2, spacing=1, align=CENTER)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
col(r1, r2, spacing=1, align=RIGHT)

lay

lay的器件参数，可以直接是矩形，也可以是(矩形，x偏移量,y偏移量)。相当于两个矩形重叠摆放。一共有两个参数

  • 参数x_align是用来控制X轴方向，有三个选项
    • LEFT：向左对齐，是默认值
    • CENTER：居中对齐
    • RIGHT：向右对齐
  • 参数y_align是用来控制Y轴方向，有三个选项
    • BOTTOM：向下对齐，是默认值
    • MIDDLE：居中对齐
    • TOP：向上对齐

注：移动到原点后，器件原有坐标会作为偏移量。m1坐标（X1,Y1），m2坐标 (X2,Y2)，lay(m1,m2) 实际效果：将 m1 m2 左下角重叠，并移动到（0，0），然后将 m1.move_by(X1,Y1), m2.move_by(X2,Y2)。

r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
lay(r1, r2, x_align=LEFT)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
lay(r1, r2, x_align=CENTER)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
lay(r1, r2, x_align=RIGHT)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
lay(r1, r2, y_align=BOTTOM)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
lay(r1, r2, y_align=MIDDLE)
r1 = Rect(M1, 1, 2)
r2 = Rect(M2, 2, 3)
lay(r1, r2, y_align=TOP)

布局结束

布局结束请手动调用place()，表明当前版图布局已经结束。此时 TED 会自动检查当前布局是否短路、满足DRC等规则要求。

自动短路检查

将两个连接不同节点的矩形摆放在一起，会报短路错误，

r1 = Rect(M1, 1000, 2000, connectTo="a")
r2 = Rect(M1, 1000, 2000, connectTo="b")
lay(r1, r2)
place()

自动打孔

TED系统会自动检查连接到同一个节点的金属层，根据DRC规则文件将其打满VIA。

@module
def demo(a):
    r1 = Rect(M1, 1000, 2000, connectTo=a)
    r2 = Rect(M2, 1000, 2000, connectTo=a)
    lay(r1, r2)
    place()

@testbench
def Top():
    demo()

@module
def Demo(a, b, c, d):
    "d" >> (m1 := PMOS(l=lmin()) % "g") >> "s"
    "d" >> (m2 := PMOS(l=2 * lmin()) % "g") >> "s"
    "d" >> (m3 := PMOS(l=4 * lmin()) % "g") >> "s"

    col(row(m1, m2, spacing=100), row(m3, m4, spacing=100), spacing=100)
    place()
