电路仿真

摘要

本文档详细介绍了 ted_sim 库，一个用于电路仿真和测量的 Python 工具集。该库提供了丰富的功能，包括执行各种电路仿真（如 AC, DC, TRAN, Noise, STB, PSS, PNoise 等），处理和分析仿真波形数据，进行参数化测量，以及支持 Monte Carlo 分析和 Data Sweep。文档首先介绍了基础的仿真命令函数（如 ac, dc, tran 等），随后讲解了核心的波形与结果处理模块（Waveform, SweepWaveform, MeasureResult, MCresult 等），接着说明了数据扫描（DataSweep, DataSweepPro）和 MDL 语法辅助类 (MDL)。之后，详细介绍了上层的测量类 Measure 和 AlpsMeasure，以及用于组织多步仿真的 AnalysisPro 类。最后，提供了完整的使用样例和常见问题解答，旨在帮助用户从基础到进阶地理解和使用该库。

1. 仿真命令函数

库中直接提供了与仿真类型对应的函数，作为执行电路仿真的基础接口。这些函数通常在 @testbench 装饰的函数内部调用，是对底层仿真器命令的封装，简化了调用过程。它们通常返回包含波形数据的 Waveform 或 SweepWaveform 对象。

核心功能:

  • 直接调用各种类型的电路仿真。
  • 接受仿真参数（如时间、频率范围、扫描设置等）。
  • 返回包含原始波形数据的对象。

常用仿真命令:

  • ac(start=100, stop=1e9, type="dec", ND=10, options="", stage=None, settings={}, **kwargs):

    • 功能: 执行 AC (交流) 分析。
    • 参数:
      • start, stop: 频率扫描范围。
      • type, ND: 扫描类型和每十倍频点数 (Hspice/Ngspice 等)。
      • settings: Spectre/Alps 特定设置 (字典)。
      • options: 附加仿真选项。
      • stage: 'pre' 或 'post'。
    • 示例 (来自 test_simcmd.py):
      # AC 仿真，测量电容
raw = ac(
    start=1,
    stop=1.001,
    settings={
        "dev": "vVdetect", # Spectre特定参数
        "freq": f"{0.5/np.pi}", # Spectre特定参数
        "step": 0.001, # Spectre特定参数
    },
)
debug(abs(raw.get_signal("iIdetect:in"))) # 获取仿真结果电流

  • dc(options="", stage=None, **kwargs):

    • 功能: 执行 DC (直流) 工作点分析。
    • 参数:
      • options: 附加仿真选项 (如 save *:oppoint depth=100)。
      • stage: 'pre' 或 'post'。
    • 示例 (来自 test_psf.py):
      @testbench
def Top():
    VSource(Vdd(), Vss(), dc=1)
    Resistor(Vdd(), "mid", r=500)
    Resistor("mid", Vss(), r=500)
    raw = dc() # 执行DC仿真
    debug(raw.get_signal_names()) # 查看保存的信号名
    debug(raw.get_signal("mid")) # 获取mid节点的直流电压

  • tran(stop=1e-8, step=1e-10, start=0.0, errpreset="moderate", options="", stage=None, settings={}, **kwargs):

    • 功能: 执行 TRAN (瞬态) 分析。
    • 参数:
      • start, stop, step: 时间扫描参数。
      • errpreset: 仿真精度 ('conservative', 'moderate', 'liberal')。
      • options: 附加仿真选项。
      • stage: 'pre' 或 'post'。
      • settings: Spectre/Alps 特定设置。
    • 示例 (来自 test_simcmd.py):
      @testbench
def tran_test():
    D = from_analogLib(2, model="vsource", type="sine", dc="0", ampl="1", freq=10e6)
    C("vout", Vss())
    ("vout", Vss()) >> D
    raw = tran(stop=1e-6) # 执行瞬态仿真到 1us
    watch("vout") # 声明要观察 vout 节点
    raw.show() # 显示 vout 波形

  • dcsweep(param, start, stop, step, dev=None, options="", stage=None, data=None, **kwargs):

    • 功能: 执行 DC Sweep (直流扫描) 分析。
    • 参数:
      • param: 扫描的参数名。
      • start, stop, step: 扫描范围和步长。
      • dev: 扫描的器件实例名 (可选)。
      • options: 附加仿真选项。
      • stage: 'pre' 或 'post'。
      • data: DataSweep 对象，用于执行多参数扫描。
    • 示例 (来自 test_psf.py):
      @testbench
def Id_Vd():
    "vd" >> NMOS(l=0.6e-6, w=6e-6) % "vg" >> Vss()
    ("vd", Vss()) >> VSource(dc=1, name="vds") # 定义vd电压源
    ("vg", Vss()) >> VSource(dc="vgs", name="vgs") # 定义vg电压源, 其dc值由datasweep控制
    # 定义DataSweep，扫描vgs从0到1.2V，步长0.1V
    datasweep = DataSweep(
        name="sweepdata", params=["vgs"], values=[np.arange(0, 1.2, 0.1)]
    )
    # 执行DC扫描，扫描vvds的dc参数从0到1.2V，步长0.01V
    # 同时应用上面定义的datasweep (扫描vgs)
    raw = dcsweep(
        dev="vvds", # 扫描vds电压源
        param="dc",
        start=0,
        stop=1.2,
        step=0.01,
        data=datasweep, # 应用vgs扫描
    )
    watch("vvds:p", calc=lambda x: -x) # 观察漏极电流(源的p端电流取反)
    raw.show() # 显示所有扫描曲线

  • noise(n1="", iprobe="", start=10, stop=2e9, options="", stage=None, settings={}, **kwargs):

    • 功能: 执行 Noise (噪声) 分析。
    • 参数:
      • n1: 输出噪声节点对 (如 "vout,0")。
      • iprobe: 输入噪声源实例名。
      • oprobe: 输出探针实例名 (Spectre)。
      • start, stop: 频率范围。
      • options: 附加仿真选项。
      • stage: 'pre' 或 'post'。
      • settings: Spectre/Alps 特定设置。
    • 示例 (来自 test_simcmd.py):
      @testbench
def noise_test():
    R("vin", "vout", r=10e3)
    V("vin", Vss(), dc=1).rename("Vin") # 定义输入电压源Vin
    # 执行噪声仿真，输出为vout和Vss之间，输入源为vVin
    raw = noise("vout,0", iprobe="vVin")
    watch("in") # 观察输入参考噪声
    watch("out") # 观察输出噪声
    raw.show() # 显示噪声谱

  • stb(start=100, stop=1e9, probe="IPRB0", options="", stage=None, **kwargs):

    • 功能: 执行 STB (稳定性) 分析。
    • 参数:
      • start, stop: 频率范围。
      • probe: 环路中断处的探针实例名。
      • options: 附加仿真选项。
      • stage: 'pre' 或 'post'。
    • 示例 (来自 test_analog.py, Measure.stb内部调用):
      @testbench
def stb_test():
    # ... 电路定义 ...
    IProbe(VINN, "vout").rename("probe") # 在环路中插入IProbe
    V(VINP, Vss(), dc=0.6)
    # Measure.stb 内部会调用 stb() 仿真函数
    raw_measure = Measure.stb(probe="iprobe", ...)
    # 如果直接调用stb():
    # raw_waveform = stb(probe="iprobe", start=1, stop=1e10)
    # debug(raw_waveform.get_signal("loopGain")) # 获取环路增益波形

  • pss(func=1e9, harms=12, tstab=0.1e-6, osc=False, options="", stage=None, **kwargs):

    • 功能: 执行 PSS (周期性稳态) 分析。
    • 参数:
      • func: 基波频率。
      • harms: 谐波数量。
      • tstab: 稳定时间。
      • osc: 是否为振荡器仿真 (需要指定 n1, n2)。
      • options: 附加仿真选项。
      • stage: 'pre' 或 'post'。
    • 示例 (来自 test_psf.py):
      @testbench
def Top():
    # ... 定义一个带时钟的电路 ...
    pss() # 执行PSS分析 (默认参数)

  • pnoise(start=1, stop=1e9, dec=100, sweeptype="default", noiseout=["usb"], stage="pre", **kwargs):

    • 功能: 执行 PNoise (周期性噪声) 分析 (通常在 PSS 之后)。
    • 参数:
      • start, stop, dec: 频率扫描范围。
      • sweeptype: 扫描类型 ('default', 'absolute', 'relative')。
      • noiseout: 输出噪声类型 (如 'usb', 'ssb')。
      • stage: 'pre' 或 'post'。
    • 注意: PNoise 通常在 AnalysisPro 流程中与 pss 结合使用。

  • pz(positive_output, negative_output, vsim_source, options="", stage=None, **kwargs):

    • 功能: 执行 PZ (零极点) 分析。
    • 参数:
      • positive_output, negative_output: 输出节点对。
      • vsim_source: 输入激励源实例名。
      • options: 附加仿真选项。
      • stage: 'pre' 或 'post'。

  • tds(stop=1e-8, step=1e-10, nproc=10, **kwargs):

    • 功能: 执行 TDS (时间分割) 瞬态分析 (加速长瞬态仿真)。
    • 参数:
      • stop, step: 时间参数。
      • nproc: 并行进程数 (分割数)。
    • 示例 (来自 test_tds.py):
      @testbench
def Top():
    # ... 电路定义 ...
    raw = tds() # 执行TDS仿真
    raw.show()

2. 波形与结果处理

仿真命令执行后，需要处理其输出。这包括处理波形数据和解析测量结果。

2.1 波形对象 (Waveform, SweepWaveform)

Waveform 类用于存储和处理单次仿真运行产生的波形数据。SweepWaveform 则用于处理 Data Sweep 或 Monte Carlo 仿真产生的多组波形数据。

Waveform 核心功能:

  • 加载波形: 可以从 PSF 文件 (Waveform.from_psf_file) 或其他 Waveform 对象加载。也可以通过 Waveform(data=...) 直接从包含 'sweeps', 'signal_names', 'signals' 的字典创建。
  • 获取信号信息:
    • get_signal_names(): 获取所有信号名称列表。
    • get_sweeps(): 获取扫描变量的值 (如时间或频率)。
  • 获取信号数据:
    • get_signal(node, pos=None, signal_type='diff'): 获取指定节点 (node) 的信号值。
      • pos: 可选，用于插值获取特定扫描点的值。
      • signal_type: 获取信号的类型 ('diff', 'VM', 'VP', 'VI', 'VR')，主要用于复数信号 ('VM': 幅度, 'VP': 相位, 'VR': 实部, 'VI': 虚部, 'diff': 复数或实数本身)。
    • get_raw_signal(name, signal_type='diff'): 获取原始信号数据 (通常是 TorchVariable 或 numpy.ndarray)。
    • get_complex_signal(name): 获取复数信号 (numpy array)。
  • 数据处理:
    • interpolate(node, x): 对信号进行线性插值。
    • sample(node, start, step): 按指定步长采样信号。
    • to_array(): 将所有信号转换为 numpy 数组 (通常用于检查数据维度)。
    • measure(mod='offline', expr={}): 使用 Ocean 表达式进行离线测量 (需要 spectre 仿真器)。
  • 绘图:
    • plot(*args, **kwargs): 绘制指定信号的波形图。
    • watch(*args, module=None, **kwargs): 声明需要观察的信号，并设置绘图参数 (如 mode, calc, ylabel, xlabel)。watch 不会立即绘图。
    • show(figname=None, path=None, **kwargs): 显示所有通过 watch() 声明要观察的信号图，并可选择保存到文件。
    • eyediagram(signame, period): 绘制眼图。
  • Monte Carlo: get_mc_result(): 获取关联的 Monte Carlo 结果对象 (MCresult)。
  • 其他: is_dc(), is_swept(), set_rawpath(), get_rawpath()。

SweepWaveform 核心功能:

  • 存储多组波形: 内部包含一个 Waveform 对象列表，每个对象对应一次 sweep 或 Monte Carlo 运行。
  • 访问单个波形: 可以通过索引访问 (raw[index]) 获取对应运行的 Waveform 对象。
  • 获取信号: get_signal(node, pos=None, signal_type='diff') 方法会返回一个包含所有 sweep/MC 运行结果的列表。
  • 绘图: plot() 和 show() (配合 watch) 方法会绘制所有 sweep/MC 运行的曲线在同一张图上。
  • 其他: 继承 Waveform 的大部分方法，并根据多波形特性进行调整。

2.2 仿真结果读取与解析 (PSFMgmt, PSFReader, MeasureResult, AlpsMeasureResult)

这些类用于从仿真器生成的特定格式文件中加载和解析数据。PSFMgmt/PSFReader 主要处理仿真日志文件 (logFile) 以定位和加载波形数据文件，而 MeasureResult/AlpsMeasureResult 则处理由 MDL 或 AlpsMeasure 生成的标量测量结果文件。

  • PSFMgmt(raw_path, en_parallel=True):

    • 功能: 管理和解析 Spectre 仿真器产生的 logFile。
    • parse(threshold=200, max_workers=4): 解析 logFile，提取仿真分析实例 (AnalysisInst) 信息，并构建父子关系树。返回父节点列表 (List[AnalysisInstParser])。支持根据节点数量自动选择并行解析。
    • get_raw_path(): 返回原始结果目录路径。
    • show_info(): 打印解析出的节点信息。

  • AnalysisInstParser: (由 PSFMgmt 内部使用)

    • 功能: 表示 logFile 中的一个分析实例。
    • 属性: analysis_inst, analysis_type, file_name, data_format, parent_analysis, properties, children, is_parent, simtype。
    • get_simtype(): 获取该分析实例对应的仿真类型。

  • PSFReader: (由 ted_sim 内部仿真流程使用)

    • 功能: 负责根据 PSFMgmt 解析的信息加载实际的波形数据 (PSF 文件)，并创建 Waveform 或 SweepWaveform 对象。
    • load(module, psfmgmt): 执行加载。
    • sim_results (属性): 存储加载后的结果字典，键为仿真类型 (如 'ac', 'tran')，值为 Waveform/SweepWaveform 对象。
    • __getitem__(key): 通过仿真类型获取结果。
    • get_all_sim_results(): 获取所有仿真结果的类型列表。

  • MeasureResult / AlpsMeasureResult:

    • 功能: 解析和存储由 Measure 或 AlpsMeasure 产生的标量测量结果 (通常是 .mt0, .ma0 等文件)。
    • load(measure_file_path): 从测量结果文件加载数据。会清理文件头信息并解析数据。
    • perf (属性): 存储解析后的结果字典，键是测量指标名称 (如 'gain', 'pm')，值是 TorchVariable 对象 (支持自动微分)。对于 Sweep 结果，值会是一个 TorchVariable 列表。
    • __getitem__(key): 通过指标名称访问结果。
    • keys(), items(): 类似字典的方法。
    • load_mc_result(mckeys, measure_file_path): (仅 MeasureResult) 加载 Monte Carlo 测量结果。
    • get_mc_result(): (仅 MeasureResult) 获取关联的 MCresult 对象。
    • AlpsMeasureResult 额外包含灵敏度信息，可通过 .jacobian() 方法访问。

2.3 Monte Carlo 结果 (MCresult, subMcResult)

MCresult 用于处理和分析 Monte Carlo 仿真产生的大量标量结果。subMcResult 则用于存储和分析单个性能指标的 MC 统计数据。

  • MCresult:

    • 加载结果:
      • load(keys, mcFile): 从 Spectre 的 .mcdata 文件加载。
      • load_with_measure(keys, content): 从 MeasureResult 解析出的内容加载。

    • 属性:
      • raw: 原始数据列表/数组。

    • 访问统计值: sub_result['min'], sub_result['mean'] 等。

3. 数据扫描 (Data Sweep) (DataSweep, DataSweepPro)

这两个类用于定义和管理参数扫描 (Data Sweep)，允许用户方便地对电路参数进行扫描仿真。

  • DataSweep(name, params, values):

    • 功能: 定义一个基本的数据扫描。
    • 参数:
      • name: sweep 的名称 (通常是 'sweepdata')。
      • params: 要扫描的参数名列表 (字符串列表)。
      • values: 参数值的 numpy 数组。注意: 输入时，values 的形状应为 (num_params, num_sweep_points)，即每一行对应一个参数的扫描值。内部存储时会转置。
    • to_netlist(): 生成用于仿真器网表的 sweep 定义语句 (内部使用)。

  • DataSweepPro(name, params, values, nproc):

    • 功能: DataSweep 的扩展，支持将 sweep 任务分割给多个进程并行执行，以加速仿真。
    • 参数:
      • name, params, values: 同 DataSweep。
      • nproc: 并行进程数 (即将扫描任务分割成的份数)。
    • build(): 根据 nproc 将 values 分割成多个子 DataSweep 对象。
    • to_netlist(): 返回一个包含多个子 sweep 网表定义的列表 (内部使用)。
    • get_datasweeps(): 获取分割后的 DataSweep 对象列表。
    • convert_to_datasweep(): (静态方法) 合并所有子 sweep 回一个 DataSweep 对象 (内部使用)。

用法:

  1. 使用 pyted.Variable 定义需要在电路中扫描的参数。
  2. 在 @testbench 函数中，创建 DataSweep 或 DataSweepPro 对象，提供 sweep 名称、参数名列表和对应的扫描值数组。
  3. 将创建的 sweep 对象赋值给 get_circuit().datasweep。
  4. 对于 DataSweepPro，在执行仿真前调用 get_circuit().parallel.start(nproc=N) 启动并行进程，仿真后调用 get_circuit().parallel.stop() 停止。
  5. 调用仿真命令函数 (如 ac, dcsweep, tran) 或 Measure 方法。这些函数会自动识别 get_circuit().datasweep 并执行扫描。
  6. 扫描仿真的结果通常是 SweepWaveform 对象 (对于仿真命令) 或包含列表的 MeasureResult 对象 (对于 Measure 方法)。

4. MDL 语法辅助 (MDL)

MDL 类提供了一系列静态方法，用于生成 Spectre MDL (Measurement Description Language) 函数的字符串表示。这有助于在 Measure 类方法的 measure 参数中构建复杂的或自定义的 MDL 测量表达式。

核心功能:

  • 将常用的 MDL 函数调用转换为 Python 方法调用。
  • 生成符合 MDL 语法的字符串，可以直接嵌入测量脚本。

主要方法:

  • MDL.risetime(sig, initval, finalval, theta1=10, theta2=90, **kwargs)
  • MDL.falltime(sig, initval, finalval, theta1=90, theta2=10, **kwargs)
  • MDL.cross(sig, dir='rise', n=1, thresh=0.6, **kwargs)
  • MDL.slewrate(sig, initval, finalval, theta1, theta2, **kwargs)
  • MDL.bw(sig, response='high', **kwargs)
  • MDL.max(arg)
  • MDL.min(arg)
  • MDL.gainBwProd(sig)
  • MDL.ph(sig, wrap='yes')
  • MDL.phasemargin(sig)

用法:

直接调用类方法生成 MDL 函数字符串，然后将其作为值传递给 Measure 类方法中的 measure 字典参数。

# 生成 risetime 函数字符串
risetime_expr = MDL.risetime(sig="V(out)", initval=1e-9, finalval=2e-9, theta1=10, theta2=90)
print(risetime_expr)
# 输出: risetime(V(out), initval=1e-09, inittype='x, finalval=2e-09, finaltype='x, theta1=10, theta2=90 )

# 在 Measure.tran 中使用自定义的 risetime 测量
@testbench
def my_tran_test():
    # ... 电路定义 ...
    raw = Measure.tran(
        stop=10e-9,
        measure={
            "my_custom_risetime": risetime_expr, # 使用生成的 MDL 字符串
            "output_max": MDL.max("V(out)") # 测量输出最大值
        }
    )
    debug(f"Custom Risetime: {raw['my_custom_risetime']}")
    debug(f"Max Output: {raw['output_max']}")

5. 测量模块 (Measure, AlpsMeasure)

这两个类提供了执行标准电路性能测量的高级接口。它们封装了底层的仿真命令（如 sim 或相应的仿真函数）和 MDL/Alps 测量语句，使用户能够通过简单的函数调用直接获取性能指标值。Measure 用于标准的 Spectre 仿真器，而 AlpsMeasure 用于 Alps 仿真器并支持灵敏度分析。

5.1 标准测量 (Measure)

核心功能:

  • 执行 AC, DC, TRAN, Noise, STB 等多种类型的仿真和测量。
  • 提供常用性能指标（如增益、带宽、相位裕度、噪声、转换速率、建立时间、延迟等）的直接计算方法。
  • 支持通过 measure 参数传入自定义 MDL 表达式进行测量。
  • 支持通过 export 参数导出特定节点或工作点信息。
  • 支持 Monte Carlo 分析和 PVT 分析 (通过环境变量控制)。
  • 自动处理仿真结果并返回 MeasureResult 对象。

常用测量方法:

  • Measure.ac(input, output, testname, start, stop, measure={}, **kwargs):

    • 功能: 执行 AC 分析并测量频域性能指标。
    • 参数:
      • input: 输入信号节点名或电压源实例名。
      • output: 输出信号节点名。
      • testname: 需要测量的预定义指标列表 (如 'gain', 'gbw', 'pm', 'bw', 'ugb', 'gain1K', 'gain1M')。
      • start, stop: 频率扫描范围。
      • measure: 自定义 MDL 测量表达式字典 (如 {"gainx1": "abs(V(vout)) @100"})。
    • 返回: MeasureResult 对象。
    • 示例 (来自 test_analog.py):
      # Gain, GBW, PM, BW, UGB 等 AC 参数测量
raw = Measure.ac(
    input="VINN",
    output="vout",
    testname=["gain", "gbw", "pm", "bw", "ugb", "gain1K", "gain1M"],
    measure={"gainx1": "abs(V(vout)) @100"},
    start=1, stop=10e9
)
debug(f"Gain: {raw['gain']}")
debug(f"GBW: {raw['gbw']}")
debug(f"Phase Margin: {raw['pm']}")

  • Measure.dc(export={}, measure={}, **kwargs):

    • 功能: 执行 DC 工作点分析并测量相关指标。
    • 参数:
      • export: 需要导出的节点电压/电流字典 (如 {"vout": "V"})，值为 'V' 或 'I'。
      • measure: 自定义 MDL 测量表达式字典 (如 {"power": "abs(I(vVsupply)*V(VDD))"})。
    • 返回: MeasureResult 对象。
    • 示例 (来自 test_analog.py):
      # 测量功耗和导出 Vout
raw = Measure.dc(
    export={"vout": "V"},
    measure={"power": "abs(I(vVsupply)*V(VDD))"},
)
debug(f"Power: {raw['power']}")
debug(f"Vout DC: {raw['vout']}")

  • Measure.noise(iprobe, oprobe, start, stop, export={"f1": ["in", "1KHz"], "thermal": ["in", "10MHz"]}, measure={}, **kwargs):

    • 功能: 执行噪声分析并提取特定频率点的噪声值。
    • 参数:
      • iprobe: 输入噪声源实例名 (通常是电压源)。
      • oprobe: 输出节点名。
      • start, stop: 频率范围。
      • export: 需要导出的噪声指标字典。键为结果名称，值为 [type, frequency] 列表，type 为 'in' (输入参考) 或 'out' (输出)。
      • measure: 自定义 MDL 测量表达式字典。
    • 返回: MeasureResult 对象。
    • 示例 (来自 test_analog.py):
      # 测量输入参考噪声 @ 1kHz 和 10MHz
raw = Measure.noise(
    iprobe="vVCM", # 输入共模电压源作为噪声输入参考
    oprobe="vout",
    export={"flicker_noise": ["in", "1e3"], "thermal_noise": ["in", "10e6"]}
)
debug(f"Flicker Noise @ 1kHz: {raw['flicker_noise']}")
debug(f"Thermal Noise @ 10MHz: {raw['thermal_noise']}")

  • Measure.sr(output, start=0, step=1e-10, stop=50e-6, export={"pSR": [...], "nSR": [...]}, measure={}, **kwargs):

    • 功能: 测量转换速率 (Slew Rate)。
    • 参数:
      • output: 输出节点名。
      • start, step, stop: 瞬态分析时间参数。
      • export: 定义 SR 测量的参数字典。键为结果名称 (如 'pSR', 'nSR')，值为 MDL slewrate 函数的参数列表 [initval, finalval, inittype, finaltype, theta1, theta2]。
      • measure: 自定义 MDL 测量表达式字典。
    • 返回: MeasureResult 对象。
    • 示例 (来自 test_analog.py):

debug(f"Negative SR: {raw['nSR']}")

  • Measure.stb(probe, start=1, stop=1e10, dec=100, measure={}, settings={}, **kwargs):

    • 功能: 执行稳定性分析 (STB) 并进行测量。
    • 参数:
      • probe: 环路中用于探测的实例名 (如 IProbe)。
      • start, stop, dec: 频率扫描参数。

debug(f"STB UGB: {raw['ugb']}")
debug(f"STB Gain @ 100Hz: {raw['gain']}")

  • Measure.settlingtime(node, initval, finalval, inittype, finaltype, theta, stop, **kwargs):

    • 功能: 测量建立时间。
    • 参数: (参考 Spectre settlingtime 函数)
      • node: 需要测量的节点名。
      • initval, finalval: 测量的起始/结束时间点或电压值。
      • inittype, finaltype: 起始/结束点类型 ('x' 时间, 'y' 电压)。
      • theta: 容差百分比。
      • stop: 瞬态仿真停止时间。
    • 返回: MeasureResult 对象 (包含 'settlingtime' 键)。

6. 仿真流程编排 (AnalysisPro)

AnalysisPro 提供了一种用于组织和执行包含多个不同分析步骤的仿真流程的高级接口。它允许用户链式调用不同的仿真方法来定义一个序列，然后一次性执行。

核心功能:

  • 链式调用: 可以通过 .dc().ac().noise()... 的方式链式定义多个仿真步骤。
  • 配置选项:
    • set_netlist_options(): 设置应用于所有分析的网表选项 (如 include, global)。
    • set_cmd_options(): 设置仿真器命令行选项 (如 +aps)。
  • 执行: run(stage=None, **kwargs) 方法按顺序执行所有定义好的仿真分析。
  • 结果访问: 返回一个字典 (perf)，键是仿真类型的小写形式 (如 'dc', 'ac', 'noise') 或带选项的名称 (如 'pss.td', 'pss.fd', 'pnoise.usb')，值是对应的 Waveform 或 SweepWaveform 对象。

支持的仿真方法 (链式调用):

  • .ac(**kwargs)
  • .dc(**kwargs)
  • .dcsweep(**kwargs)
  • .noise(**kwargs)
  • .pss(**kwargs)
  • .pnoise(**kwargs)
  • .stb(**kwargs)
  • .tran(**kwargs)
  • .manual(analysis_statements, simtype): 添加自定义仿真语句。

用法:

  1. 在 @testbench 函数中创建 AnalysisPro 实例。
  2. 链式调用所需的仿真方法，配置相应的参数。
  3. (可选) 使用 set_netlist_options 或 set_cmd_options 设置全局选项。
  4. 调用 run() 方法执行整个流程。
  5. 通过返回的字典访问各个分析步骤的结果 (Waveform/SweepWaveform)。

示例 (来自 test_analysis.py):

@testbench
def test_multi_analysis():

    # 2. 定义仿真流程：先 dc 保存工作点，再 ac，最后 noise
    analysis_runner.dc(options="save mm1")
    analysis_runner.ac(stop=10e9)
    analysis_runner.noise(positive_output_node="vout", negative_output_node="0", iprobe="vVin")

    # 3. (可选) 设置全局选项
    # analysis_runner.set_netlist_options("ahdl_include 'my_ahdl.va'")
    # analysis_runner.set_cmd_options("+aps", "+mt=8")

    # 4. 执行所有分析
    results = analysis_runner.run()

    # 5. 访问不同分析的结果
    # results 字典的键是 'dc', 'ac', 'noise'
    debug("=== DC Result ===")
    # 注意：AnalysisPro 中 dc 默认返回 op 类型结果
    debug(f"VOUT (DC): {results['op'].get_signal('vout'):.4f} V")

    debug("\n=== AC Result ===")
    ac_vout = results['ac'].get_signal("vout")
    freqs = results['ac'].get_sweeps()
    debug(f"AC Gain @ 1kHz: {T.db20(np.interp(1e3, freqs, ac_vout)):.2f} dB")
    results['ac'].watch("vout", mode="semilogx", calc=T.db20, ylabel="Gain (dB)")
    results['ac'].show(figname="AC_Gain") # 显示 AC 结果

    noise_freqs = results['noise'].get_sweeps()
    debug(f"Input Noise @ 1MHz: {np.interp(1e6, noise_freqs, noise_in)*1e9:.2f} nV/sqrt(Hz)")
    results['noise'].watch("out", mode="semilogx", ylabel="Output Noise (V/sqrt(Hz))")
    results['noise'].show(figname="Noise_Spectrum") # 显示 Noise 结果

8. 常见问题解答 (QA)

  • Q1: 如何选择使用仿真命令函数 (如 ac, tran) 还是 Measure 类?

    • A1:
      • 仿真命令函数 (ac, tran等): 用于执行基础仿真，返回包含完整波形数据的 Waveform 或 SweepWaveform 对象。当你需要详细的波形数据进行后处理、绘图或自定义计算时使用。它们是更底层的接口。
      • Measure 类: 用于执行标准性能指标的测量。它封装了仿真命令和 MDL 测量语句，直接返回计算好的标量性能值 (如增益、带宽、建立时间等) 到 MeasureResult 对象中。当你关心特定的、预定义的性能指标时，使用 Measure 更方便快捷。

  • Q2: Waveform 和 MeasureResult 有什么区别?

    • A2:
      • Waveform / SweepWaveform: 存储仿真的原始波形数据 (电压、电流随时间或频率的变化)。主要用于查看波形、进行基于波形的计算 (如插值、采样、FFT 等) 和绘图。由仿真命令函数 (如 ac, tran) 返回。
      • MeasureResult / AlpsMeasureResult: 存储通过 MDL 或 Alps 测量语句计算得到的标量性能指标 (如增益值、带宽值、相位裕度值等)。主要用于获取和分析这些具体的性能参数。由 Measure 或 AlpsMeasure 类的方法返回。

  • Q3: 如何进行参数扫描 (Data Sweep)?

    • A3:
      1. 使用 pyted.Variable 定义电路中需要扫描的参数。
      2. 创建 DataSweep (单进程) 或 DataSweepPro (多进程) 对象，指定 sweep 名称、参数名列表和对应的扫描值数组 (np.array([[param1_values], [param2_values], ...]))。
      3. 将 sweep 对象赋值给 get_circuit().datasweep。
      4. (如果用 DataSweepPro) 调用 get_circuit().parallel.start(nproc=N) 启动并行，结束后调用 get_circuit().parallel.stop()。
      5. 调用仿真命令函数或 Measure 方法。结果将包含所有扫描点的数据 (SweepWaveform 或带列表的 MeasureResult)。

  • Q4: 如何绘制仿真结果?

    • A4:
      1. 获取 Waveform 或 SweepWaveform 对象 (通常由仿真命令函数返回)。
      2. 使用 watch() 方法声明要绘制的信号以及绘图选项 (如 watch("vout", mode="semilogx", calc=T.db20))。可以多次调用 watch 添加多条曲线或图窗 (new_figure())。
      3. 调用 show(figname="...") 方法来实际生成并显示/保存图形。figname 参数用于指定保存的文件名。

  • Q5: AnalysisPro 的使用场景是什么?

    • A5: 当你需要在一个测试平台中按顺序执行多个不同类型的仿真分析 (例如，先进行 DC 工作点分析，然后基于该工作点进行 AC 分析，最后再进行 Noise 分析) 时，使用 AnalysisPro 最为方便。它允许你链式定义这个流程，并一次性执行，最后将所有分析的波形结果集中返回在一个字典中。如果只需要执行单一类型的分析或测量，直接调用仿真命令函数或 Measure 类方法通常更简单。

  • Q6: 如何获取器件的工作点信息?

    • A6: 在调用 dc() 或 Measure.dc() 时，在 options 参数中添加 save <instance_name>.*:oppoint 或 save *:oppoint 指令。返回的 Waveform 对象 (来自 dc()) 或通过 Measure.dc(export={...}) 导出的 MeasureResult 对象中会包含相应的工作点参数。对于 Waveform，可以通过 get_signal("<instance_name>:<param_name>") (如 get_signal("mm1:gm")) 获取。对于 Measure，可以通过 Measure.op(export={"mm1": "instance"}) 或类似方式导出区域等信息。AnalysisPro 的 .dc() 方法也支持 options 参数。

  • Q7: Monte Carlo 仿真如何设置和获取结果?

    • A7:
      1. 设置: 在调用仿真命令函数或 Measure 方法时，设置 monte_carlo 参数 (如 'process', 'mismatch', 'all')。可以通过 mc_num 指定运行次数，通过 corners 指定 MC 模型库。
      2. 获取结果:
        • 如果使用 Measure 方法，返回的 MeasureResult 对象包含 get_mc_result() 方法，调用此方法可获取 MCresult 对象。
        • 如果直接使用仿真命令函数，需要指定 mc_keys (测量指标名称列表) 和 mc_file (通常是 monteCarlo/mcdata 文件路径)，然后手动创建 MCresult 对象并调用 load() 方法。
      3. 分析: 使用 MCresult 对象及其包含的 subMcResult 对象进行统计分析和绘图 (调用 subMcResult 的 plot() 方法)。

  • Q8: AlpsMeasure 和灵敏度分析主要用于什么目的?

    • A8: AlpsMeasure 主要用于与 alpsmeas 仿真器配合，进行性能指标对设计变量的灵敏度分析。这对于电路优化特别有用，因为它直接计算出改变某个设计参数 (如晶体管的 m) 对性能指标 (如 gain) 的影响程度 (梯度/Jacobian)。如果不需要灵敏度分析或不使用 Alps 仿真器，应使用标准的 Measure 类。
