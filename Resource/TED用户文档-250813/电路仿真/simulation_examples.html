仿真应用案例

1. 基础仿真命令与结果解读

在 TED 中，所有仿真命令通常在 @testbench 装饰的函数内部执行。仿真结果会返回一个 Waveform 对象（或其派生类的对象，如 SweepWaveform），该对象封装了所有仿真数据和常用的数据处理方法。

1.1 直流 (DC) 分析

DC 分析用于确定电路在给定直流偏置条件下的静态工作点，或提取器件的直流参数。

1.1.1 获取电路节点工作点

这是最基础的 DC 分析，用于获取电路中各个节点的电压值。

# 引入必要的 TED 模块
from pyted import testbench, Vss, debug, dc, current_module
from ted_device import VSource, Resistor
from ted_sim import Waveform # 明确 Waveform 类

@testbench
def simple_dc_analysis():
    """一个简单的分压电路的直流分析"""
    # 电路描述
    VSource(Vdd(), Vss(), dc=1.0)  # 定义1V电源
    Resistor(Vdd(), "mid_node", r=500) # R1 = 500 Ohm
    Resistor("mid_node", Vss(), r=500) # R2 = 500 Ohm

    # 执行直流分析
    raw: Waveform = dc() # raw 对象包含了仿真结果

    # 查看可用的信号名称
    debug(f"可访问的信号名称: {raw.get_signal_names()}")

    # 获取并打印 'mid_node' 节点的电压
    # 对于直流分析，get_signal() 返回的是一个标量值 (通常封装在 TorchVariable 或 ndarray 中)
    mid_voltage = raw.get_signal("mid_node")
    debug(f"'mid_node' 节点的电压: {mid_voltage} V") # 理论值为 0.5V

    # 检查仿真类型
    assert raw.is_dc(), "仿真类型应为直流 (op)"

# 运行仿真 (假设的顶层调用)
# simple_dc_analysis()
# get_circuit().run_simulation() # 假设这是 TED 运行仿真的命令

说明与解释:

  • dc(): 执行直流工作点分析。
  • raw.get_signal_names(): 列出仿真结果中所有可供提取的信号名称。这对于不确定信号确切名称时非常有用。
  • raw.get_signal("mid_node"): 提取名为 "mid_node" 的信号值。在 DC 分析中，这通常是该节点的电压。
  • raw.is_dc(): 判断 Waveform 对象是否来自直流分析。

1.1.2 获取 MOSFET 直流参数

DC 分析的另一个重要应用是提取晶体管的详细直流工作点参数，如跨导 (gm)、输出电阻 (gds)、阈值电压 (vth) 等。

from pyted import testbench, Vss, Vdd, debug, dc, get_circuit, skip_unless, TED_SKIP_PDK, HL55, HH180
from ted_device import VSource, NMOS, Mosfet
from ted_sim import Waveform

# 根据PDK类型跳过某些测试
# @skip_unless(TED_SKIP_PDK=[HL55, HH180])
@testbench
def mosfet_dc_parameters():
    """分析NMOS管的直流工作点参数"""
    # 电路描述
    VSource("vg_node", Vss(), dc=0.8).rename("Vgs_source") # 栅极电压0.8V
    VSource("vd_node", Vss(), dc=1.0).rename("Vds_source") # 漏极电压1.0V
    
    # 实例化NMOS管
    # 假设 Mosfet() 默认参数适合观察
    # ("vd_node", "vg_node", Vss(), Vss()) >> Mosfet().rename("m1")
    # 或者使用更具体的 NMOS
    "vd_node" >> NMOS(w=5e-6, l=1e-6) % "vg_node" >> Vss()
    get_circuit().last_module.rename("m1") # 给最后实例化的NMOS命名为m1

    # 执行DC分析，关键在于 options 参数
    # 'save m1' (Spectre) 或 'save m1.*:oppoint' (HSPICE/Spectre)
    # 对于 Spectre， 'save m1' 通常足够获取参数
    # 对于 HSPICE，可能需要更详细的 .option savemod 或 .probe dc ...
    # 假设 TED 内部会根据仿真器智能处理
    # options_str = "save m1" # Spectre 风格
    # 如果使用 HSPICE 或者需要更通用的保存，可以尝试:
    # options_str = "save m1.*:oppoint depth=1" # 或者根据实际仿真器调整
    # options_str = ".option post=2\n.probe dc save m1(*)" # 另一HSPICE风格
    options_str = "save m1" # 简化，依赖TED内部处理
    if get_simulator() == "hspice": # 特定于HSPICE的选项
        options_str = ".probe dc V(*) I(*) level=1" # 或者更精细的保存选项
        # 同时可能需要在Mosfet定义中确保其参数可以被保存

    raw: Waveform = dc(options=options_str)

    # 打印部分可用信号名称，帮助定位参数
    debug(f"部分可用信号: {raw.get_signal_names()[:20]}") # 只显示前20个

    # 提取并打印关键参数
    # 注意：参数名称严格依赖于仿真器和PDK模型。
    # 以下名称是常见的 Spectre 风格。HSPICE中可能是 M1.GM, M1.GDS 等。
    # TED 的 get_signal 应该会尝试统一或直接使用仿真器名称。
    try:
        ids = raw.get_signal("m1:ids") # 漏电流
        gm = raw.get_signal("m1:gm")   # 跨导
        gds = raw.get_signal("m1:gds")  # 输出电导
        vth = raw.get_signal("m1:vth")  # 阈值电压
        region = raw.get_signal("m1:region") # 工作区 (e.g., 0:off, 1:triode, 2:saturation, 3:subthreshold)

        debug(f"MOSFET m1 参数:")
        debug(f"  Ids = {ids:.4e} A")
        debug(f"  gm  = {gm:.4e} S")
        debug(f"  gds = {gds:.4e} S (Rds = {1/gds:.2f} Ohm)")
        debug(f"  Vth = {vth:.4f} V")
        debug(f"  Region = {region} (0:off, 1:triode, 2:sat, 3:subth)")
    except Exception as e:
        debug(f"获取参数失败: {e}。请检查信号名称是否与您的仿真器/PDK匹配。")
        debug("尝试使用 `raw.get_signal_names()` 查看所有可用信号。")

# mosfet_dc_parameters()
# get_circuit().run_simulation()

说明与解释:

  • dc(options="save m1"): options 参数至关重要。它指示仿真器保存例化名为 m1 的器件的详细工作点信息。具体的语法 (save m1, save m1.*:oppoint, .probe dc m1(param)) 取决于所使用的仿真器 (Spectre, HSPICE 等) 和 PDK 的设置。TED 可能会对这些选项进行一定的抽象。
  • 参数名称 (如 m1:ids, m1:gm) 同样依赖于仿真器和模型。如果获取失败，请使用 raw.get_signal_names() 查找正确的名称。
  • region 参数可以帮助判断 MOSFET 的工作状态。

1.2 瞬态 (TRAN) 分析

瞬态分析用于观察电路信号随时间变化的特性。

from pyted import testbench, Vss, debug, tran, watch, current_module
from ted_device import VSource, Resistor, Capacitor, PulseVSource
from ted_sim import Waveform
import numpy as np

@testbench
def transient_rc_circuit():
    """一个简单的RC电路瞬态响应分析"""
    # 电路描述
    # PulseVSource(Vss(), "vin_node", v1=0, v2=1, td=1e-9, tr=0.1e-9, tf=0.1e-9, pw=5e-9, per=10e-9)
    # 为了简单，使用阶跃电压源
    VSource("vin_node", Vss(), type="pwl", value=[0,0, 1e-9,0, 1.1e-9,1]) # 在1ns时跳变到1V
    Resistor("vin_node", "vout_node", r=1e3)  # R = 1kOhm
    Capacitor("vout_node", Vss(), c=1e-12) # C = 1pF (时间常数 tau = RC = 1ns)

    # 声明需要观察的信号，这会在调用 raw.show() 时绘制出来
    watch("vin_node", ylabel="Voltage (V)") # 观察输入电压
    watch("vout_node", ylabel="Voltage (V)")# 观察输出电压 (电容电压)

    # 执行瞬态分析
    # stop: 仿真停止时间
    # step: (建议的)最大时间步长，仿真器可能会动态调整
    # start: (可选) 仿真起始时间，默认为0
    raw: Waveform = tran(stop=10e-9, step=0.1e-9)

    # 获取扫描轴 (时间轴) 和信号数据
    time_axis = raw.get_sweeps()
    vin_data = raw.get_signal("vin_node")
    vout_data = raw.get_signal("vout_node")

    debug(f"时间轴点数: {len(time_axis)}")
    debug(f"仿真结束时间: {time_axis[-1]:.2e} s")

    # 使用 sample 方法在特定时间点提取/插值数据
    # 提取 t = 2ns (相对于阶跃发生后1ns) 时的输出电压
    # RC电路充电公式: Vc(t) = V_final * (1 - exp(-t/tau))
    # 这里 t_charge = 2e-9 - 1e-9 = 1e-9 (充电1ns, 即一个时间常数)
    # Vc(1ns) = 1V * (1 - exp(-1)) approx 1 * (1 - 0.3679) = 0.6321V
    sampled_vout_at_2ns = raw.sample("vout_node", start=2e-9, step=0.1e-9)[0] # sample返回数组
    debug(f"Vout @ 2ns (sampled): {sampled_vout_at_2ns:.4f} V (Expected approx. 0.6321 V)")

    # 将波形图保存到文件，并显示 (如果环境支持)
    raw.show(figname="rc_transient_response", module=current_module())
    # 图像会保存在模块对应目录下，例如: ./fig_rc_transient_response_tran.png

# transient_rc_circuit()
# get_circuit().run_simulation()

说明与解释:

  • tran(stop, step, start): 执行瞬态分析。
    • stop: 仿真总时长。
    • step: 建议的最大输出步长。仿真器实际步长可能会根据电路变化动态调整以保证精度。
    • start: 仿真数据记录的起始时间，默认为0。
  • watch("signal_name", **plot_options): 声明要绘制的信号。plot_options 可以包括 ylabel, xlabel, calc (用于数据转换), mode (绘图模式)等。
  • raw.get_sweeps(): 在瞬态分析中，返回时间轴数组 (numpy array)。
  • raw.sample(node, start, step): 在指定的时间点对信号进行采样 (如果时间点不在原始数据中，会进行插值)。
  • raw.show(figname="filename_prefix"): 生成并保存之前用 watch 声明的信号的波形图。图片会保存在模块路径下，命名通常为 fig_{figname_prefix}_{simtype}.png。

1.3 交流 (AC) 分析

AC 分析用于评估电路在不同频率下的线性响应，如增益、相位特性等。

from pyted import testbench, Vss, debug, ac, watch, current_module, new_figure
from ted_device import VSource, Resistor, Capacitor
from ted_sim import Waveform
from ted_frontend import T # 导入 ted_frontend.T 用于 dB20 等计算
import numpy as np

@testbench
def ac_rc_lowpass_filter():
    """一个简单的RC低通滤波器AC特性分析"""
    # 电路描述: RC 低通滤波器
    # 信号源 Vin, 直流偏置0V, 交流幅度1V
    VSource("vin_node", Vss(), dc=0, ac=1).rename("Vac_in")
    Resistor("vin_node", "vout_node", r=1e3).rename("R1")  # R = 1kOhm
    Capacitor("vout_node", Vss(), c=1.59e-9).rename("C1") # C = 1.59nF
    # 截止频率 f_c = 1 / (2*pi*R*C) = 1 / (2*pi*1k*1.59n) approx 100kHz

    # 执行AC分析
    # start, stop: 频率扫描的起始和结束点 (Hz)
    # type: 扫描类型 ('dec' - 每十倍频, 'lin' - 线性, 'oct' - 每倍频程)
    # ND (或npoints): 每 decade/octave/linear interval 的点数
    raw: Waveform = ac(start=1e3, stop=10e6, type="dec", ND=20) # 1kHz 到 10MHz, 每十倍频20点

    # 获取频率轴和复数输出电压
    frequency_axis = raw.get_sweeps()
    vout_complex = raw.get_signal("vout_node") # 返回复数形式的 TorchVariable 或 ndarray

    # 1. 幅度响应 (dB20)
    #    可以直接使用 ted_frontend.T.db20 进行计算
    #    或者让 watch/plot 的 calc 参数处理
    watch("vout_node", calc=lambda v: T.db20(np.abs(v)), ylabel="Gain (dB)", mode="semilogx",
          xlabel="Frequency (Hz)", title="Magnitude Response")

    # 2. 相位响应 (角度)
    #    可以使用 np.angle(vout_complex, deg=True)
    #    或者让 watch/plot 的 calc 参数处理
    new_figure() # 开始新的图窗用于相位图
    watch("vout_node", calc=lambda v: np.angle(v, deg=True), ylabel="Phase (degrees)", mode="semilogx",
          xlabel="Frequency (Hz)", title="Phase Response")

    # 使用 get_signal 的 signal_type 参数直接获取幅度和相位
    new_figure()
    vout_magnitude = raw.get_signal("vout_node", signal_type="VM") # 获取幅度
    watch(frequency_axis, T.db20(vout_magnitude), label="Vout Mag (VM)", ylabel="Gain (dB)", mode="semilogx",
          xlabel="Frequency (Hz)", title="Magnitude via signal_type='VM'")

    new_figure()
    vout_phase_deg = raw.get_signal("vout_node", signal_type="VP") * 180 / np.pi # VP返回弧度，需转换为角度
    watch(frequency_axis, vout_phase_deg, label="Vout Phase (VP)", ylabel="Phase (degrees)", mode="semilogx",
          xlabel="Frequency (Hz)", title="Phase via signal_type='VP'")

    # 在截止频率 (100kHz) 处验证增益和相位
    fc = 100e3
    gain_at_fc_db = T.db20(np.abs(np.interp(fc, frequency_axis, vout_complex)))
    phase_at_fc_deg = np.angle(np.interp(fc, frequency_axis, vout_complex), deg=True)
    debug(f"At cutoff frequency ({fc/1e3:.0f} kHz):")
    debug(f"  Gain = {gain_at_fc_db:.2f} dB (Expected approx. -3dB)")
    debug(f"  Phase = {phase_at_fc_deg:.2f} degrees (Expected approx. -45deg)")

    raw.show(figname="rc_ac_response", module=current_module())

说明与解释:

  • ac(start, stop, type, ND): 执行 AC 分析。
    • start, stop: 频率扫描范围 (单位: Hz)。
    • type: 扫描类型，如 "dec" (decade), "lin" (linear), "oct" (octave)。
    • ND (或 npoints): 每 type 区间内的点数。
  • raw.get_signal("vout_node"): 在 AC 分析中默认返回复数电压/电流数据。
  • T.db20(value): ted_frontend.T 工具类中的函数，用于将线性值转换为 dB20 ($20 \times \log_{10}(\text{value})$)。
  • np.abs(complex_value): 计算复数的幅度。
  • np.angle(complex_value, deg=True): 计算复数的相位 (使用 deg=True 返回角度，默认为弧度)。
  • watch 中的 calc 参数: 非常强大，允许在绘图前对信号数据进行任意 NumPy 或 T 函数运算。
  • watch 中的 mode="semilogx": X 轴使用对数刻度，Y 轴使用线性刻度，适合绘制波特图。
  • new_figure(): 用于在 raw.show() 时创建新的图窗，避免所有曲线绘制在同一张图或子图上。
  • raw.get_signal(signal_name, signal_type="..."):
    • signal_type="VM": 直接获取幅度。
    • signal_type="VP": 直接获取相位 (弧度)。
    • signal_type="VR": 直接获取实部。
    • signal_type="VI": 直接获取虚部。

1.4 眼图 (Eye Diagram) 分析 (瞬态分析应用)

眼图是评估高速数字信号质量的重要手段。Waveform 对象提供了 eyediagram 方法来绘制眼图。

from pyted import testbench, Vss, debug, tran, watch, current_module
from ted_device import VSource, Resistor, Capacitor, IdealClock
from ted_sim import Waveform
import numpy as np

@testbench
def eye_diagram_example():
    """演示如何生成和绘制眼图"""
    # 假设有一个理想时钟信号作为数据源，经过一个有损信道 (RC环节)
    bit_rate = 1e9 # 1 Gbps
    period = 1 / bit_rate # 1 ns
    
    IdealClock(Vss(), "clk_in", period=period, trise=0.05*period, tfall=0.05*period)
    # 简单模拟一个有损信道 (带宽有限)
    Resistor("clk_in", "clk_out", r=50)
    Capacitor("clk_out", Vss(), c=2e-12) # C = 2pF

    # 至少需要几十个周期的数据
    num_bits_for_eye = 100
    simulation_time = num_bits_for_eye * period
    # 确保每bit有足够采样点
    samples_per_bit = 20 
    max_step = period / samples_per_bit
    
    raw: Waveform = tran(stop=simulation_time, step=max_step)

    watch("clk_out", label="Output Clock (Channel)")

    # 该方法内部会处理数据的折叠和绘图
    # 注意：eyediagram 方法直接绘图并保存，不通过 watch 和 show

    # 眼图会保存为 fig_clk_out_eyediag_tran.png (或类似名称)

2. 扫描仿真

扫描仿真是指在一个或多个电路参数、器件参数或工作条件（如温度）变化时，重复执行某种基础分析（如 DC, AC, TRAN），以观察电路性能如何随这些参数变化。

2.1 直流扫描 (DCSweep)

DCSweep 用于分析电路特性随某个直流电压源、电流源或器件参数变化的情况。

    • raw.show() 会将所有内部 Waveform 的 watched 曲线绘制在同一张图上，并自动在图例中用次级扫描参数值区分它们。

说明与解释:

  • setenv(...): 用于设置仿真环境参数。对于蒙特卡洛：
    • simulator: 必须是支持MC的仿真器，如 "spectre"。
    • monte_carlo: 指定MC类型，"process" (工艺角偏差)，"mismatch" (器件失配)，或 "all" (两者都考虑)。
    • corners: 必须包含指向MC模型库的工艺角名称 (例如，PDK中定义的 mc_lib 或 mismatch_lib)。
    • mc_analysis: 一个字典，键是自定义的性能指标名称，值是用于在每次MC迭代中评估该指标的表达式字符串 (通常是Ocean MDL语法)。这些指标最终会出现在 MCresult 中。
    • mc_num: 蒙特卡洛的运行次数。
  • Measure.dc(measure=mc_expressions): 在 setenv 配置的MC环境下，Measure 类的方法会自动执行多次仿真。它收集 mc_analysis (或其内部 measure 参数) 中定义的每个表达式在每次迭代中的值。
  • raw_mc_measure.get_mc_result(): 从 MeasureResult 对象中提取 MCresult 对象。
  • MCresult 对象 (mc_results_obj): 类似字典，键是 mc_analysis 中定义的指标名。值是 subMcResult 对象。
  • subMcResult 对象 (output_stat): 包含单个指标的所有MC运行原始数据 (.raw 或 .perf['raw'])以及统计值 (.perf['min'], .perf['max'], .perf['mean'], .perf['std'])。
    • output_stat.plot(name="...", ...): 绘制该指标的直方图，可选择显示拟合的正态分布曲线 (curve=True) 和 $\pm 1/2/3 \sigma$ 边界 (edge=True)。
