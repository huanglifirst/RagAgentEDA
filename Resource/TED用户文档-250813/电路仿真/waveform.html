摘要

本文档详细描述了 waveform.py 模块的功能和使用方法。该模块主要用于处理和分析电路仿真结果，支持多种仿真类型（如直流、交流、瞬态、PSS 等）的波形数据加载、提取、可视化以及与 PyTorch 集成以进行灵敏度分析。核心类包括 Waveform（基础波形处理）、SweepWaveform（扫描仿真波形处理）、MultiWaveform（多分析配置波形处理）和 WaveformX（多进程波形读取）。文档将依次介绍这些类的主要功能、API 接口、使用示例，并辅以完整使用样例和常见问题解答，旨在帮助用户快速理解和使用该模块。

1. Waveform 类

Waveform 类是波形处理的核心，提供了加载仿真结果、提取信号数据、进行数据转换和可视化的基础功能。

1.1. 初始化与设置 (__init__, setup)

  • __init__(rs_waveform=None, simtype=None, sec_simtype=None, module: "Module" = None, variables: Dict[str, "Variable"] = None, perturb_variables: List[float] = None, perturb_waveforms: List["Waveform"] = None, data=None)

    • 构造函数，用于创建一个 Waveform 对象。
    • 参数：
      • rs_waveform: 原始仿真器波形对象 (例如 rsted.Waveform 实例)。
      • simtype (str): 仿真类型 (如 "op", "tran", "ac", "pss" 等)。
      • sec_simtype (str, optional): 二级仿真类型，主要用于 PSS 分析 (如 "fd", "td")。
      • module ("Module", optional): 关联的 tedcore.Module 对象。
      • variables (Dict[str, "Variable"], optional): 用于灵敏度分析的变量字典。
      • perturb_variables (List[float], optional): 灵敏度分析中的扰动值列表。
      • perturb_waveforms (List["Waveform"], optional): 对应扰动变量的波形列表。
      • data (dict, optional): 直接用于初始化的数据字典。

  • setup(...)

    • 实际的初始化方法，由 __init__ 调用。参数与 __init__ 相同。
    • 初始化对象的各个属性，如 module, simtype, variables, signal_names, data 等。

1.2. 波形加载方法

1.2.1. 从数据字典加载 (from_data)

  • @staticmethod Waveform.from_data(data, simtype: str)
    • 根据给定的数据字典和仿真类型创建一个 Waveform 对象。
    • 参数：
      • data (dict): 包含波形数据的字典，应有 "signal_names" (列表), "signals" (字典, 信号名到信号值的映射), "sweeps" (扫描点数组)。
      • simtype (str): 仿真类型。
    • 返回: Waveform 对象。

1.2.2. 从 PSF 文件加载 (from_psf_file)

  • Waveform.from_psf_file(module: "Module", psf_file: str, simtype: str, sec_simtype=None)
    • 从指定的 PSF (或其他仿真器原生结果文件) 加载波形数据。
    • 参数：
      • module ("Module"): 关联的模块。
      • psf_file (str): PSF 文件路径。
      • simtype (str): 仿真类型。
      • sec_simtype (str, optional): 二级仿真类型。
    • 返回: Waveform 对象。
    • 注意: 内部使用 rsted.Waveform.new() 加载。

1.2.3. 从其他 Waveform 对象加载 (from_waveform)

  • from_waveform(self: "Waveform", waveform: "Waveform", **kwargs)
    • 将另一个 Waveform 对象的数据复制到当前对象。
    • 参数：
      • waveform ("Waveform"): 源 Waveform 对象。
      • **kwargs: 其他传递给 setup 方法的参数。
    • 返回: self。

1.2.4. 获取结果文件路径与名称

这些静态方法用于根据仿真器类型和模块信息确定仿真结果文件的标准路径和名称。

  • @staticmethod Waveform.get_result_file_path(module: "Module", simtype, pid=0)

    • 获取结果文件所在的目录路径。
    • 参数：
      • module ("Module"): 模块对象。
      • simtype (str): 仿真类型。
      • pid (int, optional): 进程 ID，默认为 0。
    • 返回: (str) 结果文件目录路径。

  • @staticmethod Waveform.get_result_file_name(module: "Module", simtype, pid=0)

    • 获取结果文件的名称。
    • 参数：
      • module ("Module"): 模块对象。
      • simtype (str): 仿真类型。
      • pid (int, optional): 进程 ID，默认为 0。
    • 返回: (str) 结果文件名。

  • @staticmethod Waveform.get_result_file(module: "Module", simtype, pid=0)

    • 获取完整的结果文件路径 (目录 + 名称)。
    • 参数：
      • module ("Module"): 模块对象。
      • simtype (str): 仿真类型。
      • pid (int, optional): 进程 ID，默认为 0。
    • 返回: (str) 完整结果文件路径。

1.2.5. 从扫描数据加载 (from_datasweep)

  • Waveform.from_datasweep(module, simtype, data, pid=0, path=None, name=None)
    • 加载扫描仿真 (DataSweep) 的结果。通常会返回一个 SweepWaveform 对象。
    • 参数：
      • module: 关联模块。
      • simtype (str): 仿真类型。
      • data ("DataSweep"): 扫描数据对象。
      • pid (int, optional): 进程 ID。
      • path (str, optional): 结果文件路径。
      • name (str, optional): 结果文件名。
    • 返回: SweepWaveform 对象 (通常情况下) 或 Waveform 对象 (特定仿真器和类型，如 spectremdl 的 DC.foreach)。

1.2.6. 通用加载方法 (load)

  • Waveform.load(module: "Module", simtype, data: "DataSweep" = None, pid=0, path=None, name=None)
    • 通用的波形加载入口。根据 data 参数是否提供，决定是加载单个仿真结果还是扫描仿真结果。
    • 如果设置了灵敏度分析环境变量 (getenv("sensitivity") 为真) 且电路中存在 PyTorch 变量，则会加载基准波形和扰动波形，并设置相关属性用于后续梯度计算。
    • 参数：
      • module ("Module"): 模块对象。
      • simtype (str): 仿真类型。
      • data ("DataSweep", optional): 扫描数据对象。如果提供，则加载扫描结果。
      • pid (int, optional): 进程 ID。
      • path (str, optional): 自定义结果文件路径。
      • name (str, optional): 自定义结果文件名。
    • 返回: Waveform 或 SweepWaveform 对象。

1.3. 信号数据获取

1.3.1. 获取信号名称列表 (get_signal_names)

  • get_signal_names(self)
    • 返回波形文件中所有可用的信号名称。
    • 返回: (List[str]) 信号名称列表。

1.3.2. 获取扫描点 (get_sweeps)

  • get_sweeps(self)
    • 获取仿真扫描的自变量值 (例如瞬态分析的时间点，交流分析的频率点)。
    • 返回: (np.ndarray) 扫描点数组。

1.3.3. 获取原始信号 (get_raw_signal)

  • get_raw_signal(self, name: str, signal_type: str = "diff")
    • 获取指定名称的原始信号数据，不做插值。
    • 对于 AC/PSS 等分析，返回复数数组。对于 TRAN/DC 等，返回实数数组。
    • 此方法会调用 to_tensor 进行可能的梯度封装。
    • 参数：
      • name (str): 信号名称。
      • signal_type (str, optional): 信号返回类型，默认为 "diff"。对于复数信号，可以是：
        • "diff": 复数形式 (默认)
        • "VM": 幅度 (Magnitude)
        • "VP": 相位 (Phase, in radians)
        • "VR": 实部 (Real part)
        • "VI": 虚部 (Imaginary part)
    • 返回: (TorchVariable or np.ndarray) 信号数据。如果启用了灵敏度分析，则为 TorchVariable。

1.3.4. 获取信号 (支持插值和类型转换) (get_signal)

  • get_signal(self, node: "CircuitNode", pos=None, signal_type="diff")
    • 获取信号值。如果 pos 提供，则在这些点上进行插值。
    • 参数：
      • node ("CircuitNode" or str): 节点对象或节点名称。
      • pos (float, List[float], np.ndarray, optional): 需要获取信号值的扫描点。如果为 None，返回所有点。
      • signal_type (str, optional): 信号返回类型，同 get_raw_signal。用于指定如何处理复数信号以及计算对应类型的梯度。
    • 返回: (TorchVariable or np.ndarray) 信号数据。
    • 示例 (来自 test_mosfet_dc0):
      # 假设 raw 是一个 Waveform 对象
# raw = dc() # 执行直流仿真
# mid_voltage = raw.get_signal("mid")
# debug(mid_voltage)

1.3.5. 转换为张量 (支持梯度计算) (to_tensor)

  • to_tensor(self, name, signal: np.ndarray, signal_type: str)
    • 将 Numpy 信号数组转换为 TorchVariable (自定义的 PyTorch Tensor 封装)。
    • 如果初始化时提供了 variables, perturb_variables, 和 perturb_waveforms (即启用了灵敏度分析)，此方法会使用差分法计算信号相对于这些变量的梯度，并将信号值和梯度信息封装在返回的 TorchVariable 中。
    • 参数：
      • name (str): 信号名称。
      • signal (np.ndarray): 原始信号数据。
      • signal_type (str): 信号类型 ("diff", "VM", "VP", "VR", "VI")，决定了如何处理复数信号及计算何种梯度。
    • 返回: (TorchVariable) 封装了信号值和可能的梯度信息的张量。

1.3.6. 获取非矢量信号/参数 (get_signal_nonvector, get_signal_params_float)

  • get_signal_nonvector(self, name)

    • 获取非矢量信号 (通常是单个值，如直流工作点参数)。
    • 参数：
      • name (str): 参数名称。
    • 返回: (np.ndarray) 通常是包含单个值的数组。

  • get_signal_params_float(self, name)

    • 获取浮点型信号参数。特定于仿真器实现 (如 Spectre, HSPICE)。
    • 参数：
      • name (str): 参数名称 (对于 HSPICE，可能包含 ":"，如 "M1:gm")。
    • 返回: (float or dict) 参数值。

1.3.7. 获取 PZ 分析的复数/浮点信号参数

  • get_complex_signal_params_for_pz(self, name)
  • get_float_signal_params_for_pz(self, name)
    • 专门用于获取 PZ (零极点) 分析结果中的参数。
    • 参数: name (str) - 参数名称。
    • 返回: 复数或浮点型参数。

1.4. 波形数据操作与检查

1.4.1. 判断是否为直流分析 (is_dc)

  • is_dc(self)
    • 判断当前波形是否来自直流 (op) 分析。
    • 返回: (bool) 如果是直流分析则为 True，否则为 False。

1.4.2. 添加额外数据 (add_data)

  • add_data(self, data: "dict")
    • 向现有 Waveform 对象中添加新的信号数据。如果 self.data 原本为 None，会先将现有通过 rs_waveform 加载的数据转存到 self.data。
    • 参数：
      • data (dict): 包含新信号数据的字典，格式同 from_data。
    • 注意: 会检查信号名称是否有重复。

1.4.3. 判断是否为扫描波形 (is_swept)

  • is_swept(self)
    • 判断原始波形数据是否包含扫描 (sweep)。
    • 返回: (bool) 如果是扫描波形则为 True。

1.4.4. 节点名称转换与检查

  • convert_node_name(self, name)

    • 转换节点名称。例如，在 HSPICE 中，电流 device_name:p 会被转换为 i(device_name)。
    • 参数: name (str) - 原始节点名。
    • 返回: (str) 转换后的节点名。

  • assert_signal_exists(self, name)

    • 检查信号是否存在于波形数据中，如果不存在则抛出异常。
    • 参数: name (str) - 信号名。

  • get_node_name(self, x)

    • 获取节点名称。如果输入是字符串，则直接使用 (可能经过 convert_node_name)；如果是 CircuitNode 对象，则调用其 name() 方法。
    • 参数: x (str or "CircuitNode") - 节点标识。
    • 返回: (str) 节点名称。

1.4.5. 数据采样 (sample, sample_binary)

  • sample(self, node, start: float, step: float)

    • 从 start 开始，以 step 为步长，对指定节点的信号进行采样，直到扫描结束。
    • 参数：
      • node: 节点对象或名称。
      • start (float): 采样起始点。
      • step (float): 采样步长。
    • 返回: (np.ndarray) 采样后的信号值数组。
    • 示例 (来自 test_waveform2 和 test_waveform3):
      # raw = tran(stop=1e-6)
# sampled_signal = raw.sample("n1", 1e-7, 2e-7) # 假设步长是基于内部插值逻辑
# sampled_current = raw.sample("iprobe1:in", 1e-7, 2e-7)
# debug(sampled_signal)
# debug(sampled_current)

  • sample_binary(self, node, start: float, step: float, th: float = None)

    • 对信号进行采样，并将结果二值化。
    • 参数：
      • node: 节点对象或名称。
      • start (float): 采样起始点。
      • step (float): 采样步长。
      • th (float, optional): 二值化阈值。如果为 None，则默认为信号最大值的 1/2。
    • 返回: (np.ndarray) 二值化后的信号值数组 (0 或 1)。

1.4.6. 数据插值 (interpolate)

  • interpolate(self, node, x)
    • 对指定节点的信号在给定的 x 点上进行线性插值。
    • 注意: 要求 x 点必须是单调递增的，并且在原始扫描范围之内。
    • 参数：
      • node: 节点对象或名称。
      • x (np.ndarray): 需要插值的点。
    • 返回: (TorchVariable) 插值后的信号值。

1.4.7. 转换为 Numpy 数组 (to_array)

  • to_array(self)
    • 将波形对象中的所有信号数据转换为一个大的 Numpy 二维数组。
    • 每一行代表一个信号，每一列代表一个扫描点。
    • 返回: (np.ndarray) 包含所有信号数据的二维数组。
    • 示例 (来自 test_mosfet_dc3):
      # raw = dcsweep(...)
# array = raw.to_array()
# debug(array.shape) # (信号数量, 扫描点数量)

1.4.8. 波形检查 (check)

  • check(self, callback, start: float, step: float, th: float = None) -> bool
    • 按指定步长采样所有信号，并对每个采样点调用 callback 函数进行检查。
    • 参数：
      • callback (function): 回调函数，接收一个包含当前采样点所有信号值的字典，返回 bool 值。如果返回 False，则检查失败。
      • start (float): 检查起始点。
      • step (float): 检查步长。
      • th (float, optional): 传递给 sample_binary (如果内部使用) 的阈值，此方法本身不直接使用。
    • 返回: (bool) 如果所有检查都通过则为 True，否则抛出异常。

1.5. 绘图与可视化

该模块使用 matplotlib.pyplot 进行绘图。

1.5.1. 声明观察信号 (watch)

  • watch(self, *args, module=None, **kwargs)
    • 声明在仿真结束后需要绘制的信号。这些声明会被记录下来，供 show() 方法使用。
    • 参数：
      • *args: 要观察的信号名称或节点对象。
      • module ("Module", optional): 关联的模块，默认为当前模块。
      • **kwargs: 传递给绘图的额外参数，例如：
        • calc (function): 一个函数，用于在绘图前对信号进行计算 (例如 calc=lambda x: -x)。
        • mode (str): 绘图模式，如 "semilogy", "loglog", "semilogx"。
        • 也可以是 label=custom_label 形式来指定图例。
    • 示例 (来自 test_mosfet_idvg):
      # raw = dcsweep(...)
# watch("vvds:p", calc=lambda x: -x, mode="semilogy") # 观察 vvds 电源的电流，取反并使用对数 Y 轴
# new_figure() # 如果需要在新图上绘制
# watch("vvds:p", calc=lambda x: -x) # 线性轴
# raw.show()

1.5.2. 显示波形图 (show)

  • show(self, text: str = None, module=None, path=None, tester=None, figname=None, **invars)
    • 根据之前 watch 声明的信号，生成并显示/保存波形图。
    • 每个 watch 调用（除非有 new_figure 分隔）会尝试在同一个子图中绘制。new_figure() (来自 pyted 库) 用于开始新的图或子图组。
    • 参数：
      • text (str, optional): 兼容性参数，可能未使用。
      • module ("Module", optional): 关联的模块，默认为当前模块。
      • path (str, optional): 保存图像的路径。如果为 None，则保存在模块的 filepath() 下。
      • tester (optional): 测试器对象，用于记录生成的图像文件名。
      • figname (str, optional): 图像文件名（不含路径和后缀）。
    • 示例 (来自 test_waveform1):
      # @testbench
# def Top():
#     Clock() >> "n1"
#     Resistor("n1", Vss(), r=100)
#     watch("n1")
#     raw = tran(stop=1e-7)
#     raw.show() # 将会绘制 n1 节点的瞬态波形

1.5.3. 基础绘图接口

  • figure(self, *args, **kwargs): 调用 plt.figure()。

  • subplot(self, *args): 调用 plt.subplot()。

  • add_curves(self, *args, mode=None, calc=None, **kwargs)

    • 准备绘图所需的数据。它获取信号数据，并根据 calc 函数进行处理。
    • 参数：
      • *args: 要绘制的信号名称或节点。
      • mode: 绘图模式。
      • calc: 计算函数。
      • **kwargs: 信号名作为键，信号源作为值的形式，如 my_label=signal_name。
    • 返回: (curves, legends)，其中 curves 是 [x0, y0, x1, y1, ...] 格式的列表，legends 是图例列表。

  • plot_curves(self, curves, legends=None, mode=None, ylabel=None, xlabel=None, save=None)

    • 实际执行绘图的底层方法。
    • 参数：
      • curves: add_curves 返回的曲线数据。
      • legends: 图例。
      • mode: "semilogx", "semilogy", "loglog", "eyediagram"。
      • ylabel, xlabel: 坐标轴标签。
      • save (str, optional): 如果提供，则以此文件名（不含路径）保存图像。

  • plot(self, *args, chart=None, title=None, index=0, mode=None, xlabel=None, ylabel=None, calc=None, **kwargs)

    • 一个更高级的绘图接口，整合了 add_curves 和 plot_curves。
    • 如果提供了 chart 对象 (用于 GUI 可视化)，则调用 visual。

  • save_plot(self, plt, module=None, figname=None, path=None, tester=None)

    • 保存当前 Matplotlib 图像。

1.5.4. 眼图绘制 (eyediagram)

  • eyediagram(self, signame, period)
    • 绘制指定信号的眼图。
    • 参数：
      • signame: 信号名称或节点对象。
      • period (float): 信号的周期，用于折叠眼图。
    • 图像会保存到默认路径，文件名为 fig_{signame}_eyediag_{simtype}.png。

1.5.5. GUI 图表可视化 (visual)

  • visual(self, chart, title, curves, legends, index, **kwargs)
    • 用于将波形数据发送到 GUI 图表组件 (如 LineChart)进行显示。
    • 此方法通常由 plot 在 chart 参数非空时内部调用。

1.5.6. 设置绘图样式 (set_plt_style)

  • set_plt_style(self, x: "dict" = {})
    • 更新 Matplotlib 的绘图样式设置。例如，可以更改背景主题。
    • 参数 x 是一个字典，键是样式设置项 (如 "plt.style.use"), 值是对应的样式名称 (如 "dark_background")。

1.6. 其他功能

1.6.1. 测量 (measure)

  • measure(self, mod="offline", expr: Dict = {})
    • (目前仅支持 Spectre) 使用外部工具 (如 Cadence Ocean/Skill 通过 OcnCal 接口) 对仿真结果进行测量。
    • 参数：
      • mod (str): 测量模式，默认为 "offline"。
      • expr (Dict): 包含测量表达式的字典。
    • 返回: (Dict) 测量结果。

1.6.2. 蒙特卡洛结果处理 (load_mc_result, get_mc_result)

  • load_mc_result(self, mckeys, mcFile)

    • 加载蒙特卡洛 (Monte Carlo) 仿真的结果。
    • 参数：
      • mckeys: 结果中的键。
      • mcFile: MC 结果文件路径。
    • 内部使用 mcresult.MCresult 类处理。

  • get_mc_result(self)

    • 返回加载的 MC 结果对象。

1.6.3. 设置/获取原始文件路径 (set_rawpath, get_rawpath)

  • set_rawpath(self, rawpath): 设置仿真原始结果文件所在的目录路径。
  • get_rawpath(self): 获取该路径。

2. SweepWaveform 类

SweepWaveform 继承自 Waveform，专门用于处理参数扫描 (DataSweep) 仿真产生的结果。一个 SweepWaveform 对象内部包含一个描述扫描参数的 sweepdata 对象和一系列 Waveform 对象，每个 Waveform 对应扫描中的一个点。

2.1. 初始化 (__init__)

  • __init__(self, sweepdata: "DataSweep" = None, waveforms: List[Waveform] = None)
    • 参数：
      • sweepdata ("DataSweep"): tedcore.DataSweep 对象，描述了扫描的参数和值。
      • waveforms (List[Waveform]): 一个 Waveform 对象列表，每个对象代表扫描中的一个仿真结果。

2.2. 从其他 SweepWaveform 对象加载 (from_waveform)

  • from_waveform(self: "SweepWaveform", waveform: "SweepWaveform")
    • 从另一个 SweepWaveform 对象复制 sweepdata 和 waveforms。

2.3. 绘图 (plot)

  • plot(self, *args, chart=None, title=None, index=0, mode=None, xlabel=None, ylabel=None, calc=None, **kwargs)
    • 重写了基类的 plot 方法。它会遍历内部所有的 Waveform 对象，并将它们的曲线绘制在同一张图上。
    • 图例会自动标注每个曲线对应的扫描参数值。
    • 示例 (来自 test_mosfet_idvg):
      # # Id_Vg 特性曲线, vds 作为扫描参数
# datasweep = DataSweep(name="sweepdata", params=["vds"], values=[np.arange(0, 1, 0.1)])
# raw = dcsweep(dev="vvgs", param="dc", start=0, stop=1.2, step=0.01, data=datasweep)
# # raw 是一个 SweepWaveform 对象
# watch("vvds:p", calc=lambda x: -x, mode="semilogy") # vvds:p 是漏极电流
# raw.show() # 会将不同 vds 值下的 Id-Vg 曲线绘制在一起

2.4. 数据获取

  • get_signal_names(self): 返回第一个内部 Waveform 的信号名称列表。
  • get_sweeps(self): 返回第一个内部 Waveform 的扫描点 (如时间或频率)。
  • get_all_sweeps(self): 返回一个列表，包含所有内部 Waveform 对象的扫描点。
  • get_signal(self, node: "CircuitNode", pos=None, signal_type="diff")
    • 返回一个列表，其中每个元素是对应内部 Waveform 在指定节点和位置的信号值。

2.5. 索引访问

  • __getitem__(self, idx): 允许通过索引访问内部的 Waveform 对象 (如 sweep_raw[0])。
  • __len__(self): 返回内部 Waveform 对象的数量 (即扫描点的数量)。

2.6. 合并扫描数据 (add_data_info, merge_split_sweepwaveform)

这些方法用于处理分块或并行执行的扫描仿真结果，之后再将它们合并回一个完整的 SweepWaveform 对象。

3. MultiWaveform 类

MultiWaveform 用于管理和访问来自多个不同分析配置 (analysis_configs) 的仿真结果。例如，一个仿真流程可能包含一个 "tran" 分析和一个 "pss.td" 分析，MultiWaveform 可以统一管理这两个分析的波形数据。

3.1. 初始化与设置 (__init__, setup)

  • __init__(self, **kwargs)
  • setup(self, analysis_configs: list = None, simtype=None)
    • 参数：
      • analysis_configs (list): 一个分析配置对象列表。每个配置对象通常包含 simtype (如 "pss") 和 option (如 "td", "fd") 等属性。
      • simtype (str): 主仿真类型 (可能用于路径确定)。

3.2. 结果文件获取

  • @staticmethod MultiWaveform.get_result_file_path(...): 继承自 Waveform。
  • @staticmethod MultiWaveform.get_result_file_name(module: "Module", analysis_config)
    • 根据分析配置 (包含 simtype 和 option) 获取特定仿真结果的文件名。例如，PSS 分析的 option 为 "td"，文件名可能是 pss.td.pss。
  • @staticmethod MultiWaveform.get_result_file(...): 获取完整结果文件路径。

3.3. 波形数据加载与合并

  • from_waveform(self, perf: dict = None)

    • 将一个包含多个波形对象的字典 (perf) 合并到 self.perf 属性中。键通常是 "simtype.option" 格式。
    • 如果值是 SweepWaveform，会进行特殊的合并处理 (通过 merge_split_sweepwaveform)。

  • @staticmethod MultiWaveform.from_datasweep(module: "Module", simtype, data, pid=0, analysis_config=None, path=None)

    • 为特定的分析配置加载扫描仿真结果。
    • 返回: SweepWaveform 对象。

  • @staticmethod MultiWaveform.load(module: "Module", simtype, data: "DataSweep" = None, pid=0, analysis_configs=None, path=None)

    • 主要的加载方法。遍历 analysis_configs，为每个配置加载对应的波形数据 (单个或扫描)。
    • 返回: 一个字典 (perf)，键为 "simtype.option"，值为对应的 Waveform 或 SweepWaveform 对象。

3.4. 访问接口

MultiWaveform 对象内部将加载的波形数据存储在 self.perf 字典中。

  • __getitem__(self, key): 通过键 (如 "pss.td") 获取对应的 Waveform 或 SweepWaveform 对象。
  • keys(self): 返回所有可用结果的键。
  • values(self): 返回所有 Waveform / SweepWaveform 对象。
  • items(self): 返回键值对。

3.5. 判断是否包含扫描 (is_sweep)

  • is_sweep(self): 检查 self.perf 中是否有任何一个是 SweepWaveform 实例。

4. WaveformX 类

WaveformX 继承自 Waveform，设计用于多进程环境下的波形数据读取和管理。它内部可以持有一个 Waveform 或 SweepWaveform 的列表。

4.1. 初始化 (__init__)

  • __init__(self, **kwargs): 调用父类构造函数，并初始化 self.waveforms = []。

4.2. 从波形加载

  • new_from_waveform(self, module, simtype, data, path, fname)

    • 加载单个波形文件，根据 get_circuit().datasweep 是否存在决定创建 Waveform 还是 SweepWaveform。

  • from_waveform(self, module, raws)

    • raws 是一个自定义对象列表，每个对象包含加载波形所需的信息 (如 simtype, data, path, fname)。
    • 此方法会遍历 raws，为每个条目加载一个 Waveform 或 SweepWaveform，并存入 self.waveforms 列表。
    • 返回: self。

4.3. 数据获取

  • get_signal_names(self): 返回第一个内部波形的信号名。
  • get_sweeps(self): 返回第一个内部波形的扫描点。
  • get_all_sweeps(self): 返回所有内部波形的扫描点列表。
  • get_signal(self, node, pos=None, signal_type="diff")
    • 如果 self.is_datasweep 为 True (表示处理的是跨进程的扫描数据)，则从所有内部波形中获取信号并合并。
    • 否则，从第一个内部波形获取信号。

4.4. 蒙特卡洛结果处理

  • load_mc_result(self, keys, mcFile): 加载 MC 结果。
  • get_mc_result(self): 获取 MC 结果。

5. 辅助函数

5.1. load_waveform

  • load_waveform(module=None, file=None, simtype=None, **kwargs)
    • 一个便捷函数，用于从单个 PSF 文件加载波形。
    • 内部调用 Waveform.from_psf_file 和 Waveform.from_waveform。
    • 参数：
      • module: 关联模块。
      • file (str): PSF 文件路径。
      • simtype (str): 仿真类型。
      • **kwargs: 传递给 Waveform.from_waveform 的其他参数。
    • 返回: Waveform 对象。

6.1. 简单直流分析

此样例展示了如何进行直流工作点分析并获取节点电压。

from pyted import testbench, Vss, debug, dc
from ted_device import VSource, Resistor

@testbench
def ResistorDCCircuit():
    VSource(Vdd(), Vss(), dc=1)       # 1V 电源 Vdd 到 Vss
    Resistor(Vdd(), "mid", r=500)     # 500 Ohm 电阻从 Vdd 到 mid 节点
    Resistor("mid", Vss(), r=500)     # 500 Ohm 电阻从 mid 节点到 Vss
    
    raw = dc()                        # 执行直流分析
    
    signal_names = raw.get_signal_names()
    debug(f"Available signals: {signal_names}") # 打印所有可用信号
    
    mid_voltage = raw.get_signal("mid") # 获取 mid 节点的电压
    debug(f"Voltage at 'mid' node: {mid_voltage}") # 预期为 0.5V

6.3. 直流扫描分析与绘图 (Id-Vg 特性曲线)

此样例展示了如何进行直流扫描 (DCSweep) 来获取 MOS管的 Id-Vg 特性，并绘制曲线。这里 vds 作为外部扫描参数。

from pyted import testbench, Vss, debug, dcsweep, watch, new_figure, DataSweep
from ted_device import NMOS, VSource
import numpy as np
# from ted_sim import run_simulation

@testbench
def MosIdVgChar():
    # 电路定义
    "vd" >> NMOS(l=1e-6, w=2e-6) % "vg" >> Vss()  # NMOS管，漏极为 vd, 栅极为 vg, 源极和衬底为 Vss
    VSource("vd", Vss(), dc="vds", name="vds_source") # Vds 电压源，值为参数 vds
    VSource("vg", Vss(), dc=1, name="vgs_source")     # Vgs 电压源，直流值为 1V (这将是扫描变量)

    # 定义外部扫描 (DataSweep for vds)
    # vds 将从 0V 到 0.9V，步长 0.1V
    vds_sweep = DataSweep(name="vds_sweep", params=["vds"], values=[np.arange(0, 1, 0.1)])

    # 执行直流扫描 (DCSweep)
    # 扫描 vgs_source 的 'dc' 参数 (即 Vgs)，从 0V 到 1.2V，步长 0.01V
    # 同时，对于每个 Vgs 点，vds 会根据 vds_sweep 的值进行变化
    raw = dcsweep(
        dev="vvgs_source",  # 注意：VSource 自动命名的规则，v<Name>
        param="dc",
        start=0,
        stop=1.2,
        step=0.01,
        data=vds_sweep      # 传入外部扫描数据
    )
    # raw 现在是一个 SweepWaveform 对象

    # 声明绘图：漏极电流 (通过 vds_source 的电流，HSPICE 中为 I(Vsource)，Spectre 为 Vsource:p)
    # 电流方向可能需要取反，具体取决于仿真器和电流定义
    # HSPICE 通常需要 I(element_name)，Spectre电流通常是 :p 后缀
    # 假设 :p 代表流出正端的电流，如果需要漏极电流（流入漏极），可能需要取反
    watch("vvds_source:p", calc=lambda x: -x, mode="semilogy", ylabel="Ids (A)", xlabel="Vgs (V)")
    new_figure() # 创建新图
    watch("vvds_source:p", calc=lambda x: -x, ylabel="Ids (A)", xlabel="Vgs (V)")

    debug(f"SweepWaveform signal names: {raw.get_signal_names()}")
    
    # 获取特定扫描条件下的信号 (示例)
    # vgs_values = raw.get_sweeps() # 这是 Vgs 的扫描点 (0, 0.01, ..., 1.2)
    # first_vds_waveform = raw[0] # 这是 vds_sweep 的第一个值 (vds=0) 对应的 Waveform
    # current_for_first_vds_at_all_vgs = first_vds_waveform.get_signal("vvds_source:p")
    # debug(f"Currents for vds={vds_sweep.values[0][0]} at Vgs points: {current_for_first_vds_at_all_vgs}")

    raw.show(figname="mos_id_vg") # 显示并保存所有 Id-Vg 曲线

# MosIdVgChar()

7. Q&A

Q1: 如何加载一个标准的仿真结果文件 (如 .psf, .tr0)？ A1: 可以使用 Waveform.load() 或 Waveform.from_psf_file() 方法。

# 假设 module 是当前电路模块对象，simtype 是仿真类型如 "tran"
# 自动确定文件名
raw = Waveform.load(module, simtype) 
# 或者指定文件路径
# raw = Waveform.from_psf_file(module, "path/to/your/result.psf", "tran")

对于扫描仿真，如果结果是按扫描点分成了多个文件，或者在单个文件中但有内部结构（HSPICE的@符号，Spectre的swp-xxx目录），应使用 Waveform.load(module, simtype, data=your_datasweep_object) 或 Waveform.from_datasweep()。

Q2: 如何获取一个特定节点 (如 "VOUT") 的电压或电流信号？ A2: 使用 raw.get_signal("VOUT")。对于电流，命名方式取决于仿真器和保存方式。例如，在 HSPICE 中，通过器件 R1 的电流可能是 i(R1) (如果用 save i(R1)) 或 R1:1 (某些默认保存)。在 Spectre 中，器件 V0 的电流通常是 V0:p (p表示正端流出的电流)。请参考 raw.get_signal_names() 查看可用的确切名称。

Q3: 如何绘制信号波形？ A3: 先使用 watch() 声明要观察的信号，然后在获取 Waveform 对象 (raw) 后调用 raw.show()。

watch("VOUT", label="Output Voltage")
watch("VIN", label="Input Voltage")
raw = tran(stop="1u")
raw.show(figname="my_transient_plot")

Q4: 交流分析 (AC) 的结果是复数，如何获取幅度 (dB20) 和相位？ A4: 使用 get_signal() 方法的 signal_type 参数，或者对返回的复数数据进行计算。

# 获取复数形式的 VOUT
vout_complex = raw.get_signal("VOUT", signal_type="diff") # 或者不指定 signal_type，默认为 "diff"

# 获取幅度 (线性)
vout_mag_linear = raw.get_signal("VOUT", signal_type="VM")
# 手动计算 dB20:
# import numpy as np
# vout_db20 = 20 * np.log10(np.abs(vout_complex))

# 获取相位 (弧度)
vout_phase_rad = raw.get_signal("VOUT", signal_type="VP")
# 手动转为角度:
# vout_phase_deg = np.angle(vout_complex, deg=True)

# 也可以在 watch 中使用 calc
# watch("VOUT", calc=lambda x: 20*np.log10(np.abs(x)), label="VOUT_dB20")
# watch("VOUT", calc=lambda x: np.angle(x, deg=True), label="VOUT_Phase_deg")

当 signal_type 不为 "diff" 时，如果启用了灵敏度分析，to_tensor 方法会计算对应类型 (幅度/相位/实部/虚部) 的梯度。

Q5: SweepWaveform 和 Waveform 有什么区别？ A5: Waveform 通常代表一次仿真运行的结果。SweepWaveform 代表参数扫描仿真 (如 dcsweep 时指定了 data 参数) 的结果集合。它内部包含多个 Waveform 对象，每个对应扫描参数的一个取值。SweepWaveform 的 plot 或 show 方法会将所有这些内部 Waveform 的曲线绘制在同一张图上，并以图例区分。

Q6: 如何进行灵敏度分析/梯度计算？ A6: 1. 在电路中定义 TorchVariable 作为设计变量。2. 设置环境变量 TED_SENSITIVITY=1 (或通过 setenv("sensitivity", True) )。3. 运行仿真，例如 dcsweep，此时 TED 后端会进行差分扰动仿真。4. 使用 raw.get_signal() 获取信号时，返回的 TorchVariable 将包含对设计变量的梯度信息，可以直接用于基于 PyTorch 的优化。Waveform 类的 to_tensor 方法是实现这一功能的关键。

Q7: 支持哪些仿真器？ A7: 从代码中的 get_result_file_path 和 get_result_file_name 等方法可以看出，主要支持 HSPICE, HSIM, ALPS, Spectre, 和 Spectremdl。不同仿真器的结果文件格式和命名约定有所不同，该模块会尝试处理这些差异。

Q8: 如何自定义绘图的样式，比如使用暗色背景？ A8: 可以使用 raw.set_plt_style() 方法。 python raw = tran(stop="1u") raw.set_plt_style({"plt.style.use": "dark_background"}) watch("VOUT") raw.show() 或者在 waveform.py 中修改 Waveform 类初始化时的 self.plt_style_settings 默认值。

Q9: sec_simtype 参数是做什么用的？ A9: sec_simtype (二级仿真类型) 主要用于某些复杂的仿真类型，如 pss (周期性稳态分析)。PSS 分析后通常可以提取时域波形 (td) 或频域谐波 (fd)。sec_simtype 允许指定要加载的是哪一种结果。例如，simtype="pss", sec_simtype="td"。这在 MultiWaveform 和新版 Waveform 加载 PSS 结果时比较重要。
