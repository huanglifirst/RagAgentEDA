PDK

功能介绍:

PDK 是一个用于管理工艺库（Process Design Kit）核心参数和配置的类，继承自 PdkBase。该类主要用于获取与半导体工艺相关的器件最小尺寸、方程计算、仿真选项设置以及工具链配置等信息。通过统一的接口提供对底层工艺数据的访问，支持多种模型（如 nch、pch）和参数动态传入，适用于电路设计、仿真和验证环节。

该类的核心功能包括：

  • 获取晶体管最小长度 lmin
  • 根据配置文件中的方程表达式计算器件参数值
  • 支持带变量的数学表达式求值（自动解析和替换）
  • 获取仿真选项默认值，并可自定义覆盖
  • 返回当前使用的工具链（toolchain）

参数定义:

参数名类型含义默认值
namestr指定 PDK 的名称，用于标识特定的工艺库版本或配置。None
modelstr指定器件模型类型，如 'nch' 表示 NMOS，'pch' 表示 PMOS。'nch'
keystr用于查找配置中特定方程的键名，例如 'sa'、'sb' 等。无
**paramsdict用户传入的参数字典，用于替换表达式中的变量占位符。支持 Variable 类型对象自动求值。无
**kwargsdict可选关键字参数，传递给 get_equation_value 或其他方法。无

返回值

  • lmin：返回当前工艺下 MOS 管的最小沟道长度（float 或 None）。
  • get_equation_value：返回方程计算结果（float 或 None）。若键不存在或表达式解析失败，返回 None。
  • get_sa, get_sb：返回特定参数（源极扩散长度、漏极扩散长度等）的浮点数值或 None。
  • get_all_lpe_params：返回一个字典，包含所有方程键对应的计算值；若未启用 load_lpe_params，则返回空字典。
  • get_equation：返回表达式求值结果（float 或原始字符串表达式）；出错时返回表达式本身。
  • get_simulation_options：返回一个包含仿真选项的字典。
  • get_toolchain：返回当前使用的工具链名称（字符串）。

def test_pdk_get_sa_sb():
    pdk = PDK(name="tsmc180")
    sa = pdk.get_sa(model='nch', w=1e-6, l=2e-7)
    sb = pdk.get_sb(model='nch', w=1e-6, l=2e-7)
    assert isinstance(sa, float) and isinstance(sb, float)

    pdk = PDK(name="tsmc180")

    pdk = PDK(name="tsmc180")
    result = pdk.get_equation(key='sa', model='nch', w=1e-6, l=2e-7)
    assert isinstance(result, float) or isinstance(result, str)

def test_pdk_simulation_options():
    pdk = PDK(name="tsmc180")
    options = pdk.get_simulation_options()
    assert 'reltol' in options and options['reltol'] == 0.001
    # 自定义仿真选项测试

    new_options = pdk.get_simulation_options()
    assert new_options['reltol'] == 0.0001

def test_pdk_toolchain():
    pdk = PDK(name="tsmc180")
    toolchain = pdk.get_toolchain()
    assert isinstance(toolchain, str)

  • test_pdk_toolchain：获取当前工具链名称，优先读取环境变量，否则调用父类方法。

lmin

功能介绍:

lmin 是一个用于获取半导体工艺中最小特征尺寸（Minimum Channel Length）的函数。该函数属于 PDK（Process Design Kit，工艺设计套件）配置的一部分，返回当前工艺节点下 MOSFET 晶体管的最小沟道长度值。此值通常作为设计中的基本几何单位，用于定义晶体管尺寸、布局间距等关键参数。

该函数是一个实例方法，需通过 get_pdk() 获取 PDK 实例后调用，其主要用途是在电路和版图设计中以标准化方式引用工艺最小长度，保证设计符合制造规范。

参数定义:

参数名类型含义默认值
modelstr指定要查询的晶体管类型模型，例如 'nch' 表示 N 型 MOS，'pch' 表示 P 型 MOS，也可支持带版本的模型如 'nch_v1'。'nch'
unitstr返回值的单位，目前支持 'm'（米），也可扩展为其他单位（如 'um'、'nm'）。'm'

返回值

  • 类型：float
  • 含义：返回当前工艺节点下指定模型的最小沟道长度值，单位由 unit 参数决定。该值通常为一个极小的浮点数（如 6.5e-9），表示工艺允许的最小晶体管长度。

from pyted import *

from tedcore.kernel import get_circuit, Serial, col, place, skip_unless
from ted_device.components.mos.mos import PMOS
from algo.router.route.__init__ import route

@skip_unless(TED_PDK=[T65])
def test_rotate03():

    @testbench
    def Top():
        l = get_circuit().get_pdk().lmin()
        w = 8 * l
        Serial('d', (mos1 := (PMOS(l=l, w=w) % 'g')), (mos2 := (PMOS(l=l, w=w) % 'g')), 's')
        spacing = mos1.mosfet_overlap_size()
        mos1.rotate(-90)
        mos2.rotate(-90)
        col(mos1, mos2, spacing=-spacing)
        place()

get_simulation_options

功能介绍:

get_simulation_options 是一个用于获取仿真配置选项的方法，通常在电路仿真环境中调用。该方法从当前对象的 config 属性中读取自定义的仿真参数设置，并与默认值进行合并，最终返回一组完整的仿真选项字典。

此方法属于类成员函数（实例方法），主要用于支持仿真器（如SPICE）所需的多种配置选项设置，涵盖精度控制、温度设定、数值格式、警告限制等常见仿真参数。通过该接口，用户可以在配置文件中覆盖默认值，实现灵活的仿真环境定制。

参数定义:

无参数。

返回值

  • 类型: dict
  • 含义: 返回一个包含所有仿真选项的字典，其中键为配置项名称，值为对应配置参数的数值或字符串。该字典可直接用于仿真器初始化或参数注入流程。

代码示例:

# 假设存在一个包含 config 配置的对象实例
config = {
    'simulationOptions': {
        'reltol': 1e-04,
        'vabstol': 1e-07,
        'temp': 50,
        'maxwarns': 10
    }

# 创建一个模拟对象，其 config 属性包含上述内容
class Simulator:
    def __init__(self, config):
        self.config = config

    def get_simulation_options(self):
        default = {'reltol': 0.001, 'vabstol': 1e-06, 'iabstol': 1e-12, 'temp': 27, 'tnom': 27, 'scalem': 1.0, 'scale': 1.0, 'gmin': 1e-12, 'rforce': 1, 'maxnotes': 5, 'maxwarns': 5, 'digits': 5, 'cols': 80, 'pivrel': 0.001, 'checklimitdest': 'psf', 'useterms': 'name'}
        if 'simulationOptions' in self.config:
            for (k, v) in self.config['simulationOptions'].items():
                default[k] = v
        return default

# 使用示例
sim = Simulator(config)
options = sim.get_simulation_options()
print(options)

get_sb 是一个用于获取特定电路模型下“sb”（可能代表某种结构参数或电学参数，如源漏接触相关的关键尺寸或电学特性）值的函数方法。该函数属于某个类的成员方法，通过调用内部的 get_equation_value 方法，基于指定的模型类型（model）和额外参数（kwargs），计算并返回对应的浮点型数值。其用途通常出现在电路设计中的参数提取或仿真数据建模场景。

参数名类型含义默认值

  • 含义：返回计算得到的 sb 值，若无法获取或计算失败则返回 None。

get_equation_value

功能介绍:

get_equation_value 是一个用于从 PDK（工艺设计工具包）配置文件中提取器件参数值的函数。该函数通过解析 YAML 格式的工艺参数定义，结合传入的变量参数，计算并返回指定方程的数值结果。

该函数属于某个类的方法（根据 self 参数推断），主要作用是从 tools/pdk/yamls/*.yaml 文件中读取特定模型（如 nch）下的方程配置，并根据提供的参数求解表达式值，适用于在电路建模和仿真过程中动态获取器件的物理参数。

函数支持表达式计算，使用了 CExpr 类进行数学表达式解析与求值，能够处理包含变量和复杂数学运算的方程。

参数定义:

参数名类型含义默认值
self类实例当前对象实例，用于访问配置数据。无
keystr需要查找的方程在配置中的键名（即方程标识符）。无
modelstr指定使用的器件模型类型，如 'nch' 表示 NMOS 器件模型。'nch'
**paramsdict可变关键字参数，用于传递方程计算所需的具体变量值。键为变量名，值为数值或 Variable 实例。无

返回值

  • 返回类型：Optional[float]
  • 含义：若指定键存在于配置中且表达式可成功求值，则返回计算结果的浮点数；若键不存在或表达式无法求值（如变量缺失），则返回 None。

代码示例:

def test_get_equation_value():
    # 模拟一个包含配置的类实例
    class MockPDK:
        def __init__(self):
            self.config = {
                'equation': {
                    'nch': {
                        'width': 'W',
                        'length': 'L * 1.5',
                        'threshold_voltage': 'Vth + 0.1'
                    }
                }
            }

    # 创建实例
    pdk = MockPDK()

    # 1. 基本参数求值：返回单个变量的值
    result1 = pdk.get_equation_value('width', model='nch', W=1.2)
    assert result1 == 1.2

    # 2. 表达式计算：带运算表达式
    result2 = pdk.get_equation_value('length', model='nch', L=0.3)
    assert result2 == 0.45

    # 3. 复合参数：包含多个变量和常量
    result3 = pdk.get_equation_value('threshold_voltage', model='nch', Vth=0.6, W=1.2)
    assert result3 == 0.7

    # 4. 变量实例作为输入
    from tedcore.kernel import Variable
    var_x = Variable(value=2.0)
    result4 = pdk.get_equation_value('width', model='nch', W=var_x)
    assert result4 == 2.0

    # 5. 未定义的键返回 None
    result5 = pdk.get_equation_value('unknown_key', model='nch')
    assert result5 is None

get_all_lpe_params

功能介绍:

get_all_lpe_params 是一个用于获取指定模型下所有 LPE（Linear Parameterized Equation，线性参数化方程）参数的方法。该方法基于配置中的方程定义，依据给定的模型名称（model）提取并返回所有相关的参数值。若环境变量 load_lpe_params 为 True，则从配置中加载对应模型的所有参数；否则返回空字典。此方法通常用于在电路设计流程中动态获取与特定模型相关的物理或电气参数。

参数定义:

参数名类型含义默认值
self任意类实例表示调用该方法的上下文对象，通常为模块级实例。无
modelstr指定要加载参数的模型名称，如 'nch'、'pch' 等。'nch'
**kwargs可变关键字参数传递给 get_equation 方法的其他额外参数，用于控制参数获取的细节。无

返回值

  • 返回一个字典，键为方程名称（str），值为其对应参数的计算结果（类型由 get_equation 的返回决定）。如果无法加载或配置中不存在该模型，则返回空字典（{}）。

代码示例:

def test_get_all_lpe_params():
    # 1. 基本使用：获取 'nch' 模型的所有 LPE 参数
    params_nch = get_all_lpe_params(model='nch')
    assert isinstance(params_nch, dict), "返回值应为字典"
    assert len(params_nch) == 0, "若 load_lpe_params 未启用，应返回空字典"

    # 2. 启用参数加载：设置环境变量后获取参数
    from tedcore import setenv
    setenv('load_lpe_params', True)
    
    params_nch_with_env = get_all_lpe_params(model='nch')
    assert isinstance(params_nch_with_env, dict), "启用后仍应返回字典"
    # 此处具体键值内容取决于实际配置中 'equation' 下 nch 的定义
    # 但假设存在 key1, key2 等方程，它们的输出应为数值或可迭代结构

    # 3. 请求不存在的模型
    empty_result = get_all_lpe_params(model='invalid_model')
    assert empty_result == {}, "针对不存在模型应返回空字典"

    # 4. 验证 kwargs 被正确传递至 get_equation
    def mock_get_equation(key, model, **kw):
        return {'key': key, 'model': model, 'extra': kw.get('context', None)}

    # 模拟原方法中的 get_equation 调用（实际在环境中应已定义）
    orig_get_equation = self.get_equation
    self.get_equation = mock_get_equation

    try:
        result_with_kw = get_all_lpe_params(model='nch', context="test_context")
        assert result_with_kw['some_key']['context'] == 'test_context', "kwargs 应正确传递"
    finally:
        self.get_equation = orig_get_equation

说明：

  • 用法1（基础获取）：调用 get_all_lpe_params(model='nch') 时，若环境变量 load_lpe_params 为 False，返回空字典；否则从配置中提取 equation['nch'] 的所有键，并通过 get_equation(k, model='nch') 构建一个参数映射字典。
  • 用法2（环境控制加载）：通过设置 setenv('load_lpe_params', True) 可开启参数加载机制，使方法返回真实配置参数。
  • 用法3（错误模型处理）：当传入的 model 不在配置文件中时，方法立即返回空字典，防止异常。
  • 用法4（传递额外参数）：支持将任意 **kwargs 传递给内部调用的 get_equation 方法，实现灵活扩展。
