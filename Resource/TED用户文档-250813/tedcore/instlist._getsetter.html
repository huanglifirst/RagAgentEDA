set_pr_boundary

功能介绍:

该函数用于设置实例列表（InstList）的物理区域边界（PR Boundary），即指定布局时的参考边界范围。该接口作用于 InstList 类型的对象，通过传入一个 Box 对象来定义边界坐标，从而影响后续的布局行为（如自动排布、放置等）。此函数在版图设计中用于精确控制子模块或实例组的排列边界，确保其在特定区域中进行对齐或约束。

参数定义:

参数名类型含义默认值
pr_boundaryBox表示物理区域边界的矩形框，包含坐标信息（左上角与右下角）。该参数用于指定布局的边界范围。无

返回值

返回调用该方法的 InstList 实例本身（self），支持链式调用。

代码示例:

以下为提供的测试用例，展示了 set_pr_boundary 的典型使用方式：

# /home/zuochang/ted/tedcore/kernel/tests/test_placer.py:484
from pyted import *

from tedcore.kernel import Node, Rect, lay, set_pr_boundary, row, place

def test11():

    @testbench
    def Top():
        a = Node()
        r1 = Rect(M1, 1000, 1000, a)
        left = lay(r1, Rect(TEXT, 1500, 1000), x_align=CENTER)
        left.set_pr_boundary(r1.box)
        r2 = Rect(M1, 1000, 1000, a)
        right = lay(r2, Rect(TEXT, 1500, 1000), x_align=CENTER)
        right.set_pr_boundary(r2.box)
        row(left, right, pr_boundary=True)
        assert r1.get_pr_boundary().x2 == r2.get_pr_boundary().x1
        place()

示例说明：

  • 创建两个 Rect 实例 r1 和 r2，并分别使用 lay 将其与一个文本层对象进行对齐排列。
  • 调用 left.set_pr_boundary(r1.box) 为 left 实例设置其 PR 边界为 r1 的边界框。
  • 同理，为 right 实例设置其 PR 边界为其自身 r2.box。
  • 使用 row(left, right, pr_boundary=True) 将两个实例按行排列，并启用共享 PR 边界特性。
  • 断言验证：左侧实例的右边界（x2）与右侧实例的左边界（x1）相等，表明它们在布局时正确对接。
  • 最终调用 place() 执行实际放置操作。

该示例展示了 set_pr_boundary 在多模块协同布局场景中的关键作用——确保子模块间通过 PR 边界进行精确对接。

get_survivors

功能介绍:

get_survivors 是一个实例列表（InstList）类的方法，用于从当前实例列表中筛选出所有仍处于“存活”状态的实例。所谓“存活”状态，是指实例调用 alive() 方法返回 True 的实例。该方法返回一个新的 InstList 实例，包含所有满足条件的存活实例，且不改变原始列表。

此方法常用于在电路设计流程中清理已销毁或无效的实例，保留有效的设计单元进行后续处理。

参数定义:

参数名类型含义默认值
selfInstList当前实例列表对象，用于遍历其中的所有实例。无

注：该方法为实例方法，不接受外部显式参数，仅通过 self 引用当前实例。

返回值

  • 类型：InstList
  • 含义：包含原始列表中所有 alive() 返回 True 的实例的新实例列表。新列表保持原有结构，但不包含已失效的实例。

代码示例:

def test_get_survivors_empty():
    instlist = InstList()
    assert get_survivors(instlist) == InstList()

def test_get_survivors_normal():
    a, b, c = Inst(), Inst(), Inst()
    a.kill()
    result = get_survivors(InstList(a, b, c))
    assert len(result) == 2
    assert b in result
    assert c in result
def test_get_survivors_all_alive():
    instlist = InstList(Inst(), Inst(), Inst())
    result = get_survivors(instlist)
    assert result is not instlist  # 返回新列表
    assert len(result) == 3
    assert all(inst in result for inst in instlist)
def test_get_survivors_all_dead():
    instlist = InstList(Inst(), Inst(), Inst())
    for inst in instlist:
        inst.kill()
    result = get_survivors(instlist)
    assert len(result) == 0
    assert result == InstList()

说明：

  • 在 test_get_survivors_empty 中，空列表返回空的 InstList，体现方法对边界情况的处理能力。
  • 在 test_get_survivors_normal 中，一个实例被手动销毁（调用 kill()）后，get_survivors 会正确排除它，仅保留存活实例。
  • 在 test_get_survivors_all_alive 中，所有实例均未被销毁，返回值为包含全部三个实例的新列表，且与原列表不为同一对象（深拷贝语义）。
  • 在 test_get_survivors_all_dead 中，所有实例都已死亡，返回空列表。

上述测例展示了该 API 的核心行为：从实例集合中筛选出存活实例并构建新集合，适用于清理无效设计单元、构建有效路径或用于后续逻辑处理。

get_layer_nboxes

功能介绍:

该函数用于获取 InstList 实例中所有实例的层级（Layer）与 NBox 集合之间的映射关系。它遍历 InstList 中每个实例，调用其 get_layer_nboxes() 方法，收集所有层级上的 NBox 列表，并将相同层级的结果合并，最终以字典形式返回。该函数是用于在电路布局中按层聚合 NBox（矩形区域）信息的核心工具，常用于后续的布线、检查或可视化处理。

参数定义:

参数名类型含义默认值
selfInstList表示要查询的实例列表，其每个元素应支持 get_layer_nboxes() 方法。无

返回值

  • 返回类型：Dict[Layer, List[NBox]]
  • 含义：键为 Layer 类型的层级对象，值为对应层级上所有 NBox 的列表。该字典表示整个 InstList 中各层上的所有 NBox 集合。

代码示例:

# /home/zuochang/ted/algo/router/pattern_route/tests/test_layout_mos.py:785
from pyted import *

from tedcore.kernel import get_layer_nboxes, place
from ted_utils import debug
from ted_device.components.mos.mos import NMOS
from algo.router.pattern_route.layout_mos.main import layout_mos

def test52():

    def get_m1_set(item: InstList) -> Set['NBox']:
        ret = set(item.get_layer_nboxes()[M1])
        debug(ret)
        return ret

    @testbench
    def Top52():
        mos = NMOS(l=1e-06, w=1e-06)
        res = layout_mos([mos], ds=1, ds_layer=M1, gate=0)
        assert get_m1_set(mos) == get_m1_set(res)
        place(via_min_number=1)

示例说明:

该测试用例展示了 get_layer_nboxes 函数在电路布局场景下的典型用法：

  1. 按层提取 NBox：定义辅助函数 get_m1_set，其功能是调用 item.get_layer_nboxes() 获取所有层级的映射，然后取出其中 M1 层对应的 NBox 列表，并转换为集合（Set）便于比较。
  2. 布局前后一致性验证：通过 layout_mos 函数对单个 NMOS 实例进行布局生成结果 res，随后调用 get_m1_set(mos) 和 get_m1_set(res) 分别获取原始实例和布局后实例在 M1 层的 NBox 集合，并断言二者相等。这验证了布局过程未改变关键层（M1）上的几何区域集合。
  3. 使用场景总结：该用例表明，get_layer_nboxes 适用于对比布局前后电路元素的几何分布是否一致，是进行自动布局逻辑正确性验证的重要手段。

此示例体现的核心用法为：通过 get_layer_nboxes() 获取整个实例列表在各层上的 NBox 分布，用于层级化查询与状态比对。

get_input

功能介绍:

get_input 是一个用于获取电路实例输入节点的函数，属于 InstList 类的成员方法。该方法返回指定实例列表中每个实例对应的输入信号节点集合。主要用于在多实例电路设计场景下，批量获取特定模块（如门电路、逻辑单元等）的输入连接信息。

此函数常用于测试或调试环境中，以验证电路连接是否正确，例如确认多个 Nand2 门的输入是否正确地连接到相应的输入节点。

参数定义:

参数名类型含义默认值
selfInstList当前实例列表（如多份Nand2模块的集合），表示一组电路单元的实例集合。无

返回值

  • 返回类型：CircuitNode
  • 含义：表示该实例列表中所有成员的输入信号节点组成的集合。若为单个实例，则返回对应输入节点；若为多实例（如 Nand2() * m），则返回包含多个输入节点的组合集合。

代码示例:

  • 连接验证：测试用例中通过断言检查返回的输入是否与预期的输入节点一致（即 a[0] 和 b[0]），用于确保电路连接正确无误。
  • 调试支持：结合 debug() 函数打印输出，有助于在仿真或测试环境中观察实际连接状态。

get_flatten_nboxes

功能介绍:

该函数用于获取 InstList 类实例中所有嵌套的 NBox 对象，并将其展平为一个单一的列表。InstList 通常表示一组电路实例的集合，而每个实例可能自身包含多个 NBox（如子模块或子结构）。此方法递归遍历整个实例结构，将所有层级的 NBox 收集并返回一个扁平化的列表。

该函数属于 InstList 类的方法，用于对实例列表中的所有结构进行深度遍历和数据提取。

参数定义:

参数名类型含义默认值
selfInstList当前实例列表的引用，表示需要展开其内部 NBox 结构的对象。无

注：self 是方法的隐式参数，代表类实例本身，在调用时无需显式传递。

返回值

  • 类型：List['NBox']
  • 含义：返回一个包含所有层级嵌套 NBox 对象的扁平列表，顺序为按 InstList 遍历顺序依次展开。

代码示例:

def test_get_flatten_nboxes():
    # 创建顶层 InstList 实例
    inst_list = InstList()

    # 新建一个 NBox 并添加到 inst_list
    top_box = NBox()
    inst_list.append(top_box)

    # 在 top_box 中嵌套另一个 NBox
    nested_box = NBox()
    top_box.nboxes.append(nested_box)

    # 调用 get_flatten_nboxes 方法
    result = inst_list.get_flatten_nboxes()

    # 断言结果包含两个 NBox 对象
    assert len(result) == 2
    assert result[0] is top_box
    assert result[1] is nested_box
def test_get_flatten_nboxes_empty():
    # 测试空的 InstList 情况
    inst_list = InstList()
    result = inst_list.get_flatten_nboxes()
    assert len(result) == 0
def test_get_flatten_nboxes_multiple_depths():
    # 构建较深嵌套结构
    root = InstList()
    box1 = NBox()
    box2 = NBox()
    box3 = NBox()

    # 嵌套层次：root -> [box1, box2]，其中 box2 内再包含 box3
    root.append(box1)
    root.append(box2)
    box2.nboxes.append(box3)

    result = root.get_flatten_nboxes()
    assert len(result) == 3
    assert result[0] is box1
    assert result[1] is box2
    assert result[2] is box3

说明：

  • 该函数适用于所有层级的 InstList 嵌套结构，能正确处理单层、多层嵌套及空列表等场景。
  • 示例展示了三种典型用法：
    1. 单一嵌套（一个顶层 NBox 包含一个子 NBox）；
    2. 空列表输入，返回空列表（边界情况）；
    3. 多级嵌套结构，正确递归提取所有 NBox。

函数通过 for inst in self: 遍历每个实例，并调用其 get_flatten_nboxes() 方法进行递归展开，最终使用 extend 将结果合并为一个扁平列表。

get

功能介绍:

get 是 InstList 类的一个实例方法，用于从 InstList 对象中按索引获取指定位置的 Instance 实例。该方法通过重载了 Python 的 __getitem__ 操作符实现，允许使用方通过类似列表访问的方式（如 inst_list[0]）来获取 InstList 中存储的各个实例。

此 API 通常用于在电路设计中访问已经创建的模块实例集合中的特定实例，以进行后续的连接、控制或分析操作。

参数定义:

参数名类型含义默认值
indexint要获取的实例在 InstList 中的位置索引，从 0 开始。无

返回值

  • 返回类型：Instance
  • 含义：返回 InstList 中指定索引位置的 Instance 实例。
