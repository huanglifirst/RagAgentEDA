reload_pdk

功能介绍:

reload_pdk 是一个用于重新加载指定名称的 PDK（Process Design Kit，工艺设计工具包）的函数。该函数旨在更新当前环境中已加载的 PDK 配置，例如重新读取层映射文件（layermap file）等资源。

然而，该功能目前未实现，由于存在一个与全局变量相关的 Bug，导致无法正确重新加载 PDK 的状态。因此，调用此函数时将始终抛出异常。

注意：虽然此函数在接口上定义为可传入 PDK 名称进行重载，但由于技术限制，当前不可用，仅作占位和未来扩展设计之用。

参数定义:

参数名类型含义默认值
pdk_namestr需要重新加载的 PDK 的名称，如 's180bcd'。无

返回值

无返回值。该函数为 None 型，但实际执行时会抛出异常。

代码示例:

# /home/zuochang/ted/tedcore/kernel/tests/test_pdk.py:5
from pyted import *

from tedcore.kernel import get_pdk, reload_pdk, skip_unless
from ted_utils import debug

@skip_unless(TED_TEST_LVL=99, TED_PDK=[T65])
def test01():
    file1 = get_pdk().get_layermap_file()
    debug(file1)
    debug(M1)
    reload_pdk('s180bcd')
    file2 = get_pdk().get_layermap_file()
    debug(M1)
    debug(file2)
    assert file1 != file2

说明：
该测试用例展示了 reload_pdk 在理想状态下的使用场景：

  • 首先通过 get_pdk().get_layermap_file() 获取当前 PDK 的层映射文件路径，并保存为 file1。
  • 调用 reload_pdk('s180bcd')，试图重新加载名为 's180bcd' 的 PDK。
  • 再次获取更新后的层映射文件路径 file2。
  • 通过对比 file1 != file2 验证 PDK 是否成功刷新。

注意：尽管测试用例中定义了这一流程，但实际运行中由于函数内部抛出异常 'There is a BUG that cannot reload global variables'，该代码段将不会正常执行完毕，无法验证重载逻辑。因此，此测例仅用于示意未来功能设计的预期行为。

lmin

功能介绍:

lmin 是一个用于获取当前工艺库（PDK, Process Design Kit）下最小特征尺寸的函数。该值表示工艺支持的最小设计线宽，单位为微米（microns），通常用于电路设计中的尺寸计算，例如确定晶体管、电阻等元件的长宽参数。此函数是EDA设计流程中不可或缺的基础查询功能，确保设计符合工艺约束。

参数定义:

无参数。

返回值

  • float：当前PDK的最小特征尺寸，单位为微米（μm）。

代码示例:

以下示例来自测试用例，展示了如何在电路设计中使用 lmin() 函数来动态计算元件尺寸：

@skip_unless(TED_PDK=[T65, T40, T180, S180BCD, S180MM])
def test05():
    l = 2 * lmin()
    w = 8 * lmin()

    @testbench
    def Top():
        from ted_builtins import VSource
        from ted_builtins import Mosfet
        ('vg', Vss()) >> VSource(dc=1.2)
        ('vd', Vss()) >> VSource(dc=1.2)
        Vdd() >> (dut := RpolySingle(l=l, w=w)) >> 'b'
        debug('tran')
        raw = tran(stop=1e-06)
        debug(raw)
        debug(raw.get_signal_names())

使用说明：

  • l = 2 * lmin()：将晶体管或电阻的长度 l 设置为最小特征尺寸的2倍，这是典型的工艺规约设计，用于保证器件可制造性。
  • w = 8 * lmin()：将宽度 w 设为最小特征尺寸的8倍，符合常见电阻或晶体管的宽长比设计规则。
  • 在 RpolySingle 模块实例化时传入 l 和 w 参数，使实际布局参数与当前PDK匹配。
  • 测试中通过 tran 进行瞬态仿真，并使用 debug 输出仿真结果，验证电路行为。

该用法体现了 lmin() 在动态依据工艺库生成合理设计参数中的核心作用。

get_pdk

功能介绍:

该函数用于获取当前电路上下文中的工艺设计套件（Process Design Kit, PDK）实例。PDK是集成电路设计中定义制造工艺约束、物理参数和器件模型的关键配置集合，包含工艺节点信息、最小特征尺寸、模型路径等。通过调用此方法，可以访问与当前设计相关的完整工艺配置数据，便于后续进行布局布线、仿真或验证操作。

该函数为类方法的一个封装，存在于 circuit 模块上下文中，通常由 get_circuit() 获取电路实例后调用其 get_pdk() 方法获得 PDK 对象。

参数定义:

无参数。

返回值

  • 类型: PDK
  • 含义: 返回当前电路所使用的工艺设计套件（PDK）对象，该对象封装了与特定制造工艺相关的所有配置信息，例如最小尺寸 lmin、模型文件路径、工艺参数等。可进一步调用其方法如 get_config_data() 或 name 等获取具体配置。

代码示例:

get_layer

功能介绍:

get_layer 是一个用于根据名称获取电路层对象的函数。该函数主要用于在集成电路设计中定位特定的工艺层（layer），支持通过原始PDK名称或声明名称进行查找。返回值为 Layer 类型对象，代表指定的电路层；若未找到匹配的层，则返回 None。

该函数的实现考虑了两种命名方式：

  • 若启用了 TED_BUILD_PDK 环境变量，函数直接返回 None（用于构建场景下跳过层查找）。
  • 否则，优先尝试以原始PDK名称查找；若失败，则尝试以 TED 声明的层名查找。

此函数广泛应用于布局设计、布线操作和验证流程中，例如在创建矩形元件、设置覆盖层、连接端口或执行自动布护环（guard ring）等操作时，需指定具体的金属层或工艺层。

参数定义:

参数名类型含义默认值
namestr要查询的层名称，可以是原始PDK中的名称或 TED 中声明的名称。无
is_original_namebool是否将 name 视为原始PDK名称（即不经过TED映射的底层名称）。当设为 True 时，函数优先使用 get_by_original_name 方法查找。False

返回值

  • 类型：Optional[Layer]
  • 含义：成功查找到对应层时返回 Layer 实例；否则返回 None。

get_corners

功能介绍:

get_corners 是一个用于获取当前电路（circuit）中所设定的 Corner 配置的函数。在模拟与全定制电路设计中，Corner 表示工艺角（Process Corner），用于描述芯片在不同制造工艺偏差下的性能表现，如 TT（典型）、FF（快快）、SS（慢慢）等。该函数返回当前环境中配置的 corner 列表，通常用于在仿真或设计验证过程中动态获取当前工艺角设置。

此函数不接收任何参数，其行为依赖于全局环境中的 corners 配置。通过 setenv(corners=[...]) 可以修改当前环境的 Corner 设置，从而影响后续对 get_corners 的调用结果。

参数定义:

无参数。

返回值

  • 类型: List[str]
  • 含义: 返回一个字符串列表，表示当前电路环境中配置的 Corner 名称（例如 ['tt_lib'] 或 ['tt1', 'tt2']）。

代码示例:

  • 接着使用 setenv(corners=['tt1', 'tt2']) 临时修改当前环境的 Corner 设置为 ['tt1', 'tt2']。
  • 在该作用域内再次调用 get_corners()，返回值变为 ['tt1', 'tt2']，验证了环境配置变更生效。
  • 测试使用 @skip_unless(TED_PDK=T65) 条件执行，确保仅在特定工艺库条件下运行。

  • 在所有涉及电路设计的测试或模块中，首先通过 get_circuit() 检查当前 PDK 是否为预期工艺（如 't65'），以确保设计兼容性。
  • 该函数作为全局入口，在实例化 InstanceCollection、调用布局工具 autolayout_euler、连接电源（Vdd, Vss）、执行 place() 布局操作前，必须确保电路已初始化。
  • 所有操作均基于该唯一全局电路对象进行状态维护和上下文管理。
