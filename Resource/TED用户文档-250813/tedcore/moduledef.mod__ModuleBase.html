ModuleBase

功能介绍:

ModuleBase 是 TED 框架中用于表示电路设计实体的核心基类。它集成了多种混合能力，支持模块的创建、管理与操作，涵盖参数配置、节点连接、实例化、布局布线、物理验证等关键功能。

该类继承了多个 Mixin 类（如 GetSetterMixin、ImportMixin、PortMixin 等），实现了对模块属性的灵活控制和扩展能力。它是所有用户自定义电路模块的基类，通过装饰器机制支持将普通函数转换为模块结构。

一个 ModuleBase 实例可以被实例化为 Instance，并可嵌套在其他模块中构成层次化设计结构，适用于复杂的模拟与全定制集成电路的设计与仿真。

参数定义:

无

返回值

返回值类型含义
ModuleModuleBase当前 ModuleBase 实例本身

代码示例:

结合所提供的测试用例，展示 ModuleBase 的典型使用方式如下：

# /home/zuochang/ted/tedcore/kernel/src/decorator/tests/test_module_decorator.py:6
from pyted import *

from tedcore.kernel import ModuleBase, Module, skip_unless

@skip_unless(TED_PDK=[T65])
def test1():

    class Analog(ModuleBase):

        def special_analog_func(self):
            print('call: special_analog_func')
            return self

    @ModuleBase()
    def Dut():
        pass
    Dut().get_name()
    try:
        Dut().to_hspice_subckt()
        assert '不该有to_spice_subckt'
    except Exception as e:
        print(e)

    @Module()
    def Dut2():
        pass
    Dut2().get_name()
    Dut2().module().to_hspice_subckt()

示例说明：

  1. 定义类继承 ModuleBase
    使用 class Analog(ModuleBase): 定义一个自定义电路模块类，继承于 ModuleBase。该类可包含用户自定义的方法（如 special_analog_func），用于实现特定功能逻辑。

  2. 使用 @ModuleBase() 装饰器创建模块
    通过 @ModuleBase() 装饰一个无参函数 Dut，将其转换为可实例化的模块。此时调用 Dut() 会产生一个 Instance 实例，该实例具备 get_name() 方法，用于获取模块名称。

  3. 调用 module() 方法
    在 Module 的实例上（如 Dut2）可以调用 .module() 获取其底层的模块类对象。这可用于访问模块级的方法，例如 to_hspice_subckt() 用于生成 SPICE 子电路文件。

  4. 异常处理检测不支持的方法
    尝试调用 Dut().to_hspice_subckt() 引发异常，并通过捕获进行验证：说明仅 Module 类型的实例才支持该方法，而 ModuleBase 实例不直接支持此功能，体现了模块层级与实例行为的区别。

  5. 模块名称获取
    通过 get_name() 方法获取模块名称，返回值为字符串类型的模块名（由函数名自动推导）。

以上示例展示了 ModuleBase 在实际设计流程中的典型用法：作为模块定义的基类、配合装饰器构建可实例化的电路结构，并支持与后续工具链（如 HSPICE 导出）集成。

set_type

功能介绍:

set_type 是用于设置模块（Module）分类标识的方法。该方法作用于模块实例，允许用户为某个模块指定其类型名称（type_name），以便在系统中进行模块的分类管理与检索。此方法是模块属性配置的一部分，通常在模块创建后调用，以明确模块所属的技术类别或功能类型。

参数定义:

参数名类型含义默认值
type_namestr模块的类型名称，用于标识该模块所属的具体分类。无

返回值

返回当前模块实例自身（self），支持方法链式调用。

代码示例:

from pyted import *

from tedcore.kernel import skip_unless

def test_category_and_type_management():

        pass

示例说明：

该用法展示了 set_type 方法在模块创建后，用于设定其类型标识的功能。通过链式调用方式（.set_category(...).set_type(...)) 可以连续配置模块的属性。此方法常用于构建具有明确功能分类的电路模块，便于后续的系统集成与管理。

set_skip_autovia

功能介绍:

该方法用于设置当前模块不进行自动打孔（auto via）处理。在电路设计中，自动打孔是指工具根据工艺规则自动生成连接不同金属层之间的通孔（via）。调用此方法后，当前模块内的金属层连接将不会触发自动打孔操作，适用于需要手动控制通孔或避免误生成通孔的特殊设计场景。

该方法是 current_module() 对象的一个成员函数，通常在模块定义内部使用，作用范围为当前模块。通过此接口可精确控制模块级的工艺处理行为。

参数定义:

无参数。

返回值

无返回值（None）。

代码示例:

# /home/zuochang/ted/pyted/test_demo.py:277
from pyted import *

from tedcore.kernel import current_module, Rect, place, skip_unless

@skip_unless(TED_PDK=T65)
def test08():

    @module
    def Demo(a):
        current_module().set_skip_autovia()
        r1 = Rect(M1, 1000, 1000, connectTo=a)
        r2 = Rect(M2, 1000, 1000, connectTo=a)
        place()

    @testbench
    def Top(a):
        Demo()

在上述测试用例中，Demo 模块通过调用 current_module().set_skip_autovia() 显式关闭了该模块的自动打孔功能。随后创建了两个矩形图形（r1 和 r2），分别位于 M1 和 M2 金属层，并与端口 a 连接，最后通过 place() 完成布局。由于已启用 set_skip_autovia，即使 M1 与 M2 之间存在需要连接的拓扑关系，系统也不会自动生成通孔。这为用户提供了一种在关键路径中防止自动插入通孔干扰设计意图的能力。

set_flatten_tag

功能介绍:

该函数用于设置当前模块的打平（flatten）标记，即指定该模块在生成或处理时是否应被展开为底层结构，而不是作为一个整体模块保留。当一个模块被标记为“打平”后，在后续的电路生成或分析流程中，其内部结构会被展开并嵌入到其上层模块中，而不再作为独立的子模块存在。

add_label

功能介绍:

添加文本标签到指定位置和层级中。该函数用于在电路设计的指定坐标（x, y）处创建一个标签对象，并将其放置在指定的图层上，通常用于标注关键位置或说明信息。若未提供坐标或坐标为空，则返回 None。

此方法属于模块实例的成员函数，作用于当前模块上下文，通过调用底层 rsmodule 的 add_label 方法完成实际创建操作。若未指定图层，默认使用 TEXT 图层。

参数定义:

参数名类型含义默认值
labelstr标签显示的文本内容。无
layerLayer标签所在的图层对象，若未提供则默认为 TEXT 图层。None
xint标签在X轴上的坐标位置。0
yint标签在Y轴上的坐标位置。0

返回值

  • 返回类型：Label
  • 含义：成功创建的标签对象；若 x 或 y 为 None，则返回 None。

该测试示例展示了在 test1 函数中使用 add_label 方法的典型用法：

  • 定义一个名为 Top 的测试基准模块（@testbench 装饰），用于构造和验证电路。
  • 在模块内部绘制一个矩形区域，图层为 TEXT，尺寸为 1000×1000 单位。
  • 使用 current_module() 获取当前模块实例，并调用其 add_label('TESTBENCH') 方法，在默认坐标 (0, 0) 处添加文本标签“TESTBENCH”，且使用默认的 TEXT 图层。

通过此示例可以看出：

  • 不传入 layer, x, y 参数时，系统会自动采用默认值；
  • 该方法无需显式创建图层，因 TEXT 为内置常量图层；
  • 标签被正确添加至当前模块上下文中，适用于设计说明、调试标注等场景。
