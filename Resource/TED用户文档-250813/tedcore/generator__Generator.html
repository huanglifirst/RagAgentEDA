Generator

功能介绍:

Generator 是 TED 框架中用于生成和管理电路设计实例的基类，继承自 EmulatedInstance。它主要负责电路仿真配置、电源电压设置、设计变量初始化与优化结果更新、版图生成与封装等核心功能。该类为电路设计流程中的自动化构建与测试提供支持，尤其适用于全流程仿真（如前仿、后仿）、PVT 测试及参数化优化。

Generator 类作为电路设计的高层抽象，集成了以下关键能力：

  • 自动根据工艺库（PDK）设置默认电源电压（VDD）。
  • 支持通过 set_pex_configs() 配置物理提取（PEX）参数。
  • 提供 build() 方法完成 DUT（Design Under Test）的构建与 PEX 操作。
  • 管理设计变量的初始值与优化后结果，支持通过 _update_variables 和 _get_initial_variables 进行状态同步。
  • 支持生成模块包装器方法（wrapper），用于封装电路设计为可复用的标准模块。
  • 提供 test_pvt() 方法执行在不同工艺、电压、温度（PVT）条件下的测试。

参数定义:

参数名类型含义默认值
pex_configsdictPEX 配置项，用于设置物理提取相关参数。空字典 {}
vddfloat设计的电源电压（VDD），由 set_vdd_voltage 方法设置。依据 PDK 自动推断：s180bcd/s180mm/t180 为 1.8V；t40/t65 为 1.2V；其他为 1.8V
variablesdict存储设计变量的初始值和最优值，键为 'initial' 和 'optimal'。{'initial': None, 'optimal': None}
sim_resultsdict仿真结果缓存，存储仿真输出数据。空字典 {}
sim_res_dictdict仿真结果的原始字典形式缓存。空字典 {}
pvt_tableAnyPVT 测试表结构，通常用于存储多种 PVT 组合信息。None

注：上述参数为类内部成员变量，用户不直接操作，但可通过公开方法进行控制。

返回值

无

代码示例:

以下测试用例展示了 Generator 类的使用方式，特别是在配置和执行 PVT 测试方面的典型用法：

def test_pvt():
    gen = Generator()
    gen.test_pvt(pvt_set=[
        {'P': ['tt_lib'], 'V': 1.2, 'T': 27},
        {'P': ['ff_lib'], 'V': 1.3, 'T': 80},
        {'P': ['ss_lib'], 'V': 1.1, 'T': -40}
    ], name="test_vdd")
def test_pvt_with_custom_kwargs():
    gen = Generator()
    kwargs = {
        'vdd': 1.2,
        'corners': ['tt_lib'],
        'temp': 27
    }
    gen.test_pvt(pvt_set=[
        {'P': ['tt_lib'], 'V': 1.2, 'T': 27},
        {'P': ['ff_lib'], 'V': 1.3, 'T': 80}
    ], name="custom_test", **kwargs)
def test_pvt_with_default_kwargs():
    gen = Generator()
    gen.test_pvt(name="default_pvt")

使用说明：

  1. 配置 PVT 测试集：
    当调用 test_pvt(pvt_set=...) 时，用户可传入一个字典列表，每个字典包含 'P'（工艺库）、'V'（电压）和 'T'（温度）三个字段。若未提供，则使用默认 PVT 集合：[['tt_lib', 1.2, 27], ['ff_lib', 1.3, 80], ['ss_lib', 1.1, -40]]。

  2. 动态传递测试参数：
    test_pvt() 方法内部会检查 self.test 是否具有 __code__.co_varnames 属性，若存在 'corners'、'vdd' 或 'temp' 参数名，则自动将当前 PVT 条件的对应字段注入到 test_kwargs 中作为参数传入测试函数。

  3. 默认行为与名称控制：
    即使不传递 pvt_set，也能运行测试；用户可通过 name 参数自定义测试结果命名规则。

  4. 灵活性设计：
    通过 **kwargs 支持额外测试参数的注入，便于与其他测试流程集成。如示例中传入的 vdd, corners, temp 均可被后续测试方法接收并使用。

注意：当前 test() 方法未实现（抛出 Exception('Unimplemented')），因此实际使用中应提供具体实现或通过继承扩展支持测试逻辑。

test

功能介绍:

test 是一个用于执行电路模块测试的函数方法，通常在电路设计类（如放大器、比较器等）中定义。该方法负责启动并完成对特定电路模块的功能与性能仿真测试，包括但不限于直流分析、交流分析、瞬态分析、噪声分析、压摆率测试和相位裕度测试等。
该方法通过可配置的参数控制测试行为，并支持在不同设计阶段（如 pre 或 post）执行测试，适用于从原型设计到后端验证的全流程测试场景。

注意：目前该方法在代码中被标记为未实现（raise Exception('Unimplemented')），实际功能依赖于具体子类或模块的重写与实现。

参数定义:

参数名类型含义默认值
figpathstr用于保存测试生成图像的输出路径，如波形图等。若指定则将结果图形保存至该路径。None
namestr测试实例的名称，可用于标识输出文件或日志中的测试任务。None
**kwargsdict其他可选参数，用于传递特定测试器（如噪声分析、压摆率测试等）的配置选项。常见包括：noisetester_settings, actester_settings, srtester_settings 等。无

返回值

  • 返回值类型为 dict 或任意可序列化的结果对象，具体取决于实现。
  • 通常包含测试结果数据，例如频率响应、增益、相位裕度、噪声密度、压摆率、功耗等关键指标，也包括仿真日志信息或异常状态。

总结

test 方法是电路设计流程中的核心验证接口，支持：

  • 灵活配置测试阶段（pre / post）
  • 自定义器件尺寸与设计参数
  • 集成多类型仿真器设置（噪声、交流、压摆率等）
  • 结合环境管理器（setenv）配置底层仿真参数
  • 与自动化测试框架兼容（如使用 @skip_unless 控制执行条件）

尽管当前声明为未实现，但其语义和使用方式已在多个项目中标准化。具体实现在继承类中通过重写完成。

report

功能介绍:

report 是一个实例方法，用于生成电路设计的报告文件，并根据是否开启优化模式（is_opt）执行不同的行为流程。该方法主要用于在电路验证或优化完成后，将相关结果保存到指定路径下，并可选择性地触发针对优化结果的测试与图表生成。

  • 若 is_opt 为 True，表示处于优化模式：
    • 从输入参数 res 中提取 Vars 字段以判断优化是否成功。
    • 若 Vars 为 None，则输出错误提示：“Optimization failure: No viable solution can be found”。
    • 否则调用 update_variables 更新当前模块的变量配置，并使用 test 方法在指定路径下运行“Optimization”测试，生成对应图表。
  • 无论是否开启优化模式，都会调用 report_content 方法将核心报告内容输出至由 path 指定的目录中。

此方法常用于电路设计流程的最终阶段，完成结果归档与可视化分析。

参数定义:

参数名类型含义默认值
resdict 或 None优化结果字典，包含变量数据和状态信息，用于判断优化是否成功。None
pathstr报告文件输出路径。若未提供，则使用默认路径：{get_ted_data()}/taco/。None
is_optbool是否处于优化模式，决定是否执行优化相关的后续处理流程。False
namestr 或 None报告文件的名称（不包含路径），用于命名生成的报告内容。None
**configdict可变关键字参数，传递给 test 方法的额外配置参数。无

返回值

无返回值（None）。

  • 最终调用 report_content 输出报告内容至指定目录。

build

功能介绍:

build 是一个用于构建电路设计实例的核心函数，通常在模块或设计生成流程中调用。该函数负责根据传入的参数获取待测设计单元（DUT, Device Under Test），并执行后续的配置与处理逻辑。在设计流程的“后处理阶段”（stage == 'post'）中，会自动调用 pex 方法进行物理提取（Parasitic Extraction）配置，最终返回配置完成后的 DUT 实例。

该函数是 TED 框架中生成和配置电路结构的关键入口，适用于全定制电路设计流程中的自动化构建与优化阶段。

参数定义:

参数名类型含义默认值
*args可变位置参数用于传递模块构造所需的位置参数，通常为端口连接或初始化参数。无
**kwargs可变关键字参数用于传递模块构造所需的命名参数，如端口配置、模块参数等。无

返回值

  • 返回类型：Module 实例（即电路设计的实例对象）
  • 含义：返回经过命名、配置并可能执行了后处理操作（如 pex）后的 DUT 实例，可直接用于后续仿真、布局布线或分析流程。

代码示例:

def test_build_dut_post_stage():
    dut = build()
    assert dut.name == 'dut'
    assert getattr(dut, 'pex_called', False)
def test_build_with_params():
    dut = build(param1=10, param2='test')
    assert dut.name == 'dut'
    assert dut.param1 == 10
    assert dut.param2 == 'test'
def test_build_in_pre_stage():
    from tedcore.kernel import setenv
    setenv('stage', 'pre')
    dut = build()
    assert dut.name == 'dut'
    assert not hasattr(dut, 'pex_called')

上述测试用例展示了 build 函数在不同场景下的行为：

  • 用法1：基础构建
    调用 build() 无任何参数，返回一个被重命名为 'dut' 的实例。若当前 stage 为 'post'，会自动触发 pex 配置。

  • 用法2：带参数的构建
    使用关键字参数传递模块配置（如 param1=10）时，这些参数会被正确传递至 get_dut() 调用中，并保留在最终返回的 DUT 实例中。

  • 用法3：在预阶段构建
    当环境变量 stage 设为 'pre' 时，pex 配置不会被执行。测试验证了该函数根据当前阶段智能决定是否执行后处理逻辑。
