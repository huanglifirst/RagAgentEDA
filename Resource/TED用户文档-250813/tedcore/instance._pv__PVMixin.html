PVMixin

功能介绍:

PVMixin 是一个混合类（Mixin），为 TED 框架中的 Instance 实例提供物理验证（Physical Verification）相关的功能支持，主要包括 LVS（Layout vs. Schematic）检查、SVS（Schematic vs. Schematic）对比检查和 DRC（Design Rule Check）检查。该类通过装饰器 @layout(default=True) 将部分方法标记为需执行布局相关操作，确保在进行物理验证时具备完整的版图数据上下文。

  • lvs：执行版图与电路图一致性检查，支持从本地或服务器端读取 GDS 文件，并可配置特定规则。
  • svs：用于两个电路图之间的对比检查（例如：设计变更前后对比）。
  • svs_from_virtuoso：对接 Cadence Virtuoso 环境，以某版图单元为基准进行 SVS 检查。
  • drc：执行设计规则检查，支持灵活配置检查范围，如仅检查特定层、特定规则或排除某些规则。

参数说明

lvs 方法

  • check_place: bool（默认为 True）：是否检查实例中是否调用了 place() 操作，用于判断布局状态的完整性。
  • gds_file：指定用于比对的 GDS 文件路径；若未传入，则使用默认生成的或已缓存的版图文件。
  • layout_cell_name：指定版图中的单元名称（cellname），在多层级结构中需明确目标。

注：该方法通过装饰器 @layout(default=True) 标记，表明其依赖于完整版图上下文。

svs 方法

  • check_place: bool（默认为 True）：是否检查布局操作被执行。
  • gds_file：用于比对的 GDS 文件路径（可选）。
  • layout_cell_name：指定要验证的版图单元名。
  • **rules：可传递任意键值对以定制 SVS 检查规则，具体支持项由底层实现决定。

svs_from_virtuoso 方法

  • cellname（必需）：待比较的 Virtuoso 中的单元名称。
  • cds_lib：CDs lib 文件路径或名称（可选），用于定位设计库。
  • library：设计库名称（可选），用于限定搜索范围。

该方法主要用于与 Cadence Virtuoso 环境集成，进行同步验证。

drc 方法

  • check_place: bool（默认为 True）：是否检查布局操作的执行状态。
  • gds_file：指定 DRC 检查所用的 GDS 文件路径；若未提供，则使用内部生成或缓存副本。
  • layout_cell_name：目标版图单元名称。
  • **rules：灵活配置 DRC 的检查范围，支持以下形式：
    • check_only=True：仅运行指定规则检查，不执行全部检查（默认为 False）。
    • all=True：开启所有物理设计规则检查。
    • other=["A", "B"]：仅检查名为 A 和 B 的工艺厂定制 DRC 实际规则（不支持反向前缀处理）。
    • _other=["A", "B"]：排除名为 A 和 B 的实际规则进行检查（即“不包含”模式）。
    • area=True：启用所有层的面积规则检查。
    • area=["M1", "M2"]：仅检查 M1、M2 层的面积规则。
    • _area=["M1"]：禁止检查 M1 的面积规则（前缀“_”表示排除）。
    • area=False 或 _area=True：禁用所有面积规则检查。
    • M1=True：对 M1 层执行所有相关规则的 DRC 检查。
    • M1=["area", "en"]：仅对 M1 层检查 area 和 en 规则（需参考相应 PDK 配置文件中的描述）。

所有规则命名遵循 pdk_resource/pdks/{pdk}/{pdk}.yaml 中定义的逻辑。

返回值说明

  • lvs：无显式返回；通常通过抛出异常或返回验证结果来体现检查状态。
  • svs：同上，实际返回取决于底层实现，但接口设计为返回布尔值表示通过与否。
  • svs_from_virtuoso：返回 bool 表示 SVS 检查是否通过。
  • drc：返回 Tuple[bool, str]：
    • 第一个元素为 bool：表示 DRC 是否通过（True 表示无错误）；
    • 第二个元素为 str：包含详细的 DRC 报告文本，若检查失败或存在警告时可提供诊断信息。

使用示例

# 假设实例名为 my_instance
result, report = my_instance.drc(
    all=True,
    area=["M1", "M2"],
    _other=["metal_overlap"],   # 排除 metal_overlap 规则
    M1=["en", "area"]
)

if not result:
    print("DRC failed:\n", report)
else:
    print("DRC passed!")

# LVS 检查（使用默认 GDS 和单元名）
try:
    my_instance.lvs(check_place=True, gds_file="output.gds", layout_cell_name="top")
    print("LVS passed.")
except Exception as e:
    print(f"LVS failed: {e}")

# SVS 对比两个设计
my_instance.svs(gds_file="diff_design.gds", layout_cell_name="subcell")

# 从 Virtuoso 中获取单元并对比
my_instance.svs_from_virtuoso(cellname="adder_8bit", cds_lib="my_lib.cdslib", library="tech1")

svs

功能介绍:

svs 是用于对当前实例执行 SVS（Short Verification Suite）检查的API方法。该方法将当前实例与指定的另一个 SPICE 文件中的顶层单元（top cell）进行对比，验证两者在电路连接、器件参数等方面是否一致。此功能常用于模拟电路设计流程中，确保不同版本或设计路径下的电路拓扑结构正确无误。

svs 方法是 Instance 类的一个成员函数，它内部调用了模块（module）的 svs 方法来完成实际的对比逻辑。

参数定义:

参数名类型含义默认值
other_sp_filestr另一个 SPICE 文件的路径，用于与当前实例进行 SVS 检查对比。无
other_top_cellnamestr在 other_sp_file 所指定的 SPICE 文件中，作为顶层单元名称的 cell 名字。无

返回值

  • bool：如果 SVS 检查通过，返回 True；否则返回 False。

代码示例:

from pyted import *

from tedcore.kernel import Dut
from ted_utils import get_virtuoso_config

@skip_unless(TED_PDK=T65)
def test05():

    @testbench
    def Top():
        from ted_interface.virtuoso import export_lvs_netlist_from_virtuoso
        dut = Dut(l=1e-06, w=1e-06, m=2, model='nch')
        params = {'l': '1000n', 'w': '1000n', 'simM': 2}
        sp_file = export_lvs_netlist_from_virtuoso('nch', **params)
        dut.svs(sp_file, get_virtuoso_config()['temp_cell'])

说明：

  • 该测试用例展示了如何在特定工艺（T65）环境下，使用 dut.svs() 方法执行 SVS 检查。
  • 首先通过 Dut 创建一个 NMOS 管实例，并设置其尺寸参数。
  • 接着调用 export_lvs_netlist_from_virtuoso 函数从 Virtuoso 工具导出 SPICE 网表文件，保存为 sp_file。
  • 最后调用 dut.svs() 方法，将当前实例与导出的 SPICE 文件中的顶层单元（由 get_virtuoso_config()['temp_cell'] 指定）进行 SVS 对比。
  • 该流程常用于验证设计变更后的电路是否与原始网表一致。

pex

功能介绍:

pex 是一个用于对电路实例（Instance）执行物理提取（Parasitic Extraction, PEX）检查的函数。它基于实例所在的模块（Module），调用底层工具进行寄生参数提取，并可选择性地验证版图与网表的一致性（LVS）。该函数支持多种提取规则和配置选项，适用于全定制模拟电路设计流程中，确保布局布线后的电路在电气特性上满足设计要求。

pex 本质上是实例对象的一个方法，其内部通过调用 self.module().pex(...) 实现具体的功能执行，因此它绑定于某个已定义的模块实例。

参数定义:

参数名类型含义默认值
levelstrPEX 分析的精度级别。常见的取值包括 'rcc'（推荐）和其他支持的等级。'rcc'
pex_netlist_typestr生成的网表类型，例如 'sp' 表示 SPICE 格式网表。None
ticer_freqfloatTicer 工具使用的频率值（用于 AC 分析），单位为 Hz。0
check_placebool是否检查模块中是否调用了 place() 函数。若未调用，则提示错误。True
power_namestr电源信号名称，用于标识版图中的电源端口。'_vdd_'
ground_namestr地信号名称，用于标识版图中的地端口。'_vss_'
netlist_ground_nameOptional[str]生成网表中定义的接地名称，可覆盖默认的 ground_name。None
rulestr使用的提取规则，仅限 'pex' 或 'starrc'。'pex'
ignore_capstr是否忽略电容参数的提取。可选值如 'all' 表示忽略所有电容。'all'
starrc_postsimbool在使用 starrc 规则时，是否启用后仿真（post-simulation）模式。True
check_lvsbool是否进行 LVS（Layout vs. Schematic）验证检查。True
is_tvfbool是否使用 TVF（Tanner Verilog Format）语法生成网表。True
gds_filestr指定要进行 PEX 的 GDS 文件路径；若不指定，则使用实例对应的默认 GDS 文件。None
layout_namestr指定版图中 Cell 名称，用于定位布局文件。若未指定，则使用模块的默认 layout 名称。None
virtual_connectOptional[str]虚拟连接设置，通常在处理特殊连接逻辑时使用。None

返回值

  • bool：表示 PEX 检查是否成功通过。
    • 返回 True 表示 PEX 流程执行完成且无错误或通过了所有验证。
    • 返回 False 表示执行失败或检查未通过。

代码示例:

以下是从提供的测试用例中提取出的典型使用方式，展示了 pex 在不同场景下的调用方式和行为：

示例一：基础 PEX 检查（无参数配置）

from pyted import *

@module
def Dut(d: Input, s: Output):
    'd' >> (m1 := NMOS()) % 'g' >> 's'
    's' >> (c1 := MomCap()) >> 's'

@testbench
def Top():
    dut: Instance = Dut()
    dut.pex()  # 简单调用，使用默认参数

说明：该用例中仅调用 dut.pex()，不指定任何参数，使用所有默认配置进行 PEX 提取。适用于常规流程。

示例二：条件性执行 PEX（基于仿真阶段判断）

from pyted import *

@module
def Inv(in: Input, out: Output):
    ('in',) >> Inv() >> ('out',)

@testbench
def Top():
    V(Vdd(), Vss(), dc=1.2).rename('vdd')
    Clock(period=1e-07) >> 'in' >> (dut := Inv()) >> 'out'
    
    if get_sim_stage() == 'post':
        dut.pex()  # 仅在 post 阶段执行 PEX 检查
    raw = tran(1e-06)
    watch('in')
    watch('out')
    raw.show()

说明：通过 get_sim_stage() 判断仿真阶段，只在 'post' 时调用 pex()，这常用于后仿流程中自动进行物理提取验证。

示例三：显式配置 PEX 规则与参数

lvs

功能介绍:

lvs 是一个用于对电路实例执行 LVS（Layout vs. Schematic）检查的 API。该方法通过对比版图（GDS）与电路网表（Spice）的一致性，验证设计是否正确。该 API 作为 Instance 类的一个成员方法，调用底层工具完成 LVS 检查流程。它支持自动导出 Spic e 和 GDS 文件、指定顶层单元名称、跳过电源引脚检查、虚拟连接设置等高级选项，并可选择在检查失败时抛出异常或静默返回。

重要说明：LVS 检查依赖于外部工具（如 Calibre），需确保环境已正确配置支持 LVS 验证流程。

参数定义:

参数名类型含义默认值
sp_filestrSpice 文件路径。如果为 None，则从 TED 内部导出 Spic e 文件。None
top_cellnamestrSpice 文件中顶层单元名称。此参数必须与 sp_file 配合使用。None
check_placebool是否检查模块中是否调用了 place() 方法。若未调用，可能提示布局未完成。True
gds_filestr手动指定 GDS 文件路径。如果为 None，则从 TED 内部导出 GDS 文件。None
layout_namestrGDS 文件中顶层单元名称。此参数必须与 gds_file 配合使用。None
virtual_connectOptional[str]虚拟连接的节点名。在版图中未物理连接，但程序认为已连接。用于处理某些特殊结构（如预留连接点）。None
power_namestr电源引脚名称（默认为 _vdd_）。_vdd_
ground_namestr地引脚名称（默认为 _vss_）。_vss_
skip_vddbool是否跳过对 Vdd 引脚的检查。False
skip_vssbool是否跳过对 Vss 引脚的检查。False
ccistr是否使用 CCI（Calibre Connectivity Interface）。取值只能是空字符串 "" 或 "cci"。""
interruptbool当 LVS 检查失败时，是否抛出异常。若为 True，则检查失败会中断执行；否则仅返回 False 表示失败。True

返回值

  • 类型：bool
  • 含义：返回 True 表示 LVS 检查通过，返回 False 表示检查未通过。

drc

功能介绍:

drc 是用于对电路实例（Instance）执行设计规则检查（Design Rule Check, DRC）的 API。该方法会根据当前工艺库（PDK）定义的设计规则，检测版图是否存在违反设计规范的情况，例如最小线宽、最小间距、面积限制等。

该 API 作为 Instance 类的一个成员函数，通过调用底层模块的 DRC 检查逻辑来实现。它支持灵活配置检查范围，可针对特定金属层、特定规则类型或所有规则进行检查，并返回检查结果状态和错误报告信息。适用于在电路设计流程中验证布局合理性。

参数定义:

参数名类型含义默认值
check_placebool是否检查 place() 操作后的位置是否合规（如间距、对齐等）。若为 True，则需确保电路已经完成布局放置。True
gds_filestr可选的 GDS 文件路径，用于指定外部版图输入进行 DRC 检查。如果不提供，则使用内部生成的版图数据。None
layout_cell_namestr若指定了 GDS 文件，该参数用于指定其中的单元名称。None
**rulesdict可变关键字参数，用于控制 DRC 检查的具体行为：
- 传入键如 area=True 表示启用面积规则检查。
- 支持针对单个金属层（如 M1=True）或规则组进行开关控制。
- 具体可支持的规则名由当前 PDK 定义，例如 min_width, spacing, area, overlap 等。		无

代码示例说明

在以下测试用例中展示了 drc 的典型使用场景：

  • 在多个模块（如 OTA5T、CP_OP2、ParallelResistor、SeriesResistor）中调用 .drc() 验证版图是否合规。
  • 结合 area=True 参数检查金属层面积规则（如最小/最大面积限制）。
  • 与 place()、route() 等布局操作协同使用，确保在完成物理放置和布线后执行 DRC。

返回值

一个元组：

  • 第一个元素为布尔值：
    • True 表示无 DRC 违规，设计通过检查；
    • False 表示存在违反设计规则的情况。
  • 第二个元素为字符串，指向 DRC 报告的错误位置（如 GDS 中的具体坐标、单元名称或描述文本），用于定位问题。

测试用例说明

测试文件功能描述

  • 推荐在完成 place() 和 route() 后调用此函数，以获得最精确的物理布局检查结果。
  • 若需调试特定问题，可启用如 area=True 或具体金属层检查来缩小排查范围。
  • 可结合 debug(dut.to_hspice()) 进行综合验证，确保仿真、DRC 和 LVS 均通过。
