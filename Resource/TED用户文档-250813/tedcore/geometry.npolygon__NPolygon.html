NPolygon

功能介绍:

NPolygon 是 TED 系统中用于表示多边形几何形状的核心类。它封装了电路设计中常见的多边形数据结构，包含坐标点、所属图层（Layer）、可选的电路节点（Node）、名称、标签（Tags）等属性，并支持多种几何操作和转换。

该类主要用于电路版图设计中的多边形管理与处理，例如判断点是否在多边形内部、将复杂多边形分解为矩形块（NBox）、进行多边形平移、绘制以及与其他多边形比较。NPolygon 会自动将输入的顶点列表调整为逆时针方向以保证几何一致性，并支持与底层 Rust 实现（RustPolygon）集成，从而提高性能。

类内部维护了与 Rust 层的桥接对象 _rs_polygon，用于执行高效的几何运算。所有创建的 NPolygon 实例会自动注册到当前模块的 shapes 列表中，便于统一管理。

参数定义:

参数名类型含义默认值
lyLayer多边形所属的图层对象。无
pointsList[Tuple[int, int]]构成多边形的一系列坐标点（x, y）。顶点需按顺序排列，且会被自动校正为逆时针方向。无
nodeOptional[CircuitNode]与该多边形关联的电路节点对象，用于连接或标识电气特性。若为字符串，则视为节点名进行查询。若为 None，则不绑定任何节点。None
namestr多边形的可选名称。若未提供，则系统自动生成如 p0, p1 等命名。None
skip_pvbool是否跳过物理验证（PV）检查，用于性能优化或特定情况下的忽略校验。False
tagsOptional[Set[str]]为该多边形添加的标记集合，可用于分类、筛选或元数据管理。None
create_rs_polygonbool是否创建底层 Rust 实现的 RustPolygon 对象，用于高性能计算。若为 False，则仅保留 Python 层数据。True

返回值

  • __init__：无返回值（构造函数）。
  • __eq__：返回 bool，判断两个 NPolygon 实例是否在所有属性上完全一致。
  • __hash__：返回 int，基于图层、顶点、节点和跳过PV标志生成哈希值。
  • new_by_rs_polygon：返回 NPolygon，从一个 RustPolygon 对象构建一个新的 NPolygon 实例。
  • new_by_nboxes：返回 List[NPolygon]，将一组 NBox 转换为对应的多边形列表。
  • get_rs_polygon：返回 rsted.RustPolygon，获取底层 Rust 多边形对象。

to_polygon

功能介绍:

to_polygon 是一个实例方法，用于将当前对象（假设为某个可表示为多边形的几何实体）转换为一个独立的 Polygon 实例。该方法会基于当前对象的层（layer）、点列表（points）和节点信息（node）创建一个新的 Polygon 对象，并保留原始对象的标签（tags）及其是否跳过物理验证（skip_pv）的配置。

此方法适用于从非标准多边形结构（如 NPolygon 等复合几何体）中提取出标准多边形实例，便于后续在电路设计流程中使用或进行操作。

参数定义:

无参数。

返回值

  • 类型: Instance
  • 含义: 返回一个新创建的 Polygon 实例，该实例继承了原对象的层信息、顶点坐标、节点数据，并保持原有的标签和物理验证跳过设置。

from tedcore.kernel import NBox, Box

def test03():

    polygons = NPolygon.new_by_nboxes([r1, r2])

to_nboxes

功能介绍:

to_nboxes 是 NPolygon 类的一个实例方法，用于将一个正交多边形（orthogonal polygon）按照指定的切割方向（水平或垂直）分割为多个矩形区域，并返回这些矩形对应的 NBox 对象列表。该方法适用于处理由多个矩形拼接而成的复杂形状（如非凸多边形），通过几何切割将整体分解为若干个最小的矩形单元，便于后续的物理设计、布局分析或可视化操作。

该方法支持两种切割方向：

  • HORIZONTAL：沿水平方向进行切割，将多边形分解为上下堆叠的矩形。
  • VERTICAL：沿垂直方向进行切割，将多边形分解为左右并排的矩形。

若传入的 orientation 参数非上述两个有效值，则会抛出 ValueError 异常。

参数定义:

参数名类型含义默认值
orientationstr切割方向，可选值为 HORIZONTAL 或 VERTICAL。HORIZONTAL

返回值

返回一个 List[NBox]，包含由原正交多边形切割得到的所有矩形块（即 NBox 对象），每个 NBox 保留原始多边形的层（layer）、节点（node）信息、标签（tags）及跳过PV检查标志（skip_pv）。

to_nbox

功能介绍:

to_nbox 是 NPolygon 类的一个实例方法，用于将一个具有恰好5个顶点的多边形（NPolygon）转换为一个最小边界矩形（NBox）。该方法通过遍历多边形的前4个点（第5个点为闭合点，与第一个点重复），计算这些点在x和y方向上的最大值与最小值，从而确定矩形的左下角坐标、宽度与高度。生成的 NBox 对象保持原始 NPolygon 的图层（layer）、节点（node）、名称（name）以及标签（tags）等元信息。

此方法主要用于在几何处理中将不规则多边形简化为紧凑的包围矩形，适用于布局布线、碰撞检测、区域计算等场景。前提是输入的 NPolygon 必须包含且仅有5个点，其中第1个和第5个点相同（闭合多边形），其余三个点构成矩形轮廓。

参数定义:

无参数。此方法为实例方法，无需额外传入参数。

返回值

  • 类型：NBox
  • 含义：返回一个由当前 NPolygon 的最小边界盒生成的 NBox 对象，其几何范围与原始多边形一致，同时保留原始图形的元信息（如图层、节点、名称等）。

功能介绍:

该函数用于判断一个给定坐标点是否位于当前多边形的内部（包含边界）。此方法属于 NPolygon 类的实例方法，主要应用于半导体版图设计中的几何查询场景，例如在特定图层上检查某一点是否落在某个多边形区域内。

函数通过调用外部库 shapely 来实现精确的几何判断逻辑：首先确认目标图层与当前多边形所属图层一致，若不一致则直接返回 False；否则将输入的坐标转换为 Point 对象，当前多边形顶点列表构建为 Polygon 对象，并使用 covers 方法判断该点是否被包含在多边形中（包括边界）。

参数定义:

参数名类型含义默认值
layerLayer要检查的图层对象，用于验证当前多边形是否属于该图层无
xfloat点的 X 坐标值无
yfloat点的 Y 坐标值无

返回值

  • 类型：bool
  • 含义：若点位于多边形内部或边界上（且图层匹配），返回 True；否则返回 False。

代码示例:

from pyted import *

from tedcore.kernel import NPolygon

def test01():
    poly = NPolygon(M1, [(0, 0), (100, 0), (100, 100), (0, 100), (0, 0)])
    assert poly.is_point_in(M1, 50, 50)
    assert poly.is_point_in(M1, 0, 100)
    assert not poly.is_point_in(M1, -1, -1)
    assert not poly.is_point_in(M1, 101, 101)
    assert not poly.is_point_in(M2, 50, 50)

说明：

  • 第一个断言 assert poly.is_point_in(M1, 50, 50)：测试点 (50, 50) 位于正方形内部，且图层为 M1，因此返回 True。
  • 第二个断言 assert poly.is_point_in(M1, 0, 100)：测试点在多边形的顶点上（边界），属于有效包含范围，返回 True。
  • 第三个断言 assert not poly.is_point_in(M1, -1, -1)：测试点位于多边形外部，返回 False。
  • 第四个断言 assert not poly.is_point_in(M1, 101, 101)：测试点超出正方形范围，返回 False。
  • 第五个断言 assert not poly.is_point_in(M2, 50, 50)：虽然坐标 (50, 50) 在多边形内，但图层为 M2，与 poly 所属的 M1 不一致，因此返回 False。

总结用法：

  1. 验证点是否在指定图层上某多边形内部或边界（使用相同图层）；
  2. 图层不匹配时直接拒绝判断，确保几何检查与图层语义一致性；
  3. 支持边界点判定（包含边界的covers语义），适用于版图规则检查等场景。

is_intersecting

功能介绍:

该方法用于判断当前多边形（NPolygon）是否与另一个多边形（NPolygon）或矩形框（NBox）存在交集。该方法在几何计算中常用于检测两个二维图形之间的空间重叠关系，广泛应用于版图设计、布局分析等场景。
该方法属于 NPolygon 类的成员函数，通过调用 shapely 库中的 Polygon.intersects 方法实现精确的几何交集判断。

在判断过程中，首先检查两个对象是否处于同一层（layer），若不同层则直接返回 False；若同层，则将两者的顶点坐标转换为 shapely.Polygon 对象，并调用其内置的 intersects 方法进行几何交集检测。

参数定义:

参数名类型含义默认值
otherUnion['NPolygon', 'NBox']另一个待比较的多边形或矩形框对象，用于与当前对象进行交集判断。无

返回值

  • 类型: bool
  • 含义: 如果当前多边形与传入的对象存在空间交集（重叠、接触等），则返回 True；否则返回 False。

代码示例:

def test03():
    r1 = NBox(layer=M1, w=5, h=5)
    r2 = NBox(layer=M1, w=5, h=10).move_to(5, 0)
    polygons = NPolygon.new_by_nboxes([r1, r2])
    p1 = polygons[0]
    assert p1.is_intersecting(NBox.new_by_box(Box(10, 10, 6, 7), M1))
    r1 = NBox(layer=M1, w=5, h=5).move_to(6, 7)
    r2 = NBox(layer=M1, w=5, h=10).move_to(11, 7)
    polygons = NPolygon.new_by_nboxes([r1, r2])
    polygons[0].to_polygon()

说明：

  • 第一个测试用例中，创建了两个 NBox（分别位于 (0,0) 和 (5,0)），并组合成一个 NPolygon。随后调用 p1.is_intersecting 方法判断其是否与位置在 (10,10)、尺寸为 6×7 的新矩形相交，断言返回 True，说明两区域存在空间重叠。
  • 第二个测试用例中，创建了两个新的 NBox，分别移动到 (6,7) 和 (11,7)，并再次组合成 NPolygon，随后调用 to_polygon() 将其转换为标准多边形对象（此操作不涉及 is_intersecting 的逻辑，仅用于验证数据转换）。
  • 通过上述示例可见，is_intersecting 方法可灵活应用于由多个矩形组成的复合多边形与单个矩形之间的交集检测。

has_slant_line

功能介绍:

该方法用于判断当前多边形对象是否包含任意斜线（即非水平或垂直的边）。
该方法属于 NPolygon 类的成员函数，用于在几何处理过程中检测多边形的边是否具有斜向特征。如果多边形中存在至少一条既不水平也不垂直的边（即两个端点的 x 坐标和 y 坐标均不同），则返回 True；否则返回 False。

参数定义:

参数名类型含义默认值

注：该方法为无参实例方法，不接收外部输入参数。

返回值

  • 类型：bool
  • 含义：若多边形中存在至少一条非水平且非垂直的边（即斜线），则返回 True；否则返回 False。

draw_from_list

功能介绍:

该静态方法用于绘制一组多边形（NPolygon 或 NBox 类型）并将其保存为图像文件。方法会自动计算所有多边形的边界框，并在图中以随机颜色显示每个多边形的轮廓，支持可选的文字标签标注索引编号。最终生成的图像保存到指定路径或默认路径下，返回原始传入的多边形列表。

该方法适用于可视化多个几何形状（如电路版图中的单元或多边形区域）的位置关系和布局结构。

filenamestr输出图像文件的保存路径和名称，若未指定则自动创建默认路径并使用默认文件名。None

返回值

  • 返回原始传入的 polygons 列表，即输入的多边形列表。该返回值用于链式调用或后续处理。
