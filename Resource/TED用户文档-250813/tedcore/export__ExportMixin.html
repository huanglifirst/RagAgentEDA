ExportMixin

功能介绍:

ExportMixin 是一个混合类（mixin），用于为 TED 框架中的 Module 类提供多种导出功能。该类不独立使用，而是作为基类被其他模块类继承，从而扩展其导出能力。通过该 mixin 提供的方法，可以将电路模块及其子实例导出为多种工业标准格式，包括但不限于 LEF、DEF、DB、Milkyway 和宏文件（Macro），支持缓存控制、服务器同步等功能，适用于模拟与全定制集成电路设计流程中的物理实现阶段。

所有方法均基于 ted_interface 中的相应导出器类（如 LIBExporter, LEFExporter, DEFExporter 等）进行封装执行。这些导出操作通常用于生成后续工具（如 P&R 工具 ICC、IC Compiler）可读入的设计文件。

参数定义:

参数名类型含义默认值
selfModule当前模块对象，表示待导出的电路模块。无
use_cachebool是否使用缓存文件进行导出（减少重复计算）。True
from_serverbool是否从服务器获取或上传数据（用于分布式协作环境）。True
instancesList[Instance]要处理的子实例列表，若未提供则使用模块自身的所有实例。None
instlistInstList用于导出 DEF 的特定实例集合。None
tech_lef_file_pathstr技术库 LEF 文件路径。无
def_file_pathstr输出的 DEF 文件路径。无
exclude_netsList[str]需要排除的网络名称列表（在导出 DEF 时忽略）。空列表
netsList[str]仅保留指定的网络列表（在导出 DEF 时使用）。空列表
blockage_filestr阻挡区域定义文件路径（如用于指定固定布局区域）。空字符串

返回值

方法名返回类型含义
export_lib()Any导出当前模块的库文件内容，返回具体导出器定义的结构（通常为路径或序列化数据）。
export_db()str返回数据库（DB）文件路径。
export_lef()str返回 LEF 文件路径。
export_milkyway()Any返回 Milkyway 格式输出信息，通常为字符串或列表。
export_children_milkyway()List[str]返回子模块导出的 Milkyway 路径组成的列表，每个元素格式为 'MW_PATH/NAME_HASH'。
export_children_db()List[str]返回子模块导出的 DB 文件路径组成的列表。
to_lef_macro()str返回宏形式 LEF 文件的路径。
export_def()str返回生成的 DEF 文件路径。
export_new_def()Tuple[str, List[str], List[str]]返回元组，依次为：DEF 文件路径、技术 LEF 路径列表、宏 LEF 路径列表。

说明：

  • export_lef() 用于导出单个模块的 LEF 宏描述。
  • export_db() 可以选择是否使用缓存、是否从服务器获取数据。

  • export_new_def() 是高级接口，允许用户自定义技术 LEF、宏 LEF 与过滤网络等参数，生成更灵活的 DEF 输出。

  • to_lef_macro() 将模块转换为独立宏 LEF 形式供系统使用。

to_verilog_module

功能介绍:

to_verilog_module 是一个实例方法，用于将当前模块（Module）及其内部有效实例转换为标准的 Verilog 模块代码字符串。该方法主要用于电路设计自动化流程中，将 Python 构建的 TED 模块结构导出为可被综合工具识别的 Verilog 代码。

该函数会遍历当前模块的所有实例，筛选出处于活动状态且非基础几何结构（is_base_geometry() 返回 False）的实例，并调用其 to_verilog() 方法生成子模块代码。随后将所有端口信息（输入、输出、双向）按 Verilog 语法组织，最终拼接成完整的模块定义。

此方法是 TED 框架中用于生成 Verilog 综合文件的关键接口之一，适用于全定制电路设计流程的后端导出阶段。

参数定义:

参数名类型含义默认值
instancesList[Instance]指定要包含在 Verilog 模块中的实例列表。若未提供，则自动筛选出所有活动且非基础几何的实例。None

返回值

  • 类型：str
  • 含义：返回一个完整的 Verilog 模块代码字符串，包含模块头、端口声明和内部实例代码。

代码示例:

# /home/zuochang/ted/tests/ted_tutorial/code/tutorial/test1.py:78
from pyted import *

from tedcore.kernel import Node, current_module
from ted_device.components.mos.mos import NMOS
from ted_utils import debug

def test_export():

    @testbench
    def Top():
        (d, g, s) = Node(3)
        d >> (n1 := NMOS()) % g >> s
        current_module().auto_rename()
        debug(n1.to_hspice())
        current_module().to_hspice_subckt()
        current_module().to_verilog_module()
        current_module().to_hspice_netlist()

该测试用例展示了 to_verilog_module 方法在实际电路设计流程中的典型使用方式：

  • 首先定义一个测试基准（@testbench）模块 Top，创建三个节点 d, g, s。
  • 通过 D >> (n1 := NMOS()) % G >> S 的语法构建一个 NMOS 管，并完成端口连接。
  • 调用 current_module().auto_rename() 自动为模块中的对象重命名，确保名称符合 Verilog 命名规范。
  • 接着调用多个导出方法：to_hspice(), to_hspice_subckt(), to_verilog_module(), 和 to_hspice_netlist()，表明该流程支持多种仿真格式输出。
  • 其中 to_verilog_module() 用于生成当前模块的 Verilog 模块代码，其结果可直接用于综合或仿真工具。

此示例说明了：

  1. 在模块构建完成后调用 to_verilog_module() 可导出对应的 Verilog 模块；
  2. 当前模块包含一个有效的 NMOS 实例，该实例将在 Verilog 输出中生成对应实例化代码；
  3. 方法会自动处理端口声明（输入/输出/双向），并基于 name_hash() 为模块命名；
  4. 支持通过传入自定义实例列表来控制导出内容，便于分层或条件性导出。

to_systemc

功能介绍:

该函数用于将一个 TED 框架中的模块（Module）导出为 SystemC 语言格式的头文件，以便在 SystemC 仿真环境中使用。它会根据模块的端口定义、内部子电路实例以及连接拓扑，生成符合 SystemC 规范的类定义代码。

该函数是 Module 类的一个方法，作用是将模块结构转换为可被 SystemC 编译器识别的 .h 文件格式，包含信号声明、子模块实例化、参数设置和连接初始化等元素。导出的文件适用于基于 TLM（Transactional Level Modeling）或 SC-AMS（SystemC Analog/Mixed-Signal）仿真场景。

函数特别处理以下内容：

to_spectre_netlist

功能介绍:

to_spectre_netlist 是一个实例方法，属于 Module 类的成员函数。该方法用于将当前模块（Module）及其内部电路结构转换为 Spectre 仿真工具可用的 netlist 格式文本。生成的 netlist 包含了完整的电路描述、模型引用、参数定义、包括文件指令、仿真控制命令以及 PVT（工艺-电压-温度）角配置支持，适用于后续在 Spectre 仿真器中进行电学仿真。

该函数会根据当前电路环境变量、PDK 配置、模块包含的外部依赖及仿真选项，自动生成符合 Spectre 语法规范的 netlist 文件内容。它整合了全局和局部的 include 指令、参数导出、模型文件引用、以及多种仿真条件（如 PVT 裕度）的 alter 块。

参数定义:

参数名类型含义默认值
selfModule当前模块实例，方法所属对象。无
simcmdstr额外附加的 Spectre 仿真命令（如 .tran, .ac 等）。''
optionsstr附加到 netlist 尾部的选项字符串，用于设置仿真器全局参数或特殊指令。''
simulator_optionsdict[str, Any]仿真器选项字典，覆盖默认配置（如 temp、vdd 等），支持动态注入仿真环境变量。若为 None，则使用初始化的选项集合。None

返回值

  • 类型：str
  • 含义：返回一个完整的 Spectre Netlist 字符串，包含如下内容：
    • 包含文件指令（include 和 ahdl_include）
    • 全局参数定义（来自 Variable 和 TB_Variable 的导出）
    • 模拟器模型文件引用
    • 网表主体（模块电路结构的 Spectre 表达式）
    • 仿真命令（simcmd）
    • 用户自定义选项（options）
    • PVT 角配置块（alt* 块，若设置了 PVT 环境变量）

to_networkx_graph

功能介绍:

该函数用于将一个 TED 框架中的电路模块（Module）转换为一个有向图（DiGraph），并利用 NetworkX 和 Matplotlib 可视化该电路的拓扑结构。此函数会分析模块中所有器件（如 NMOS、PMOS 管）的连接关系，将其转化为节点与边的形式，其中：

  • 每个 MOS 管被表示为一个中心节点，并标注其类型（NMOS 或 PMOS）；
  • 每个引脚（G、D、S）作为子节点，带有明确的标签（如 NMOS.g、PMOS.s）；
  • 器件内部连接关系以带属性的边表示（如 CTNS 表示 NMOS 的栅极到源极连接）；
  • 同一网络（net）上的多个引脚之间通过 wire 类型的边进行连接，反映电气连通性；

最终生成的图结构可用于电路分析、可视化与调试。

该函数适用于已定义并实例化的 TED 模块对象，支持展平（flatten）处理，以递归解析嵌套模块的底层结构。

参数定义:

参数名类型含义默认值
selfModule当前待转换的 TED 模块实例，包含完整的电路结构信息。无
flattenbool是否对模块进行展平处理（递归展开嵌套子模块）；若为 True，则将所有子模块的器件整合到顶层进行建模。False

返回值

  • 类型: networkx.DiGraph
  • 含义: 返回一个有向图对象，包含了电路中所有器件、引脚及连接关系的完整拓扑信息，支持进一步分析或绘图。

代码示例:

    def my_circuit(D, S, G, VDD, VSS):
        D >> NMOS() % G >> S

    inst = my_circuit(D=Input(), S=Output(), G=Ctrl(), VDD=Supply('VDD'), VSS=Supply('VSS'))

    # 调用 to_networkx_graph 生成并显示电路图

    @module
    def sub_circuit(D, S, G):
        D >> NMOS() % G >> S

    @module
    def top_circuit(IN, OUT, CLK):
        sub_inst = sub_circuit(D=IN, S=OUT, G=CLK)
        pass

    # 实例化顶层模块
    top_inst = top_circuit(IN=Input(), OUT=Output(), CLK=Ctrl())

    # 调用 to_networkx_graph 并启用展平模式
    graph = to_networkx_graph(top_inst, flatten=True)

def test_to_networkx_graph_nested():
    @module
    def leaf(D, S, G):
        D >> NMOS() % G >> S

    @module
    def mid(A, B, C):
        inst1 = leaf(D=A, S=B, G=C)

    @module
    def top(T, N, W):
        inst2 = mid(A=T, B=N, C=W)

    # 实例化顶层模块并转换为网络图（仅展平）
    top_inst = top(T=Input(), N=Output(), W=Ctrl())
    graph = to_networkx_graph(top_inst, flatten=True)

  • test_to_networkx_graph 展示了基本用法，对单级模块使用 flatten=False，仅分析当前模块的直接器件。
  • test_to_networkx_graph_flatten 与 test_to_networkx_graph_nested 演示了嵌套模块在 flatten=True 模式下的处理：所有子模块被展开并合并到同一图中，实现整个电路的完整可视化。

此函数主要用于调试与理解复杂电路结构，尤其适合验证连接是否正确，或进行电路拓扑分析。图中每个节点标记其角色（如 NMOS、NMOS.d 等），边标记连接属性（如 CTNS、wire），帮助用户快速识别关键路径和网络连通性。

功能介绍:

  • 类型: str
  • 含义: 返回一个字符串形式的 HSPICE 子电路网表描述，格式符合 .subckt 指令规范，包含子电路名称、端口列表和内部元件定义。

to_hspice_netlist_body

功能介绍:

该函数用于生成适用于HSPICE仿真工具的网表主体内容。它根据当前模块是否启用了网表展平（flatten_netlist）配置，决定调用不同的网表生成逻辑：

export_def

功能介绍:

export_def 是一个用于导出电路模块定义路径的接口方法，属于模块（Module）类的成员函数。该方法通过调用 DEFExporter 工具类，将当前模块中所有实例的布局信息转化为标准的 DEF 格式文件路径，主要用于后续的物理设计流程（如布线、时序分析等）。

export_def 本质上是生成一个描述电路布局结构的 DEF 文件，该文件包含各个实例的位置、尺寸以及互连关系。此方法通常在完成自动布局（auto_place）或手动放置（place）后调用，以输出可用于下游工具处理的物理设计数据。

参数定义:

参数名类型含义默认值
instlistInstList可选，指定需要导出的实例集合。若不提供，则默认导出当前模块中所有已定义和放置的实例。None

返回值

返回一个字符串，表示生成的 DEF 文件路径。

export_db

功能介绍:

export_db 是一个用于导出模块设计数据库（Database）的实例方法，属于 Module 类的成员函数。该方法通过调用底层 LIBExporter 模块，将当前模块及其子节点结构化信息导出为一个可被EDA工具识别的数据库文件（通常是标准单元库格式），便于后续物理实现、仿真或验证。

该方法主要用于在芯片设计流程中完成逻辑/物理层级的封装与输出。支持从服务器加载数据、利用缓存加速导出过程，并可根据模块是否为内置标准单元来决定导出策略。

参数定义:

参数名类型含义默认值
use_cachebool是否使用本地缓存，若启用可跳过重复导出过程，提高效率。True
from_serverbool是否从远程服务器获取数据（例如数据库、工艺库等）。如果为 False，则仅依赖本地资源。True

返回值

  • 类型：str
  • 含义：返回生成的数据库文件路径（文件名包含时间戳或唯一标识），该路径可用于后续工具直接读取。
