Circuit

功能介绍:

Circuit 是 TED 框架中用于管理电路设计环境和操作的主类。它负责电路的初始化、重置、PDK（工艺技术文件）加载、配置管理、模块跟踪以及电路相关操作。该类维护了对图层（layers）、模块（modules）、变量（variables）的引用，并提供了环境配置和仿真控制的方法。通过 Circuit 类，用户可以统一管理整个电路设计流程中的各种状态与资源。

该类在初始化时会构建内部数据结构，如路由层（route_layers）、通孔层（via_layers）、混合层（mix_layers）、观察层（route_obs_layers）等，并支持 PyTorch 变量创建以用于优化和梯度计算。此外，它还提供对并行计算、仿真控制、配置哈希检查等的集成管理。

Circuit 是 TED 框架中电路上下文的核心容器，所有模块实例化、变量声明、连接操作均依赖于该类维护的全局状态。

参数定义:

无直接输入参数。该类为单例管理模式（通常只存在一个实例），其状态通过内部属性维持。

无直接返回值（方法不返回结果）。

代码示例:

def test_circuit_init():
    circuit = Circuit()
    circuit.init()
    assert circuit.is_build is True
    assert len(circuit.route_layers) > 0
    assert len(circuit.via_layers) > 0
def test_circuit_reset():
    circuit = Circuit()
    circuit.reset()
    assert circuit.reset_count == 1
    assert circuit.is_build is False
def test_create_torch_variables():
    from tedcore.kernel.circuit import get_circuit
    circuit = get_circuit()
    circuit.create_torch_variables()
    assert circuit.torch_variables is not None
    assert circuit.torch_variables.requires_grad is True
def test_circuit_init_layers():
    circuit = Circuit()
    circuit.init_layers()
    assert len(circuit.route_layers) == len(rsted.Global.get_route_layers())
    assert len(circuit.via_layers) == len(rsted.Global.get_via_layers())
    assert len(circuit.mix_layers) == len(circuit.route_layers) + len(circuit.via_layers)

说明：

  1. init() 方法：用于初始化电路环境，包括 PDK 加载、日志打印、重置状态并构建图层信息。该方法调用后使 is_build = True，表示电路已进入可用状态。

  2. reset(is_init=False, skip_config=False) 方法：重置当前电路的配置与状态，可选择跳过某些初始化流程（如配置加载）。常用于重新开始一个设计流程或在测试中重建环境。

  3. init_layers() 方法：从全局配置中获取路由层和通孔层，并构建混合层列表。该方法构建了电路布线所需的所有基础图层结构。

  4. create_torch_variables() 方法：将所有电路变量转换为 PyTorch 张量，以支持反向传播与优化。该方法在需要进行参数优化或梯度计算时调用。

以上测试用例展示了 Circuit 类的主要功能使用方式：

  • 成功初始化电路环境；
  • 重置状态并验证重置行为；
  • 创建带有梯度追踪能力的 PyTorch 变量；
  • 构建正确的图层结构（路由、通孔、混合层）。

setenv

功能介绍:

setenv 是一个用于在电路仿真环境中配置参数的上下文管理器方法，通常作为 Circuit 类的一个成员函数提供。它允许用户在特定代码块内动态地设置电路仿真所需的各种环境参数，例如仿真器类型（simulator）、工艺角（corners）、工艺电压温度（PVT）等。通过 with 语句使用时，可临时更改当前仿真环境的配置，退出上下文后恢复为原来的状态。

该方法会根据传入的参数，自动补充必要的模型文件路径（如 HSpice、Spectre 的 .inc 和 .scs 文件），尤其当 corners 参数被指定时，系统会从对应的工艺角信息中提取并更新模型引用。最终返回一个包装了所有参数的 Environment 实例，供底层仿真系统使用。

参数定义:

参数名类型含义默认值
**parametersdict传递到环境配置中的任意键值对参数，如 simulator, PVT, corners 等。无

返回值

返回一个 Environment 类型的实例，该实例封装了所有传入的仿真环境参数及其扩展后的模型路径信息。

代码示例:

以下两个测试用例展示了 setenv 在不同场景下的使用方式：

示例 1：设置仿真器类型并运行优化任务

from pyted import *

from tedcore.kernel import sim_acc, get_circuit
from ted_optimization.tacomark import TACO_Benchmark

@skip
def test():
    sim_acc(2, 6, True)
    with get_circuit().setenv(simulator='spectremdl'):
        TacoB = TACO_Benchmark()
        opt_task = BGR_Opt_Problem
        optimizers = {ML_Sizer: {'batch_episode': 40, 'max_episode_len': 50, 'num_steps': 4, 'inference': True, 'summaray_path': f'data/autockt/BGR', 'agent_path': 'data/autockt/RLopt_BGR_1727280062/agent.pth'}}
        TacoB.run_optimizer_cmp(opt_task, optimizers)

说明：

  • 此用例中通过 setenv(simulator='spectremdl') 将当前仿真器设置为 spectremdl。
  • 所有在 with 块内的操作（包括创建 TACO_Benchmark 实例和调用优化函数）都将基于该指定的仿真器运行。
  • 适用于执行仿真驱动的自动电路优化任务。

示例 2：配置复杂的 PVT 工艺角并分层设置环境

from pyted import *

from tedcore.kernel import get_circuit

@skip_unless(TED_PDK=[S180MM])
def test1():
    with get_circuit().setenv(simulator='spectremdl', PVT={'V': [3.3 * 0.9, 3.3, 3.3 * 1.2], 'T': [27], 'P': {'mos': ['ss', 'ff'], 'bjt': ['bjt_tt', 'bjt_ss', 'bjt_ff'], 'res': ['res_tt', 'res_ss']}}):
        ckt = Bgr_Off()
        xk = ckt.test(vdd=3.3)
        with get_circuit().setenv(PVT={'T': [40], 'P': {'mos': ['ss'], 'bjt': ['bjt_tt', 'bjt_ss'], 'res': ['res_tt', 'res_ss']}}):
            xk.build()

说明：

  • 外层 setenv 配置了完整的 PVT 环境，包括电压（V）在三个极端值范围、温度（T）为 27°C，以及各类器件的工艺角。
  • 在该上下文中创建电路实例 Bgr_Off() 并执行测试。
  • 内层嵌套使用 setenv(PVT={...}) 进一步细化环境，仅将温度改为 40°C，且限制 mos 为 'ss' 角。
  • 最终调用 xk.build() 时使用的是内层指定的参数。
  • 展示了 setenv 的上下文嵌套能力，支持局部覆盖全局配置。

总结：setenv 支持以下典型用法：

  1. 仅指定仿真器：with get_circuit().setenv(simulator='spectremdl'): ...
  2. 配置完整 PVT 参数：with get_circuit().setenv(PVT={...}): ...
  3. 嵌套设置以实现局部环境替换：多层 with setenv(...) 语句可实现精细控制。
  4. 自动扩展模型路径（如 corners 被指定时）：内部逻辑自动加载对应 .inc、.scs 等文件。

get_pdk

功能介绍:

该函数用于获取当前电路上下文中的工艺设计套件（Process Design Kit, PDK）实例。PDK是集成电路设计中的核心配置对象，包含了工艺节点相关的参数、模型路径、最小尺寸等关键信息。通过调用此函数，可获取当前项目所使用的PDK对象，进而访问其内部配置数据或执行与工艺相关操作。

此API为一个普通函数，返回值是一个PDK实例对象，用于统一管理和操作当前设计所依赖的工艺信息。

参数定义:

无参数

返回值

  • 类型: PDK
  • 含义: 返回当前电路上下文中配置的工艺设计套件（PDK）对象。该对象封装了工艺相关的所有元数据，如最小尺寸 lmin、模型路径、工艺名称等，可用于后续的参数查询与配置。

def test_euler01():

参数定义:

无参数

返回值

  • str: 返回表示电路配置的MD5哈希值的字符串，长度为32个十六进制字符。

ahdl_include

功能介绍:

该函数用于向电路（circuit）中添加一个AHDL（Altium Hardware Description Language）包含文件路径。通过调用此方法，可以在电路设计过程中引入外部的AHDL文件，以便在后续的综合或仿真中使用该文件中的定义或代码。此方法属于circuit类的一个成员函数，用于维护电路所依赖的外部包含文件集合。

参数定义:

参数名类型含义默认值
incfilestr要添加的AHDL包含文件的路径。可以是相对路径或绝对路径。无

返回值

无返回值（None）。

代码示例:

def test_ahdl_include():
    circuit = Circuit()
    circuit.ahdl_include("include/defs.hdl")
    assert "include/defs.hdl" in circuit._ahdl_includes

该测试用例展示了如何调用 ahdl_include 方法，将一个名为 defs.hdl 的AHDL包含文件路径加入到电路中。调用后，可验证该路径是否已被正确添加至内部的 _ahdl_includes 集合中。此方法适用于在电路设计阶段引入共享的定义或常量文件，确保不同模块之间的代码一致性与复用性。
