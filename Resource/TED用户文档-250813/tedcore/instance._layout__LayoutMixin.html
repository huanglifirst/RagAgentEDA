LayoutMixin

功能介绍:

LayoutMixin 是一个用于电路设计中实例（Instance）布局操作的混合类（mixin），提供了一系列与布局相关的实用方法，主要用于计算、获取、调整和转换实例的几何信息及布线结构。该类主要服务于全定制集成电路设计流程中的版图生成与优化任务。

通过继承 LayoutMixin，实例可以具备以下能力：

  • 计算自身在父模块中的边界框（box）。
  • 获取指定层上的边界区域（get_box）。
  • 将实例转换为非对齐的 NBox 表示（to_nbox）。
  • 清理实例中的 via 矩形（clear_via）或整个布局（clear_layout）。
  • 生成包围整个实例的 NBox（bounding_nbox）。
  • 在实例周围添加垂直方向的 bus 线（add_vbus），支持多种参数配置，如位置、间距、宽度、优先级等。

该类中的方法均基于 TED 框架的底层版图操作机制，广泛应用于物理设计阶段的自动化布线与修正。

参数定义:

参数名类型含义默认值
selfInstance当前实例对象，用于调用本方法。无
layerLayer 或 None指定获取该层上的边界框；若为 None，则返回整体边框。None
argsCircuitNode 或 str 的可变参数列表要连接的电路节点，决定生成多少条 bus 线；顺序与 bus 顺序一致。无
viasint用打孔数表示 bus 宽度（如 via=2 表示两列 via），与 width 不能同时指定。1
widthint 或 None以实际尺寸定义 bus 宽度（单位为 nanometer），与 vias 不能同时指定。None
locationLEFT, CENTER, RIGHTBus 相对于实例的位置：左对齐、居中或右对齐。LEFT
spacingintBus 与实例之间的间距（单位为 nanometer）。300
bus_spacingint 或 NoneBus 线之间的间距，若为 None 则使用层级最小间距。None
priorityint 或 NoneBus 的优先级，数值越高优先级越高，高优先级线会优先连接。None
extendTuple[int, int]扩展方向和大小，第一项为方向（TOP, MIDDLE, BOTTOM），第二项为偏移量。(MIDDLE, 0)
overlapbool是否允许重叠放置，若为 True 则 spacing 参数失效。False
to_originbool操作完成后是否将结果移回原点（即坐标归零）。True

注意：

  • vias 和 width 二者只能使用其一，不能同时指定。
  • location 必须为 LEFT, CENTER, 或 RIGHT 之一。

返回值

  • box: 返回一个 Box 对象，表示实例在父模块中的边界框。
  • get_box(): 返回指定层上的 Box；若无对应层，则返回整体 box。
  • to_nbox(): 返回一个 NBox 对象，表示实例的非对齐版图区域。
  • clear_via(): 返回清理后剩余的矩形集合（InstList）。
  • clear_layout(): 返回清除布局后生成的新实例列表（List[Instance]）。
  • bounding_nbox(): 返回一个 NBox 对象，表示整个实例的包围区域。
  • add_vbus(): 返回添加 bus 后生成的实例列表（InstList）。

代码示例:

def test_layout_mixin():
    # 创建一个 Instance 实例并配置其子模块
    inst = create_instance_with_layout()

    # 1. 计算并获取实例的边界框（Box）
    box = inst.box
    assert isinstance(box, Box)
    print(f"Instance box: width={box.w}, height={box.h}, x1={box.x1}, y1={box.y1}")

    # 2. 获取指定层上的盒子
    layer = get_layer_by_name("metal1")
    specific_box = inst.get_box(layer=layer)
    print(f"Box on layer 'metal1': {specific_box}")

    # 3. 将实例转换为 NBox
    nbox = inst.to_nbox()
    assert isinstance(nbox, NBox)
    print(f"Converted to NBox: {nbox}")

    # 4. 清理实例中的 via 矩形，并返回处理后的矩形集合
    cleared_insts = inst.clear_via()
    print(f"Number of cleared rects: {len(cleared_insts)}")

    # 5. 清除整个布局并获取新的实例列表
    new_insts = inst.clear_layout()
    print(f"Layout cleared, produced {len(new_insts)} instances")

    # 6. 获取整个实例的包围 NBox
    bound_nbox = inst.bounding_nbox()
    assert isinstance(bound_nbox, NBox)
    print(f"Bounding NBox: {bound_nbox}")

    # 7. 在实例周围添加垂直 bus 线（使用 vias 定义宽度）
    node1 = CircuitNode("VDD")
    node2 = CircuitNode("GND")
    buses = inst.add_vbus(node1, node2, layer=layer, vias=3, spacing=500, location=CENTER, priority=1)
    print(f"Added {len(buses)} bus lines")

    # 8. 使用 width 参数定义宽度，并启用扩展
    buses_expanded = inst.add_vbus(node1, layer=layer, width=200, extend=(TOP, 150), overlap=True, to_origin=False)
    print(f"Added expanded buses: {len(buses_expanded)}")

to_nbox

功能介绍:

to_nbox 是一个装饰为布局函数（@layout）的实例方法，用于将当前 Instance 对象转换为一个 NBox 类型的对象。该方法仅在当前实例表示一个矩形几何形态时有效，即调用前需确保 self.is_rectangle() 返回 True。方法内部通过 NRect 类对实例的布局信息进行封装，并最终调用 NRect 的 to_nbox 方法完成转换。该 API 常用于在电路版图设计中将具有矩形形状的实例转换为更高级别的边界盒（Bounding Box）抽象，便于后续的空间运算和布局分析。

参数定义:

参数名类型含义默认值
selfInstance当前实例对象，必须是一个矩形形状的实例。无

注意：该函数为实例方法，无需显式传参；参数列表中仅 self 表示调用者自身。

返回值

  • 返回类型：NBox
  • 含义：表示当前 Instance 在版图中的边界盒（即最小外接矩形），包含位置和尺寸信息，适用于空间布局管理与几何运算。

代码示例:

def test_to_nbox():
    from tedcore.kernel import Instance, NBox, NRect

    # 创建一个矩形实例
    rect_inst = Instance()
    rect_inst.set_shape(NRect(0, 0, 10, 20))  # 设定为 x=0, y=0, 宽10, 高20 的矩形
    assert rect_inst.is_rectangle()  # 确保是矩形

    # 调用 to_nbox 方法
    nbox = rect_inst.to_nbox()
    
    # 验证返回值
    assert isinstance(nbox, NBox)
    assert nbox.x == 0
    assert nbox.y == 0
    assert nbox.width == 10
    assert nbox.height == 20

该测试用例展示了 to_nbox 的典型使用方式：

  • 用法一：将矩形实例转换为 NBox 对象
    当一个 Instance 实例具有矩形形状时（通过 set_shape 设置），调用 to_nbox() 方法可将其转换为 NBox，用于获取其完整的边界信息。测试中验证了转换后的 NBox 对象的坐标和尺寸与原 NRect 一致。

  • 用法二：基于断言前置检查
    代码中通过 assert self.is_rectangle() 确保调用前实例为矩形，避免非矩形对象引发错误。测试用例中显式调用 is_rectangle() 保证合法输入，体现该方法的使用前提。

注意：调用 to_nbox 前必须确保实例是矩形形态，否则会触发断言失败。

pguard

功能介绍:

pguard 是一个用于在电路设计中为指定单元（cell）添加 P 型保护环（Guard Ring）的函数式 API。该函数作用于 Instance 类型对象，旨在在模拟或全定制集成电路设计中，增强电路对噪声和闩锁效应（latch-up）的抗干扰能力，尤其适用于 P 沟道 MOS 管（PMOS）等敏感结构。

该函数根据输入参数配置保护环的类型（P 型或 N 型）、与顶层金属层的连接方式、间隔距离、接触孔数量、遮挡层（obstruction layer）以及是否验证单元类型。最终返回一个 GuardRing 实例，表示已创建的保护环结构。

该函数通过装饰器 @layout 注册为布局操作，可链式调用后续方法如 .route() 和 .lvs()，进行布线和版图验证。

参数定义:

参数名类型含义默认值

pg

功能介绍:

pg 是一个用于在电路实例（Instance）上添加 P 型保护环（Guard Ring）的装饰器函数。该函数主要针对模拟或全定制集成电路设计中的保护环结构生成，用于提升电路的抗干扰能力、减少寄生效应和改善噪声性能。

该函数本质上是调用 self.pguard 方法的封装，通过指定参数来配置保护环的类型（P型或N型）、与外界的连接方式、间距、顶层金属层、过孔数量以及是否添加遮挡层等。此功能适用于需要严格控制版图布局和电气特性的高频、高精度或低噪声电路设计场景。

参数定义:

参数名类型含义默认值
connectToOptional['CircuitNode']保护环需要连接的外部节点，如电源（VDD）或地（VSS）。若不指定，则默认不连接。None
typOptional[str]指定保护环的类型，取值为 'p' 表示 P 型保护环，取值为 'n' 表示 N 型保护环。'p'
marginUnion[int, List[int]]保护环到目标单元（cell）之间的间距。若为整数，则表示四周均等的间距；若为长度为4的列表，则分别代表左、上、右、下四个方向的间距。格式：[left, top, right, bottom]。None
viasOptional[int]保护环中允许容纳的 VIA1 过孔数量，用于控制垂直连接的密度。None
obsOptional['Layer']需要添加的遮挡层（Obstruction Layer），用于在版图中屏蔽特定区域。None
top_metalUnion['Layer', List['Layer']]保护环所使用的最高金属层。若为单个层，则所有方向均使用该层；若为长度为4的列表，则对应左、上、右、下方向分别使用不同金属层。格式：[left, top, right, bottom]。None
check_cellbool是否检查被保护单元（cell）的类型是否正确，以确保其支持保护环添加操作。True

返回一个 guardring 实例对象，代表创建的保护环结构，可用于后续连接或布局操作。

nguard

功能介绍:

nguard 是一个用于在集成电路设计中为指定单元（cell）添加 N 型保护环（Guard Ring）的函数式 API。该操作通常用于增强电路的抗噪声能力与隔离性能，尤其是在模拟和混合信号设计中，通过将敏感区域包围在 N 型保护环内，防止寄生效应或外部干扰影响电路功能。

该 API 是一个装饰器函数，应用于 Instance 类型的实例上，能够根据参数配置生成符合工艺要求的 N 型保护环结构。它支持灵活设置保护环与单元之间的间距、顶层金属连接方式、过孔数量、屏蔽层以及连接节点等选项。

viasOptional[int]指定保护环上的过孔数量，即能容纳多少个 VIA1，用于确保电流路径连通性。None

功能介绍:

ng 是一个用于在电路实例（Instance）上添加 N 型保护环（Guard Ring）的布局函数。该函数通过调用底层 nguard 方法实现，主要用于在模拟或全定制集成电路设计中为指定单元（cell）添加 N 型保护环，以增强电路的抗干扰能力，减少寄生效应和噪声影响。

该函数是一个装饰器函数（由 @layout 标记），用于定义布局行为，适用于 TED 框架中的模块实例。调用此函数后，会在指定单元周围生成符合要求的 N 型保护环结构，并根据参数配置其扩展距离、层数、过孔数量及障碍层等。

该 API 是对 self.nguard 方法的封装，用于实现保护环的布局逻辑。

参数定义:

参数名类型含义默认值
connectToOptional['CircuitNode']保护环需连接的电路节点。若不指定，则保护环可能不进行外部连接。None
typOptional[str]指定保护环类型，如 "p" 表示 P 型保护环，"n" 表示 N 型保护环（默认）。"n"
marginUnion[int, List[int]]指定单元到保护环的间距。若为整数，则表示四边均等距离；若为长度为 4 的列表，则分别代表左、上、右、下四个方向的距离（对应：← ↑ → ↓）。None
viasOptional[int]保护环的过孔密度，表示可容纳多少个 VIA1 过孔用于连接多层金属。None
obsOptional['Layer']添加的障碍物层（即掩膜层），用于在布局中标识不可布线区域或其他物理限制。None
top_metalUnion['Layer', List['Layer']]保护环所使用的最上层金属。若为单个 Layer，则整个保护环使用同一层；若为列表，则按 [left, top, right, bottom] 的顺序分别指定各边的顶层金属。None
check_cellbool是否检查传入的 cell 类型是否符合预期（如是否为合法的子模块、是否支持添加保护环）。True
_kwargs**_kwargs保留参数，用于后续扩展或兼容其他布局配置。——

返回值

  • 返回一个 guardring 实例对象，代表生成的保护环结构，可用于后续连接、检查或输出到版图中。

功能介绍:

cover 是一个用于在电路实例（Instance）上添加矩形覆盖区域的函数。该操作通常用于在特定层上为实例（如晶体管、单元等）生成覆盖形状，以满足设计规则或实现特定的物理布局需求。

该函数主要用于在电路版图设计中对已有单元或结构进行扩展性覆盖，常用于添加金属覆盖、保护区域、隔离区等场景，特别适用于与 guardring（保护环）、col（列布局）等其他布局操作配合使用。

bounding_nbox

功能介绍:

bounding_nbox 是一个用于获取电路实例（Instance）边界框的函数型布局方法。该函数返回一个表示该实例几何边界的 NBox 对象，NBox 通常用于描述电路单元在版图中的空间位置、尺寸和层级信息。

此函数通过调用实例自身的 box 属性（表示其几何区域）、层信息、控制节点（若有）、名称、是否跳过物理验证（PV）、优先级以及标签等元数据来构造一个完整的 NBox 实例。该操作仅支持矩形或多边形类型的实例，因此在执行前会进行合法性校验。

@layout 装饰器表明此函数用于自动布局流程中，属于版图生成阶段的辅助功能。

参数定义:

无参数。

返回值

  • 类型: NBox
  • 含义: 返回一个基于当前实例几何信息（边界框）及属性构造的新 NBox 实例，包含层、位置、名称、控制节点、跳过 PV 标志、优先级和标签等完整上下文信息。

代码示例:

def test_bounding_nbox_basic():
    inst = Instance()
    inst.set_box(Box(0, 0, 10, 20))  # 设置边界框
    inst.set_layer("METAL1")
    inst.set_name("cell_1")
    inst.set_priority(5)
    inst.add_tag("logic")
    
    nbox = bounding_nbox(inst)
    assert isinstance(nbox, NBox)
    assert nbox.get_box().equals(Box(0, 0, 10, 20))
    assert nbox.get_layer() == "METAL1"
    assert nbox.get_name() == "cell_1"
    assert nbox.get_priority() == 5
    assert nbox.get_tags() == {"logic"}

def test_bounding_nbox_polygon():
    poly_inst = Instance()
    poly_inst.set_polygon(Polygon([Point(0, 0), Point(10, 0), Point(10, 5), Point(0, 5)]))
    poly_inst.set_layer("POLY")
    poly_inst.set_name("poly_cell")
    
    nbox = bounding_nbox(poly_inst)
    assert isinstance(nbox, NBox)
    assert nbox.get_box().equals(Box(0, 0, 10, 5))

def test_bounding_nbox_skip_pv():
    inst = Instance()
    inst.set_box(Box(100, 100, 200, 300))
    inst.set_layer("DIFF")
    inst.set_name("diff_cell")
    inst.set_skip_pv(True)

在以上测试用例中，展示了 bounding_nbox 函数的典型用法：

  • 基本使用：为一个矩形实例设置 box、layer、name、priority 和 tag 后调用 bounding_nbox，验证返回的 NBox 是否正确携带这些属性；
  • 多边形支持：传入具有 Polygon 造型的实例，函数依然能正确提取其边界框并构造 NBox；
  • 特殊属性处理：测试了 is_skip_pv() 和 get_priority() 等属性在 NBox 中的传递是否准确。

该 API 在自动布局流程中可用于快速获取任意实例的几何表示，便于后续的空间分析、对齐、布线等操作。

add_vbus

功能介绍:

add_vbus 是一个用于在电路实例中添加垂直方向布线（bus）的函数型装饰器方法。该函数根据指定的节点列表生成一组垂直的金属连线（bus），并将其布置在实例的特定位置，支持对布线宽度、间距、优先级、扩展长度等参数进行精细化控制。

该方法通常用于在电路布局中自动创建多根平行的垂直信号线（bus），以连接多个节点。其行为会根据参数配置决定布线的物理尺寸（通过 vias 或 width 指定）、相对位置（如左、中、右）、与其他元件的距离（spacing）、各 bus 之间的间距（bus_spacing）以及是否自动裁剪等。

该函数返回一个 InstList 实例，表示生成的布线整体，可用于后续的连接或布局操作。

参数定义:

参数名类型含义默认值
self'Instance'当前实例对象，用于绑定操作上下文。无
*argsUnion['CircuitNode', str]用于连接的节点列表，每个节点将作为生成 bus 的一个端点。输入的节点顺序即为生成 bus 的顺序。无
layerOptional['Layer']指定 bus 所在的物理层（如 M1、M2 等）。如果未指定，则由底层算法自动推断。None
viasint使用打孔数量来表示 bus 宽度，即以“via单元格”为单位计算宽度。不能与 width 同时使用。1
widthOptional[int]以实际物理尺寸（单位：纳米）表示 bus 宽度。不能与 vias 同时使用。None
locationint指定 bus 相对于实例的布局位置，可选值为 LEFT、CENTER 或 RIGHT。LEFT
spacingintbus 与实例之间的最小间距（单位：纳米）。300
bus_spacingOptional[int]bus 之间彼此的间距（单位：纳米），若为 None，则使用该层的最小间距。None
priorityOptional[int]bus 的优先级，值越大表示优先级越高。高优先级的布线会优先完成连接。None
extendTuple[int, int]布线的扩展大小，第一个元素为方向（TOP、MIDDLE、BOTTOM），第二个元素为扩展量（单位：纳米）。例如 (TOP, 100) 表示向上扩展 100nm；(MIDDLE, 100) 表示上下各扩展 100nm。(MIDDLE, 0)
overlapbool是否允许重叠放置（即布线覆盖已有实体），若为 True，则 spacing 参数将失效。False
to_originbool是否将生成的布线移动到原点位置（即坐标 (0, 0)）。True
auto_cutbool是否自动裁剪超出范围的部分。若设置了 extend，则该参数可能导致 cut 失效。False

返回值

InstList：一个包含所有生成的 bus 实例及其组合结构的整体实例列表，可以用于后续布局、连接或验证操作。

add_tap

功能介绍:

add_tap 是一个用于在电路实例中添加 TAP（终端接触孔）的函数。TAP 通常用于为晶体管或电路区域提供电源或接地连接，以确保良好的电气性能和工艺鲁棒性。该函数通过指定连接节点、类型、方向、间距等参数，智能地将 TAP 添加到目标实例周围，并返回由 TAP 和实例共同构成的整体结构。

此函数是 Instance 类的一个装饰器方法（使用 @layout 修饰），属于版图布局阶段的操作，常用于全定制模拟电路设计中，如在标准单元、宏模块或复杂电路结构的边界处添加 TAP。

参数定义:

参数名类型含义默认值
connectToCircuitNodeTAP 需要连接的电路节点，即 TAP 的连接目标。无
typstrTAP 的类型，如 "p" 表示 P 型 TAP，"n" 表示 N 型 TAP。具体类型需在 guardring_type.yaml 文件中定义。None
directionintTAP 相对于实例的位置方向，可为 TOP, BOTTOM, LEFT, 或 RIGHT，表示添加到实例的哪一侧。TOP
spacingOptional[int]TAP 与实例之间的间距（以最小单元为单位）。若为 None，则自动计算最小安全间距。None
top_metalLayerTAP 使用的最顶层金属层（如 M1, M2 等）。若为 None，则默认使用 M1 层。None
viasOptional[int]TAP 的宽度通过 via 数量来描述，即采用多少个 via 串联形成通孔通道。若为 None，则根据工艺规则自动选择合适数量。None

返回值

返回类型为 GuardRing，表示由 TAP 和其连接的实例共同组成的一个完整保护环结构（Guard Ring）或终端环结构。

代码示例:

def test_add_tap():
    # 创建一个实例并添加不同类型的 TAP
    inst = Instance()  # 假设已定义实例
    node = CircuitNode()

    # 添加顶部 P 型 TAP，并使用自动间距和 M1 层
    guard_ring_p = add_tap(inst, connectTo=node, typ="p", direction=TOP)

    # 添加底部 N 型 TAP，并指定固定间距和 M2 层
    guard_ring_n = add_tap(inst, connectTo=node, typ="n", direction=BOTTOM, spacing=5, top_metal=M2)

    guard_ring_left = add_tap(inst, connectTo=node, typ="p", direction=LEFT, vias=3, top_metal=M1)

说明：

  • 在第一个示例中，add_tap 添加了 P 型 TAP 到实例顶部，使用默认的 spacing=None（自动计算最小间距），并默认使用 M1 层。
  • 第二个示例中，指定了固定 spacing=5 和 top_metal=M2，用于控制工艺精度和连接可靠性。
  • 第三个示例使用 vias=3 明确设置通孔宽度，适用于高密度连接需求。

这些实例展示了 add_tap 的多种典型用法：

  1. 添加 P/N 型 TAP 到顶部或底部；
  2. 指定特定的间距和金属层；
  3. 通过 vias 参数控制 TAP 导通能力。

add_label

功能介绍:

add_label 是一个装饰器函数（由 @layout 定义），用于为电路实例（Instance）添加标签（Label）。该函数会基于传入的参数在对应的层级（Layer）上生成文本标签，通常用于在版图中标识特定实例或元素。若未指定层级，则自动创建一个 TEXT 层级的标签。

当调用此方法时，系统会自动生成并附加一个带有指定名称的标签到实例，并返回包含该标签和原实例的组合列表（InstList），同时将新生成的标签重命名为以 [label] 包裹的形式，便于在版图中识别。

参数定义:

参数名类型含义默认值
labelstr标签名称。若为 None，则自动使用实例的名称作为标签名。None
layerOptional['Layer']指定标签所绘制的版图层级。若为 None，表示系统将自动创建一个 TEXT 层级用于显示标签。None

返回值

  • 类型：InstList
  • 含义：包含原始实例与新添加的标签组成的整体实例列表，其中标签部分已重命名为 [label] 格式。
