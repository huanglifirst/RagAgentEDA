Variable

功能介绍:

Variable 是 TED 框架中用于定义可变参数的核心类，主要用于电路设计中的参数化建模与仿真。它支持数值变量的声明、边界设置、整数类型约束、枚举值指定以及自动导出网表和灵敏度扫描数据等功能。

该类具备以下核心能力：

  • 声明一个具有名称、初始值、步长（step）和可选枚举值的参数；
  • 支持整数型（is_int=True）或浮点型变量的区分；
  • 可设置上下界（set_bound），用于灵敏度分析或优化场景；
  • 提供对变量进行数学运算（如 +, *, // 等）的能力，返回新 Variable 实例；
  • 支持将所有变量导出为 HSpice 网表参数格式；
  • 生成适用于敏感性分析的数据扫描配置（export_data_sweep_for_sensitivity）；
  • 能够注册到全局电路上下文中，避免重复命名冲突。

Variable 可作为模块参数传递，也可用于表达式计算，是实现参数化设计、自动化仿真流程的重要基础组件。

参数定义:

参数名类型含义默认值
namestr变量的名称，唯一标识符。若未提供且无默认值时，自动命名格式为 _param{index}。None
valueint 或 float变量的初始值。必须是整数或浮点数。None
stepfloat变量变化的步长，仅对整数型变量有效；浮点型变量允许步长为非整数。None
enumsList[Union[int, float]]该变量可取的离散值列表（即枚举），若设置则忽略上下界与步长，仅在枚举集中选择值。[]
is_intbool标记是否为整数类型变量。影响导出行为和灵敏度分析策略。False
force_rebuildbool若为 True，允许覆盖已存在的同名变量；否则会抛出异常。False
callbackcallable当变量值改变时触发的回调函数，可自定义行为（如更新视图、刷新计算等）。None

注意：value 与 name 必须至少指定一个。若仅传入 value 而无 name，则系统自动为其生成名称。

返回值

  • 所有方法返回值类型为具体类型（如 float, str, list, Variable 等），详见各方法描述。
  • 特别说明：
    • get_values() 返回一个列表，包含在上下界及步长范围内所有可能的取值。
    • set() 方法返回一个链式调用支持的 Variable 实例（通过 self）。
    • export_data_sweep_for_sensitivity() 返回一个 DataSweep 对象，用于敏感性分析。
    • export_netlist_parameters() 返回字符串，表示所有变量在网表中的参数定义。

update_all_variables

功能介绍:

该函数是一个静态方法，用于批量更新电路中所有变量的值。它接收一个包含新值的列表 x，并将这些值逐个分配给电路当前变量集合中的对应变量。
函数会首先对每个元素进行评估（若为 Variable 类型则调用其 eval() 方法获取实际数值），然后将更新后的值通过 set_value 方法写入电路中的相应变量对象中。
该函数主要用于在电路仿真或调试过程中，统一刷新多个变量的状态，常用于调试信息输出、参数初始化或动态注入新值等场景。

参数定义:

参数名类型含义默认值
xList[Union[Variable, float, int]]包含待更新变量值的列表，元素可以是 Variable 实例或数值类型（如 float、int）；若为 Variable 实例，则会调用其 eval() 方法获取实际值。无

返回值

无返回值（None）

    with new_circuit() as c:
        a = Variable('a', 1.0)
        b = Variable('b', 2.0)

set_bound

功能介绍:

set_bound 是 Variable 类的一个实例方法，用于设置变量的取值边界（下界 lb 和上界 ub）。该方法的作用是为变量定义一个有效的数值范围，通常用于敏感性分析、参数扫描或优化场景中对变量取值范围的约束。

若变量已定义了枚举值列表（_enums），则调用此方法会扩展原有枚举值：将 [lb, ub] 范围内以 step_copy 为步长生成的新数值添加到原枚举列表中，并更新内部索引边界 _lb, _ub 和步长 _step。如果未定义枚举值，则直接设置连续型变量的边界。

该方法支持浮点和整数类型变量，适用于构建参数扫描数据集（通过 export_data_sweep_for_sensitivity() 导出）或作为后续计算中范围约束的基础。

参数定义:

参数名类型含义默认值
lbfloat 或 int变量的下界（最小值）无
ubfloat 或 int变量的上界（最大值）无

返回值

返回当前 Variable 实例本身，支持链式调用。

lt

功能介绍:

lt 是 Variable 类的一个实例方法，用于创建一个限制变量值上限的约束条件。该方法返回一个新生成的 Variable 对象，其表达式为 min(当前变量, maximum)，即取当前变量与指定最大值中的较小者作为结果值。此操作常用于对变量输出进行上限控制，确保其不超过设定的最大阈值。

由于该方法定义在 Variable 类中，因此它适用于所有继承或实例化自该类的变量对象。

参数定义:

参数名类型含义默认值
maximumAny表示要限制的上限值，用于与当前变量进行比较并取较小值。无

返回值

返回一个新创建的 Variable 对象，其表达式为 min(self, maximum)，实际值为 min(self.eval(), maximum)。

参数名类型含义默认值
minimumAny用于比较的最小值，与当前变量值取最大值。无

返回值

返回一个 Variable 类型的对象，该对象封装了表达式 max(当前变量, minimum) 及其计算后的值（通过 eval() 计算得到的最大值）。

代码示例:

from pyted import *

from tedcore.kernel import Variable

def test_max_min():
    max_v = 10
    min_v = 2
    v1 = Variable(name='v1', value=8, step=1).set_bound(2, 10)
    v2 = Variable(name='v2', value=6, step=1).set_bound(2, 10)
    v3 = Variable(name='v3', value=4, step=1).set_bound(2, 10).int()
    c1 = max(v1 * v2 / v3, max_v)
    c2 = min(v1 * v2 / v3, min_v)
    print(c1, c2)
    d1 = (v1 * v2 / v3).gt(min_v)
    d2 = (v1 * v2 / v3).lt(max_v)
    print(d1(), d2())

解释：

  • v1, v2, v3 为具有初始值和取值范围的变量，均设置了上下限约束为 [2, 10]。
  • v1 * v2 / v3 表达式计算得 8 * 6 / 4 = 12。
  • c1 = max(v1 * v2 / v3, max_v) 等价于 max(12, 10)，结果为 12。
  • c2 = min(v1 * v2 / v3, min_v) 等价于 min(12, 2)，结果为 2。
  • d1 = (v1 * v2 / v3).gt(min_v) 使用 gt 方法，将表达式的值与 min_v=2 比较，取最大值，即 max(12, 2) = 12。调用 d1() 可获取最终结果。
  • d2 = (v1 * v2 / v3).lt(max_v) 采用类似逻辑但取最小，此处仅为说明链式操作的可用性。

该示例展示了在动态建模场景中，通过 gt 方法实现对变量输出施加下限的行为，适用于约束设计参数或校验电路响应边界。

export_data_sweep_for_sensitivity

功能介绍:

该静态方法用于生成一个用于灵敏度分析的数据扫描（DataSweep）对象，其目的是为电路中的所有变量构建一个标准化的参数扫描方案。该方法会自动分析当前电路中所有变量的边界值（下界和上界）以及当前取值，并根据变量类型（浮点型或整型）计算出一个合理的扰动方向与幅度，生成一组用于灵敏度测试的参数组合。

具体行为如下：

  • 为每个变量确定一个扰动方向（向高或向低）：优先选择与较远边界的方向。
  • 对于浮点型变量，扰动幅度为当前值的10%（即 v() * 0.1）。
  • 对于整型变量，使用其步长（step）作为扰动单位。
  • 构建一个数据集，其中第一行为原始参数值，后续行每行仅有一个变量被小幅调整，其余保持原值，形成一维敏感度扫描。

最终返回的 DataSweep 对象可用于后续的仿真或分析流程，支持对电路性能随参数微小变化的分析。

参数定义:

参数名类型含义默认值
无输入参数-此方法为静态方法，无需外部传入参数。内部通过 get_circuit().variables 获取当前电路中所有变量的集合。无

返回值

  • 返回类型：DataSweep
  • 含义：一个包含用于灵敏度分析的参数扫描数据的对象，其结构为：
    • name: 名称为 'sweepdata' 的扫描数据集；
    • params: 所有变量名组成的列表；
    • values: 一个二维数组，第一列为原始值，其余每行表示仅有一个变量发生微小扰动后的取值。

eval

功能介绍:

eval 是 Variable 类的一个实例方法，用于计算并返回该变量的当前值。该方法首先检查内部 _eval 属性是否为可调用对象（如函数或方法），若是则执行它获取值；否则直接使用 _eval 的原始值作为结果。随后对获取到的值进行处理（通过 process 方法），最终返回处理后的值。此方法常用于在敏感性分析或参数扫描场景中动态获取变量的实际数值。

参数定义:

无参数。

返回值

  • 类型：任意类型，具体取决于 _eval 的返回结果及 process 的处理逻辑。
  • 含义：经过计算和处理后的变量实际值，通常为浮点数、整数或对应的数据结构。
