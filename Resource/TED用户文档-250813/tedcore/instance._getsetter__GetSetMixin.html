GetSetMixin

功能介绍:

GetSetMixin 是一个用于电路设计中实例（Instance）的混合类（mixin），提供了一系列用于获取实例几何属性和最小间距信息的辅助方法。该类主要服务于全定制电路设计场景，帮助用户获取实例在平面布局中的位置、尺寸以及基于工艺规则的最小间距要求。

  • get_h_min_spacing 和 get_v_min_spacing 方法分别用于计算水平方向和垂直方向上所需的最小间距，以满足设计规则检查（DRC）中的层间间距约束。它们通过分析当前实例所处的所有嵌套单元（NBox），结合工艺表格中的间距规则来动态计算。
  • 提供多个坐标与尺寸获取方法，如 get_x1、get_y1、get_x2、get_y2 等，用于读取实例的左下角与右上角坐标，以及宽度（w）和高度（h）属性。
  • get_position 方法返回一个包含 (x1, y1) 的元组，表示实例左下角的位置。

所有这些方法均基于 self.box 对象获取几何信息，其中 box 是实例的封装矩形区域（通常代表物理边界），其属性包括 x1, y1, x2, y2, w, h 等。

说明：该类为工具类混合使用，不应当被单独实例化，而是作为其他模块或实例的基类混入实现功能扩展。

参数定义:

参数名类型含义默认值
selfInstance当前的电路实例对象，包含几何信息（box）和层级结构（如NBox）无

注：所有方法均不接受外部输入参数，参数意义由 self 实例承载。

返回值

方法名返回类型含义
get_h_min_spacingint返回水平方向最小间距（单位为工艺节点单位），若无约束则返回 0
get_v_min_spacingint返回垂直方向最小间距（单位为工艺节点单位），若无约束则返回 0
get_x1, get_y1, get_x2, get_y2int分别返回实例左下角的 x 坐标与 y 坐标，右上角的 x 坐标与 y 坐标
get_w, get_hint返回实例的实际宽度（w）和高度（h），单位为工艺节点单位
get_positionTuple[int, int]返回左下角坐标 (x1, y1) 的元组

代码示例:

def test_getset_mixin():
    # 假设有一个实例对象 instance，其 box 属性包含以下值：
    # x1 = 10, y1 = 20, x2 = 30, y2 = 40
    # w = 20, h = 20

    # 模拟场景：实例位于某个设计中，内部有多个 NBox（包含层信息）

    # 获取左下角坐标
    x1 = instance.get_x1()
    assert x1 == 10

    y1 = instance.get_y1()
    assert y1 == 20

    # 获取右上角坐标
    x2 = instance.get_x2()
    assert x2 == 30

    y2 = instance.get_y2()
    assert y2 == 40

    # 获取宽度与高度
    w = instance.get_w()
    assert w == 20

    h = instance.get_h()
    assert h == 20

    # 获取位置元组
    pos = instance.get_position()
    assert pos == (10, 20)

    # 计算最小水平间距（H方向）
    h_min_spacing = instance.get_h_min_spacing()
    # 假设工艺中 AA 层间 H 方向有 5 单位的最小间距，且当前实例与相邻 NBox 距离为 3 单位，则返回 max(0, 5 - (20-?))，具体取决于上下文
    # 该值由 internal `__get_min_spacing_helper` 动态计算

    # 计算最小垂直间距（V方向）
    v_min_spacing = instance.get_v_min_spacing()
    # 同上，根据 V 方向的层间规则判断最小间距

说明：

  • 上述测例展示了 GetSetMixin 中各方法的基本调用方式。
  • get_x1, get_y1, get_w, get_h 等方法直接返回 box 对象的属性值，适用于快速获取几何信息。
  • get_position 是对 get_x1 和 get_y1 的封装，简化坐标访问逻辑。
  • get_h_min_spacing 与 get_v_min_spacing 的行为依赖于内部的 __get_min_spacing_helper 方法，其计算过程涉及：
    • 获取所有扁平化的 NBox（get_flatten_nboxes()）
    • 合并同一层的 NBox（merge_by_layer）
    • 遍历每个有效层（仅当层为 route 层或 AA/TAP）进行间距检查
    • 根据实际坐标差值 diff 与工艺表格中查得的最小间距对比，最终返回最大满足距离需求的结果。
  • 此类设计用于支持自动布局布线或 DRC 逻辑，确保各实例之间保持安全距离。

set_priority

功能介绍:

set_priority 是一个用于设置实例（Instance）优先级的方法，属于 Instance 类的成员函数。该方法允许用户在电路设计过程中动态调整特定实例的优先级值，影响后续布局、布线或综合等流程中的处理顺序。优先级高的实例将在调度或优化时被优先考虑。

该方法为链式调用设计（返回 self），因此支持连续操作，例如：r1.set_priority(2).some_other_method()。

参数定义:

参数名类型含义默认值
priorityint指定实例的新优先级值，整数类型，数值越大表示优先级越高。无

返回值

返回实例本身（Instance），支持方法链式调用。

set_group

功能介绍:

set_group 是一个用于设置实例（Instance）所属分组的方法。在 TED 框架中，实例可以被归类到不同的逻辑分组中，便于管理和组织电路设计中的模块实例。该方法允许用户为某个实例指定一个分组名称，并将此信息持久化存储于实例内部。

调用此方法后，实例将记录其所属的分组名，后续可通过 get_group 方法获取该值。该方法以链式调用方式返回实例本身，支持方法连写。

参数定义:

参数名类型含义默认值
groupstr指定该实例的分组名称，用于在设计中组织和管理实例。无

返回值

返回调用该方法的 Instance 实例本身（即 self），支持链式调用。

get_port_rect_list

功能介绍:

该函数用于获取实例（Instance）的所有端口矩形信息。返回值为经过排序后所有端口矩形的列表，每个端口可能包含一个或多个矩形区域（通常表示物理布局中的引脚或接触孔区域）。此接口常用于电路版图分析、设计规则检查（DRC）、自动布线或可视化工具中，以获取实例在物理空间上的端口位置分布。

该函数是装饰为 @layout 的方法，说明其行为与版图布局相关，返回值默认为空列表（通过 default=[] 指定），且不接收外部输入参数。

参数定义:

参数名类型含义默认值
selfInstance当前实例对象，代表一个电路模块的具体实例。无

返回值

  • List[Instance]：按端口名称升序排列的所有端口矩形组成的列表。每个元素为一个表示矩形的 Instance 对象（通常用于表示物理位置和尺寸），多个端口可能共享同一组矩形，因此最终结果是一个扁平化后的矩形列表。

  • 用法一：空端口矩形
    当实例未定义任何端口矩形时（如 port_rects 为空字典），返回空列表。适用于初始化状态或无物理引脚的模块。

  • 用法二：多端口多矩形场景
    如果一个实例包含多个端口，且每个端口有多个矩形，则函数会按端口名称进行升序排序（sorted(self.port_rects.keys())），然后将每个端口下的所有矩形依次展开并合并成一个扁平列表返回。此行为保证了输出顺序的一致性和可预测性。

  • 用法三：结果列表的结构
    返回的列表中，每一个元素均为 Instance 类型的对象，代表一个独立的矩形区域（常见于版图中的金属层、接触孔等）。该结构便于后续进行几何运算、碰撞检测或导出为CAD工具可识别的数据。

get_flatten_nboxes

功能介绍:

该函数用于获取一个实例（Instance）在经过打平（flatten）处理后所有NBox（非重叠盒子，用于表示布局中的几何区域）的集合。它会根据实例所属模块的旋转角度和翻转状态，对每个子实例的边界框（bounding box）进行坐标变换，并将其相对于实例的左下角进行偏移定位。

该函数是 Instance 类的一个方法，主要用于电路设计工具中对布局数据进行几何提取与整理。在处理嵌套模块或复杂结构时，能够递归地获取所有子单元的最终物理位置信息。

若当前实例是一个形状（shape），即自身为一个基本图形，则直接返回其包围盒；否则遍历其打平后的子实例，仅对矩形和多边形类型的子实例进行处理，将其包围盒通过逆变换转换到正确的方向，并移动到相对于父实例左下角的位置。

参数定义:

参数名类型含义默认值
self'Instance'当前实例对象，调用该方法的实例。无

注意：此方法无显式输入参数，仅依赖于实例自身的状态（如模块、旋转角度、翻转标志等）。

返回值

  • 类型：List['NBox']
  • 含义：打平后所有非重叠盒子的列表，每个 NBox 包含一个几何区域及其位置信息，表示电路中各个子单元在最终布局中的实际占用空间。

代码示例:

  • 返回结果为经过坐标变换和位置校正后的全部 NBox 列表。
