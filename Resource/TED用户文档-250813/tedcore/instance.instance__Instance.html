Instance

功能介绍:

Instance 是 TED 框架中用于表示电路模块实例的核心类，继承自多个混合（Mixin）基类，提供了丰富的功能支持，包括导出、节点管理、属性读写、布局操作、排序、几何信息处理、电源电压管理、端口管理、参数设置、标签标记以及调试支持等。该类封装了实例在电路设计中的完整行为与状态，适用于全定制集成电路设计中对模块实例的创建、操作与管理。

每个 Instance 实例对应一个模块的实际部署（即“实例化”），其内部通过 rsinst 原始对象承载底层数据，并支持克隆、打平（flatten）、拆分（split）等高级操作，以实现电路结构的重新布局和分析。

参数定义:

参数名类型含义默认值
rsinstAny用于初始化实例的底层原始对象，通常包含电路结构与属性信息。无
moduleModule实例所属的模块对象（可选），若未提供将通过上下文自动获取。None

返回值

  • __eq__: 返回布尔值，表示两个实例是否同名。
  • __hash__: 返回整数，表示实例的哈希码。
  • setup: 无返回值。
  • ignore_in_icc: 返回布尔值，指示该实例在 ICC（综合工具）中是否被忽略。
  • split: 返回 List[InstList]，将当前实例拆分为多个子实例列表。
  • flatten: 根据指定名称或标签，返回打平后的 Instance 列表。
  • clone: 返回新的 Instance 实例，与原实例参数相同。

代码示例:

# 测试用例来源: kernel/src/moduledef/tests/test_flatten.py#flatten1

def test_flatten():
    # 定义一个模块并创建实例
    @module
    def example_module(name, tag):
        return instance('sub_inst', name=name, tag=tag)

    # 创建顶层实例
    top = instance('top_module')

    # 子模块中嵌套多个带标签的子实例
    sub1 = example_module('inst1', tag='A')
    sub2 = example_module('inst2', tag='B')
    sub3 = example_module('inst3', tag='A')

    # 将 top 模块中的相关子实例按 tag 打平
    flat_instances = top.flatten(['A'], category='tag')
    
    # 验证结果
    assert len(flat_instances) == 2
    assert all(inst.name() in ['inst1', 'inst3'] for inst in flat_instances)
# 使用示例：按名称打平子实例

def test_flatten_by_name():
    @module
    def sub_module(name):
        return None

    top_inst = instance('top')

    # 创建多个命名的子实例
    inst1 = instance('sub1', module=sub_module)
    inst2 = instance('sub2', module=sub_module)

    # 按名称打平实例
    flattened = top_inst.flatten(['sub1'], category='name')
    
    assert len(flattened) == 1
    assert flattened[0].name() == 'sub1'

说明:

  • flatten 方法支持两种分类方式：
    • 当 category='tag'，依据子实例的 tag 属性进行筛选与打平。
    • 当 category='name'，依据子实例的 name 属性进行筛选与打平。
  • 打平过程会递归展开嵌套结构，并返回所有匹配条件的原始 Instance 实例列表，用于重新布局或分析。
  • 在测试用例中展示了对 tag 和 name 两类条件的使用场景，体现了其在模块化电路设计中的灵活性和实用性。
  • clone() 方法可创建当前实例的副本，常用于复制电路结构而不改变原结构。

to_subclass

功能介绍:

to_subclass 是一个实例方法，用于将当前的 Instance 实例转换为其对应的子类类型。该方法会尝试根据实例所属模块的路径和名称，动态导入并实例化一个继承自原始类型的子类（通常为特定用途的封装类）。若无法完成转换，则直接返回原实例对象。

此方法主要用于在 TED 框架中实现动态类型升级或类型适配，例如将通用的硬件实例（如 Instance）转换为具有明确语义或功能特性的子类（如 NRect 等具体布局/元件类）。转换成功与否取决于是否存在对应模块路径、可导入的对象，以及该对象构造函数是否接受 inst 参数。

参数定义:

无

参数名类型含义默认值
无-无参数-

返回值

  • 返回类型：Instance（或其子类）
  • 含义：若转换成功，返回由对应模块导入的子类实例；否则返回原 Instance 实例。

代码示例:

def test_to_subclass_rect():
    mod = NRect()
    inst = Instance(mod)
    assert isinstance(inst.to_subclass(), NRect)  # 转换为 NRect 类型

def test_to_subclass_no_module_path():
    inst = Instance(module=None)
    assert isinstance(inst.to_subclass(), Instance)  # 无路径时保持原类型
def test_to_subclass_is_rectangle():
    mod = NRect()
    inst = Instance(mod)
    inst.set_type('RECTANGLE')
    assert isinstance(inst.to_subclass(), NRect)

def test_to_subclass_with_module_path():
    module_path = "tedcore.kernel.NRect"
    name = "NRect"
    # 模拟导入逻辑
    call = import_object(module_path, name)
    assert callable(call) and inspect.isclass(call)
    inst = Instance(module=None)
    assert isinstance(inst.to_subclass(), NRect)

get_sim_current

功能介绍:

get_sim_current 是一个实例方法，用于在 TED 电路仿真环境中注册当前实例的电流采集需求。该方法通过向电路对象的 module_currents 列表中添加一条记录，标记需要对当前实例进行电流仿真分析。

该方法支持传入可变数量的端口参数（Node），表示该实例上特定端口的电流采集点，若不传参则默认采集整个实例的电流。使用此方法后，仿真器将在运行时自动收集对应电流数据，便于后续查看或分析电路功耗、信号行为等特性。

说明：该功能主要用于仿真阶段的数据采样，不影响电路结构本身。

参数定义:

参数名类型含义默认值
*NodeTuple[Instance, ...]可选的端口实例列表，用于指定需要采集电流的具体端口。若不提供，则默认对整个实例进行电流采集。无

注：Node 应为在 TED 框架中定义的 Instance 实例对象，通常是从模块实例化后获得的端口引用。

返回值

无返回值（返回类型为 None）。

代码示例:

def test_get_sim_current():
    from tedcore.kernel import Instance, TEDModuleParser

    # 创建一个简单的模块
    @module
    def SimpleMOS(G, D, S):
        return Instance(SimpleMOS, G=G, D=D, S=S)

    # 实例化模块并进行电流采集配置
    mos1 = SimpleMOS()
    mos1.get_sim_current()  # 不传参，采集整个实例的电流

    # 针对特定端口（例如漏极D）进行电流采集
    mos2 = SimpleMOS()
    d_port = Instance(SimpleMOS, G=..., D=..., S=...)  # 确保d_port为有效D端口引用
    mos2.get_sim_current(d_port)  # 指定在D端口采集电流

    # 多个端口同时采集
    mos3 = SimpleMOS()
    g_port = Instance(SimpleMOS, G=..., D=..., S=...)
    s_port = Instance(SimpleMOS, G=..., D=..., S=...)
    mos3.get_sim_current(g_port, s_port)  # 同时采集G和S端口的电流

说明：

  • 第一个用例 mos1.get_sim_current()：无参数调用，表示对该实例整体进行电流仿真采集。
  • 第二个用例 mos2.get_sim_current(d_port)：传入一个具体的端口实例（如漏极D），指定在该端口位置采集电流。
  • 第三个用例 mos3.get_sim_current(g_port, s_port)：支持多个端口同时注册，系统将为每个端口生成对应的电流观测点。

上述用法展示了 get_sim_current 的三种典型场景：

  1. 无需参数，对整个实例进行电流采集；
  2. 单个端口参数，精确指定某端口电流监测；
  3. 多个端口参数，实现多个信号节点的并行电流监控。

create_pv

功能介绍:

create_pv 是一个用于创建物理验证（Physical Verification, PV）实例的方法，属于某个类的成员函数。该方法基于当前对象（通常为模块或实例上下文）构建并返回一个物理验证对象，用于后续的版图一致性检查、设计规则检查（DRC）、布局与原理图一致性检查（LVS）等流程。

调用此方法时，可选择性地指定实例间的连接关系 inst_connect，这些连接用于描述电路中不同模块实例之间的信号或控制连接。该函数内部通过导入 lay 模块，并利用其提供的功能来完成物理验证的创建。

参数定义:

参数名类型含义默认值
inst_connectOptional[List[Tuple[str, str]]]实例连接列表，表示不同实例之间的端口连接关系，格式为 (source_inst, target_inst) 形式，用于构建物理连接拓扑。None

返回值

  • 类型：PhysicalVerification
  • 含义：返回一个已初始化的物理验证对象，可用于执行 DRC、LVS 等后续物理验证操作。

代码示例:

def test_create_pv_with_empty_connect():
    inst = MyModule()
    pv = inst.create_pv()
def test_create_pv_with_non_empty_connect():
    inst = MyModule()
    connections = [('inst1', 'inst2'), ('inst3', 'inst1')]
    pv = inst.create_pv(inst_connect=connections)

说明：

  1. 无连接参数：当不提供 inst_connect 参数时，使用默认空列表，表示当前模块实例之间没有额外的连接关系，默认按默认拓扑处理。

  2. 有连接参数：当传入非空的 inst_connect 时，系统将根据提供的连接元组列表建立实例间的连接关系，用于指导物理验证引擎构建完整的设计图结构。

该方法通常在模块实例化后、进行版图生成或检查前调用，确保物理验证能够正确识别电路内部连接逻辑。
