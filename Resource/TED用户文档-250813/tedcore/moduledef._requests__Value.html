Value

功能介绍:

Value 是一个用于描述电路模块实例化所需信息的数据模型类，继承自 BaseModel。该类主要用于封装模块的路径、名称、关键标识、配置参数、端口连接信息、输入参数及环境变量等，为模块实例化和后续处理提供完整上下文。

此类通常在电路设计流程中用于定义一个模块（Module）的详细需求，包含其所属路径、名字、内部状态键值（key）、配置项、输入/输出/控制端口的连接信号列表以及运行所需的参数与环境。适用于构建自动化模块生成、连接与仿真流程。

参数定义:

参数名类型含义默认值
module_pathstr模块在项目中的路径，用于定位模块文件。无
module_namestr模块的名称，表示模块的具体类型或功能。无
keyint用于标识该模块实例的唯一键值，通常用于内部状态管理或映射。无
configDict[Any, Any]模块运行所需的配置参数，支持任意类型的键值对。无
terminalsTuple[List[str], List[str], List[int]]包含三部分的元组：输入端口名称列表、输出端口名称列表、控制端口索引列表。无
argsDict[str, Any]模块构造时传入的参数，键为参数名，值为对应参数。无
envDict[Any, Any]运行环境变量，用于配置模块在特定环境下的行为或资源访问。无

返回值

无返回值。该类为数据封装用途，本身不执行计算或逻辑，仅定义数据结构。

代码示例:

def test_Value_basic():
    value = Value(
        module_path="/example/module/path",
        module_name="SampleModule",
        key=42,
        config={"vdd": 1.8, "temp": 25},
        terminals=(["in1", "in2"], ["out1"], [0]),
        args={"width": 100, "length": 50},
        env={"tool": "ted-capture", "device": "t7"}
    )
    assert value.module_path == "/example/module/path"
    assert value.key == 42
    assert value.config["vdd"] == 1.8
    assert value.terminals[0] == ["in1", "in2"]

本测试用例展示了如何完整创建一个 Value 实例，涵盖所有字段的赋值。

  • 使用 module_path 和 module_name 指明模块的位置与名称。
  • 通过 key 设置实例的唯一标识符。
  • 通过 config 定义运行时配置（如电源电压和温度）。
  • terminals 元组中：前两个列表分别代表输入和输出端口名，第三个列表为控制端口索引，示例中包含1个控制端接口（索引0）。
  • args 用于传入模块的构造参数。
  • env 设置运行环境变量。

该用法适用于在自动化电路生成系统中，统一描述一个模块实例的所有元信息，便于后续调度、连接与仿真处理。
