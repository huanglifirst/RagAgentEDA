GeometryMixin

功能介绍:

GeometryMixin 是一个用于电路设计中实例（Instance）几何操作的混合类，提供了一系列对实例及其边界进行位置调整的方法。该类主要用于在版图布局（layout）阶段精确控制实例的位置，支持基于坐标、中心点、边界等不同参考点的移动操作。

所有方法均针对 Instance 类型对象，通过修改其内部的 inst（实例本身）和 box（包围盒）的几何位置来实现位移。其中：

  • move_to：将实例整体移动到指定坐标点。
  • move_by：根据偏移量对实例进行平移。
  • move_pr_boundary_to：以物理边界（PR boundary）为参考，将实例移动到目标位置。
  • move_layer_boundary_to：基于某个图层的边界定位，调整实例位置，可选择是否启用回退机制。
  • move_center_to：将实例中心移动到指定坐标点。
  • move_cx_to：仅设置实例中心横坐标。

上述方法均返回自身对象（self），支持链式调用。

参数定义:

参数名类型含义默认值
xint横向目标坐标或偏移量。无
yint纵向目标坐标或偏移量。无
pr_boundarybool是否使用物理边界作为参考进行移动。False
layerOptional['Layer']指定的图层，用于获取图层边界。None
enable_move_to_fallbackbool当目标图层无边界时是否启用 move_to 回退机制。True
cxint中心点的横坐标（仅用于 move_center_to 和 move_cx_to）。无
cyint中心点的纵坐标（仅用于 move_center_to）。无

返回值

所有方法均返回 Union['NRect', 'Instance'] 类型，但在实际使用中，都返回当前实例对象（self），支持链式操作。

注意：由于 NRect 是包围盒类型，返回类型为联合类型是为了兼容某些上下文中的多态调用，但在此混合类中，所有方法最终返回的是 Instance 实例本身。

代码示例:

# 示例1: 使用 move_to 将实例移动到 (100, 200)
inst.move_to(100, 200)

# 示例2: 使用 move_by 对实例进行平移（偏移量为 50, -30）
inst.move_by(50, -30)

# 示例3: 将实例的物理边界移动到 (150, 250)
inst.move_pr_boundary_to(150, 250)

# 示例4: 将实例的某个图层边界移动到 (300, 400)，启用回退机制
inst.move_layer_boundary_to(layer=layer_a, x=300, y=400)

# 示例5: 将实例中心点移动到 (200, 300)
inst.move_center_to(200, 300)

# 示例6: 仅设置实例中心横坐标为 250
inst.move_cx_to(250)

说明：

  • move_to 适用于精确控制实例的左上角坐标。
  • move_by 适合连续移动或增量调整，当偏移量为零时直接返回自身。
  • move_pr_boundary_to 用于以物理边界作为基准进行定位，特别适用于对齐工艺层边缘。
  • move_layer_boundary_to 可按特定图层边界定位，若该图层未定义边界，则依据 enable_move_to_fallback 决定是否回退到 move_to。
  • move_center_to 通过计算宽高中心点实现移动，并返回当前实例。
  • move_cx_to 支持仅修改中心横坐标，适用于在保持纵坐标不变的情况下水平调整。

所有方法内部均使用 round_int 对坐标值进行整数舍入，确保数值符合版图精度要求。

rotate

功能介绍:

rotate 是一个实例方法（属于 Instance 类），用于将电路设计中的实例（Instance）进行旋转操作。该方法以逆时针方向为正向旋转，支持对实例整体进行 90 度的倍数旋转。若旋转角度为 360 的整数倍，则直接返回原实例，不做任何变换。

旋转过程中可指定旋转中心点；若未指定（即 center 参数为 None），则默认以实例包围盒（bounding box）的左下角坐标作为旋转中心。旋转后会自动更新实例的布局与位置信息，确保其拓扑结构正确。

该方法常用于布图设计（layout）中调整元件方向，例如在版图布局、路由或模块封装时，需要将某个基本单元按特定角度对齐。

参数定义:

参数名类型含义默认值
angleint旋转角度，必须是 90 的倍数（正负均可），单位为度。逆时针方向为正。无
centerTuple[int, int]旋转中心点坐标 (x, y)，若为 None 则以实例左下角为旋转中心。None

返回值

返回旋转操作完成后的该 Instance 实例本身，以便支持链式调用。

move_center_to

功能介绍:

move_center_to 是一个用于移动实例（Instance）位置的函数，其作用是将实例的中心坐标平移到指定的二维坐标点 (cx, cy)。该函数通过计算实例当前尺寸（宽 w 和高 h），将其左上角新坐标设为 (cx - w//2, cy - h//2)，然后调用 move_to 方法实现最终位置移动。
此函数适用于在版图设计中精确控制元件或子模块的位置布局，尤其当需要以中心点为基准进行对齐或定位时非常有用。

该 API 为装饰器 @layout 所修饰的函数，属于 TED 框架中的布局操作方法，通常用于实例级别的几何变换。

参数定义:

参数名类型含义默认值
cxint目标位置的横坐标（X轴），表示要将实例中心移动到的水平位置。无
cyint目标位置的纵坐标（Y轴），表示要将实例中心移动到的垂直位置。无

返回值

返回类型为 Union['NRect', 'Instance']，表示该函数执行后返回当前实例对象本身（即 self）。

  • 若当前实例是 NRect 类型（矩形区域），则返回 NRect 实例；
  • 否则，返回原始的 Instance 对象。
    这使方法支持链式调用，便于连续操作。

move_by

功能介绍:

move_by 是一个用于对实例（Instance）进行位移操作的函数。它将实例在二维平面内按照指定的横向（x方向）和纵向（y方向）偏移量移动，并同步更新实例内部的几何边界框（box）信息。该函数适用于所有继承自 Instance 类的对象，如模块实例、单元实例等。

该函数属于 Instance 类的实例方法，通过装饰器 @layout 标记为布局级操作函数，在设计流程中常用于调整元件位置或对齐布局结构。

执行移动后，实例自身保持不变（即返回自身），因此支持链式调用。当移动偏移量为 (0, 0) 时，直接返回原对象，不进行任何操作。

参数定义:

参数名类型含义默认值
xint横向（X轴）的位移量，单位为设计库中的物理长度单位（如nm）。正数表示向右移动，负数表示向左移动。无
yint纵向（Y轴）的位移量，单位同上。正数表示向上移动，负数表示向下移动。无

返回值

  • 返回类型：Union['NRect', 'Instance']
  • 含义：函数返回调用 move_by 的实例对象本身（即 self），便于链式调用，支持连续位移操作。

flipx

功能介绍:

flipx 是一个用于对电路实例（Instance）沿 X 轴进行翻转操作的方法。该方法会以实例的原点为中心，将实例在 X 轴方向上进行镜像变换，即水平翻转。此操作常用于布局布线中，调整元件的朝向或对称性，以满足版图设计的规则或匹配需求。

该方法属于 Instance 类的一个成员函数，执行后会修改当前实例的几何位置与方向信息，并返回自身以支持链式调用。

参数定义:

无参数。该方法不接受任何输入参数。

返回值

  • 类型：Instance
  • 含义：返回翻转操作后的实例对象本身，支持链式调用（如 .flipx().rotate(90)）。

correct_pos

功能介绍:

该函数用于校正实例（Instance）的位置，使其精确对齐到网格点上。通过调整实例的坐标位置，确保其位于设计规则所定义的格点上，避免因微小偏移导致的布局错误或制造问题。该操作适用于需要高精度布图的场景，如模拟集成电路设计中的全定制流程。

函数支持三种不同的取整方式：四舍五入（round）、向上取整（ceil）、向下取整（floor），用户可通过参数指定期望的对齐策略。

此函数是实例对象（Instance）的方法，修改当前实例的位置并返回自身以支持链式调用。

参数定义:

参数名类型含义默认值
funcstr指定取整方式，仅支持 "round"（四舍五入）、"ceil"（向上取整）和 "floor"（向下取整）。"round"

返回值

返回当前经过位置校正后的实例对象（Instance），便于链式调用。
