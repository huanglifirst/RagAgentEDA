InstanceMixin

功能介绍:

InstanceMixin 是一个混入类（mixin class），用于为模块类提供与实例（Instance）相关的操作能力。该类定义了一系列方法，用于管理模块中实例的插入、连接、重命名、延迟初始化等常见操作。通过继承此混入类，模块可以具备对子电路实例进行完整生命周期管理的能力。

  • insert_instance：将一个实例对象插入到模块内部，并可选择性覆盖已有实例。
  • initialize_delayed_instances：初始化所有延迟加载的实例，并返回成功初始化的实例列表。
  • record_instance：将实例记录到模块的设备映射表中，确保唯一命名和引用管理。
  • connect_instance 和 connect_instances：建立两个或多个实例之间的连接关系。
  • disconnect_instance 和 disconnect_instances：断开两个或多个实例之间的连接。
  • rename_instance：为指定实例重新命名，并自动更新内部映射表。
  • auto_inst_rename：根据当前作用域中局部变量的名称，自动为子电路实例重命名，主要用于代码生成环境下的名称冲突处理。

该类的设计旨在支持 TED 框架下模块化电路设计中的实例管理与连接逻辑。

参数定义:

参数名类型含义默认值
instInstance要插入或操作的电路实例对象。无
forcebool是否强制覆盖已存在的同名实例，若为 True 表示即使名称冲突也会替换；否则抛出异常。True
namestr实例在模块中登记的唯一名称。无
leftInstance要连接的第一个实例对象。无
rightInstance要连接的第二个实例对象。无
srcUnion[list, InstList]包含多个实例对象的列表或 InstList 对象，用于成批执行连接/断开操作。无
frameframePython 的栈帧对象，用于获取当前代码上下文的局部变量信息。None
frame_pathstr当前模块在调用层级中的路径标识，用于辅助命名冲突分析。''

返回值

  • initialize_delayed_instances：返回类型为 List[Instance]，包含所有成功初始化的实例对象。
  • rename_instance：无返回值（None）。
  • auto_inst_rename：无返回值（None）。

代码示例:

@module
def example_module(in_: Input, out: Output):
    # 实例定义
    inst1 = Instance("my_inst", module=example_submodule)
    inst2 = Instance("another_inst", module=other_submodule)

    # 插入实例
    insert_instance(inst1, force=True)

    # 连接两个实例
    connect_instance(inst1, inst2)

    # 批量连接
    connect_instances([
        inst1,
        inst2
    ])

    # 重命名实例
    rename_instance(inst1, "renamed_inst")

    # 断开连接
    disconnect_instance(inst1, inst2)

    # 释放资源后重命名
    inst1.deactivate()
    rename_instance(inst1, "final_name")

    # 延迟初始化实例
    delayed_instances = [delayed_inst1]
    initialized = initialize_delayed_instances()

    # 自动重命名（通常在代码生成或执行上下文中调用）
    auto_inst_rename()

说明：

  • insert_instance 一般用于将一个预先创建的 Instance 对象插入到模块中，若存在同名实例且 force=False，会触发异常。
  • initialize_delayed_instances 适用于那些尚未立即初始化的实例，可在后续流程中批量激活。
  • record_instance 在实例注册前检查是否已存在同名实体，并通过断言确保命名冲突问题不被忽略。
  • connect_instance 和 connect_instances 分别用于单个和多个实例间的连接。输入参数为两个 Instance 对象时，它们的底层 inst 字段会被传递给 RSM 基础模块进行实际连接。
  • rename_instance 支持命名冲突检查，特别是当启用了重复名称检测时（即 is_duplicate_name_check_enabled() 为 True），会抛出异常提示具体冲突路径，帮助开发者定位代码问题。
  • auto_inst_rename 按照当前帧中的局部变量名自动重命名子电路实例，避免因变量名重复导致的命名污染。该功能常用于自动化代码生成环境。
  • 所有方法均通过 self.rsmodule 调用底层资源管理模块的接口，保证操作一致性。

注意：实际使用中需确保 Instance 对象为有效的模块实例，并且其生命周期处于有效状态（即 alive() 返回 True）。

merge_rects

功能介绍:

该方法用于合并同一层（layer）上间距小于指定值（spacing）的矩形实例。合并规则如下：

  • 矩形宽度一致且 x1 坐标相同，可合并；
  • 矩形高度一致且 y1 坐标相同，可合并；
  • 一个矩形完全包含另一个，也可合并。

此方法适用于对布线层（如 M1）中密集排列的矩形进行优化合并，以减少冗余形状、简化布局。
该方法是 Module 类的实例方法，作用于当前模块中的所有矩形实例。

参数定义:

参数名类型含义默认值
layerLayer指定需要合并的层，仅对此层内的矩形进行处理。无
spacingOptional[int]矩形之间的最大允许间距。若为 None，则自动使用该层的最小间距值；小于该间距的矩形将被合并。None

返回值

  • 返回类型：InstList
  • 含义：合并后生成的新矩形实例列表，每个实例代表一个合并后的矩形区域。

代码示例:

以下两个测试用例展示了 merge_rects 方法在不同场景下的典型使用方式：

用法一：合并两个相邻的相同层矩形（间距为0）

@skip_unless(TED_TEST_LVL=99)

    @module
    def MergeRects():
        Rect(M1, 200, 100, connectTo=Vdd())
        Rect(M1, 200, 100, connectTo=Vdd())
        place()

    mod = MergeRects().module()
    instances = mod.get_instances()

说明：

  • 在模块中定义了两个大小相同（200×100）、位于 M1 层的矩形；
  • 通过调用 merge_rects(M1, spacing=0)，将间距为0的矩形合并为一个更大型的矩形；
  • 合并后输出的实例应是一个 RRect 类型（即合并后的规则矩形）；
  • assert instances[0].module().is_rrect() 验证了合并结果确实生成了一个矩形。

用法二：合并两个水平间距为1000的矩形

@testbench
def Top():

  • 使用 row 构造两个间距为1000的 TEXT 层矩形；
  • 通过 merge_rects(TEXT, spacing=1000) 对其进行合并操作；
  • 合并后返回的矩形区域 box 应与原始两个矩形构成的整体边界一致（即从 r1 左下角到 r2 右上角）；
  • 验证了 left_bottom_coord() 和 right_top_coord() 与预期一致，说明合并正确处理了空间关系。

instances

功能介绍:

instances 是模块（Module）类的一个成员方法，用于获取当前模块中所有的实例（Instance）对象集合。该方法可以返回模块内所有直接或间接创建的实例，支持通过可选的过滤函数对实例进行筛选。

此方法适用于需要遍历模块中所有子实例、分析电路结构或进行批量操作的场景。例如，在构建电路图、提取特征、检查布局或执行自动化测试时非常有用。

参数定义:

参数名类型含义默认值
funcOptional[Callable[['Instance'], bool]]可选的筛选函数，用于过滤符合条件的实例。该函数接收一个 Instance 对象作为输入，并返回 True 表示保留该实例，返回 False 则跳过。若未提供，则返回所有实例。None

返回值

  • 返回类型：List['Instance']
  • 含义：包含当前模块中所有满足条件的实例对象组成的列表。若未提供筛选函数，则返回全部实例；若提供了筛选函数，则仅返回通过该函数测试的实例。

功能介绍:

get_top_vias 是一个用于获取当前模块中顶层金属层上所有有效通孔（Via）的接口函数。该函数通过调用 self.rsmodule.get_top_vias() 获取当前模块中位于最高金属层上的通孔列表，主要用于在版图设计或布线后检查通孔生成情况。该接口常用于测试或自动化验证场景，帮助判断通孔是否成功生成、是否存在被跳过（skip_via）或被障碍物（V1_OBS）覆盖等情况。

  • 类型: list
  • 含义: 返回当前模块中顶层金属层上所有有效通孔对象的列表。每个元素代表一个通孔实例，若无有效通孔则返回空列表。

get_instances

功能介绍:

获取当前模块中所有的实例（Instance）集合。该方法用于查询某一模块下包含的所有子实例对象，可选地根据提供的过滤函数对实例进行筛选。

此方法为模块类的成员函数，通常在模块上下文中调用，例如通过 current_module() 获取当前作用域的模块实例后调用该方法。其主要用途包括：遍历模块内所有组件、基于特定条件筛选实例（如仅获取几何形状实例）、或进行后续操作（如连接、路由等）。

参数定义:

参数名类型含义默认值
funcOptional[Callable[['Instance'], bool]]用于筛选实例的布尔函数，输入为一个 Instance 实例，返回 True 表示保留该实例。若为 None，则不进行过滤，返回所有实例。None

返回值

  • 类型：List['Instance']
  • 含义：包含当前模块中所有满足条件的实例对象列表。若未提供筛选函数或筛选函数始终返回 True，则返回模块中所有已创建的实例。

flatten_instance

功能介绍:

flatten_instance 是一个用于展开电路实例（Instance）的函数，主要用于将嵌套或复杂的模块实例结构展平为一组独立的基础实例。该函数在电路设计流程中常用于生成净表（netlist）或进行后续处理前的实例规范化。

该函数属于类方法，其行为由所属类（如 Module 或相关上下文）的 rsmodule 提供支持。它接收一个原始实例对象 inst，并根据指定的目的（如 'netlist'）对其进行展平处理，并可选地重置实例的位置和变换属性。

功能核心包括：

  • 展开嵌套的模块实例结构（由 self.rsmodule.flatten_instance 实现）。
  • 可选择将所有实例移动至坐标原点 (0, 0)。
  • 可选择清除翻转和旋转状态，然后应用新的旋转角度。
  • 返回一组经过处理的新 Instance 对象。

此函数适用于需要标准化电路实例布局的场景，例如生成统一格式的 Netlist 或进行物理布局前处理。

参数定义:

参数名类型含义默认值
instInstance需要展平的原始实例对象。无
purposestr展平的目的，常见的有 'netlist'，表示用于生成净表。'netlist'
reset_positionbool若为 True，则将展平后的所有实例移动到坐标 (0, 0)。False
reset_transformbool若为 True，则清除实例原有的翻转（flip）与旋转状态。False
rotationint统一应用于所有展平后实例的旋转角度（以度为单位）。0

返回值

  • 返回类型：List['Instance']
  • 含义：一个包含经过展平、位置调整、变换重置和旋转处理后的 Instance 对象列表。

    # 创建一个模块和实例
    @module
    def example_module(A, B, C):
        return A >> B % C

    m = example_module()

    inst = Instance(m)

    # 测试1: 展平为 netlist 用途，不重置位置与变换
    result1 = flatten_instance(inst, purpose='netlist', reset_position=False, reset_transform=False, rotation=0)
    assert len(result1) == 1

    # 测试2: 重置位置到 (0, 0)，但不重置变换
    result2 = flatten_instance(inst, purpose='netlist', reset_position=True, reset_transform=False, rotation=0)
    assert all([i.pos == (0, 0) for i in result2])

    # 测试3: 重置变换，并应用90度旋转
    result3 = flatten_instance(inst, purpose='netlist', reset_position=False, reset_transform=True, rotation=90)
    assert all([i.get_rotation_angle() == 90 for i in result3])

    # 测试4: 同时重置位置与变换，并设置旋转角度
    result4 = flatten_instance(inst, purpose='netlist', reset_position=True, reset_transform=True, rotation=45)

    assert all([i.get_rotation_angle() == 45 for i in result4])

上述测试用例展示了 flatten_instance 函数在不同使用场景下的行为：

  • 用法一（不重置）：仅展平实例，保持原始位置和变换状态。
  • 用法二（重置位置）：展平后将所有实例移动至原点 (0, 0)。
  • 用法三（重置变换并旋转）：清除翻转与旋转状态，并统一设置新旋转角度（如90度）。
  • 用法四（全面重置）：同时重置位置、变换，并应用指定旋转角（如45度），适用于需要标准化布局的场景。

该函数在模块实例处理链中最常应用于生成最终可执行或分析的电路结构。

flatten

功能介绍:

flatten 是一个用于将电路模块（Module）进行“打平”处理的接口方法，主要作用是将嵌套的层次化电路结构展开为单一平面结构。该操作根据传入的 purpose 参数来决定打平的具体目标和行为，适用于不同设计流程中的需求场景。

该方法属于模块（Module）实例的一个成员方法，返回一个全新的已打平的模块对象（Module），用于后续的网表生成、仿真、版图导出等操作。其主要用途包括：

  • 生成用于 LVS（物理结构与电路网表一致性验证）的网表；
  • 构建用于仿真的网表；
  • 打平为电路级结构，支持在排布、布线等流程中使用；
  • 生成 GDS 或 LEF 等工艺文件所需的矩形化结构；
  • 支持特定功能如单元放置（cplace）、自动布局、MOS 管处理等。

flatten 的结果模块保留原始电路的层级逻辑，但移除了嵌套结构，成为一整个扁平化的模块。

参数定义:

参数名类型含义默认值
purposestr指定打平的用途，决定如何展开电路结构。可选值包括：'netlist', 'netlist_sim', 'circuit', 'routing', 'rect', 'lef_marco', 'euler', 'cplace', 'auto_place', 'cplace2', 'mos', 'layer', 'bottom_cell', 'unit'。不同值对应不同的输出目标：如网表、版图、布线数据等。'netlist'

返回值

  • 类型：Module
  • 含义：返回一个已经完成打平处理的模块实例，该模块不再包含任何嵌套层级结构，所有子电路均已展开为单一平面结构。

connect_instances

功能介绍:

该函数用于将多个已存在的实例（Instance）进行连接操作，通常在电路布局或路由阶段使用。它接收一个实例列表（或列表形式的实例集合），并将这些实例在当前模块中进行逻辑或物理上的连接管理。该方法内部通过调用 rsmodule.connect_instances 完成实际连接，适用于需要批量处理多个实例连接关系的场景。

此函数是 Module 类的一个成员方法，属于模块级操作接口，主要用于在电路设计流程中完成实例之间的连通性配置。

参数定义:

参数名类型含义默认值
srcUnion[list, 'InstList']一个包含待连接实例的列表，或已封装为 InstList 对象的实例集合。支持原始 Python 列表或 InstList 类型输入。无

返回值

无返回值（None）。
