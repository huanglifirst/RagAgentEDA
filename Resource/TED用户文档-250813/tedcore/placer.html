row_with_fixed_size

功能介绍:

row_with_fixed_size 是一个用于在固定宽度范围内排列多个电路实例（或实例列表）的布局函数。该函数将一组输入项目（items）按行排列，确保整体布局的总宽度不超过指定的 fixed_size。函数会自动计算并分配各个实例之间的间距，使得所有项目均匀分布在给定宽度内，并满足最小步进约束（5nm 对齐）。若未提供 fixed_size，则降级为标准 row 布局，不进行固定尺寸控制。

该函数适用于全定制模拟电路设计中的紧凑布局场景，例如在金属层或有明确物理尺寸限制的区域中排列晶体管、电阻等组件。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']一组待布局的电路实例或实例列表，这些实例将被排列成一行。无
alignOptional[int]指定行内元素对齐方式（如左对齐、居中等），若为 None 则按默认对齐策略处理。None
fixed_sizeOptional[int]行布局的总宽度上限（单位：nm）。此值必须不小于任意一个 item 的宽度，否则抛出断言错误。None
pr_boundarybool是否在最终输出中保留边界区域（例如用于工艺规则检查）。False
layerOptional['Layer']指定布局所使用的物理层（如金属层），若为 None 则不指定层。None

返回值

返回一个表示已排列完成的行布局结果的结构，通常兼容于 TED 框架中的后续连接与布局操作。该返回值可以作为子模块或进一步布图的输入，其内部包含所有原始 items 的正确位置和间距。

代码示例:

def test_row_with_fixed_size():
    # 示例 1: 固定尺寸布局
    inst1 = Instance(MOS, w=50, l=20)
    inst2 = Instance(MOS, w=60, l=20)
    inst3 = Instance(MOS, w=70, l=20)
    result1 = row_with_fixed_size(inst1, inst2, inst3, fixed_size=250, align=0)

    # 示例 2: 使用 InstList 类型
    inst_list = InstList([Instance(MOS, w=80, l=20), Instance(MOS, w=90, l=20)])
    result2 = row_with_fixed_size(inst_list, fixed_size=300)

    # 示例 3: 不设置 fixed_size，等价于 row
    result3 = row_with_fixed_size(inst1, inst2, align=3, pr_boundary=True)

说明：

  • 用法一（固定宽度布局）：当 fixed_size 被明确指定时，函数计算所有 items 总占宽，然后将剩余空间平均分配给各间距（除首尾外），并确保间距为 5nm 的整数倍。若任一项宽度超过 fixed_size，则抛出断言错误。
  • 用法二（实例列表传入）：支持直接传入 InstList 类型作为 items，函数会将其展开处理。
  • 用法三（无 fixed_size）：当 fixed_size=None 时，自动使用 row() 函数实现默认布局，此时不进行固定尺寸约束。

该 API 在实际使用中可用于快速生成符合版图物理限制的紧凑行排布结构，并保证间距合规与对齐一致。

row_with_different_spacing

功能介绍:

row_with_different_spacing 是一个用于在版图设计中按指定间距排列一组实例（Instance）的布局函数。它将多个输入实例（items）按照给定的间距（spacings）依次排成一行，并支持对齐方式、边界处理和层设置。

该函数为 @layout 装饰器修饰的布局函数，用于在版图生成过程中实现非均匀间距的水平排列。其核心逻辑是：

  1. 首先将第一个实例作为基点，计算所有实例整体的最小左上角坐标（x1, y1）；
  2. 依次将后续每个实例以指定间距与前一个实例进行 row() 排列；
  3. 每次排列后调整位置至当前最小左上角；
  4. 最终将整个组合移动到原点 (0, 0)，并返回最终的实例。

该函数适用于需要精确控制各单元之间不等间距排列的设计场景，如多级缓冲器、非对称布线或特殊物理布局结构。

注意：items 中元素的数量必须比 spacings 多一个（即：len(items) == len(spacings) + 1），因为间距是位于两个相邻实例之间，因此共有 n-1 个间距对应 n 个实例。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']待排列的多个实例列表，每个元素需为一个 Instance 或 InstList。这些实例将按照顺序水平排列。无
spacingsList[int]每两个相邻实例之间的间距值列表，长度应为 len(items) - 1。单位为设计规则中的最小刻度（如 nm）。无
alignOptional[int]对齐方式，用于指定排列时的对齐基准。可选值包括：TOP, BOTTOM, CENTER, LEFT, RIGHT 等。默认为 None，表示不进行强制对齐。None
pr_boundarybool是否输出边界（boundary）信息。若为 True，则在生成的实例中包含其边界区域。False
layerOptional['Layer']指定布局所使用的工艺层，用于标注或定义层信息。若为 None，则使用默认层。None

返回值

返回一个 Instance 或 InstList 类型对象，表示所有输入实例按照指定间距与对齐方式组合后形成的总体实例。该结果实例可直接用于后续布局操作（如 place()）。

代码示例:

# /home/zuochang/ted/tedcore/kernel/tests/test_placer.py:227
from pyted import *

from tedcore.kernel import get_circuit, Rect, row_with_different_spacing, place

def test_row_with_different_spacing():
    if get_circuit().get_pdk().name != 't65':
        return

    @testbench
    def Top():
        items = [Rect(M1, i * 100, i * 100) for i in range(1, 6)]
        spacings = [i for i in range(100, 500, 100)]
        row_with_different_spacing(*items, spacings=spacings, align=TOP)
        place()

使用说明：

  • items 为五个 Rect 实例，每个位于不同位置 (i * 100, i * 100)，分别构成一组可排布的矩形结构。
  • spacings = [100, 200, 300, 400] 表示四个相邻实例之间依次设置 100、200、300、400 的间距（单位：nm）。
  • align=TOP 指定所有实例在垂直方向上以顶部对齐方式排列，确保其 Y 坐标统一为最高值。
  • 调用 row_with_different_spacing 将五个矩形按非等距方式排成一行。
  • 最后调用 place() 实际执行版图放置。

此例展示了在 t65 工艺库下，如何使用 row_with_different_spacing 实现不等间距排列功能，适用于需要精确控制单元之间空间关系的版图设计场景。

row_by_target

功能介绍:

row_by_target 是一个用于电路布局的排列函数，主要用于将一个或多个实例（Instance 或 InstList）按指定目标对象的位置和对齐方式，在水平方向上进行排布。该函数以某个目标实例（target_inst）为基准，将其他实例放置在目标实例的左侧或右侧，并根据指定的垂直对齐方式（底部、中部、顶部）调整其 Y 坐标位置。

此函数支持自动对齐、间距设置、层控制以及网格对齐功能。当 reverse=True 时，传入的实例列表将被反转后再排布；若未提供间距或对齐参数，则使用默认值。最终通过 move_layer_boundary_to 将实例移动到目标位置并贴合指定层。

该 API 是一个顶层布局函数，适用于全定制 IC 设计中基于参考对象进行相对定位的场景。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']要被排布的一个或多个实例（可以是单个实例或实例列表）。无
target_instUnion['Instance', 'InstList']作为布局参考的目标实例，所有 items 将相对于该实例进行位置计算。无
target_posUnion[str, int]指定 items 相对于 target_inst 的放置方向：
- 'LEFT' 或 'HEAD'：位于目标左侧；
- 'RIGHT' 或 'TAIL'：位于目标右侧。		LEFT
spacingOptional[int]实例之间的水平间距（单位：长度）。若未指定则默认为 0。None
alignOptional[int]垂直对齐方式，用于控制 items 在 Y 方向相对于 target_inst 的位置：
- BOTTOM：底部对齐；
- MIDDLE：中心对齐；
- TOP：顶部对齐。		None
reversebool是否反转 items 列表再进行排布（即从最后一个开始排列）。False

row

功能介绍:

row 是一个用于在电路布局中将多个实例（Instance）按照水平方向排列成一行的布局函数。该函数能够根据指定的间距、对齐方式和方向，自动计算每个实例的位置，完成线性排布。它支持多种布局选项，包括不同间距、反向排列、固定总宽度约束、边界标记、图层控制等，适用于模拟电路设计中的模块化布局与对齐需求。

此函数以 @layout 装饰器标记，属于底层布局接口，通常在模块或测试平台中用于实现整齐的物理布线结构。当输入为多个实例时，这些实例将被组织成一个 InstList 类型的集合，并设置其排列类型为 'row'。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']待排列的一个或多个实例或实例列表。支持单个实例、列表或混合传入。无
spacingUnion[int, List[int]]实例之间的间距。若为整数，表示所有相邻实例间均采用相同间距；若为列表，则每个元素指定对应相邻实例间的间距。0
alignOptional[int]对齐方式：可选值包括 TOP、MIDDLE 或 BOTTOM（CENTER 也会被自动转换为 MIDDLE）。控制行内所有实例在垂直方向上的对齐位置。None
reversebool是否将输入的实例顺序反向排列，默认为 False，即保持原始顺序；设为 True 则逆序排列。False
as_modulebool若设为 True，则返回结果被视为一个模块（Module），可用于后续模块嵌套操作。False
fixed_sizeOptional[int]指定生成的排布整体宽度上限。若设定此值，则自动计算各实例间间距以满足总宽度限制，忽略 spacing 参数。None
is_confinedbool是否将该排列视为受限区域（通常用于后续布线或检查约束）。False
check_placebool是否在排布前校验各实例的放置状态。True
target_instUnion['Instance', 'InstList']指定一个参考实例，用于确定整体布局的位置偏移（即移动整个行使其与该实例对齐）。None
pr_boundarybool是否在布局后添加边界标记（PR Boundary）。若为 True，则不能同时指定 layer。False
layerOptional['Layer']指定布局所在的图层（Layer），用于绘制边界或定义工艺层信息，仅当 pr_boundary=False 时可用。None
forcebool是否强制跳过最终对齐修正步骤。若为 False，会调用 align_fin 进行自动微调以优化布局。False

返回值

返回一个 InstList 类型的对象，该对象封装了排布后的所有实例，并具有如下特性：

  • 类型标记为 'row'，表明其为水平排列结构。
  • 可作为模块嵌套、连接或后续路由操作的基础。

代码示例说明（基于提供的测试用例）

以下是一些典型用法：

layc_by_target

功能介绍:

layc_by_target 是一个用于电路布局的函数，主要用于将一个或多个实例（Instance 或 InstList）在版图上进行居中对齐布局，使其相对于目标实例（target_inst）在水平和垂直方向上均保持中心对齐。该函数基于 lay_by_target 实现，内部调用并传入了默认的对齐方式（水平居中 CENTER 和垂直居中 MIDDLE），适用于需要精确相对定位的场景。

此函数是 TED 框架中用于自动布图的辅助工具，能够简化多个电路单元间的对齐操作，提高设计效率。它返回原输入实例或实例列表，位置已经被更新以满足指定的对齐要求。

参数名类型含义默认值

target_instUnion[Instance, InstList]作为对齐基准的目标实例。所有 items 将相对于该实例进行中心对齐（水平和垂直）。无
layerOptional[Layer]指定将布局操作作用于的版图层。若为 None，则不指定特定层，按默认逻辑处理。None

返回值

返回类型为 Union[Instance, InstList]，即与输入的 *items 类型相同：如果传入的是单个实例，则返回单个实例；如果传入的是实例列表，则返回一个实例列表，这些实例的位置已根据目标实例完成中心对齐。

lay_by_target

功能介绍:

lay_by_target 是一个用于电路布局的函数型API，其主要功能是将一个或多个指定的实例（items）相对于某个目标实例（target_inst）进行对齐布局。该函数支持在指定层上按照水平和垂直方向的对齐方式（左对齐、居中、右对齐、上对齐、居中对齐、下对齐），自动计算目标位置，并将待布局对象移动到对应位置。

函数的核心行为包括：

  • 若传入多个实例，会先使用 lay 函数进行合并布局；
  • 然后根据 x_align 和 y_align 参数指定的对齐规则，将其放置于以 target_inst 为参考的指定位置；
  • 支持在启用网格强制（force_on_grid）时自动将位置对齐到工艺网格；
  • 最终通过 move_layer_boundary_to 方法完成实际的移动操作。

此API适用于需要精确对齐器件、金属层或其他结构于目标对象上的场景，例如连接线与晶体管源漏极的对齐。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']待布局的一个或多个实例。若传入多个实例，则先进行合并布局。无
target_instUnion['Instance', 'InstList']参考对齐的目标实例，所有对齐操作均以该实例的边界框（box）为基准。无
x_alignOptional[int]水平方向对齐方式：LEFT、CENTER、RIGHT 或 NOT_MOVE。表示待布局对象相对目标实例在X轴上的对齐方式。LEFT
y_alignOptional[int]垂直方向对齐方式：BOTTOM、MIDDLE、TOP 或 NOT_MOVE。表示待布局对象相对目标实例在Y轴上的对齐方式。BOTTOM
layerOptional['Layer']指定操作所在的图层。若未提供，则使用默认图层。如果指定图层，将仅更新该图层的边界位置。None

返回值

返回值类型为 Union['Instance', 'InstList']，即函数最终处理后的实例对象。当输入只有一个 items 时，返回该单个实例；当输入多个实例时，返回合并后布局的结果实例。

lay

功能介绍:

lay 是一个用于对电路实例集合进行重叠布局的函数型API，主要用于根据指定的对齐方式将多个 Instance 或者 InstList 实例叠加排列（如并排、居中对齐等），生成一个新的组合实例。该函数支持横向和纵向的对齐控制，并可选择是否将其作为模块返回、是否移动到原点、是否强制调整位置等。

此API的核心用途是在电路设计中实现对多个图形或器件单元的自动布局，常用于多层金属布线、晶体管阵列排列、测试结构构建等场景。其行为受 x_align 和 y_align 参数控制，可实现左对齐、居中对齐、右对齐（横向），以及顶对齐、居中对齐、底对齐（纵向）。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']要进行布局合并的实例集合，可以是单个 Instance 或 InstList 对象。无
x_alignOptional[int]横向对齐方式，可选值为 LEFT（左对齐）、CENTER（居中对齐）、RIGHT（右对齐）或 MIDDLE（等价于 CENTER）。None
y_alignOptional[int]纵向对齐方式，可选值为 TOP（顶对齐）、MIDDLE（居中对齐）、BOTTOM（底对齐）或 CENTER（等价于 MIDDLE）。None
as_modulebool是否将返回的实例视为一个模块（Module），若为 True，则生成的对象具有模块属性。False
to_originbool是否将布局后的结果整体移动到坐标原点 (0, 0)。True
is_confinedbool是否限制该布局在某一边界内（用于后续检查或生成受限区域）。False
check_placebool在合并前是否验证每个实例已正确放置，防止重复放置错误。True
pr_boundarybool是否在输出时打印边界的几何信息，常用于调试布局范围。False
layerOptional['Layer']指定用于边界绘制的层（仅当 pr_boundary=True 有效），不可与 pr_boundary=True 同时设置非 None 值。None
forcebool是否强制跳过最终的对齐微调步骤，直接返回布局结果。False

返回值

返回一个 Union['Instance', 'InstList'] 类型的对象，即根据输入项合成后的新实例或实例列表。如果 as_module=True，则返回的实例会以模块形式存在；否则为普通实例集合（InstList）。

get_cplace_insts

功能介绍:

get_cplace_insts 是一个用于电路布局设计的函数，其主要功能是从给定的实例列表中提取出可以进行“cplace”（即紧凑放置）操作的实例列表。该函数会处理两种类型的输入：单个 Instance 实例或 InstList 列表。对于每个输入实例，函数根据其类型和属性判断是否满足“可放置”条件，并据此生成新的 InstList 列表。

函数内部会对输入的实例进行如下处理：

  • 若为 Instance 类型且是端口矩形或基础几何图形，直接将其封装为 InstList；
  • 若为普通 Instance 但不满足直接放置条件，则尝试展开其层级结构（通过 flatten_instance）并递归处理；
  • 所有被处理过的 Instance 将被标记为“已销毁”（由 inst.kill() 实现），防止后续误用；
  • 最终返回的所有 InstList 都会自动添加标签以便调试或标识。

该函数常用于电路布局前的预处理阶段，用于筛选出可参与布放优化的元件集合。

参数定义:

参数名类型含义默认值
instsList[Union['Instance', 'InstList']]输入的实例列表，包含单个 Instance 实例或 InstList 对象。无

返回值

  • 返回类型：List['InstList']
  • 含义：一个封装了所有可进行“cplace”操作的实例的 InstList 列表。每个 InstList 包含至少一个 Instance，且均已添加标签（add_label()），可用于后续布局或调试。

colr_by_target

功能介绍:

colr_by_target 是一个用于在电路布局中实现自定义排列的函数，专门用于将一组实例（或单个实例）按照指定目标实例的位置进行从右向左的垂直排列（即“右侧对齐”布局）。该函数结合了 col_by_target 的功能，并通过设置 reverse=True 实现从右到左的排列顺序。

该函数主要应用于全定制集成电路设计中的版图布放（placement）场景，能够根据一个目标实例（target_inst）的位置，将其他指定实例以特定方向（如从尾部开始向右对齐）进行对齐和间距控制，适用于多模块或元件在水平方向上的对称或规则布局。

说明：该函数是 col_by_target 的封装，其核心逻辑为：以目标实例的某个位置点作为对齐基准（如顶点、尾部等），将传入的其它实例按照逆序排列方式从右向左堆叠，实现类似“右侧对齐”的布局效果。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']待排列的一组实例或单个实例，将这些实例按目标对齐方式放置。无
target_instUnion['Instance', 'InstList']作为布局对齐基准的目标实例，所有其他实例的排列将围绕它进行。无
target_posUnion[str, int]定义目标实例上的参考位置点，可取值为字符串 'HEAD', 'TAIL', 或整数索引（表示图层中某条边的位置）。默认为 'TOP'，即使用顶部边界作为对齐基准。'TOP'
spacingOptional[int]实例之间的间距（单位：纳米），若未指定则不强制间距。None
alignOptional[int]对齐方式，如 LEFT, RIGHT, CENTER 等，控制排列时的对齐轴线。例如 RIGHT 表示所有实例右边缘对齐。None
layerOptional['Layer']指定布局所处的物理层（如 M1、M2），用于确保布线正确性。None

返回值

返回类型为 Union['Instance', 'InstList']，即：若输入仅包含一个实例，则返回单个 Instance；若输入为多个实例，则返回一个 InstList（实例列表）。返回值可用于后续连接或嵌套布局。

col_with_fixed_size

功能介绍:

col_with_fixed_size 是一个用于在电路布局中将一组实例（Instance 或 InstList）垂直排列成一列的布局函数。该函数的核心功能是：在保证所有子项高度总和不超过指定 fixed_size 的前提下，自动计算并分配各实例之间的间距，使得整体列的高度恰好不超过 fixed_size，从而实现紧凑且规范的排列。

该函数特别适用于需要对多个电路模块进行等高或限高布局的场景，例如构建模块化的功能单元、信号链路或电源树结构。如果未指定 fixed_size，则退化为标准的 col 布局，不进行高度限制。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']一组需要垂直排列的实例或实例列表。每个 item 必须具有 .box.h 属性，表示其高度（以纳米为单位）。无
alignOptional[int]排列时的对齐方式（如左对齐、右对齐等），可使用整数值指定对齐基准。若不指定，则默认居中对齐。None

pr_boundarybool是否在布局结果中添加边界标记（用于可视化或后端处理）。默认为 False。False

返回值

    inst1 = Instance('M1', box=Box(h=50, w=30))
    inst2 = Instance('M2', box=Box(h=60, w=40))

    total_height = sum(item.box.h for item in [inst1, inst2, inst3])

  2. 忽略高度限制：当 fixed_size=None 时，行为退化为标准 col，不进行额外约束。

功能介绍:

col_with_different_spacing 是一个用于在版图布局中将多个实例垂直排列的函数，支持为每两个相邻实例之间设置不同的间距。该函数适用于需要精细控制多个元件垂直分布间距的设计场景，例如多级晶体管堆叠、模块化单元排列等。函数会自动计算所有元件位置的最小边界，并将整体布局移至原点 (0, 0)，便于后续与其他模块集成。

col_by_target

功能介绍:

col_by_target 是一个用于在版图布局中将一组元件（或实例）按照指定目标位置进行垂直对齐的函数。该函数能够根据目标实例的位置和方向，自动计算并设置当前元件集合的放置坐标，实现与目标元件在指定方向上的对齐排列。

此函数常用于电路设计中的模块布局，支持多种对齐方式（左对齐、居中对齐、右对齐），可指定间距、层别，并具备强制对齐网格的功能。若输入参数为单个实例，则直接使用该实例；否则将多个实例垂直堆叠后进行位置调整。最终返回已放置的实例或实例列表，其位置由目标实例和用户设定的对齐方式共同决定。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']要放置的一组元件（实例）或实例列表，将被垂直排列并相对于目标实例定位。无
target_instUnion['Instance', 'InstList']作为对齐基准的目标实例，所有布局操作都将围绕此实例进行。无
target_posUnion[str, int]目标位置，决定当前元件集合应位于目标实例的上方还是下方。支持取值为 'HEAD' / TOP 表示顶部上方，'TAIL' / BOTTOM 表示底部下方。TOP
spacingOptional[int]当前元件与目标实例之间的垂直间距（单位：最小栅格尺寸的倍数）。0
alignOptional[int]水平对齐方式，控制当前元件集合在水平方向上的对齐位置。支持取值为 LEFT、CENTER、RIGHT。LEFT
reversebool是否反转输入实例列表的顺序再进行排列（适用于倒序布局）。False
layerOptional['Layer']指定操作所用的版图层，若不指定则使用默认层。None

返回值

返回类型为 Union['Instance', 'InstList']，即根据输入 items 的数量返回单个实例或实例列表，并且该对象已依据目标实例的位置和参数完成坐标移动。

col

功能介绍:

col 是一个用于在电路布局中将一组实例（Instance）按列方向排列的布局函数。它能够将多个实例垂直堆叠，支持自定义间距、对齐方式、反转顺序等功能，适用于模块化电路设计中的元件组织与排布。

该函数会返回一个 InstList 类型的对象，代表排列后的整体集合，并支持后续的放置（place）和路由（route）操作。其主要用途包括：将多个晶体管、电容、逻辑单元等按列布局，实现紧凑的版图设计，并可用于构建带保护环（guardring）、端口对齐等复杂结构。

该函数为装饰器 @layout 修饰的函数，属于 TED 框架中的核心布局工具之一。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']要排列的实例或实例列表，可以是单个实例或多个实例集合。无
spacingUnion[int, List[int]]实例之间的间距设置。若为整数，则所有相邻实例间保持相同间距；若为列表，则每个间隔使用对应值。0
alignOptional[int]列内元素的水平对齐方式，可选 LEFT、CENTER、RIGHT（或 MIDDLE）。默认为 LEFT。None

array_place

功能介绍:

array_place 是一个用于电路布局的函数，采用阵列方式对传入的实例进行排列布线。该函数会根据传入的实例数量和指定的方向（水平或垂直），自动计算出最优的行列布局结构，并将这些实例按行或列进行分组排列，最终形成一个完整的布局模块。

该函数适用于全定制电路设计中需要大规模重复单元阵列排列的场景。例如，在构建标准单元库、存储器阵列或者并行处理模块时，能够自动实现规则化布图。

函数内部通过 math.ceil(math.sqrt(len(items))) 来确定行列尺寸，并以指定间距对各实例进行间隔放置。最终输出结果是一个由多个 row 或 col 布局组成的复合模块。

参数定义:

参数名类型含义默认值
*itemsUnion['Instance', 'InstList']要进行阵列布局的实例列表，可以是单个实例或多个实例组成的可迭代对象。无
spacingint实例之间的间距（单位为栅格长度）。0
directionUnion[VERTICAL, HORIZONTAL]布局方向：HORIZONTAL 表示按行优先排布，VERTICAL 表示按列优先排布。HORIZONTAL

返回值

返回一个由 col 包裹的布局模块，内部通过多个 row 子模块进行分组排列，最终形成阵列布图。

示例说明：

  • 创建了 6 个 NMOS 实例组成的列表 nmos。
  • 调用 array_place(*nmos, spacing=1000, direction=HORIZONTAL)：
    • 因为传入了 6 个实例，且方向为 HORIZONTAL，计算得：
      • 列数 col_size = ceil(sqrt(6)) = ceil(2.45) = 3
      • 行数 row_size = ceil(6 / 3) = 2
    • 最终布局为 2 行 × 3 列，每行放置 3 个 NMOS，行间间距为 1000。
  • 调用 .to_gds() 将最终布图导出为 GDS 文件。

该示例展示了 array_place 在实际设计中用于构建标准单元阵列的基本用法。当需要将多个相同或相似的电路模块按规则阵列排列时，可直接使用此函数，无需手动编写复杂的行/列组合逻辑。
