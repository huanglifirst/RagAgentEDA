DebugMixin

功能介绍:

DebugMixin 是一个用于调试 TED（Tsinghua Electronic Design）框架中实例（Instance）的混合类（Mixin），提供一系列辅助方法来查看和输出实例的连接关系、调用路径、生成代码表示及调试信息。该类不独立使用，通常被其他实例类继承以增强其调试能力。

通过该类提供的方法，用户可以：

  • 打印布线结果（全局或本地）；
  • 获取实例的调用路径和堆栈帧信息；
  • 生成清晰的调试字符串，展示输入、模块名、参数、控制信号及输出的完整调用链；
  • 跟踪实例创建的代码路径；
  • 将实例转换为可读代码表示（虽当前返回空字符串）。

此混合类主要面向开发人员在设计电路时进行快速诊断与调试。

参数定义:

参数名类型含义默认值
selfInstance当前的实例对象，用于访问其端口、模块信息及控制信号等。无

注：所有方法均以 self 作为隐式参数，属于实例方法。

返回值

方法名返回值类型含义
print_route_resultAny调用模块的 print_route_result(path) 方法，输出全局布线结果。具体返回值由模块决定。
print_local_route_resultstr返回本地图布线结果的字符串表示。
showInstance返回当前实例本身，用于链式调用。
debugstr返回一个格式化的调试字符串，包含输入、模块调用（含控制信号与参数）、输出信息的组合。
get_tracestr返回该实例生成的代码路径（即 _code 属性内容），若无则返回空字符串。
get_call_pathstr返回完整的调用路径，按行分隔的帧信息字符串（文件名:行号）。
to_codestr将实例转换为代码表示形式，当前实现返回空字符串。
trace_call_path2None无返回值，仅用于追踪调用栈（将当前调用帧信息记录到 self.frame_msg 列表中）。

代码示例:

# 测试用例: 验证 debug 方法的输出格式
def test_debug_output():
    module = Module("TestModule")
    inst = Instance(module)
    inst.set_input([Input("in1"), Input("in2")])
    inst.set_output([Output("out1")])
    inst.set_ctrl([Ctrl("en")])
    inst.set_all_func_parameters({
        "param1": 1,
        "param2": "hello"
    })
    # 验证 debug 输出
    assert inst.debug() == "in1,in2 >> TestModule(en, param1=1, param2='hello') >> out1"

# 测试用例: 验证 get_call_path 获取调用路径
def test_get_call_path():
    inst = Instance(Module("Dummy"))
    inst._code = "module = Module('test')"
    # 模拟 trace_call_path2 调用
    inst.frame_msg = []  # 初始化帧消息列表
    inst.trace_call_path2()
    assert len(inst.frame_msg) > 0

# 测试用例: 验证打印布线结果
def test_print_route_result():
    module = Module("Test")
    inst = Instance(module)
    result = inst.print_route_result(path="/debug")
    # 此处验证方法是否可被调用，不验证具体输出内容
    assert isinstance(result, (type(None), str))

# 测试用例: 验证 get_trace 返回代码路径
def test_get_trace():
    inst = Instance(Module("test"))
    inst._code = "x = 1"
    assert inst.get_trace() == "x = 1"

# 测试用例: 验证 to_code 返回空字符串
def test_to_code():
    inst = Instance(Module("test"))
    assert inst.to_code() == ""

说明：

以上测试用例展示了 DebugMixin 的各项功能：

  • debug 方法将输入、模块调用（含控制和参数）、输出整合为一行可读的链式表达；
  • get_call_path 通过追踪调用栈生成调试路径信息；
  • print_route_result 和 print_local_route_result 可用于打印布线状态，适用于复杂连线场景；
  • get_trace 提供实例创建时的源码路径；
  • to_code 为预留扩展接口，当前未实现具体逻辑；
  • trace_call_path2 在测试环境中被主动调用以模拟调试追踪行为。

这些方法协同工作，为用户提供完整的实例级调试支持，特别适用于在电路模块组装过程中排查连接关系和逻辑问题。

print_route_result

功能介绍:

print_route_result 是一个实例方法，用于输出布线结果的详细信息。该方法属于 Instance 类的成员函数，通过调用其所属模块（Module）的 print_route_result 方法来实现布线结果的打印功能。主要用于在电路设计流程中调试和查看布线后的连接关系、路径信息等结果，帮助开发者确认布线是否符合预期。

参数定义:

参数名类型含义默认值
pathstr 或 None指定输出的布线路径或文件路径。若不提供，则打印当前模块的默认布线结果。None

返回值

无返回值（None）。

代码示例:

# /home/zuochang/ted/projects/empyrean/tests/test_channel_route.py:6
from pyted import *

from tedcore.kernel import place, moduledef, skip_unless
from ted_utils import get_ted_path
from algo.router.route.__init__ import route

@skip
@skip_unless(TED_PDK=S180BCDEPI)
def test01():

    @moduledef(import_file=f'{get_ted_path()}/projects/empyrean/tests/layout/SDADC_CORE0327/layout/SDADC_CORE0x21a7471a')
    def DUT():
        place()
        route_config = f'{get_ted_path()}/projects/empyrean/tests/layout/SDADC_CORE0327/setting.json'
        route(config_file=route_config)
    dut = DUT()
    dut.to_gds()
    dut.print_route_result()

使用说明：

  • 该示例展示了在完成布局（place）和布线（route）流程后，调用 dut.print_route_result() 打印当前模块的布线结果。
  • 布线配置通过 route(config_file=...) 加载外部设置文件，确保布线逻辑按预设参数执行。
  • dut.to_gds() 将设计导出为 GDSII 格式文件，为后续流程准备。
  • dut.print_route_result() 在此调用中未传入 path 参数，表示打印当前实例的默认布线结果，适用于快速查看所有连接路径状态。

典型使用场景：

  • 用于验证已执行布线操作后连线是否完整、无冲突或未连接；
  • 可在自动化测试流程中用于输出日志信息，作为成功布线的判定依据之一。

debug

功能介绍:

debug 是一个实例（Instance）类的成员方法，用于生成该实例在电路中的连接信息的调试字符串。该方法以可读格式输出实例的输入端口、控制端口、参数配置以及所属模块的调用方式，并通过 >> 符号分隔各部分信息，便于在仿真或设计过程中快速查看实例的接线状态和配置参数。

该方法特别适用于在复杂电路中排查连接错误或验证实例化是否正确。其输出结果为一个字符串，包含输入端口列表、模块调用信息（含控制端与参数）、以及输出端口列表的组合，形式上清晰表达电路中的信号流向和模块实例化细节。

参数定义:

参数名类型含义默认值
selfInstance当前实例对象，代表一个模块的具体实例。无

注：该方法不接受外部参数，仅通过 self 访问所在实例的内部状态。

返回值

  • 类型：str
  • 含义：返回一个格式化的调试字符串，包含实例的输入端口、控制端口与参数、模块调用信息、输出端口等信息，以 >> 分隔各部分。其中：
    • 输入端口以逗号分隔，若超过1个则包裹在圆括号中。
    • 输出端口同样处理。
    • 控制端口和参数字符串如不为空，将拼接为模块调用的参数列表。
    • 最终结构：输入部分 >> 模块调用部分 >> 输出部分。

代码示例:

def test_debug_basic():
    # 创建一个空实例（假设已存在模块）
    inst = Instance(module=Module(name="test_module"))
    
    # 设置输入端口
    inp_node = Node(name="in1", port_type="Input")
    inst.add_input(inp_node)
    
    # 设置输出端口
    oup_node = Node(name="out1", port_type="Output")
    inst.add_output(oup_node)
    
    # 测试 debug 输出
    result = inst.debug()
    assert result == "in1 >> test_module() >> out1"
def test_debug_with_ctrl_and_params():
    # 模块定义包含参数和控制端
    module = Module(name="test_ctrl_mod")
    module.add_param("scale", 2.0)
    
    inst = Instance(module=module)
    ctrl_node = Node(name="enable", port_type="Ctrl")
    inp_node = Node(name="data_in", port_type="Input")
    oup_node = Node(name="data_out", port_type="Output")
    
    inst.add_ctrl(ctrl_node)
    inst.add_input(inp_node)
    inst.add_output(oup_node)
    
    result = inst.debug()
    assert result == "data_in >> test_ctrl_mod(scale=2.0) >> data_out"
def test_debug_with_multiple_ports():
    module = Module(name="multi_port_mod")
    inst = Instance(module=module)
    
    # 多个输入端口
    inp1 = Node(name="in_a", port_type="Input")
    inp2 = Node(name="in_b", port_type="Input")
    inst.add_input(inp1)
    inst.add_input(inp2)
    
    # 多个输出端口
    oup1 = Node(name="out_x", port_type="Output")
    oup2 = Node(name="out_y", port_type="Output")
    inst.add_output(oup1)
    inst.add_output(oup2)
    
    result = inst.debug()
    assert result == "(in_a,in_b) >> multi_port_mod() >> (out_x,out_y)"
def test_debug_with_ctrl_and_params_and_multiple_ports():
    module = Module(name="complex_mod")
    module.add_param("vth", 0.7)
    
    inst = Instance(module=module)
    
    # 多个输入和控制端口
    inst.add_input(Node(name="din1", port_type="Input"))
    inst.add_input(Node(name="din2", port_type="Input"))
