InstList

功能介绍:

InstList 是一个用于管理和操作电路设计中多个实例（Instance）集合的类。它封装了对多个芯片实例进行批量处理的能力，支持常见的操作如添加、移动、旋转、翻转、重命名、获取边界框（box）、生成实例化模块（to_instance）等。它还支持从不同数据源初始化，包括原始实例列表、rsted.InstList 实例，或其它包含实例的对象（比如模块的实例列表）。
该类内部维护一个 rs_instlist 对象（底层 C++ 接口），以高效管理实际的物理布局数据，并通过 _instances 列表保持高层 Python 层的实例引用。此外，它支持设置特定的顺序索引（schematic_sequence），用于控制在原理图中显示或处理的顺序。

该类主要用于全定制集成电路设计流程中的布局阶段，常见于模块化封装、自动布线前准备、电路重构等场景。

参数定义:

参数名类型含义默认值
itemsOptional[Union[List, Tuple, Set, rsted.InstList]]初始的实例列表或集合，可以是 Python 的 list/tuple/set，也可以是底层 rsted.InstList 实例。如果传入的是 rsted.InstList，则直接使用该对象进行初始化。None
nameOptional[str]给此 InstList 设置的名称，可用于标识或调试用途。None
schematic_sequenceOptional[Union[Tuple, List]]用于指定在原理图中显示时元件的顺序索引。若未提供，则默认按 0 到 len-1 排序；若已指定，将使用该序列进行重命名和迭代顺序控制。None
childrenOptional[Union['Instance', 'InstList']]表示当前列表作为子项嵌套于某个父实例或集合中，用于层级结构管理。None
rs_instlistOptional[rsted.InstList]底层 C++ 层面的实例列表对象，若未提供，则自动创建一个新的 rsted.InstList 实例。None

返回值

无直接返回值，构造函数和 setup 方法均为初始化操作，不返回任何值。

to_instance

功能介绍:

to_instance 是实例集合（InstList）类的一个方法，用于将当前实例集合整体转换为一个独立的 Instance 实例对象。该方法主要用于在电路设计中，将多个子元件或图层结构合并为一个可复用的单元模块。

当调用此方法时，系统会根据集合中的元素创建一个统一的实例。如果参数 kill_instance 设置为 True，则会先销毁原集合中的所有实例，并将合并后的单一实例作为新的集合内容；若设置为 False，则保留原始实例（但需在后续调用 place() 之前手动调用 kill() 以避免命名冲突）。

该方法适用于需要对多个元件进行聚合封装的场景，如在构建标准单元（Standard Cell）、模块化电路或高阶子模块时。

参数定义:

参数名类型含义默认值
kill_instancebool是否销毁原集合中的所有实例。若为 True，会释放原有实例并创建新实例；若为 False，则保留原始实例，需用户主动调用 kill() 管理资源。True

返回值说明

smartbus 方法的返回值为 InstList 类型的对象，该对象表示添加了智能布线（智能 bus）后的电路布局结果。

具体说明如下：

  • 返回类型：InstList

    • InstList 是 TED 框架中用于表示一组实例（如矩形、单元等）的容器类，常用于管理和操作物理布局中的对象。
    • 返回的 InstList 包含了经过自动布线处理后的所有 Rect 实例或相关布局元素。

  • 属性说明：

    • name='lay'：表示该 InstList 的名称为 'lay'，通常用于标识其为“布局”层级。
    • placement_type = 'lay'：表明其放置方式为布局模式（layout placement）。
    • is_confined = False：表示该实例列表不受约束（可自由移动或调整位置），意味着它可能在后续布线中被修改。
    • rs_instlist：底层由 Rust 实现的高效数据结构，用于存储实际的物理几何信息和布局数据。

  • 功能意义：

    • 当调用 .smartbus(...) 方法时，系统会根据指定参数（如 hlayer, vlayer, method, min_length 等）自动计算最优路径，并在指定金属层上生成连接各节点的 bus（总线），以实现高效、紧凑的布线。
    • 最终返回的 InstList 包含了原有的原始实体（如 Rect），以及新插入的 bus 布线结构（通常为多个 Rect 或规则形状）。

rotate

功能介绍:

rotate 是 InstList 类的一个成员方法，用于将实例列表（InstList）整体绕指定中心点进行旋转操作。旋转方向为逆时针方向（即正角度表示逆时针旋转）。该方法支持旋转角度必须是 90 的整数倍（如 90°、180°、270°、-90° 等），若旋转角度为 360° 的倍数，则直接返回自身，不进行任何操作。

该函数会更新实例列表中所有实例的位置和方向，并重新计算其 bounding box（边界框）。如果未指定中心点（center），则以实例整体的几何中心为旋转中心；若指定了 center，则以该点为中心进行旋转。旋转后，对象仍保持其原始逻辑关系与连接状态。

参数定义:

参数名类型含义默认值
angleint旋转角度（单位：度），必须是 90 的整数倍。正数表示逆时针，负数表示顺时针。无
centerTuple[int, int]旋转中心点坐标 (x, y)，若为 None 表示使用实例整体的几何中心作为旋转中心。None

返回值

返回旋转后的 InstList 实例本身（即 self），支持链式调用。

功能介绍:

push 是 InstList 类中的一个实例方法，用于将一个电路实例（Instance）添加到内部的实例管理结构中。该方法负责维护两个核心数据结构：self.schematic_sequence 和 self._instances，确保其长度一致，并为新加入的实例分配唯一的序列键值。此外，它还会调用另一个组件 rs_instlist 的 push 方法以同步实例信息。

  • 否则，直接将原始实例对象追加。

pguard

功能介绍:

pguard 是一个用于为电路单元（cell）添加 P 型保护环（Guard Ring）的函数。该函数通过在指定的 cell 外围构建一个 P 型保护环，以增强其抗干扰能力、防止闩锁效应（latch-up），并保证电路在不同工艺和电压条件下的稳定性。该函数适用于全定制模拟集成电路设计中对关键模块进行安全隔离。

pguard 函数返回一个 GuardRing 实例，表示已成功创建的保护环结构。其行为依赖于输入参数，能够灵活控制保护环的位置、尺寸、层叠结构、连接方式以及是否启用检查机制。

功能介绍:

调用此方法后，系统将在指定单元周围生成一个 N 型导电环结构，其边界与原始单元之间保持预设间距（margin），并通过指定金属层（top_metal）和 via 通孔数量（vias）连接至底层。保护环可以与电源或地线节点相连（通过 connectTo 参数指定）。该方法支持对 MOS 管等标准单元进行自动布局优化，同时可选进行单元类型检查。

该函数属于某个模块实例的成员方法，通常由 layout_mos 或类似模块封装的布局流程调用。

connectToOptional['CircuitNode']保护环连接的目标节点，如电源（VDD）、地（VSS）等。若不指定，则默认不连接。None

move_to

功能介绍:

move_to 是一个实例移动方法，用于将当前实例（如几何对象、布局单元等）整体移动到指定的坐标点 (x, y)。该方法支持两种移动模式：普通移动和边界对齐移动。

  • 当 pr_boundary=False 时（默认），会先将输入的坐标四舍五入为整数，然后通过底层 rs_instlist.move_to(x, y) 和 box.move_to(x, y) 同步更新实例的位置与包围盒。
  • 当 pr_boundary=True 时，调用 move_pr_boundary_to(x, y) 方法进行特殊边界对齐移动（常用于规则检查或物理实现阶段）。

该方法为链式调用设计，返回自身对象（self），便于连续操作。

参数定义:

yint目标坐标 y。无
pr_boundarybool是否启用边界对齐移动模式。当为 True 时，将调用 move_pr_boundary_to 方法；否则执行常规移动。False

返回值

  • 类型：InstList
  • 含义：返回当前实例对象（self），支持链式调用。

flipy

功能介绍:

flipy 是 InstList 类的一个成员方法，用于将实例列表（InstList）沿 Y 轴进行翻转（即上下翻转）。该操作会改变实例在布局中的垂直方向，并更新其坐标位置。翻转后，所有实例的位置和边界框信息都会根据新的布局重新计算。此方法常用于电路设计中对模块或单元进行对称变换。

调用 flipy 后，内部调用 rotate_after_flip(180, True) 实现旋转，并通过 compute_box() 重新计算整体包围盒（bounding box）以反映翻转后的几何变化。

参数定义:

无参数。

返回值

  • 返回类型：InstList
  • 含义：返回执行翻转操作后的 InstList 实例本身，支持链式调用。

  • 范围内查询（lookup_in_rectangle）
  • 自动分离（separate_h, separate_v）并兼容偏移控制

该方法是实现自动布图（Auto-placement）、DRC 检查与动态布局优化的关键工具。

cover

功能介绍:

cover 是 InstList 类的一个成员方法，用于在当前的实例列表（InstList）上添加一个或多个矩形覆盖层（coverage）。该操作会在指定的层（layers）上创建一个扩展后的矩形区域，并将其覆盖到当前实例中。支持单个图层或图层列表作为输入，可对覆盖区域进行横向和纵向的扩展（expanded_w 和 expanded_h），并可选择性地连接到指定的电路节点（connectTo）。该方法通常用于在电路设计中实现如保护环（guard ring）、电源/地网络布线、层级覆盖等操作。

参数定义:

参数名类型含义默认值
layersUnion['Layer', List['Layer']]要添加的图层，可以是单个 Layer 对象，也可以是一个包含多个 Layer 的列表。无
expanded_wint扩展的宽度（单位：纳米），表示矩形在水平方向上向外扩展的距离。正值向右扩展，负值向左扩展。0
expanded_hint扩展的高度（单位：纳米），表示矩形在竖直方向上向外扩展的距离。正值向下扩展，负值向上扩展。0
connectToOptional['CircuitNode']可选的连接目标节点，用于将新添加的覆盖层连接到指定电路节点（如电源、地等）。如果为 None，则不进行连接。None

返回值

  • 返回类型：InstList
  • 含义：返回一个更新后的 InstList 实例，包含了新添加的覆盖层图形。该方法支持链式调用，即可以连续调用 cover() 方法。

auto_connect

功能介绍:

auto_connect 是一个实例方法，用于自动将当前实例列表中的金属形状（metal shapes）连接到已命名的节点上。该方法会分析所有金属形状之间的连通性，并为每个连通区域找到其关联的已命名节点（named node），然后将该连通区域内的所有形状统一连接到对应节点上。

此功能确保每个连通分量中的形状都至少有一个控制端（ctrl）连接到了一个已命名节点，从而保证电路拓扑的正确性。若某个连通区域没有与任何已命名节点相连，则抛出异常并提供详细错误信息。

该方法适用于在版图设计中对自动连接进行校验和补全，特别适合用于路由后的金属层自动绑定逻辑。

参数定义:

无参数。

返回值

无返回值（None）。

代码示例:

该示例展示了 auto_connect 方法的基本调用方式。虽然在此测试中未实际触发错误，但其逻辑是：

  • 创建一个 InstList 实例，包含一个 DummyInstance（占位实例），模拟电路中的组件集合。
  • 调用 auto_connect() 方法，触发自动连通性识别与连接过程。
  • 使用 try-except 捕获可能抛出的异常，这表明该方法在无法找到合适目标节点时会主动抛错。

add_tap

功能介绍:

add_tap 是用于在电路实例中添加 TAP（测试端口或注入端口）的函数方法。TAP 通常用于测试、电源分布或电荷平衡等场景，尤其在标准单元或宏模块中常见。该函数将根据指定的连接点、类型、方向和金属层配置，生成一个完整的 TAP 结构，并将其与实例进行连接。

该方法属于类方法，适用于电路结构中的实例对象（如 row 等），可在实例上链式调用以添加多个 TAP。

参数定义:

参数名类型含义默认值
connectToCircuitNodeTAP 需要连接的电路节点（如信号节点或电源节点），通常是字符串标识符或电路引脚。无
typstrTAP 的类型，例如 "p" 表示 P 型 TAP，"n" 表示 N 型 TAP，具体类型由 guardring_type.yaml 中定义。None
directionintTAP 相对于实例的位置方向，可选值为 TOP, BOTTOM, LEFT, RIGHT，用于确定 TAP 的布局方位。TOP
top_metalLayer 或 list[Layer]TAP 所使用的最上层金属，若为 None，则默认使用 M1 层；当指定方向时，会根据不同方向自动设置为对应的金属层排列。None
spacingOptional[int]TAP 与实例之间的间距，若为 None，则系统将自动计算最小安全间距。None
viasOptional[int]TAP 的宽度通过 via 数量来描述，表示接触孔的个数，用于控制 TAP 大小。None

返回值

返回类型为 GuardRing，表示由 TAP 与实例共同构成的整体结构（即带有 TAP 保护环或测试口的完整单元）。
