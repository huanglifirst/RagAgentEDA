DelayedInstance

功能介绍:

DelayedInstance 是一个延迟实例化类，用于在电路构建过程中延迟创建实例，允许在真正需要时才进行实例化操作。该类支持各种电路连接操作符和变换操作，如连接、旋转、翻转等，适用于模拟电路设计中对模块实例的延迟处理。

通过将模块创建函数（module_func）与参数（params）封装，在调用 initialize() 方法前不会立即生成物理实例。这在构建复杂电路架构时非常有用，可以提前定义连接关系，最后统一完成实例化。

该类继承并封装了常见的电路操作逻辑，支持：

  • 使用 >> 进行输出到输入的连接；
  • 使用 % 连接控制信号（如栅极）；
  • 支持翻转和旋转等几何变换；
  • 提供参数动态配置能力。

参数定义:

参数名类型含义默认值
module_funcCallable模块创建函数，用于生成具体实例的函数。无
**paramsdict模块实例化时所需的参数，包括端口和模型参数。无

返回值

  • initialize() 方法返回一个 Instance 实例，表示已成功创建的电路模块实体。
  • 若实例已死亡（非活跃状态），则返回 None。

代码示例:

以下测试用例展示了 DelayedInstance 的典型使用方式和行为：

# /home/zuochang/ted/tedcore/kernel/src/instance/tests/test_delayed.py:5
from pyted import *

from tedcore.kernel import DelayedInstance

def test01():

    @testbench
    def Top():
        inst = DelayedInstance()
        'a' >> inst % 'b' >> 'c'
        inst.set_vdd('d')
        inst.set_vss('e')
        inst.set_module(NMOS)
        mos = inst.initialize()
        assert mos.get_input() == inst.get_input()
        assert mos.get_output() == inst.get_output()
        assert mos.get_ctrl() == inst.get_ctrl()
        assert mos.get_supplies() == inst.get_supplies()
        assert mos.get_module_name() == inst.get_module_name()

该示例中，首先创建了一个空的 DelayedInstance 实例，随后通过连接操作符配置输入、控制和输出信号：

  • 'a' >> inst % 'b' >> 'c' 表示将信号 'a' 连接到实例的输入端口，并将控制信号 'b' 传给控件；输出再连接到 'c'。
  • set_vdd('d') 和 set_vss('e') 为电源引脚指定电位信号。
  • set_module(NMOS) 指定该实例对应的模块类型为 NMOS 管。
  • 调用 initialize() 实际创建实例并完成全部连接。
  • 验证生成的实例与延迟实例具有相同的输入、输出、控制、电源及模块名称，说明连接正确并已同步。
# /home/zuochang/ted/tedcore/kernel/src/instance/tests/test_delayed.py:47
from pyted import *

from tedcore.kernel import DelayedInstance

def test03():

    @testbench
    def Top():
        inst = DelayedInstance(l=1e-06, m=1)
        inst.set_module(NMOS)
        inst.set_params(w=1e-06, m=2)
        mos = inst.initialize()
        assert mos.get_parameter('l') == 1e-06
        assert mos.get_parameter('w') == 1e-06
        assert mos.get_parameter('m') == 2

该示例展示了参数管理机制：

  • 初始化时传入 l=1e-06, m=1，作为默认参数。
  • 调用 set_params(w=1e-06, m=2) 动态修改参数，其中 m 被更新为 2，而 l 保持不变。
  • 最终通过 initialize() 实例化，并检查实例的参数值，确认参数合并行为正确（先使用初始化时传入的值，后由 set_params 覆盖）。

综合上述两例可知，DelayedInstance 提供了灵活的延迟创建机制与强大的参数配置能力，在电路设计中适用于构建复杂连接结构但推迟具体实例化的时间点。

set_vss

功能介绍:

set_vss 是 DelayedInstance 类的一个成员方法，用于设置该实例的接地节点（VSS 端口）。在模拟电路设计中，VSS 表示电源地（Ground），是电路工作的参考电位。通过此方法，可以将一个或多个物理节点指定为当前延迟实例的地连接点。该方法支持传入一个或多个 CircuitNode 类型的参数，并将其转换为内部引用存储在实例中。调用后返回自身实例（链式调用支持），便于连续配置。

参数定义:

参数名类型含义默认值
vssCircuitNode 或其可迭代对象一个或多个代表地节点的电路节点（如接地引脚），将被设置为本实例的 VSS 连接点。无

返回值

返回当前 DelayedInstance 实例自身，支持方法链式调用。

set_vdd

功能介绍:

set_vdd 是 DelayedInstance 类的一个成员方法，用于设置该实例的电源节点（VDD）。该方法通过传入一个或多个电路节点（CircuitNode）作为电源输入，将其绑定到当前延迟实例的 VDD 供电端口。此操作通常在电路实例化前完成，以确保模块能够正确获取供电信息。

该方法接受可变长度的位置参数 *vdd，表示一个或多个电源节点，并将其统一转换为内部表示形式后赋值给 self._vdd。调用后返回 self 本身，支持链式调用。

参数定义:

参数名类型含义默认值
vddCircuitNode 或 CircuitNode 的可变参数列表（如 *vdd: 'CircuitNode'）一个或多个电路节点，作为电源（VDD）输入。这些节点将被用于为该延迟实例提供正电源。无

返回值

返回当前 DelayedInstance 实例的引用（self），以便支持方法链式调用。

功能介绍:

返回值

  • 返回类型：Optional['DelayedInstance']
  • 含义：返回该实例自身（self），用于链式调用支持。当旋转角度为360°倍数时，直接返回 self；否则也返回修改后的 self。
