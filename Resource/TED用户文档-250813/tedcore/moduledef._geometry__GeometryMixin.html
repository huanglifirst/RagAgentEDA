GeometryMixin

功能介绍:

GeometryMixin 是一个混合类（Mixin），用于为模块（Module）提供与几何信息相关的操作能力。该类通过封装底层 rsmodule 对象的接口，为上层用户提供统一、便捷的几何属性查询与设置方法。主要功能包括获取模块尺寸、边界框（Box）、面积、Z轴范围等几何信息，以及对工艺规则（PR）边界进行自动设置、手动设定或扩展。

该类不独立存在，而是作为其他模块类的混入组件，通过继承实现其功能。所有几何操作均基于 rsmodule 提供的底层数据接口完成，确保与设计数据库的一致性。

参数定义:

参数名类型含义默认值
pr_boundaryBox工艺规则边界（PR Boundary）的坐标区域，包含左上角和右下角坐标。无

返回值

返回值类型含义
Box表示一个矩形区域，包含其宽度、高度及四个顶点坐标，用于定义几何边界。
Tuple[int, int]返回模块的宽度和高度，以整数元组形式表示。
Dict[Layer, Box]返回图层与对应边界盒的映射关系，键为图层对象，值为该图层的几何边界。
bool表示模块是否具备尺寸信息或是否存在工艺规则边界。
int表示模块的高度（H）、宽度（W）或面积（Area）。
Module返回自身实例，支持链式调用（方法返回 self）。

代码示例:

def test_geometry_mixin():
    from tedcore.kernel import Module, Box, Layer

    # 定义一个模块并启用 GeometryMixin 功能
    class TestModule(Module):
        def __init__(self):
            super().__init__()
            self._size = (100, 200)  # 模拟设置尺寸
            self._pr_boundary = (0, 0, 50, 100)
            self._layer_boundary = {Layer(1): (0, 0, 50, 100)}

        def get_size(self):
            return self._size

        def has_size(self):
            return self._size is not None

        def get_rsmodule(self):
            return self

        def set_pr_boundary(self, x1, y1, x2, y2):
            self._pr_boundary = (x1, y1, x2, y2)

        def get_pr_boundary(self):
            return self._pr_boundary

        def get_layer_boundary(self):
            return self._layer_boundary

    # 创建实例
    module = TestModule()

    # 测试 box 属性：获取由尺寸构建的边界框
    assert module.box == Box(100, 200, 0, 0)

    # 测试 get_size 和 size：获取模块尺寸
    assert module.size() == (100, 200)
    assert module.get_size() == (100, 200)

    # 测试 has_size：判断是否有定义尺寸
    assert module.has_size() is True

    # 测试 get_area：计算模块面积
    assert module.get_area() == 20000

    # 测试 get_w 和 get_h：分别获取宽度和高度
    assert module.get_w() == 100
    assert module.get_h() == 200

    # 测试 get_zspan：获取Z轴范围（假设为(0, 1)）
    assert module.get_zspan() == (0, 1)

    # 测试 has_pr_boundary 和 get_pr_boundary
    assert module.has_pr_boundary() is True
    assert module.get_pr_boundary() == Box.new_by_coords(0, 0, 50, 100)

    # 测试 get_layer_boundary
    layers = module.get_layer_boundary()
    assert isinstance(layers, dict)
    assert len(layers) == 1
    assert Layer(1) in layers
    assert layers[Layer(1)] == Box.new_by_coords(0, 0, 50, 100)

    # 测试 auto_set_pr_boundary：自动设置PR边界为整个模块范围
    module.auto_set_pr_boundary()
    assert module.get_pr_boundary() == Box.new_by_coords(0, 0, 100, 200)

    # 测试 set_pr_boundary：手动设置PR边界
    new_box = Box(10, 10, 90, 190)
    module.set_pr_boundary(new_box)
    assert module.get_pr_boundary() == Box.new_by_coords(10, 10, 90, 190)

    # 测试 expand_pr_boundary：扩展PR边界
    module.expand_pr_boundary(x=5, y=10)
    assert module.get_pr_boundary() == Box.new_by_coords(5, 0, 95, 200)

    # 测试 set_move_bottom_left_to_origin（设置是否强制左下角对齐原点）
    module.set_move_bottom_left_to_origin(True)
    # 此方法无返回值，仅作用于内部状态

说明与行为分析

  • box 属性：基于当前模块的尺寸自动生成一个以(0,0)为原点、大小与模块一致的边界框。
  • get_size() 与 size()：提供对模块尺寸的访问，其中 get_size() 使用 @layout 装饰，支持布局上下文管理。
  • has_size()：判断模块是否已定义有效尺寸，用于安全操作前检查。
  • get_area()：计算面积，基于 width × height 的数学公式。
  • get_zspan()：返回模块在Z轴上的范围（如层次深度），用于设计规则检查。
  • get_pr_boundary() 与 set_pr_boundary()：用于获取和设置工艺规则边界，支持通过 Box 实例传参。
  • auto_set_pr_boundary()：将 PR 边界自动扩展为覆盖整个模块的区域，适用于标准单元建模。
  • expand_pr_boundary(x, y)：对当前 PR 边界进行水平（x）和竖直（y）方向扩展，常用于预留制造公差或安全距离。
  • get_layer_boundary()：获取各图层的几何边界映射，便于多图层布局分析。
  • set_move_bottom_left_to_origin(bool)：控制是否在布局时将模块左下角强制对齐原点，影响输出坐标系。

该混合类适用于全定制电路设计中需要频繁操作几何数据的场景，提升代码可读性与工程效率。

size

功能介绍:

该函数用于获取模块（Module）的尺寸信息，返回模块的宽度和高度。此方法为模块实例提供了一个统一的接口来查询其在物理布局中的大小，适用于电路设计中对模块空间占用的预估与配置。

该API以函数形式存在，是模块类（Module）的一个成员方法，调用者需通过模块实例访问该方法。

参数定义:

无

返回值

返回值类型含义
Tuple[int, int]模块的尺寸，顺序为（宽度, 高度），单位为逻辑单元（grid units）

set_pr_boundary

功能介绍:

set_pr_boundary 是一个用于设置模块原始布局边界（Placement Region Boundary）的成员方法。该方法将传入的矩形区域 pr_boundary 作为模块的布局区域边界，该边界将影响后续布局操作中的空间分配与对齐行为。

此方法属于模块（Module）类的扩展功能，用于在模块定义中显式指定其物理布局范围（即“放置区域”），适用于需要控制多个模块在行排布（row）或对齐时边界关系的场景。该方法在调用后返回自身模块对象，以支持链式操作。

参数定义:

参数名类型含义默认值
pr_boundaryBox表示模块布局边界的矩形区域，包含四个坐标 (x1, y1, x2, y2)，用于定义该模块在版图中的放置范围。无

返回值

返回调用该方法的模块实例（Module 类型），以支持链式调用。

代码示例:

以下测试用例展示了 set_pr_boundary 的典型使用方式：

# /home/zuochang/ted/tedcore/kernel/tests/test_placer.py:418
from pyted import *

from tedcore.kernel import Rect, lay, current_module, place, Node, row

def test08():

    @module
    def Dut(a):
        r1 = Rect(M1, 1000, 1000, a)
        left = lay(r1, Rect(TEXT, 1500, 1000), x_align=CENTER)
        current_module().set_pr_boundary(r1.box)
        place()

    @testbench
    def Top():
        a = Node()
        left = Dut() % a
        right = Dut() % a
        row(left, right, pr_boundary=True)
        assert left.get_pr_boundary().x2 == right.get_pr_boundary().x1

该示例中：

  • Dut 模块内部定义了一个矩形 r1，并使用 lay 在其上放置一个文本图层。
  • current_module().set_pr_boundary(r1.box) 将 r1 的边界作为当前模块的 pr_boundary 设置。
  • 在顶层测试中，创建两个 Dut 模块实例 left 和 right，并通过 row(left, right, pr_boundary=True) 进行行排布。
  • 验证条件 assert left.get_pr_boundary().x2 == right.get_pr_boundary().x1 验证了两个模块在排布时边界紧密对齐，体现了 pr_boundary 设置对布局对齐的影响。
# /home/zuochang/ted/tedcore/kernel/tests/test/test_placer.py:507
from pyted import *

from tedcore.kernel import Rect, layc, current_module, place, Node, row

def test12():

    @module
    def Dut(a):
        r1 = Rect(M1, 1000, 1000, a)
        layc(r1, Rect(TEXT, 1500, 1500))
        current_module().set_pr_boundary(r1.box)
        place()

    @testbench
    def Top():
        a = Node()
        d1 = Dut() % a
        d2 = Dut() % a
        d1.move_to(100, 100, pr_boundary=True)
        row(d1, d2, pr_boundary=True)

该示例中：

  • Dut 模块使用 layc 将文本图层居中放置于矩形 r1 上。
  • 同样调用 set_pr_boundary(r1.box) 为模块设置布局边界。
  • 在顶层测试中，将 d1 使用 move_to(100, 100, pr_boundary=True) 移动到指定位置，并在移动时更新其 pr_boundary。
  • 然后通过 row(d1, d2, pr_boundary=True) 将两个模块按布局边界对齐排布，表明 set_pr_boundary 能够与移动操作协同工作，确保布局连贯性。

综上，set_pr_boundary 支持两种典型用法：

  1. 在模块内部显式设置其边界，用于后续布局对齐。
  2. 配合 move_to(..., pr_boundary=True) 使用，实现模块在移动后自动维护其边界关系。

set_move_bottom_left_to_origin

功能介绍:

该函数用于控制是否将模块的左下角强制移动到原点（坐标 [0, 0]）。当启用此功能后，模块在布局时会自动调整其位置，使其中心或边界对齐至原点，尤其适用于需要严格对齐到原点的布局场景。该接口作用于当前模块对象的内部状态管理器（rsmodule），并传递参数给底层实现。

参数定义:

参数名类型含义默认值
is_movebool若为 True，则开启强制将模块左下角移动至原点的功能；若为 False，则关闭该功能。无

返回值

无返回值（None）

代码示例:

    assert module.rsmodule.get_move_bottom_left_to_origin() is False

get_w

功能介绍:

get_w 是一个实例方法，用于获取模块（Module）的宽度尺寸。该方法通过调用内部 rsmodule 对象的 get_l() 方法来返回当前模块的宽度值。虽然方法名名为 get_w，但其实现是基于 get_l() 的，说明其实际返回的是模块在长度维度上的参数，可能为设计中的“宽度”或等效尺寸，具体含义依赖于底层模块定义的语义。

该方法适用于所有继承自具备 rsmodule 属性的类实例，常用于 MOS 管、标准单元等电路模块的尺寸查询，便于在版图设计与仿真分析中获取元件几何参数。

参数定义:

无参数。

返回值

返回值类型含义
int模块的宽度尺寸（以最小单位为基准）

get_size

功能介绍:

get_size 是一个用于获取模块（Module）尺寸的API，返回模块在布局区域内的宽度和高度。该方法属于模块实例的成员函数，通常在电路布局完成后调用，以获取整个模块所占据的几何范围。该方法基于底层模块对象（rsmodule）的尺寸信息进行计算，适用于需要测量或验证布局大小的场景。

参数定义:

无参数。

返回值

  • 类型：Tuple[int, int]
  • 含义：返回一个元组，包含模块的宽度（第一项）和高度（第二项），单位为工艺设计规则中的最小单元（如纳米或栅长等），表示布局区域的实际尺寸。

auto_set_pr_boundary

功能介绍:

该方法用于自动设置模块的PR（Physical Region）边界。当模块具有尺寸信息时，它会根据模块的尺寸自动计算并设置PR区域的上下左右边界。PR边界定义了模块在物理设计中的有效范围，通常用于布局与布线阶段的约束管理。
此函数是模块类（Module）的一个成员方法，调用后可自动将当前模块的尺寸信息映射为PR边界参数，并传递给底层的RSMODULE组件进行设置。

参数定义:

无参数。

返回值

  • Module 类型：返回调用该方法的模块实例本身（支持链式调用）。

代码示例:

# /home/zuochang/ted/develop/tedgpt/src/agents/ted_doc/tests/test_module2.py:107
from pyted import *

from tedcore.kernel import Rect, current_module, skip_unless

@skip_unless(TED_TEST_LVL=99)
def test_pr_boundary_operations():

    @module
    def PRBoundary():
        Rect(M1, 100, 100)
        current_module().auto_set_pr_boundary()
    mod = PRBoundary().module()

解释：

  • 定义了一个名为 PRBoundary 的模块，该模块中创建了一个尺寸为 100×100 的矩形（Rect(M1, 100, 100)），表示一个具有明确物理尺寸的结构体。
  • 调用 current_module().auto_set_pr_boundary()，将当前模块的尺寸自动用于设置PR边界。
  • 方法内部检查该模块是否拥有有效尺寸（通过 has_size() 判断），若有，则提取其宽度和高度，并以 (0, 0) 为左下角点、x2, y2 为右上角点，调用 get_rsmodule().set_pr_boundary(x1, y1, x2, y2) 设置PR边界。
  • 最终通过 .module() 获取模块实例并赋值给 mod，完成整个流程。

此测试用例展示了在具有尺寸信息的模块中，如何使用 auto_set_pr_boundary 方法自动配置PR边界，适用于需要精确控制物理区域范围的电路模块设计场景。
