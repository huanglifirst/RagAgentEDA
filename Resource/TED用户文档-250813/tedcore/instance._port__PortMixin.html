PortMixin

功能介绍:

PortMixin 是一个混入类（Mixin），用于为电路模块的实例（Instance）提供端口配置和连接操作的支持。该类扩展了实例对象的功能，允许用户通过链式调用的方式设置输入、输出和控制端口，并支持实例的复制与连接操作。

此混入类主要用于 TED（Tsinghua Electronic Design）框架中，帮助开发者以直观的面向对象语法完成电路图中模块实例的端口绑定、拓扑连接以及多实例生成。其方法设计遵循函数式编程思想，支持链式调用（如 inst.input(A).output(B)），从而简化复杂电路结构的构建流程。

参数定义:

参数名类型含义默认值
xCircuitNode 或 tuple[CircuitNode, ...]要设置或连接的节点，可以是单个电路节点，也可以是一组节点的元组。无

注：所有 *x 参数均表示可变参数（支持传入多个节点），其中节点类型需为 CircuitNode 或由其构成的元组。

返回值

所有方法均返回 self，即当前实例对象本身，以支持链式调用（如 inst.input(A).output(B)）。

代码示例:

以下测试用例展示了 PortMixin 提供的核心功能使用方式：

def test_port_mixin():
    class TModule(Module):
        pass

    inst = TModule()
    # 1. 测试 input 方法的用法
    inst.input(A)
    
    # 2. 测试 output 方法的用法
    inst.output(B)

    # 3. 测试 ctrl 方法的用法
    inst.ctrl(C)

    # 4. 测试 replace_ctrl 方法的用法
    inst.replace_ctrl(D)

    # 5. 测试字符串表示
    print(str(inst))

    # 6. 测试 % 连接操作
    inst % E

    # 7. 测试 * 复制实例生成序列
    inst_list = inst * (1, 10)
    assert len(inst_list) == 10

    # 8. 测试 >> 右移连接（输出 → 输入）
    other_inst = TModule()
    inst >> other_inst

    # 9. 测试 << 左移连接（输出 → 输入，逆向）
    other_inst << inst

    # 10. 测试 rrshift（反向右移连接）
    inst1 = TModule()
    inst2 = TModule()
    inst1 >> inst2
    inst2 >> inst1  # 验证是否支持双向连接逻辑

说明：

  • input(A) 设置实例的输入节点为 A，支持传入单个或多个节点。
  • output(B) 设置输出节点为 B，用于表示该实例的输出信号源。
  • ctrl(C) 将控制端设为 C，常用于配置晶体管栅极等控制信号。
  • replace_ctrl(D) 替换当前的控制节点为 D，通常用于动态更新控制路径。
  • str(inst) 返回实例的字符串表示形式，便于调试或日志输出。
  • % 操作符用于将一个节点连接到实例的控制端口（如控制信号注入）。
  • * 生成多个完全相同的实例，并保持独立端口连接，适用于阵列构建。
  • >> 和 << 用于实现标准数据流向连接：Output >> Input，支持双向操作语义。
  • rrshift 与 lshift 分别处理左右两端的连接顺序，确保复杂网络连接可正确解析。

vss

功能介绍:

vss 是一个实例方法，用于获取电路中与地（VSS）相连的节点。该方法返回一个 CircuitNode 类型的对象，代表当前实例所连接的地节点。通常在构建和分析模拟电路时，用于访问模块或器件的接地端口。

参数定义:

无参数

返回值

  • 类型: CircuitNode
  • 含义: 返回与该实例关联的地（VSS）节点对象，用于表示电路中的接地连接点。

vdd

功能介绍:

vdd 是 Instance 类的一个实例方法，用于获取该实例所关联的电源节点（VDD 节点）。该方法返回一个表示电路中电源连接点的对象，即 CircuitNode 类型的实例，通常用于访问或验证当前实例连接到的 VDD 电源信号。

此方法在电路建模和仿真上下文中非常关键，特别是在需要确认模块或器件是否正确连接到电源时，可用于判断器件的供电状态。

参数定义:

该方法无输入参数。它是 Instance 类的成员函数，无需传入任何参数即可调用。

参数名类型含义默认值
（无）——无参数。该方法为实例方法，依赖于 self 实例自身上下文。——

返回值

  • 返回类型：CircuitNode

set_port

功能介绍:

set_port 是实例（Instance）类的一个方法，用于在特定条件下将当前矩形实例（Rect）设置为端口（port）矩形。该方法仅在以下条件满足时生效：

  • 当前模块是测试基准（Testbench），或
  • 当前实例所属的模块名称为 'RRect'（金属性层矩形），且其控制节点（Ctrl node）是端口类型的节点。

调用该方法后，会读取该节点的名称，并通过 set_port(name, self) 将当前矩形标记为端口，以便在后续布局或物理验证中被识别为端口。若不满足上述条件，则直接返回自身实例。

该方法主要用于自动识别和配置特定类型的矩形对象为端口，尤其适用于在全定制电路设计中的版图自动生成流程中对金属层端口的定义。

参数定义:

参数名类型含义默认值
selfInstance当前实例对象，通常为 Rect 或 RRect 类型实例。无

注：此方法为实例方法，不接受外部显式参数。其行为依赖于当前模块上下文和节点类型。

返回值

  • 返回调用该方法的实例本身（self），类型为 Instance。
  • 用于支持链式调用，如 rect.set_port().some_other_method()。

port

功能介绍:

port 是 Instance 类的一个成员方法，用于获取实例指定端口的端口矩形集合（即端口对应的物理形状或几何区域）。该方法返回一个 InstList 类型的对象，其中包含与给定端口名称匹配的所有端口实例。在模拟和全定制电路设计中，端口通常对应于晶体管的栅极、漏极、源极等物理连接点，通过此方法可以获取这些连接点的空间布局信息（如坐标、尺寸等），便于进行布线、验证或布局优化。

该方法支持同一端口名称下存在多个端口实例的情况（例如多指器件中的多个并联栅极），因此返回的 InstList 可能包含多个 Instance 对象，每个代表一个具体的端口矩形。

参数定义:

参数名类型含义默认值
keystr端口名称，用于指定要查询的端口类型（如 'g' 表示栅极、'd' 表示漏极、's' 表示源极等）。支持任意自定义的端口名。无

返回值

  • 返回类型：InstList
  • 含义：表示指定端口名称下的所有端口矩形实例组成的列表。每个元素是一个 Instance 对象，包含了其对应的物理几何信息（如通过 .box 可获取边界框坐标）。

has_port

功能介绍:

该函数用于判断一个实例（Instance）是否包含指定名称的端口。在 TED 框架中，实例是模块（Module）的具体实例化对象，其端口定义来源于对应模块的端口声明。通过此接口，用户可以在运行时动态检查某个实例是否具备特定端口，适用于条件性连接、配置验证等场景。

该函数为实例对象（Instance）的成员方法，接收一个字符串类型的端口名称作为参数，返回布尔值以表示是否存在该端口。

参数定义:

参数名类型含义默认值
namestr要查询的端口名称。无

返回值

  • bool：若实例包含指定名称的端口，则返回 True；否则返回 False。

代码示例:

def test_has_port():
    # 创建一个模块并定义其端口
    @module
    def example_module(A: Input, B: Output):
        pass

    # 实例化该模块
    inst = example_module()

    # 测试存在端口的情况
    assert has_port(inst, "A") is True
    assert has_port(inst, "B") is True

    # 测试不存在端口的情况
    assert has_port(inst, "C") is False

在上述测试用例中展示了 has_port 函数的两种典型使用方式：

  1. 查询已定义端口：传入实例中存在的端口名称（如 "A" 或 "B"），函数返回 True，表示该实例确实包含此端口。
  2. 查询未定义端口：传入一个不存在的端口名称（如 "C"），函数返回 False，说明该端口未被当前实例所持有。

此功能可用于在动态连接或电路构造逻辑中进行安全检查，避免因错误端口名导致运行时错误。

  • get_input() 在模块实例上被调用，返回该实例所有输入端口对应的节点。

get_ctrl

功能介绍:

get_ctrl 是 Instance 类的一个成员方法，用于获取该实例的控制端口节点（Control Node）。在电路设计中，控制端通常指器件的栅极（Gate）等用于控制通断的输入信号。此方法返回一个 CircuitNode 对象，代表与该实例关联的控制信号接口。

参数定义:

参数名类型含义默认值
selfInstance调用该方法的实例对象。无

注：self 是类方法的隐式第一个参数，表示当前实例，无需显式传入。

返回值

  • 类型：CircuitNode
  • 含义：返回该实例的控制端口所连接的节点（即控制信号所在的位置），通常对应 MOS 管的栅极节点。

all_terminals

功能介绍:

该函数用于获取实例中包含电源节点（VDD/VSS）在内的全部电路节点。返回值为一个 CircuitNode 类型的对象，其内部封装了所有与实例相关的电路节点，包括但不限于电源端口（如 VDD、VSS）以及普通信号端口。该接口主要用于电路分析、节点遍历或验证电路完整性等场景。

此函数属于实例（Instance）的成员方法，用于在模块实例化后查询其全部电路连接节点信息。
