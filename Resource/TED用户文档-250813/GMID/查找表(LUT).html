查找表 (LUT)

本文档旨在说明 lut.md 文件中包含的 Python 代码的功能和用法。这些代码主要围绕查找表（LUT）的生成、测试、数据处理和模型预测等功能展开，特别是在电容和 MOSFET 器件参数方面。

一、 电容查找表生成与测试 (capacitance 目录)

这部分代码主要用于生成和测试 Metal-Oxide-Metal (MOM) 电容以及通用电容模型的查找表。

1. MOM 电容查找表生成 (lut_generator_momcap.py)

此脚本用于生成 MOM 电容的查找表。

主要功能:

  • 定义 MOM 电容模块 (_momcap, d_momcap)：基于输入的物理参数（如 finger 长度、宽度、间距、数量，金属层等）构建 MOM 电容的电路表示。
  • 定义 MOM 电容测试器 (MomCapTester)：继承自 TesterBase，用于设置测试环境（如 vdd）、定义测试平台 (testbench) 和性能指标提取 (performance)。
    • testbench: 设置交流（AC）仿真环境，施加激励，连接待测器件（DUT），并使用探针 (IProbe) 测量电流，最后进行 PEX（寄生参数提取）和 AC 仿真。
    • performance: 定义如何从仿真结果中提取性能指标，这里是电容值 (cap) 和总对地电容 (tcap)。MomCapPerformance 类实现了具体的提取逻辑。
  • 定义 MOM 电容查找表生成器 (Lut_Gen_MomCap)：
    • 初始化：设置查找表保存路径、金属层组合、扫描参数（finger 数量 sweep_fnum、finger 长度 sweep_fl）和 finger 宽度 (fw)。
    • run 方法：遍历金属层组合和扫描参数，为每种组合运行 MomCapTester 进行仿真，提取电容值 (cap, tcap)，并将结果存储在 lut 字典中。最后，使用 dill 将 lut 字典序列化并保存到指定路径的文件中（文件名格式为 [pdk_name]_MomCap.dill）。
  • 测试函数 (test_run)：提供了一个使用 Lut_Gen_MomCap 生成 MOM 电容查找表的具体测例。

API 用法示例 (来自 test_run):

# 初始化 MOM 电容查找表生成器
glut = Lut_Gen_MomCap(
    metal_layer=[M3, M4, M5], # 指定用于生成电容的金属层
    sweep_fnum=np.arange(10, 200, 10), # 扫描的 finger 数量范围
    sweep_fl=[6.5e-6, 8.5e-6, 10.5e-6], # 扫描的 finger 长度列表
    fw=0.1e-6, # 固定的 finger 宽度
)
# 运行查找表生成
glut.run()

2. 通用电容查找表生成 (lut_generator.py)

此脚本用于生成通用电容模型的查找表，支持更灵活的参数扫描。

主要功能:

  • 定义通用电容查找表生成器 (Lut_Gen_Capacitance)：
    • 初始化：设置查找表保存路径，初始化 lut 字典（用于存储查找表信息、插值函数和参数）。
    • cap_build 方法：构建测试模块。接收器件模型 (model)、固定参数 (param)、扫描变量 (variable) 和额外的连接端口 (extConnTo)。使用 creat_sweep 函数生成扫描点，并定义一个包含 DataSweep 和器件子电路 (Subckt) 的 pyted 模块 (test_cap)。
    • run 方法：运行 CapacitanceTester 进行仿真，提取电容值，创建查找表信息 (creat_info) 和插值函数 (create_interpolate)，最后将 lut 字典保存到文件中（文件名格式为 [pdk_name]_[model].dill）。
    • creat_info 方法：使用 prettytable 生成包含 PDK 名称、模型名称、参数扫描范围、电容范围等信息的表格，并存储在 lut["info"] 中。
    • create_interpolate 方法：使用 scipy.interpolate 中的 griddata 和 interp1d 创建插值函数。interpolate_C 用于根据输入的参数值插值计算电容值，interpolate_val 用于根据目标电容值反向查找最接近的参数组合。
  • 定义电容测试器 (CapacitanceTester)：继承自 TesterBase，设置 AC 仿真环境，使用电压源 (V) 和电流探针 (IProbe) 来测量电容值。
  • 定义电容性能提取 (CapacitancePerformance)：继承自 Performance，定义如何从仿真结果中提取电容值。
  • 辅助函数 (creat_sweep)：根据输入的多个参数范围，生成所有可能的参数组合（笛卡尔积）。

API 用法示例 (来自 test.py 中的 test3):

# 初始化通用电容查找表生成器
gen = Lut_Gen_Capacitance()
# 构建测试模块
gen.cap_build(
    extConnTo=[Vss()], # 额外的连接端口
    model="crtmom", # 电容模型名称
    param={ # 固定的参数
        "w": 100e-9,
        "s": 100e-9,
        "stm": 3,
        "spm": 5,
        "mf": 1,
        "mismatchflag": 0,
    },
    variable={ # 需要扫描的变量及其范围
        "nv": np.arange(10, 102, 2),
        "nh": np.arange(10, 102, 2)
    }
)
# 运行查找表生成
gen.run()

3. 电容查找表数据读取与使用 (lut_data.py, lut_momcap_data.py)

这两个脚本定义了用于加载和使用已生成的电容查找表的类。

  • lut_data.py 中的 LUTData_Cap：
    • load 方法：加载通用电容查找表文件 (.dill)。优先使用用户指定的 lut_path，否则按默认规则查找路径 ([global_home]/matfiles/capacitance/ 或 [ted_data]/lookuptable/capacitance/)。加载失败会抛出异常。
  • lut_momcap_data.py 中的 LUTData_MomCap：
    • load 方法：加载 MOM 电容查找表文件 (.dill)，查找逻辑与 LUTData_Cap 类似。
    • lookup 方法：根据顶层 (top_layer)、底层 (bottom_layer) 金属和目标电容值 (c)，查找最合适的 finger 数量 (fnum) 和 finger 长度 (fl)。
      • 首先从加载的 _LUT 字典中提取对应金属层组合的数据。
      • 使用 creat_fun 创建插值函数（基于 scipy.interpolate.interp1d）。
      • 如果目标电容 c 超出查找表范围，使用 find_min_n_within_range 计算需要并联的电容数量 (c_num)，并相应调整目标电容值。
      • 使用插值函数计算 fnum 和 fl。
      • 返回 fnum (整数)，fl (保留两位小数的科学计数法) 和 c_num (整数)。
    • creat_fun：内部辅助函数，创建用于插值计算 fnum 和 fl 的函数。
    • find_min_n_within_range：内部辅助函数，计算将一个超出范围的值 a 分成多少份 (n) 可以使其落在 [range_min, range_max] 区间内。

API 用法示例 (来自 test_momcap.py 中的 test_read):

# 查找表文件路径
file = f"{get_ted_data()}/lookuptable/capacitance/t65_MomCap.dill"
# 检查文件是否存在
if not os.path.exists(file):
    return
# 加载 MOM 电容查找表数据
lut_data = LUTData_MomCap().load(file)

# 定义顶层和底层金属
mbottom = M3
mtop = M4
# 定义目标电容值
cap_target = 7e-15

# 使用 lookup 方法查找参数
fnum, fl, c_num = lut_data.lookup(top_layer=mtop, bottom_layer=mbottom, c=cap_target)

debug(f"查找结果: finger_num = {fnum}, finger_l = {fl}, 并联数量 = {c_num}")

4. 电容查找表测试脚本 (test 目录)

包含 test_momcap.py 和 test.py，用于测试 MOM 电容和通用电容查找表的功能和精度。

  • test_momcap.py (test_read):
    • 加载 MOM 电容查找表。
    • 定义目标电容值。
    • 使用 LUTData_MomCap 的 lookup 方法查找参数。
    • 使用 ted_device 中的 get_c_from_momcap 函数（假设存在）或直接仿真来验证查找到的参数对应的电容值，并计算误差。
  • test.py:
    • test(): 测试基本的 DataSweep 和 Subckt 调用。
    • test2(): 使用 CapacitanceTester 测试单个电容实例。
    • test3(): 使用 Lut_Gen_Capacitance 生成通用电容查找表。
    • test4(): 加载通用电容查找表，使用 griddata 插值计算指定参数下的电容值，并与仿真结果比较，计算误差。
    • test5(): 加载通用电容查找表，使用 interpolate_val 根据目标电容反向查找参数，并进行仿真验证和误差计算。

二、 MOSFET 查找表生成与模型预测 (lut 目录)

这部分代码主要用于生成 MOSFET 器件的 DC 参数查找表，并基于这些查找表训练机器学习模型进行预测。

1. 查找表生成 (lookuptable_generator_hspice.py, lookuptable_generator_enhance.py, lookuptable_generator_hspice_new.py)

这些脚本提供了不同的方法来生成 MOSFET 的 DC 参数查找表，主要利用仿真器（HSPICE 或 Spectre）进行扫描。

  • lookuptable_generator_hspice.py (旧版):

    • lookuptable_Generator_hspice 类：
      • 初始化：设置扫描参数范围 (vdsrange, vgsrange, vbsrange, Lrange) 和数据保存路径。
      • muti_run 方法：使用多进程 (multiprocessing.Pool) 并行运行仿真。遍历 Lrange，为每个 L 值调用 do_run。将结果合并并保存为 .pkl 和 .csv 文件。
      • do_run 方法：调用 newsim 进行仿真。
      • newsim 方法：定义 pyted 测试平台 (@testbench)，包含电压源 (VSource) 和 MOSFET 器件。设置 DataSweep 进行 vgs, vds, vbs 的扫描。调用 dcsweep 运行 HSPICE 仿真，并使用 ModelPrint 提取 DC 参数（如 ids, vth, vdsat, gm, gds, 电容等）。将提取的参数整理成 NumPy 数组返回。
    • 主要特点：使用多进程加速，依赖 HSPICE 的 ModelPrint 功能提取参数。

  • lookuptable_generator_enhance.py:

    • lookuptable_Generator_enhance 类：
      • 初始化：与旧版类似，设置扫描参数和路径。
      • muti_run 方法：与旧版类似，使用多进程调用 get_perf。
      • get_perf 方法：实例化并运行 MosFetDCTester。
    • MosFetDCTester 类 (继承自 TesterBase)：
      • setup: 初始化测试参数，包括器件模型、L 值、电压扫描范围等。
      • testbench: 定义测试平台，与旧版类似。
      • performance: 返回 DCpara 性能提取对象。
      • runsim: 设置 DataSweep，根据仿真器类型（HSPICE 或 Spectre）设置仿真命令 (dcsweep) 和参数提取方式 (ModelPrint 或 save options)。
    • DCpara 类 (继承自 Performance)：
      • performance: 返回包含 dcpar 的字典。
      • get_DCpar: 根据仿真器类型，从仿真结果 (raw) 中提取 DC 参数，计算衍生参数（如 fug, gmoverid, self_gain），并将所有参数整理成 NumPy 数组返回。
    • 主要特点：使用 TesterBase 和 Performance 框架，兼容 HSPICE 和 Spectre，可以获取更全面的 DC 参数（特别是 Spectre）。

  • lookuptable_generator_hspice_new.py:

    • 与 lookuptable_generator_hspice.py 类似，但增加了对 Wrange 的扫描支持，并调整了多进程和数据处理逻辑。

API 用法示例 (来自 lookuptable_generator_enhance.py 中的 get_perf 调用):

# 假设 Lseq[0] 是当前的 L 值，model 是模型名称
# vdsrange, vgsrange, vbsrange, Lrange 是 NumPy 数组
# batch 和 G 是用于性能提取的参数
res = MosFetDCTester(
    l=Lseq[0], # 当前扫描的 L 值
    model=model, # MOSFET 模型
    vdsrange=self.vdsrange, # VDS 扫描范围
    vgsrange=self.vgsrange, # VGS 扫描范围
    vbsrange=self.vbsrange, # VBS 扫描范围
    Lrange=self.Lrange, # L 扫描范围 (可能冗余)
    batch=self.batch, # 批处理大小 (用于 DCpara)
    G=self.G, # VGS 扫描点数 (用于 DCpara)
).run()

# 提取 DC 参数结果
dc_parameters = res["dcpar"]

2. 机器学习模型训练 (mlmodel.py)

此脚本定义了 MLModelTrain 类，用于训练机器学习模型来预测 MOSFET 参数。

主要功能:

  • MLModelTrain 类：
    • 初始化 (__init__)：接收数据路径 (datapath) 和 MOSFET 类型 (mostype)。加载 .pkl 格式的查找表数据。定义目标参数列表 (targetlist)。
    • prepare_data 方法：初始化参数范围、符号标志 (isneg)、零值标志 (iszeros)、浮点数精度 (eps)、正负主导标志 (ispos) 和归一化偏移 (alpha)。调用 check_data 检查数据特性。根据 target (单个参数或 "multpre") 选择需要的数据列。
    • check_data 方法：检查查找表数据中是否存在 0 值和符号不一致的情况，并记录相关信息用于后续的 log10 处理。
    • filter 方法：根据工作区 (typ: "all", "satuation", "linear", "cutoff") 筛选数据。使用 Vth（可能通过 Vthpredict 类预测得到，或者直接从数据中读取）来判断器件工作区。
    • split 方法：将数据随机划分为训练集和验证集（默认 90%/10%）。根据 log 标志对目标参数进行 log10 转换（调用 utils.log10，处理 0 值和负值）。根据 std_ 标志对输入和输出数据进行标准化（调用 utils.std）。使用 utils.build_X 对输入特征进行多项式扩展。
    • train 方法：调用 split 准备数据。使用选定的机器学习模型 (mlmodel) 训练模型 (fit)。调用 savemodel 保存模型和参数。使用训练好的模型预测训练集和验证集的结果。调用 inv_std_log 将预测结果反向转换。调用 testmodel 评估模型在训练集和验证集上的误差。记录模型信息、性能指标（时间、文件大小、误差）到 .csv 文件。如果 mult=True，则遍历 targetlist 分别评估和记录。调用 plotdistri 和 ploterror 绘制误差分布图和误差曲线图。
    • testmodel 方法：计算预测值 (yp) 和真实值 (y) 之间的相对误差，找出最大误差点及其对应的输入参数 (x) 和 Vth。返回最大误差点的索引、最大误差值、平均误差值以及最大误差点的输入参数和 Vth。
    • savemodel 方法：将训练好的模型 (self.model) 保存为 .pkl 文件，并将标准化、log 转换、多项式扩展相关的参数 (self.para) 保存到另一个 .pkl 文件中。文件名包含 PDK、模型类型、目标、模型名称和工作区类型。同时在 temp 目录下保存备份。
    • predict 方法：根据保存的参数对新输入数据进行预处理（标准化、特征扩展），使用加载的模型进行预测，并进行反向转换（反标准化、反 log10）。
    • inv_std_log 方法：predict 的辅助函数，用于将预测结果进行反向转换。
    • testtime 方法：测试模型预测单个向量和多个向量所需的时间。
    • summarydoc, simpletabel, summarytabel, summarypic: 生成 Markdown 格式的训练报告，包含模型信息、误差统计表格和误差图。
    • ploterror 方法：绘制特定输入条件下（固定 L, VBS, VDS）模型预测误差随 VGS 变化的曲线图。需要重新运行仿真获取更精细的真实值。
    • plotdistri 方法：使用 seaborn 绘制预测误差（绝对误差和相对误差）的分布图。

API 用法示例 (来自 test_train.py 中的 _test_nch):

key = "pch_v1" # MOSFET 类型

    mostype=key,
)
# 运行训练 (使用 deeplearning 中的 'gpumlp_1' 模型, 预测所有参数, 不分区训练)

# 生成训练报告

3. 机器学习模型预测 (modelpredict.py, modelpredict_old.py)

这两个脚本定义了 MLModelPred 类，用于加载训练好的模型并进行预测。modelpredict_old.py 似乎是一个旧版本。

主要功能 (modelpredict.py):

  • MLModelPred 类：
    • 初始化 (__init__)：接收目标参数 (target)、MOSFET 类型 (mostype)、预测方法 (method: "div" 或 None) 和模型名称 (modelname)。根据 method 调用 loaddivmodel 或 loadallmodel。
    • loadallmodel 方法：加载 "all" 类型（不分区）的模型和参数文件。
    • loaddivmodel 方法：加载 "satuation", "cutoff", "linear" 三种分区类型的模型和参数文件。
    • load 方法：内部函数，用于加载指定的模型和参数 .pkl 文件。
    • inv_std_log 方法：反向转换函数，与 MLModelTrain 中类似。
    • filter 方法：根据 Vth（通过 Vthpredict 预测）将输入数据划分到饱和区、线性区和截止区。
    • preddiv 方法：如果 method="div"，调用此方法。分别对三个区域的数据调用 pred 方法进行预测，然后合并结果。
    • pred 方法：核心预测函数。根据加载的参数对输入数据进行预处理（标准化、特征扩展），调用模型的 predict 方法，然后调用 inv_std_log 进行反向转换。
    • predict 方法：外部调用的主预测接口。根据 method 调用 preddiv 或 pred。

API 用法示例 (来自 test_predict.py 中的 test_nch_div_ids):

# 初始化模型预测器 (分区预测 'div', 目标为 'multpre', 模型为 'gpumlp')

        [0.00000226, -0.7, 0.05, 0],
        [0.00000226, 0.6, 0.7, 0],

# 提取 ids 预测结果

4. Vth 预测 (vthpred.py)

定义 Vthpredict 类，用于训练模型预测 MOSFET 的阈值电压 Vth。

主要功能:

  • Vthpredict 类：
    • 初始化 (__init__)：接收 MOSFET 类型 (modeltype)、数据路径 (datapath) 和模型保存路径 (savepath)。
    • train 方法：加载数据 (loaddata)。使用 sklearn.tree.DecisionTreeRegressor 训练 Vth 预测模型。可选地进行测试 (test)。保存模型 (save)。
    • save 方法：将训练好的决策树模型保存为 .pkl 文件。
    • loaddata 方法：从 .pkl 文件加载查找表数据，提取输入特征 (L, VGS, VDS, VBS) 和目标 Vth。
    • loadmodel 方法：加载已保存的 Vth 预测模型。
    • pred 方法：加载模型并对新的输入数据进行 Vth 预测。
    • test 方法：生成新的测试数据集，运行仿真获取 Vth 真实值，与模型预测值比较，计算误差。

5. PyTorch 模型 (pytorchmodel.py)

定义了一个基于 PyTorch 的深度神经网络（DNN）模型。

主要功能:

  • DNN 类 (继承自 torch.nn.Module)：定义了一个多层感知机（MLP）结构，包含输入层、多个隐藏层（带有 ReLU 激活函数）和输出层。
  • DNNModel 类：封装了 DNN 模型，提供了训练 (fit) 和预测 (predict) 的接口。
    • 初始化 (__init__)：设置网络结构参数、训练超参数（学习率 lr、epoch 数 epoch、批大小 batch_size）、设备 (cuda 或 cpu)。初始化 DNN 模型、优化器 (torch.optim.Adam) 和损失函数 (torch.nn.MSELoss)。
    • addbatch 方法：使用 torch.utils.data.DataLoader 将 NumPy 数据转换为 PyTorch 的 DataLoader，用于分批训练。
    • fit 方法：训练 DNN 模型。将数据加载到指定设备，循环遍历 epoch 和 batch，执行前向传播、计算损失、反向传播和优化器更新。
    • predict 方法：使用训练好的模型进行预测。将输入数据分批加载到设备，执行前向传播，并将结果转换回 NumPy 数组。

6. 模型字典 (modeldict.py)

三、 电阻查找表 (resistor 目录)

这部分代码用于生成、测试和使用电阻模型的查找表。

1. 电阻查找表生成 (lut_generator.py)

  • Lut_Gen_Resistor 类：
    • 初始化 (__init__, setup)：接收 L 和 W 的扫描范围 (Lrange, Wrange)、电阻模型 (model)、额外连接 (extConnTo) 和保存路径。定义测试模块 (test_Resistor)。
    • run 方法：使用 ResistorTester 进行仿真，获取电阻值。将结果（R, L, W）存入 lut 字典。调用 creat_info 和 create_interpolate。保存 lut 字典到 .dill 文件。
    • creat_info 方法：生成包含 PDK、模型、L/W/R 范围的表格信息。
    • create_interpolate 方法：创建插值函数。interpolate_R 用于根据 W, L 计算 R；interpolate_WL 用于根据 R 计算 W, L。
  • ResistorTester 类：继承自 TesterBase，设置 DC 仿真环境，使用 DataSweep 扫描 L 和 W，通过电压源和电流探针计算电阻值。
  • ResistorPerformance 类：从仿真结果中提取电阻值。
  • create_sweep 函数：生成 L 和 W 的笛卡尔积扫描点。

API 用法示例 (来自 test_run.py 中的 test_run_t65):

# 初始化电阻查找表生成器
lut = Lut_Gen_Resistor(
    Lrange=np.arange(1, 15.1, 0.1), # L 扫描范围 (单位 um)
    Wrange=np.arange(0.5, 5.05, 0.05), # W 扫描范围 (单位 um)
    model="rppolywo", # 电阻模型
)
# 运行查找表生成
lut.run()

2. 电阻查找表数据处理 (lut_data.py)

  • LUTData_Res 类：
    • load 方法：加载电阻查找表 .dill 文件。
    • info 方法：打印查找表信息。
    • get_L, get_W, get_R, get_model, get_info, get_module：获取查找表中的相应数据。
    • lookup 方法：核心查找接口。根据传入的参数（R, W, L）调用不同的内部查找函数。
    • lookup_r：根据 W, L 插值计算 R。
    • lookup_size：根据 R 插值计算 W, L 组合。
    • lookup_l：根据 R, W 插值计算 L。
    • lookup_w：根据 R, L 插值计算 W。
    • lookup_size_min：使用 KD 树（假设 KDTree 类已定义）查找在给定误差范围内，面积 (L*W) 最小的尺寸组合。
  • LUTErrorAnalyzer 类：用于分析查找表的误差。
    • 初始化：接收 LUTData_Res 对象和采样数量。
    • run 方法：随机采样目标电阻 R，使用 lookup 查找对应的 W, L。运行 ResistorTester 进行仿真获取实际电阻值。计算绝对误差和相对误差。生成概览表格 (toverview) 和统计结果表格 (tstatres)。绘制目标 R 与仿真 R 的散点图并保存。
    • report 方法：将分析结果（表格和图片链接）格式化为 Markdown 报告并保存。
  • KDNode, KDTree 类：实现了 KD 树数据结构，用于 lookup_size_min 中的最近邻搜索和范围搜索。

四、 BJT 查找表 (bjt 目录)

这部分代码用于生成、加载和使用 BJT（双极结型晶体管）的查找表，主要关注 Vbe 和 Ie 的关系以及温度特性。

1. BJT 查找表生成 (lookuptable_generator_bjt.py)

  • lookuptable_generator_bjt 类：
    • 初始化 (__init__, setup)：设置查找表保存路径。
    • run 方法：接收温度列表 (templib)、Ie 扫描范围 (ierange)、BJT 模型 (model) 和单位尺寸 (uni)。使用多进程并行运行 get_perf 获取不同温度下的数据。整理数据格式并保存到 .pkl 文件。
    • get_perf 方法：设置当前温度环境，运行 BJTDCTester 获取 Vbe 数据，返回包含 Ie 和 Vbe 的字典。
  • BJTDCTester 类：继承自 TesterBase，设置 DC 扫描环境，使用电流源 (I) 扫描 Ie，测量对应的 Vbe。
  • BJTDCpar 类：继承自 Performance，从仿真结果中提取 Vbe。

API 用法示例 (来自 test_bjt_lut_gen.py 中的 test1):

# 定义 Ie 扫描范围
ie = np.arange(0.01, 50, 0.005)
# 运行 BJT 查找表生成 (使用默认温度 [-40, 27, 125] 和默认模型 pnp5)
lookuptable_generator_bjt().run(ierange=ie)

2. BJT 查找表数据加载与使用 (load.py, lutmodel.py)

  • load.py 中的 LUTData_BJT 类：
    • 初始化 (__init__, get)：加载 BJT 查找表 .pkl 文件。如果文件不存在，则调用 lookuptable_generator_bjt 自动生成。存储加载的数据和支持的温度列表。
    • lookupvbe：根据输入的 Ie 和温度，使用线性插值查找对应的 Vbe。支持调整晶体管数量 n（调整电流）。
    • lookupie：根据输入的 Vbe 和温度，使用线性插值查找对应的 Ie。支持调整晶体管数量 n。
    • get_TC：计算指定 Ie 和晶体管数量 n 下 Vbe 的温度系数 (TC)。可选 ztd 参数决定是否进行 ZTD（零温度系数）补偿（调整 Ie）。
    • lookupvbe_t：查找特定温度下的 Vbe，可选 ZTD 补偿。
  • lutmodel.py 中的 LutBjt 类：封装了 LUTData_BJT，提供了更面向对象的接口。
    • 初始化 (__init__, setup)：接收 Ie 或 Vbe、数量 n、温度 temp、模型 model 和 ztd 标志。初始化 LUTData_BJT 实例。
    • get_vbe, get_Tc, get_ie, get_vbe_T：调用 LUTData_BJT 的相应方法。
    • 属性 (vbe, Tc, ie)：使用 @property 装饰器，惰性计算并缓存结果。
    • show：打印 BJT 的主要参数。

API 用法示例:

五、 总结与注意事项

  • 该代码库提供了一套相对完整的工具，用于生成、测试、处理和使用电容、MOSFET 和 BJT 的查找表。
  • 查找表生成依赖于 pyted 框架和外部仿真器（HSPICE, Spectre）。
  • MOSFET 部分包含了基于传统查找表插值和基于机器学习模型预测两种方法。
  • 机器学习部分使用了 sklearn 和 pytorch。
  • 代码中大量使用了 numpy 进行数值计算和 dill / pickle 进行数据序列化。
  • 测试脚本提供了丰富的使用示例和验证流程。
  • 在使用机器学习模型前，需要先生成查找表数据并进行训练。
  • 使用查找表时，注意输入参数的单位和范围，特别是 L (通常是 um) 和电流/电压值。
  • P 管的电压通常使用负值，但在查找表和模型内部可能已做处理，需根据具体实现确认。

七、 Q&A

Q1: 这些查找表 (LUT) 是如何生成的？精度如何？

A1:

  • 生成方法:
    • 传统查找表 (TD): 通过在指定的 L, W (电阻) 或 L, VGS, VDS, VBS (MOSFET) 范围内进行大量的电路仿真 (使用 HSPICE 或 Spectre)，直接记录仿真得到的器件参数 (如 R, Id, gm, Cgg 等)，形成多维数据网格。
    • 机器学习查找表 (ML/MULTML): 首先使用仿真生成大量的 DC 参数数据点（类似于 TD 方法，但可能更密集或范围更广）。然后，使用这些数据训练机器学习模型（如 MLP 神经网络）。模型学习输入参数 (L, VGS, VDS, VBS) 与输出参数之间的复杂关系。MULTML 指的是使用一个模型同时预测多个输出参数。
  • 精度:
    • TD: 精度主要取决于仿真网格的密度和插值算法（通常是线性插值）。网格越密，插值精度越高，但生成时间和存储空间也越大。在网格点之间，线性插值会引入误差。
    • ML/MULTML: 精度取决于训练数据的质量、模型的复杂度和训练效果。理论上，训练良好的 ML 模型可以比线性插值更精确地捕捉非线性关系，可能在网格点之间提供更高的精度。但模型训练本身也可能引入误差，并且在训练数据范围之外的外插预测通常不可靠。check_acc 和 check_acc_plot 函数可用于评估查找表的精度。

Q2: ML 和 MULTML 方法有什么区别？

A2:

  • ML: 指的是为每个需要预测的输出参数（如 Id, gm, gds 等）单独训练一个机器学习模型。
  • MULTML: 指的是训练一个单一的机器学习模型，该模型能够同时预测多个输出参数。这可能在模型结构和训练效率上有所不同。从代码实现上看 (mlmodel.py vs multoutputmlmodel.py)，它们使用了不同的训练和预测类。

Q3: 如何选择使用 TD, ML, 还是 MULTML 方法？

A3:

  • TD: 优点是直观，生成的数据直接来自仿真。缺点是如果需要高精度，网格需要很密集，导致生成时间长、存储空间大；线性插值可能引入较大误差。适用于对精度要求不高或非线性不强的场景。
  • ML/MULTML: 优点是模型可以学习复杂的非线性关系，可能提供更高的插值精度，并且模型文件通常比密集的 TD 数据网格更小。缺点是需要额外的模型训练步骤，模型训练的好坏直接影响精度，且外插能力有限。MULTML 可能在训练效率和模型大小上相对于为每个参数单独训练 ML 模型有优势。适用于对精度要求较高，且计算资源允许进行模型训练的场景。
  • 选择建议: 如果已有训练好的 ML/MULTML 模型，通常推荐使用它们以获得更好的精度和性能。如果需要快速生成或对精度要求不高，TD 是一个备选方案。可以通过运行 check_acc 或 check_acc_plot 来比较不同方法的精度。

Q4: 代码中 L 和 W 的单位是什么？

A4:

  • 在 lookup 等函数调用时，文档和示例（如电阻查找）明确指出 L 和 W 的单位通常是 微米 (um)，例如 L=1.5 代表 1.5um。
  • 在内部仿真脚本（如 lookuptable_generator_hspice.py）或 pyted 器件实例化时，L 和 W 通常需要传入国际标准单位（米），例如 l=1.5e-6。请注意区分 API 调用时的单位和内部实现时的单位。

Q5: 如果我需要的参数组合超出了查找表的范围怎么办？

A5:

  • TD: 查找表使用插值，如果查询点在数据网格范围内，会进行插值。如果超出了范围，插值函数 (RegularGridInterpolator) 的行为取决于 bounds_error 和 fill_value 的设置。默认情况下，超出范围可能会报错或返回 NaN。代码中的 apply_bound 函数似乎会在插值前将超出范围的输入限制在边界值，这意味着结果可能是边界处的值，精度无法保证。
  • ML/MULTML: 机器学习模型在外插（预测训练数据范围之外的点）时通常表现不佳，结果可能非常不准确或不可靠。代码中也使用了 apply_bound 函数对输入进行限制，同样会导致结果是边界预测值。
  • 建议: 尽量确保您的查询参数在查找表生成的范围内。如果确实需要超出范围的预测，请谨慎对待结果，其精度无法保证。可能需要重新生成覆盖所需范围的查找表。

Q6: 如何重新生成或扩展查找表？

A6:

  • 重新生成: 找到对应的生成脚本（如 lut_generator.py, lookuptable_generator_hspice.py 等），修改其中的扫描范围参数（如 Lrange, vgsrange 等），然后重新运行该脚本的测试函数（如 test_run_t65, test_all_process 等）。
  • 扩展: 如果只是想增加数据点或扩展范围，可以修改扫描参数后重新生成。目前代码库似乎没有提供增量更新查找表的功能，通常需要完全重新生成。

Q7: 如何处理 P 管的负电压？

A7: 代码在多个地方对 P 管的电压进行了处理。通常，外部调用 API 时（如 lookup），用户应传入实际电路中的电压值（P 管 VGS, VDS 通常为负，VBS 通常为正）。在内部进行查找或仿真时，代码（如 lookupfz 或 run_sim）会将 P 管电压转换为与 N 管一致的符号（例如，VGS, VDS 取绝对值或反号，VBS 取绝对值或反号）以匹配查找表数据的存储方式或仿真器的要求。最终返回的结果会是参数的绝对值或根据需要转换回实际符号。使用时需注意函数文档或具体实现细节。
