电路优化

摘要

本文档详细介绍了 ted_optimization 库，一个用于模拟电路自动化设计的 Python 优化框架。该库基于 pyted 框架，提供了多种优化算法和工具，包括基于梯度的方法、演化算法（如差分进化 DE、CMA-ES）、贝叶斯优化（如 MACE、Turbo、BOES）、以及基于强化学习（RL）的电路尺寸自动优化方法。此外，库还提供了网表级优化的 API、结果可视化工具、PVT 角点剪枝以及设计初始化等辅助功能。本文档将按照模块的重要性和使用频率进行组织，首先介绍核心优化器基类和通用的优化方法，然后重点介绍基于强化学习的尺寸优化器和网表级优化 API，最后补充通用工具、使用样例和 Q&A。

1. 核心优化器基类 (opt_base.py)

该模块定义了优化器的基础框架和通用功能。

  • BaseOptimizer 类:
    • 所有优化算法类的基类，提供了优化流程的基本框架。
    • 主要功能:
      • 初始化优化器，设置通用参数（如最大时间、最大迭代次数、性能改进阈值等）。
      • 管理优化变量信息（维度、上下界、步长、枚举值等）。
      • 定义了核心的优化方法接口 (solve, iteration_in_ade)，子类需要实现这些接口。
      • 提供了仿真接口 (update, batched_update, eval_in_ade)，用于调用 pyted 的仿真器执行电路评估。
      • 集成了通用的代价函数 (opt_cost_function)，结合目标和约束计算代价值。
      • 实现了停止标准的检查 (check_stopping_criteria) 和状态更新 (update_stopping_criteria_state_after_iter)。
      • 提供了记录评估结果和更新最优解的功能 (record_evaluations, _update_best_x)。
      • 包含了后处理 (post_process) 和解的合法化 (legalize_x) 方法。
      • 使用 Tracer 类（来自 tacomark.py）记录优化过程和 TopK 设计点。
    • 关键属性:
      • tester: CombinedTester 实例，用于执行电路仿真评估。
      • problem: Problem 实例，定义了优化目标和约束。
      • stopping_criteria: 字符串，指定停止优化的条件。
      • tracer: Tracer 实例，用于记录优化过程。
      • lb, ub, steps, enums: 优化变量的下界、上界、步长和枚举列表。
      • dim: 优化变量维度。
      • best_x, min_cost: 记录优化过程中找到的最优解及其对应的代价值。
      • best_meetsAll_x, min_cost_meetsAll: 记录满足所有约束条件下的最优解。
      • stop_opt: StopOpt 实例，用于控制优化循环的停止。
  • StoppingCriteria 类:
    • 定义了多种静态方法作为优化过程的停止标准。
    • 支持的标准:
      • all_specs_met: 首次满足所有性能指标。
      • out_time_limit: 超过最大优化时间限制。
      • out_iterations_limit: 超过最大迭代次数限制。
      • no_improvement_with_iterations: 性能指标在一定迭代次数内没有显著提升。
      • iterations_after_all_specs_met: 满足所有指标后，继续迭代指定的次数。
  • StopOpt 类:
    • 简单的状态类，用于标记优化是否应该停止。
  • save_bestX_decorator 装饰器:
    • 自动应用于 BaseOptimizer 子类的 solve 方法，确保在返回结果前进行后处理并更新最优解到电路变量。

2. 通用优化方法

库中集成了多种成熟的通用优化算法，可以直接应用于电路优化问题。

2.1 差分进化 (evolution/de/ea.py)

  • DE 类 (继承自 BaseOptimizer):
    • 实现了差分进化（Differential Evolution）算法，具体为 DE/best/1/bin 策略。
    • 基于 geatpy 库实现。
    • 主要参数:
      • pop_size: 种群规模。
      • maxgen: 最大进化代数。
      • mutOper_F: 变异算子的缩放因子 F。
      • recOper_XOVR: 重组算子的交叉概率。
      • maxtrappedcount: 找到满足约束的解后继续优化的代数（用于寻找更优解）。
    • 使用: 初始化 DE 类实例，传入 tester 和 problem，然后调用 solve() 方法启动优化。
    • 样例: benchmarks/tests/test_taco_demo.py#taco_demo
  • 其他多目标优化算法:
    • NSGA_II, NSGA_III, MOEAD: 基于 geatpy 实现了常见的多目标优化算法，接口与 DE 类似。

2.2 贝叶斯优化 (bayes/)

  • MACE 类 (bayes/mace/mace.py, 继承自 BaseOptimizer):
    • 实现了 MACE (Mixed-Integer Accelerated Bayesian Optimization) 算法，适用于包含整数变量的优化问题。
    • 基于 hebo 库实现。
    • 主要参数:
      • init_point: 初始采样点数。
      • maxiters: 最大迭代次数。
      • batch_num: 每次迭代建议的样本数量。
    • 使用: 初始化 MACE 类实例，传入 tester 和 problem，然后调用 solve() 方法。
    • 样例: benchmarks/tests/test_taco_demo.py#taco_demo
  • Turbo 类 (bayes/turbo/turbos.py, 继承自 BaseOptimizer):
    • 实现了基于信赖域的贝叶斯优化算法 (Turbo-1 或 Turbo-m)。
    • 主要参数:
      • method: 'turbo-1' 或 'turbo-m'。
      • init_point: 初始采样点数。
      • maxiters: 最大迭代次数。
      • batch_size: 批处理大小。
      • n_trust_regions: (仅 Turbo-m) 信赖域数量。
    • 使用: 初始化 Turbo 类实例，传入 tester 和 problem，然后调用 solve() 方法。
    • 样例: benchmarks/tests/test_taco/test_ckt.py#test_turbo
  • GeneralBO, Batch_BO, BO_LCB 类 (bayes/vanilla_bo/bayesian.py):
    • 提供了更通用的贝叶斯优化实现，支持约束处理。
    • GeneralBO 基于 bayes_opt 库，支持非线性约束。
    • Batch_BO 基于 hebo 库，支持多目标和约束优化。
    • BO_LCB 基于 hebo 库，使用 LCB (Lower Confidence Bound) 作为采集函数。
  • BOBYQA 类 (bayes/pdfo/bobyqa.py, 继承自 BaseOptimizer):
    • 基于 pdfo 库封装了 Powell 的 BOBYQA (Bound Optimization BY Quadratic Approximation) 算法，适用于无导数的有界约束优化。
    • 主要参数:
      • init_num: 初始插值点数 (npt)。
      • max_eval: 最大函数评估次数 (maxfev)。
    • 使用: 初始化 BOBYQA 类实例，调用 solve() 方法。
  • BOES 类 (bayes/boes/boes.py):
    • 实现了结合贝叶斯优化 (Turbo-1) 和协方差矩阵自适应演化策略 (CMA-ES) 的混合优化算法 BOES。
    • 主要参数:
      • pop_size: 种群/批处理大小。
      • maxiters: 最大迭代次数。
    • 使用: 初始化 BOES 类实例，调用 solve() 方法。

2.3 CMA-ES (evolution/cmaes/cmaes.py)

  • CMAES 类 (继承自 BaseOptimizer):
    • 实现了协方差矩阵自适应演化策略 (Covariance Matrix Adaptation Evolution Strategy, CMA-ES) 算法。
    • 基于 cma-es 的 Python 实现 (cma 库)。
    • 主要参数:
      • init_num: 种群大小 (popsize in cma options)。
      • max_eval: 最大函数评估次数 (maxfev in cma options)。
      • sigma0: 初始标准差。
    • 使用: 初始化 CMAES 类实例，调用 solve() 方法。
    • 样例: benchmarks/tests/test_taco/test_ckt.py#test_cmaes

2.4 基于梯度/灵敏度的优化 (sensitivity/)

  • GD 类 (sensitivity/gd.py):
    • 实现了基于梯度下降 (Gradient Descent) 的优化方法，适用于需要梯度信息的场景。
    • 支持拉格朗日乘子法处理约束。
    • 需要目标函数 func 和梯度函数 sens_func。
    • 主要参数:
      • maxiter: 最大迭代次数。
      • learning_rate: 学习率。
      • gamma: 约束项的惩罚因子。
  • SLSQP_Opt 类 (sensitivity/gd.py):
    • 封装了 SciPy 中的序列最小二乘二次规划 (Sequential Least Squares Programming, SLSQP) 算法。
    • 适用于需要梯度信息的约束优化问题。
    • 需要目标函数 func、雅可比矩阵函数 jac 和约束函数列表 constraints。
  • SensOpt 类 (sensitivity/sens.py):
    • 实现了基于灵敏度分析的优化方法。
    • 通过计算性能对设计参数的灵敏度矩阵，迭代调整参数以满足约束。
    • 依赖于 pyted 框架提供的灵敏度信息（需要电路支持）。

3. 基于强化学习的电路尺寸优化 (ml_designer/)

该模块提供了一套基于强化学习（RL）进行模拟电路尺寸优化的工具。

  • ML_Sizer 类 (ml_designer/ml_sizer.py, 继承自 BaseOptimizer):
    • 核心类，使用 RL 代理进行电路尺寸优化。
    • 封装了 RL 环境 (CktEnv) 和 RL 算法（目前主要是 PPO）。
    • 主要功能:
      • 管理 RL 训练和推理流程。
      • 通过 CktEnv 与电路仿真器交互。
      • 调用 RL 代理 (PPO) 生成动作（参数调整）。
    • 主要参数:
      • batch_episode: 并行训练的 episode 数量。
      • max_episode_len: 每个 episode 的最大步数。
      • num_steps: PPO 算法中每个 rollout 的步数。
      • max_iterations: 最大仿真交互次数（用于训练或推理）。
      • agent: 指定使用的 RL 代理，目前支持 'ppo'。
      • inference: 布尔值，True 表示推理模式，False 表示训练模式。
      • agent_path: 推理模式下，预训练模型的路径。
      • summaray_path: TensorBoard 日志和模型保存路径。
    • 使用:
      • 训练: 初始化 ML_Sizer (inference=False)，调用 learn() 方法。
      • 推理: 初始化 ML_Sizer (inference=True, agent_path='path/to/model')，调用 solve() 方法。
    • 样例: benchmarks/tests/test_rl/test_bgr_auto.py#test_rl
  • PPO 类 (ml_designer/ppo.py):
    • 实现了近端策略优化 (Proximal Policy Optimization, PPO) 算法。
    • 包含 Actor-Critic 网络结构 (Agent 类)。
    • 负责学习策略（如何根据状态选择动作）和价值函数。
    • 集成了 RND (Random Network Distillation) 作为内在奖励模块 (intrinsic_reward_modules/rnd.py) 以鼓励探索。
    • 支持使用离线训练的 DQN 模型 (offline_rl.py) 作为 Vanguard 进行初始化指导（可选）。
  • CktEnv 类 (ml_designer/cktenv.py, 继承自 BaseOptimizer):
    • 定义了符合 gymnasium 接口的电路优化强化学习环境。
    • 主要功能:
      • 将电路状态（性能指标、目标性能、当前参数索引）转换为 RL 代理的观测空间 (observation space)。
      • 将 RL 代理的动作（离散索引变化）转换为实际的电路参数调整。
      • 调用仿真器获取电路性能。
      • 根据性能和目标计算奖励 (reward)。
      • 管理 episodes 的重置和状态转换。
    • 状态空间: 通常包括归一化的当前性能、归一化的目标性能、归一化的参数索引。
    • 动作空间: MultiDiscrete，每个维度对应一个可调参数，动作值为离散的参数索引变化量（如 [-5, -1, 0, 1, 5]）。
    • 奖励函数: 使用 rewards.py 中定义的奖励函数（如 Settaluri_Reward）计算奖励。
  • rewards.py:
    • 定义了用于 RL 的奖励函数类。
    • Settaluri_Reward: 基于 Keertana Settaluri 等人的论文实现的奖励函数，根据性能与目标的相对偏差计算奖励，并在满足所有目标时给予额外奖励。
    • Mohsen_Reward: (未完全实现) 可能基于其他文献的奖励函数。
  • RL_Monitor 类 (ml_designer/rl_monitor.py):
    • 使用 TensorBoard 记录和监控 RL 训练过程，如平均回合奖励、回合长度、成功率等。
    • 提供可视化功能 (visualization) 分析训练完成情况。
  • 其他辅助模块:
    • htask_agent.py: 实现了分层任务代理，可结合快速（廉价）和精确（昂贵）仿真进行训练，先用快速仿真初步训练，再用精确仿真微调。
    • nnd_aware_sizing.py: InitDesignGenerator 类，利用已有的设计数据库，通过 K-近邻 (KNN) 找到与目标性能相似的设计，为优化提供更好的初始点。
    • pvt_pruner.py: PVT_Pruner 类，当考虑 PVT (Process, Voltage, Temperature) 变化时，使用 K-Means 聚类对所有 PVT 角点进行聚类，并选出每簇中最差的角点，以减少需要仿真的角点数量。PVT_Encoder 用于将 PVT 角点编码为数值特征。
    • trick.py: 包含一些 RL 技巧或辅助工具，如状态归一化 (State_Normalization)、离策略轨迹合成 (Offpolicy_Transition_Synthesis, trajectory_synthesis) 等。

4. 网表级优化 API (netlist_opt_api/)

该模块提供了直接基于电路网表（如 SPICE, Spectre MDL, Ocean Script）进行优化的接口。

  • Netlist_Tester_V2 类 (netlist_opt_api/parallel_sim.py):
    • 用于网表级优化的主要 Tester 类。
    • 功能:
      • 复制基础设计文件夹到临时仿真目录。
      • 根据优化算法提供的参数更新网表文件或参数文件。
      • 调用相应的仿真器（目前支持 Spectre MDL, ALPS, Ocean）执行仿真。
      • 解析仿真结果（通常是 .measure 或 .mt0 文件，或通过 Psf2Meas 从波形转换）。
    • 参数:
      • base_folder: 包含完整设计（网表、模型库、仿真设置等）的基础文件夹路径。
      • spfile 或 ocn_path: 主仿真网表文件或 Ocean 脚本的路径（相对于 base_folder）。
      • cktname: 电路名称，用于生成临时目录和日志/结果文件名。
      • perf_class: （仅 Spectre）用于从波形数据中提取性能指标的自定义类（需继承自 spectre_perf.Performance）。
    • 使用:
      1. 准备一个包含完整仿真所需文件的基础设计文件夹 (base_folder)。
      2. 定义 pyted.Variable 来指定优化变量及其范围和步长。
      3. 如果使用 Spectre 仿真器且需要从波形提取性能，需定义一个 Performance 子类。
      4. 实例化 Netlist_Tester_V2。
      5. 将其传递给选择的优化器（如 DE, MACE, CMAES 等）的 tester 参数。
    • 样例: ted/ted_optimization/tests/test_netlist_opt.py#test_ldo, test_tft, test_ocean_opt
  • CombinedTester 类 (netlist_opt_api/parallel_sim.py):
    • 用于组合多个 Tester 实例（可以是 Netlist_Tester_V2 或其他 pyted Tester）。
    • 当一次迭代需要运行多个不同仿真（如 DC, AC, Tran）以获取不同性能指标时使用。
    • build 方法会依次（并行）运行所有包含的 Tester，并将结果合并。
    • 使用: tester = CombinedTester([tester_dc, tester_ac, tester_tran])
    • 样例: ted/ted_optimization/tests/test_netlist_opt.py#test_ldo
  • OceanPreprocess 类 (netlist_opt_api/ocn_preprocess.py):
    • 用于预处理从 Cadence Virtuoso ADE XL 导出的 Ocean 脚本。
    • 功能:
      • 修改脚本中的工程目录和结果路径，使其适用于优化框架。
      • 解析脚本中的 ocnxlSweepVar 命令，提取优化变量信息（名称、初始值、范围、步长），并创建 pyted.Variable。
      • 将扫描变量的设置分离到 param.ocn 文件中，主脚本通过 load("param.ocn") 加载。
      • 清空脚本中 desVar 的默认值（因为优化器会控制变量值）。
      • 修改 cds.lib 文件，使其指向优化运行时使用的库路径。
      • 检查并删除 .cdslck 锁文件。
      • 删除原始仿真结果数据。
    • 使用: 在 Netlist_Tester_V2 初始化时，如果提供了 ocn_path，会自动调用 OceanPreprocess 进行处理。
    • 样例: ted/ted_optimization/tests/test_netlist_opt.py#test_ocn_preprocess
  • 波形与性能提取 (spectre_waveform.py, spectre_perf.py):
    • Waveform, SweepWaveform 类: 用于读取和处理 Spectre 仿真产生的 PSF (或 PSF Bin) 波形数据。
      • 提供 get_signal 方法获取指定节点的电压或电流波形。
      • 支持读取 sweep 仿真结果 (SweepWaveform)。
    • Performance 类: 从 Waveform 对象中提取性能指标的基类。用户需要继承此类并实现 performance 方法，在该方法中编写从波形数据计算所需性能指标的逻辑。
    • Psf2Meas 类: 在 Netlist_Tester_V2 内部使用，当使用 Spectre 仿真器时，调用用户提供的 perf_class (继承自 Performance) 从 PSF 波形数据中提取性能，并将结果写入 .mt0 格式的测量文件，供后续处理。
    • 样例: ted/ted_optimization/tests/test_spectre_waveform.py, test_netlist_opt.py (LDO 示例中的 No2/3/4Performance 类)
  • utils.py:
    • 包含 data_sweep (用于更新 ALPS 网表中的 .data 段) 和 write_param (用于更新 Ocean 脚本的 param.ocn 文件) 等辅助函数。

5. 通用工具与可视化

  • utils.py:
    • variables_all_info(): 从 pyted 的 get_circuit().variables 中提取所有优化变量的维度、上下界、步长、枚举值、类型（实数/整数）和名称。
    • tune_vars_with_step(): 根据步长和边界将连续的变量值调整（量化）为合法的离散值。
    • cost_func(): 通用的代价函数计算，结合目标和约束的归一化值进行计算。
    • parallel_sim(): 使用 pyted 的 DataSweep 功能并行执行多个参数点的仿真。
    • lhs_design_samples(): 使用拉丁超立方采样 (Latin Hypercube Sampling, LHS) 生成初始设计点。
    • set_random_seed(): 设置随机种子以保证可复现性。
  • visua.py:
    • Performance_Visualization 类: 提供了可视化优化结果的功能。
      • draw_n_iters_fom: 绘制 FOM (Figure of Merit) 随迭代次数变化的收敛曲线。
      • draw_foms_cmp: 比较不同优化方法或运行的 FOM 收敛曲线。
      • draw_radars: 使用雷达图可视化多个性能指标的结果。
  • tacomark.py:
    • Tracer 类: 用于跟踪和记录优化过程。

from pyted import sim_acc, Variable, Problem, get_circuit, setenv
from ted_optimization import DE
from ted_optimization.netlist_opt_api import Netlist_Tester_V2 # 假设使用网表优化
import numpy as np

# 1. 设置仿真精度和并行度
sim_acc(n_dim=10, n_process=10, parallel=True)
setenv(simulator='spectre') # 或 'alps', 'spectremdl', 'ocean'

# 2. 定义优化变量 (假设在 Netlist_Tester_V2 初始化时通过 ocean 脚本解析或手动定义)
# Variable(name='w1', value=1e-6, step=0.1e-6).set_bound(0.5e-6, 5e-6)
# Variable(name='l1', value=1e-6, step=0.1e-6).set_bound(0.5e-6, 5e-6)
# ... 其他变量

# 3. 准备 Tester (以 Netlist_Tester_V2 为例)
# 假设 LDO 电路，包含三个测试台，分别测静态电流、负载调整率、线性调整率
# 需要预先定义 No2Performance, No3Performance, No4Performance 类 (继承自 Performance)

tester1 = Netlist_Tester_V2(base_folder=base_folder_path + '/No2Tester',
                            spfile=base_folder_path + '/No2Tester/pre/test_op.scs', # 假设网表路径
                            cktname='LDO_op', perf_class=No2Performance)
tester2 = Netlist_Tester_V2(base_folder=base_folder_path + '/No3Tester',
                            spfile=base_folder_path + '/No3Tester/pre/test_loadreg.scs',
                            cktname='LDO_loadreg', perf_class=No3Performance)
tester3 = Netlist_Tester_V2(base_folder=base_folder_path + '/No4Tester',
                            spfile=base_folder_path + '/No4Tester/pre/test_linereg.scs',
                            cktname='LDO_linereg', perf_class=No4Performance)
tester = CombinedTester([tester1, tester2, tester3])

# 4. 定义优化问题
def ldo_problem(tester):
    # 从 tester 获取性能指标 (指标名称需与 Performance 子类中定义的 key 一致)
    Iq = tester['Quiescent_Current']
    LR = tester['Load_Regulation']
    Vout = tester['vout'] # 假设 No4Performance 返回了 Vout
    LineReg = tester['Line_Regulation']

    return Problem(
        objective=Iq.minimize().rename('Quiescent_Current'),
        constraint=(
            LR.lt(0.133).rename('Load_Regulation'),
            Vout.lt(1.1).rename('vout_ub'),
            Vout.gt(0.8).rename('vout_lb'),
            LineReg.lt(0.2).rename('Line_Regulation'),
            Iq.lt(14.2e-6).rename('Iq_ub'),
            Iq.gt(8e-6).rename('Iq_lb')
        )
    )

# 5. 初始化并运行 DE 优化器
de_configs = {'pop_size': 50, 'maxgen': 100} # DE 特定参数
opt = DE(tester=tester, problem=ldo_problem, **de_configs)
result = opt.solve()

# 6. 输出结果
if result['success']:
    print("优化成功!")
    print("最优参数:", result['Vars'])
    # 可以加载 tracer 查看最优性能
else:
    print("优化失败或未完成。")

7. Q&A

  • Q1: 如何选择合适的优化算法？
    • A1: 选择取决于问题的特性：
      • 如果问题维度不高（<20），且计算梯度可行或成本不高，可以考虑基于梯度的方法 (GD, SLSQP_Opt) 或基于灵敏度的方法 (SensOpt)。
      • 对于黑盒优化问题（梯度未知或计算成本高）：
        • 维度较低时（<20），贝叶斯优化 (MACE, Turbo, GeneralBO) 通常效率较高。
        • 维度较高时（>20），演化算法 (DE, CMAES) 或基于 RL 的方法 (ML_Sizer) 可能更适用。
        • 如果问题包含整数变量，MACE 或 DE (设置 vartypes) 是合适的选择。
        • 如果需要多目标优化，考虑 NSGA_II, NSGA_III, MOEAD 或 Batch_BO (设置 num_obj > 1)。
        • BOBYQA 适用于无导数且有界约束的问题。
        • BOES 结合了 BO 和 ES 的优点，可能在某些问题上表现良好。
      • 如果设计空间探索和自动学习设计策略是目标，可以尝试基于 RL 的 ML_Sizer。
  • Q2: 如何定义优化问题 (Problem 类)？
    • A2: Problem 类需要定义 objective (优化目标) 和 constraint (约束条件)。
      • objective: 使用从 tester 获取的性能指标变量（如 gain = tester['gain']），并调用其 minimize() 或 maximize() 方法。可以 .rename() 以便记录。
      • constraint: 一个包含多个约束表达式的元组。每个约束使用性能指标变量和比较运算符（如 lt (<), gt (>), le (<=), ge (>=)）以及目标值定义，例如 gain.gt(60)。同样可以 .rename()。
      • 对于 RL (ML_Sizer)，还需要提供 agent 参数，传入一个 RLAgent 实例来配置 RL 环境规格。
  • Q3: 如何处理离散或整数优化变量？
    • A3:
      • 定义 pyted.Variable 时，设置 is_int=True 和 step=1。
      • 对于枚举类型，设置 enums=[val1, val2, ...]。
      • 优化器需要能处理整数/离散变量：
        • DE：在其 problem 初始化时，会自动从 Variable 获取 vartypes。
        • MACE：设计上支持混合整数优化。
        • 其他优化器（如 CMAES, Turbo, BOBYQA）主要针对连续变量，但内部的 legalize_x 方法 (tune_vars_with_step) 会在每次评估前将连续值映射到最近的合法离散/整数值，但这可能不是最高效的方式。
  • Q4: 如何使用网表级优化 API (Netlist_Tester_V2)？
    • A4: 参考 样例 1。关键步骤是准备好包含所有仿真所需文件的 base_folder，正确指定 spfile 或 ocn_path，定义优化变量，并（如果需要从波形提取性能）提供 Performance 子类。
  • Q5: 如何记录和可视化优化过程？
    • A5:
      • 大多数优化器（继承自 BaseOptimizer）接受 tracer 参数。传入 Tracer 实例 (from ted_optimization.tacomark import Tracer) 可以记录 FOM 和 Top-K 设计点。例如 tracer = Tracer()，然后 opt = DE(..., tracer=tracer)。
      • RL_Monitor 用于记录 RL 训练过程，日志保存在 summaray_path，可用 TensorBoard 查看。
      • Performance_Visualization 类 (visua.py) 提供了绘制 FOM 收敛曲线和雷达图的方法。你需要从 Tracer 或其他记录中获取数据，然后调用相应绘图方法。
