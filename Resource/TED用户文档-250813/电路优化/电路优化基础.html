电路优化基础

TACO简介

TACO专注于自动化的电路级sizing和优化，能够根据给定工艺下的电路设计指标，利用优化算法快速求解出元件的物理参数，如晶体管的宽度和沟道长度、电容电阻值等。我们设计TACO框架的主要目标有三个：

  1. 为TED用户提供模拟优化器支持
  2. 降低模拟电路优化研究门槛，为算法研究者提供框架支持
  3. 针对TACO构建多个优化测例，便于算法验证与比较

TACO框架提供了一系列易于使用的电路环境交互和优化器接口，TED内核为其提供了底层支持。接下来，我们将首先介绍TACO的整体架构，然后分别从优化流程选项、设计变量、多电路类别支持、优化问题定义和内建优化器这五个方向详细说明TACO中的关键模块设计。

框架简介

与传统面向网表的电路优化工具不同，TACO更加易用和可扩展。TACO主要由仿真环境和优化引擎两部分构成：

  • 仿真环境：负责电路仿真与电路性能获取。
  • 优化引擎：主要涉及对于抽象出的优化问题求解任务。

针对典型的线性/非线性模拟电路类别，TACO当前提供了五类Tester以支持电路仿真与性能获取，包括运放电路（如single-ended OTA, fully-differential OTA, transimpedance amplifier, ring amplifier）、比较器电路、开关电路、电荷泵电路和带隙基准。对于电路仿真性能的获取，TACO提供了两种方式：measure和psf parser。measure主要借助仿真器自带的measure命令计算电路性能，而psf parser则通过对仿真信号的解析按照性能定义进行计算。目前，TACO支持使用spectremdl和alps仿真器的measure方法。TACO的输入不再是SPICE网表，而是在TED语法规则下封装好的Generator Class。

TACO优化引擎为使用者提供了简洁明了的电路优化问题定义与抽象接口，并集成了学术界/工业界常用的电路优化算法。用户可以在TED的电路描述模块中，使用Variable类完成优化变量的定义。借助Problem、Objective、Constraint三个类完成优化问题的抽象与定义。从基准算法中选择优化器，即可实现对定义好的优化问题进行求解。

设计变量

模拟电路晶体管尺寸自动优化的迭代过程中需要不断对电路网表中的设计变量进行更新，仿真更新后的网表以获取对应电路性能。TACO引入了Variable类来实现电路优化变量的定义、边界约束、算术操作与更新等功能，在TED电路模块构建中基于Variable中如下两个方法导出更新的电路优化变量至电路网表：

  • export_netlist_parameters()：应用于正常模式下优化变量导出。
  • export_data_sweep_for_sensitivity()：应用于Sensitivity-Based Optimizer下的优化变量导出。

另外，对于涉及到并行仿真的情况，本工作还设计了DataSweep这个类来进行辅助。DataSweep可以将电路参数矩阵导入到SPICE网表完成参数更新并进行批量化仿真，只需要一行简单的代码就可以完成该操作。针对分布式并行仿真管理，进一步开发了DataSweepPro进行支持。

以下是使用DataSweep类的Python代码示例：

get_circuit().datasweep = DataSweep("sweepdata", params, Vars)

相较于常见优化框架, TACO使用者只需要在电路描述时定义好优化变量, 在优化算法中需要更新电路参数时, 调用Variable.update_all_variables(x)就可以更新电路网表中的优化变量。下面以一个简单的MOSFET参数化为例介绍如何使用Variable定义优化变量。这里定义了三个变量$l1$, $w1$和$m1$, 并且确定了其初始值、步长step值(离散化处理使用)和边界范围。其中因为$m1$为mos管的Multiplier, 因此必须为正整数, 所以需要通过int()方法将其转换为整数形式。这样, 定义的NMOS实际$l=l1u$, $w=w1u$。

# varible unit: um  
u = 1e-6  
l1 = Variable(name="l1", value=0.5, step=0.05).set_bound(0.1, 2)  
wl = Variable(name="wl", value=10, step=0.1).set_bound(1, 20)  
m1 = Variable(name="m1", value=5, step=1).set_bound(1, 20).int()  
Vdd() >> NMOS(l=l1 * u, w=wl * u, m=1) % "Vin" >> Vss()

多电路类别支持

为了满足常见模拟/混合信号电路优化的需求, TED提供了常见的tester, 包括运放、比较器、开关、电荷泵和带隙基准电路。对于指定类别的模拟电路, 相应的tester类内实现了仿真该电路的不同testbench以及获取仿真结果的Performance模块。例如, 运放tester下的子tester涵盖了dc, ac, sr, noise, thd等仿真。TACO之所以将不同的tester独立实现而非尽可能在一次仿真启动中完成, 主要是考虑如下两个因素。一方面, 独立的tester从软件设计的角度来看更加便于继承与维护; 另一方面, 不同的子tester可以像积木一样根据实际情况灵活组合起来以实现定制化tester。在实际优化应用中, 面对非全面性能驱动下的电路优化情况, 用户就可以利用TACO下的CombinedTester将不同的tester combine在一起。

用户可以像访问字典一样直接获取tester中的仿真结果，以BGRTester和OTATester为例说明对于不同的类别的仿真提供的关键的性能访问键。

优化问题定义

模拟电路优化工作首先需要完成的工作就是优化问题的抽象，然后对抽象出来的数学问题借助算法和设计经验进行优化。简单来说，模拟电路优化问题的抽象就是将设计性能指标转化为约束/无约束的数学形式。在TACO中，我们可以做到优化问题的所见即所得。借助Problem、Objective和Constraint三个类，可以快速简洁地定义新的电路优化问题函数。需要注意的是，为避免了不同电路性能间量纲差别影响优化算法收敛，Constraint内部实现了约束函数的归一化操作。因此，在优化算法中我们不需要再单独对约束函数进行归一化处理。

下面我们以一个五管OTA电路为例介绍如何定义优化问题，该电路优化问题的数学形式如下：

最大化 DC Gain
约束条件：

  • Power ≤ 0.4 μW
  • Bandwidth ≥ 20MHz
  • Phase Margin ≥ 60°

电路性能可以从tester中以字典访问形式获取，均为TorchVariable类型。这样，只需要参照上述数学形式便可以完成这样一个有约束优化问题的定义。

def problem(tester: OTATester):  
    from frontend import Problem  
  
    gain = tester["gain"]  
    bw = tester["bw"]  
    pm = tester["pm"]  
    power = tester["power"]  
    gainDB = gain.db()
    return Problem(  
        objective=gainDB.maximize().rename("Av"),  
        constraint=(  
            power.lt(0.4e-3).rename("Power"),  
            bw.gt(20e4).rename("BW"),  
            pm.gt(60).rename("PM"),  
        ),  
    )

内建优化器

TACO目前集成了若干个具有代表性的baseline优化算法，大致可以分为四类：遗传算法、基于灵敏度分析的优化方法、贝叶斯优化算法以及深度强化学习驱动的方法。这些算法在学术界研究中常用作实验基准，对于一般规模电路，往往能够取得不错的优化效果。目前实现的算法如下：

  • 遗传算法

    • DE (Differential Evolutionary)
    • NSGA_II (Non-Dominated Sorting Genetic Algorithm-II)

  • 基于灵敏度分析的优化方法

  • 贝叶斯优化

    • Basic BO with LCB, UCB, EI, POI acquisition functions
    • MACE (Multi-objective Acquisition Ensemble Bayesian Optimization)
    • Batch MOBO (Multi-Objective Bayesian Optimization)

  • 深度强化学习

    • DQN (Deep Q-Network)
    • PPO (Proximal Policy Optimization)

TACO中的baseline算法在不同电路优化case下需要根据问题实际情况调节算法中的参数，以达到理想的优化效果。除此之外，need_prophetVars为布尔型参数，取决于优化器算法本身能否支持利用人工设计的先验解来加速优化收敛速度。

从用户角度介绍如何使用与扩展TACO

在这一节将从用户的视角来介绍如何使用TACO来优化自己的电路以及如何将自定义优化器或tester对接到TACO框架。

使用TACO

输入输出文件

TACO的输入不是spice网表，而是基于TED（Technology-Enabled Design）语言描述的电路模块（封装成生成器形式）。TED提供了特定领域的语法和高度可重用的参数化内置单元来简化电路描述过程，提高开发效率。同时，TED特有的VPDK（Virtual Process Design Kit）使得电路设计可以在所有支持的工艺下实现工艺通用性。这些特点使得TED代码描述的电路设计更利于实现参考设计的复用与共享。

下面我们以五管运放为例，介绍如何描述TACO支持的电路。需要注意的是，TACO目前仅考虑模拟电路前端优化，因此在这个电路描述中，不涉及版图部分。我们构建了名为Single_Stage_PMOS_Input_OTA5T的类，继承自OpampGenerator的父类。在__init__()内完成了优化变量的定义与约束，在core()内实现了核心电路的描述，并在wrapper()内完成外围电路的搭建。

对于优化成功的电路，TACO可以导出该电路的优化报告，包括最终的优化变量值、优化后的电路性能及仿真波形。

from pyted import *
from pyted.modules import *
from pyted.optimization import *


class Single_Stage_PMOS_Input_OTA5T(OpampGenerator):
    def __init__(self):
        super().__init__("diso")
        lmin = self.lmin
        u = 1e-6

        self.M1 = (
            Variable(name="l1", value=0.5).set_bound(
                lmin * 1e6, 100 * lmin * 1e6) * u,
            Variable(name="w1", value=16).set_bound(
                2 * lmin * 1e6, 400 * lmin * 1e6)
            * u,
            Variable(name="m1", value=14, is_int=True).set_bound(1, 20).int(),
        )
        self.M3 = (
            Variable(name="l3", value=0.9).set_bound(
                lmin * 1e6, 100 * lmin * 1e6) * u,
            Variable(name="w3", value=8).set_bound(
                2 * lmin * 1e6, 200 * lmin * 1e6)
            * u,
            Variable(name="m3", value=1, is_int=True).set_bound(1, 20).int(),
        )
        self.M0 = (
            Variable(name="l0", value=0.5).set_bound(
                lmin * 1e6, 100 * lmin * 1e6) * u,
            Variable(name="w0", value=5).set_bound(
                2 * lmin * 1e6, 200 * lmin * 1e6)
            * u,
            Variable(name="m0", value=6, is_int=True).set_bound(1, 20).int(),
        )

    def core(self, VINP, VINN, IDC: Ctrl, Vo: Output):

        vtail, vcur = Node(2)
        l1, w1, m1 = self.M1
        l0, w0, m0 = self.M0
        l3, w3, m3 = self.M3

        Vdd() >> (pm7 := PMOS(l=l0, w=w0, m=1) % (IDC)) >> IDC
        Vdd() >> (pm0 := PMOS(l=l0, w=w0, m=m0) % (IDC)) >> vtail
        vcur >> (nm3 := NMOS(l=l3, w=w3, m=m3) % (vcur)) >> Vss()
        Vo >> (nm4 := NMOS(l=l3, w=w3, m=m3) % (vcur)) >> Vss()
        vcur >> (nm1 := PMOS(l=l1, w=w1, m=m1)) % (VINP) >> vtail
        Vo >> (nm2 := PMOS(l=l1, w=w1, m=m1)) % (VINN) >> vtail

    def wrapper(self, VINP, VINN, Vo: Output):

        IDC = Node()
        dut = self.build()
        (VINP, VINN) >> dut % IDC >> Vo
        (IDC, Vss()) >> ISource(dc=10e-6)
        C(Vo, Vss(), c=5e-12).rename("load")

应用TACO优化

TACO的终极目标还是希望可以给模拟电路自动综合提供一个强大的自动优化工具，用户只需要编写简单的python代码就可以完成电路模块/系统的自动化设计或优化。对于应用TACO完成优化的电路，需要按照TED下生成器形式进行设计，这样描述好的电路就可以直接调用优化器进行优化。

如下例子中我们使用TACO来对前面提到的单级五管运算放大器进行性能优化，整个功能的实现只需几十行简单的python代码。实例化的Single_Stage_PMOS_Input_OTA5T提供单端运算放大器的完整的tester，CombinedTester可用于组合所需的子tester。假设现在我们只关注该电路的直流增益、相位裕度、带宽和1/f噪声，我们可以将gainTester和noiseTester组合起来。借助三个精心设计的类：Problem、Objective和Constraint，使用者可以快速定义电路优化问题。这里，我们选择差分进化作为优化算法来优化电路，优化器的参数根据实际情形与需求可以灵活调整。

@skip_unless(TED_PDK=T65)
def test01():
    with get_circuit().setenv(simulator="spectremdl"):
        ckt = Single_Stage_PMOS_Input_OTA5T()
        ota_tester = ckt.test()
        tester = CombinedTester([ota_tester.gainTester])

        # define optimization problem
        def problem(tester: OTATester):
            gainDB = tester["gain"].db()
            pm, bw = tester["pm"], tester["bw"]
            return Problem(
                objective=gainDB.maximize().rename("Av"),
                constraint=(
                    bw.gt(30e4).rename("BW"),
                    pm.gt(60).rename("PM"),
                ),
            )

        # configure optimizer
        configs = {"pop_size": 30, "maxgen": 10}
        opt = tester.optimize(
            problem=problem,
            optimizer=DE,
            **configs,
        )
        res = opt.solve()
        # export the optimization report
        if res["success"]:
            ckt.report(res=res, is_opt=True)  # 确保save_path已定义
