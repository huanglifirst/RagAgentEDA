仿真优化底层数据结构 (ted_frontend)

摘要

本文档详细介绍了 ted_frontend 库的主要功能和使用方法。ted_frontend 提供了一套用于电路设计自动化、仿真控制、参数优化和结果处理的 Python 工具。核心功能包括：

  • 变量定义与操作: 支持定义可优化变量 (Variable, TorchVariable)，并进行算术运算。
  • 仿真器接口: 封装了仿真流程 (Simulation)，支持多种仿真类型（tran, ac, dc 等），并特别集成了 Xyce 仿真器的灵敏度分析功能 (Xyce)。
  • 结果处理与可视化: 提供 Result 类用于处理仿真原始数据，提取信号，并进行绘图 (plot, show)。
  • 优化问题定义: 允许用户定义优化目标 (Objective) 和约束 (Constraint)，构建优化问题 (Problem)。
  • PyTorch 集成: 利用 TorchVariable 和 T 辅助类，将电路参数和仿真结果与 PyTorch 的自动微分机制结合，用于基于梯度的优化。
  • 强化学习接口: 包含 RLAgent 及相关类，用于支持基于强化学习的电路优化代理。

文档将按照模块的重要性和使用频率进行组织，并结合代码中的测试用例提供详细的使用说明和示例。

1. 变量 (variable, torch_variable)

变量是 ted_frontend 中进行参数化设计和优化的基础。主要有两种变量类型：Variable (核心库 tedcore 提供，前端扩展) 和 TorchVariable (前端特有，集成 PyTorch)。

1.1 Variable (扩展自 tedcore.Variable)

Variable 用于表示设计中的参数，如晶体管的宽度、长度、偏置电压等。

  • 创建变量:

    from pyted import Variable

# 创建名为 'a'，值为 2 的变量
a = Variable(name="a", value=2)
# 创建名为 'b'，值为 3 的变量
b = Variable(name="b", value=3)

  • 算术运算: 变量支持标准的算术运算，运算结果仍然是表达式树。

    c = a + b  # c 的表达式为 "(a)+(b)"
d = a * 2 + b * b - a # d 的表达式为 "(((a)*(2))+((b)*(b)))-(a)"

  • 求值: 使用 .eval() 方法计算表达式的当前值。

    assert c.eval() == 5
assert d.eval() == 11

    示例来源: test_variable.py::test1

  • 设置边界和步长: 可以为变量设置优化时的取值范围和步长。

    # 设置变量 l 的边界为 [0.5, 5]
l = Variable(name="l", value=1).set_bound(0.5, 5)

# 设置变量 m 的边界为 [1, 10]，步长为 1，并指定为整数类型
m = Variable(name="m", value=2, step=1).set_bound(1, 10).int()

    示例来源: test_variable.py::test_float, test_variable.py::test_int

  • 离散枚举值: 可以指定变量只能从一组离散值中选取。

    v = Variable(name="v", value=3, enums=[1, 3, 4, 7])

    示例来源: test_variable.py::test_discrete_enums

  • 特殊操作: 支持 max, min, clamp, round 等操作。

    from pyted import Variable
from ted_device import Mos

lmin = 1e-6
w_min, w_max = Mos.w_bound() # 假设 Mos 器件有宽度边界方法

w1 = Variable(name="w1", value=8e-6, step=1e-6).set_bound(lmin, 100 * lmin)
l1 = Variable(name="l1", value=2e-6, step=0.1e-6).set_bound(lmin, 100 * lmin)
l2 = Variable(name="l2", value=1e-6, step=0.1e-6).set_bound(lmin, 100 * lmin)

# 计算表达式，并限制在 [w_min, w_max] 范围内，最后四舍五入到两位有效数字
w = (5 * w1 * l1 / l2).clamp(w_min, w_max).round(2)

    示例来源: test_variable.py::test_clamp

  • 回调函数: 支持在变量更新时执行回调函数。

    def callback(x):
    return 2 * x + 1

# 创建变量时指定回调
v1 = Variable(name="v1", value=2, step=1, callback=callback).set_bound(1, 100)

    示例来源: test_variable.py::test_callback

1.2 TorchVariable

TorchVariable 继承自 torch.Tensor，专门用于需要利用 PyTorch 自动微分功能的场景，例如基于梯度的优化。它重载了许多操作符和方法，使其行为类似于普通的 torch.Tensor，但增加了与 ted 框架集成的特性。

  • 创建: 通常由仿真结果或其他计算自动生成，或者手动包装 torch.Tensor。

    from ted_frontend.src.torch_variable import TorchVariable
from torch import tensor

a = TorchVariable(tensor(1.0)) # 从 tensor 创建

    示例来源: test_torch_variable.py::test1

  • 运算: 支持 Tensor 的标准运算。

    b = a + 1 # 结果仍然是 TorchVariable

    示例来源: test_torch_variable.py::test1

  • 与优化目标/约束结合: TorchVariable 可以直接用于定义优化目标和约束。

    gain = TorchVariable(...) # 假设 gain 是一个 TorchVariable
objective = gain.maximize() # 定义最大化目标
constraint = gain.gt(60)    # 定义大于 60 的约束

  • 自动命名: auto_rename() 方法尝试根据变量在代码中的名称自动为其命名，便于在目标和约束中引用。

  • 梯度计算: .jacobian() 方法计算该变量相对于电路中所有 torch_variables 的雅可比矩阵 (梯度)。.backward() 方法触发反向传播（需要启用 sensitivity 选项）。

  • 与 NumPy 转换: .to_numpy() 方法将其转换为 NumPy 数组。

1.3 VariableDict

提供一个便捷的方式来管理一组相关的 Variable。

  • 创建:

    from ted_frontend.src.variable import vardict

params = vardict(w=1e-6, l=1e-6, m=1)

  • 批量设置边界:

    params.set_bound(lb=0.5e-6, ub=10e-6)

    示例来源: variable.py

2. 仿真 (simulator, xyce)

仿真模块负责执行电路仿真并获取结果。

2.1 Simulation 类

Simulation 类是通用的仿真接口，根据配置选择后端仿真器（如 HSpice、Spectre 或 Xyce）。

  • 执行仿真: 提供了类方法来运行不同类型的仿真。

    from ted_frontend.src.simulator import Simulation

# 瞬态仿真
tran_raw = Simulation.tran(stop=100e-9, output=["vin", "vout"])

# AC 仿真
ac_raw = Simulation.ac(output="vout")

# DC 工作点仿真
dc_raw = Simulation.dc(output="vout")

# 噪声仿真
noise_raw = Simulation.noise(...)

# 零极点仿真
pz_raw = Simulation.pz(...)

    示例来源: simulator.py, test_xyce.py::test1, test_xyce.py::test2

  • 返回结果: 仿真方法返回一个 Result 对象 (下一节介绍)，用于后续处理。

2.2 Xyce 类 (特定仿真器)

Xyce 类专门用于与 Xyce 仿真器交互，特别是利用其灵敏度分析功能。当 simulator 环境设置为 xyce 时，Simulation 类内部会调用 Xyce。

  • 初始化: 在 Simulation 内部自动完成，用户通常不直接创建 Xyce 对象。初始化时会生成 Xyce 网表。
  • 生成网表: generate_netlist 方法根据当前电路 (current_module()) 和仿真参数生成 Xyce 仿真网表 (.sp 文件)，并包含灵敏度分析所需的指令 (.sens)。
  • 运行仿真: run 或 sim 方法调用 Xyce 执行仿真。
  • 加载结果: loadResult 和 loadSensPrn 方法读取 Xyce 输出的 .prn 文件，解析信号值和灵敏度数据。
  • 获取梯度: get_gradient 方法从灵敏度数据中提取输出信号相对于设计变量的梯度矩阵。
  • PyTorch 集成 (forward): forward 方法是核心，它执行仿真、加载结果、获取梯度，并利用梯度和当前变量值构建一个与 PyTorch 自动微分兼容的 TorchVariable 输出。这样，即使 Xyce 本身不支持反向传播，也能通过其前向灵敏度分析结果，将梯度信息传递给 PyTorch 优化器。
    # Xyce.forward() 内部逻辑示意
# 1. self.sim() -> 调用 Xyce 运行网表
# 2. self.loadResult() -> 加载 .prn 文件
# 3. gradient = self.get_gradient() -> 提取 d(output)/d(variable)
# 4. output = self.signal_dict['output_signal'] -> 获取仿真输出值
# 5. variables = self.variables # PyTorch Tensor 形式的设计变量
# 6. delta = output - gradient.dot(variables.numpy()) # 计算截距 y - (dy/dx) * x
# 7. torch_output = torch.matmul(tensor(gradient), variables) + tensor(delta) # 构建包含梯度的 TorchVariable
# return torch_output
    示例来源: test_xyce.py::test1, test_xyce.py::test2

3. 结果处理 (result)

Result 类用于封装和处理仿真返回的原始数据。

  • 获取信号:

    # tran_raw 是 Simulation.tran() 的返回值 (Result 对象)
# 获取 'vout' 信号的 NumPy 数组
vout_signal = tran_raw.get_signal("vout")

# 获取扫描变量 (如时间或频率)
sweep_values = tran_raw.get_sweeps()

# 获取所有信号名称
signal_names = tran_raw.get_signal_names()

    示例来源: test_xyce.py::test1

  • 插值采样: 可以对信号进行插值并在指定点采样。

    # 在 [start, stop) 区间内，以 step 为步长，对 'vout' 信号进行采样
sampled_vout = tran_raw.sample("vout", start=10e-9, step=1e-9)

    示例来源: result.py

  • 绘图: 提供了便捷的绘图方法。

    # 绘制 vin 和 vout 信号
tran_raw.plot("vin", "vout", name="inv_tran") # 保存为 inv_tran.png

# 在对数坐标轴上绘制
ac_raw.plot("vout", mode="loglog", name="ota_ac")

# 显示所有通过 watch() 注册的波形
tran_raw.show()

    示例来源: test_variable.py::test_w_m_nf, result.py

4. 优化问题定义 (problem, constraint, objective)

这组模块用于定义优化问题，包括目标函数和约束条件。

4.1 Objective 类

继承自 TorchVariable，用于定义优化目标。

  • 创建: 通常通过 TorchVariable 的 maximize() 或 minimize() 方法创建。

    gain = TorchVariable(...) # 来自仿真结果
power = TorchVariable(...)

obj1 = gain.maximize(weight=1.0) # 最大化 gain，权重为 1
obj2 = power.minimize(weight=0.5) # 最小化 power，权重为 0.5

    示例来源: torch_variable.py

  • 显示: .display() 方法返回目标的描述字符串，如 "maximize gain (70.5)".

4.2 Constraint 类

继承自 TorchVariable，用于定义优化约束。

  • 创建: 通常通过 TorchVariable 的 gt() (大于等于) 或 lt() (小于等于) 方法创建。

    pm = TorchVariable(...)  # 相位裕度
gbw = TorchVariable(...) # 增益带宽积

con1 = pm.gt(60, weight=1.0)  # 约束 pm >= 60
con2 = gbw.gt(100e6, weight=1.0) # 约束 gbw >= 100MHz
con3 = pm.lt(90, weight=0.8)   # 约束 pm <= 90

    示例来源: torch_variable.py

  • 约束函数: 内部使用 eval 方法将约束转化为一个标量值，该值在满足约束时 <= 0，违反约束时 > 0。通常使用类似 sigmoid 或线性函数进行转换，并乘以权重。

  • 显示: __str__ 方法提供约束的可视化表示，并根据满足情况（满足、接近、违反）使用不同颜色。

    # 示例输出:
# pm(75.2) >= 60  (绿色，满足)
# gbw(95.1M) >= 100M (黄色，接近)
# area(1.2e-10) <= 1e-10 (红色，违反)

4.3 Problem 类

Problem 类用于将优化目标和约束组合在一起，定义一个完整的优化问题。

  • 创建:

    from ted_frontend import Problem

# 单目标优化
problem1 = Problem(
    objective=gain.maximize(),
    constraint=(
        pm.gt(60),
        gbw.gt(100e6),
    )
)

# 多目标优化 (需要确保目标转化为最小化形式)
problem2 = Problem(
    objective=(
        power.minimize(), # 最小化功耗
        -gbw.maximize()  # 最大化 GBW (取反转化为最小化)
    ),
    constraint=(gain.db().gt(60),)
)

    示例来源: problem.py 中的文档字符串

  • 显示: 创建 Problem 对象时，会打印出优化问题的描述信息。

5. PyTorch 辅助工具 (torch)

T 类提供了一系列静态方法，封装了 torch 和 numpy 中的常用数学函数，使其能同时处理 torch.Tensor 和普通 Python 数字/NumPy 数组。这在编写需要同时兼容这两种类型的计算（例如性能指标计算）时非常有用。

  • 用法:
    from ted_frontend.src.torch import T
import torch

a = torch.tensor([-1.0, 2.0])
b = np.array([0+1j, 1+0j])

abs_a = T.abs(a) # torch.abs
abs_b = T.abs(b) # np.abs

angle_b_rad = T.angle(b) # np.angle (radians)
angle_b_deg = T.angle(b, deg=True) # np.angle (degrees)

phase_b = T.phase(b) # 计算相位 (度)，包含解环绕 T.rad2deg(T.unwrap(T.angle(b)))

db20_a = T.db20(a) # 20 * T.log10(T.abs(a))
    示例来源: torch.py

6. 强化学习代理 (agent)

agent.py 文件定义了与强化学习 (RL) 相关的类，用于基于 RL 的电路优化。

  • MetricSpec: 定义单个性能指标的规格，包括名称、目标范围 (Range)、归一化值、优化方向 (beta：1 表示最大化，-1 表示最小化)、权重、缩放因子和上限。
  • MetricSpecs: 包含多个 MetricSpec 的列表。
  • RLAgent: 表示 RL 优化代理。
    • 初始化: 需要提供 MetricSpecs (TT corner)、奖励函数 (reward)、代理模型路径 (agent_path) 和奖励阈值。
    • PVT 支持: gen_pvt_specs 方法可以根据环境变量 PVT 自动生成所有 PVT 角的 MetricSpecs。

样例 3: 定义优化问题

from pyted import testbench, setenv, debug
from ted_frontend import Simulation, Problem, TorchVariable, T

# 假设已经执行了 AC 仿真并得到了 ac_raw (Result 对象)
# 假设 gain_db, phase_deg, power 是从仿真结果计算得到的 TorchVariable

@testbench
def define_optimization_problem():
    # 假设这些变量已经从仿真中获取并是 TorchVariable
    # gain_db = ...
    # phase_deg = ...
    # power = ...
    # freq = ac_raw.get_sweeps() # NumPy array

    # --- 计算性能指标 ---
    # 找到 0dB 频率 (GBW 频率近似)
    gain_abs = T.abs(ac_raw.get_signal("vout")) # 幅度
    zero_db_freq_index = T.nonzero(gain_abs < 1)[0] # 第一个小于1的点
    if len(zero_db_freq_index) > 0:
        gbw_freq = freq[zero_db_freq_index[0]]
        # 需要确保 gbw_freq 也是 TorchVariable 并包含梯度
        # (这部分需要更复杂的插值和梯度传递，这里简化处理)
        gbw = TorchVariable(torch.tensor(gbw_freq)) # 假设 gbw 是 TorchVariable
    else:
        gbw = TorchVariable(torch.tensor(0.0)) # 无法计算

    # 计算相位裕度 (在 0dB 频率处的相位)
    if len(zero_db_freq_index) > 0:
         phase_at_gbw = T.phase(ac_raw.get_signal("vout", pos=gbw_freq)) # 插值获取相位
         pm = 180.0 + phase_at_gbw # 相位裕度
         pm = TorchVariable(pm) # 假设 pm 是 TorchVariable
    else:
         pm = TorchVariable(torch.tensor(-180.0))

    # --- 定义优化问题 ---
    # 目标：最大化 GBW，最小化 Power
    # 约束：Gain > 60dB, PM > 60 degrees

    # 假设 dc_gain_db 是从 DC 或低频 AC 仿真得到的 TorchVariable
    # dc_gain_db = ...

    opt_problem = Problem(
        objective=(
            gbw.maximize(weight=1.0), # 最大化 GBW
            power.minimize(weight=0.5)  # 最小化 Power (权重较低)
        ),
        constraint=(
            # dc_gain_db.gt(60), # 假设有 DC 增益约束
            pm.gt(60, weight=1.0),  # PM 约束
        )
    )

    debug("Optimization Problem Defined.")
    # 后续可以将 opt_problem 传递给优化器

8. Q&A

  • Q1: Variable 和 TorchVariable 有什么区别？何时使用？

    • A1: Variable 是基础的参数表示，主要用于构建电路和传递参数给仿真器。它通过 .eval() 进行求值。TorchVariable 继承自 torch.Tensor，专门用于需要梯度信息的场景，例如基于梯度的优化。它与 PyTorch 的自动微分系统集成，可以通过 .backward() 或 .jacobian() 获取梯度。当你需要进行基于梯度的优化时，应确保你的性能指标最终表示为 TorchVariable。

  • Q2: 为什么 Xyce 仿真需要特别处理 (Xyce 类)？

    • A2: Xyce 支持前向灵敏度分析（计算 d(output)/d(parameter)），但本身不像某些现代框架那样直接支持反向自动微分。Xyce 类通过运行 Xyce 的灵敏度分析，获取前向梯度，然后利用这些梯度构建等效的 TorchVariable，从而将梯度信息桥接到 PyTorch 的优化流程中。这样可以在不修改 Xyce 的情况下，利用其仿真结果进行梯度优化。

  • Q3: 如何获取仿真信号的梯度？

    • A3:
      1. 使用 Xyce: 设置 setenv(simulator='xyce', sensitivity=True)，运行仿真 (Simulation.ac, Simulation.tran 等) 并指定 output 参数。返回的 Result 对象中包含的信号（如果计算正确）将是 TorchVariable，可以直接调用 .jacobian() 或 .backward()。
      2. 其他仿真器 (不支持灵敏度): 对于不支持内置灵敏度或未实现接口的仿真器，通常无法直接获取梯度。可能需要依赖无梯度优化方法，或使用有限差分法估计梯度（但这通常效率较低且精度不高）。

  • Q4: Result.show() 和 Result.plot() 有什么不同？

    • A4: Result.plot() 用于绘制用户明确指定的若干信号，并保存为单个图像文件。Result.show() 用于显示在代码中通过 watch() 函数注册的所有波形组。watch() 通常在 testbench 中用于分组记录需要观察的信号，show() 则在仿真结束后根据这些记录自动生成并保存多个绘图页面。

  • Q5: 如何定义多目标优化问题？

    • A5: 在创建 Problem 对象时，将 objective 参数设置为一个包含多个 Objective 对象的元组。请注意，通常优化器解决的是最小化问题，因此对于需要最大化的目标，需要取反（例如 gbw.maximize() 内部会处理为 -gbw）或手动取反传递给 Problem。

  • Q6: ted_frontend 主要适用于哪些场景？

    • A6: 主要适用于模拟和混合信号电路设计的自动化流程，包括：
      • 参数化电路生成。
      • 调用 SPICE 类仿真器进行仿真验证。
      • 基于仿真结果的性能指标自动计算。
      • 使用梯度信息（通过 Xyce 灵敏度或未来可能的其他方式）或无梯度方法进行电路参数优化。
      • 集成强化学习进行电路自动设计探索。
