电路优化进阶

灵敏度分析

灵敏度方法通过分析电路性能与设计变量之间的敏感性关系来求得“梯度”以完成优化。对于目标函数𝑂是性能𝑝的函数，电路中 设计变量为𝑥，那么我们可以得到如下灵敏度公式：

实际实现时，∂𝑂/∂𝑝 很容易得到，而 ∂𝑝/∂𝑥 的计算则比较关键。目前有两种方法可以得到，第一个方法是借助内建支持计算灵敏度的SPICE仿真器——xyce(开源)/ALPS(商用)，另一个方法就是通过构建微扰方程，利用差分法计算灵敏度矩阵。实际使用中更倾向于后者，因为该方法针对不同的SPICE仿真器。

基于灵敏度分析的电路优化

当我们能够获得电路性能与设计参数之间的灵敏度矩阵，就可以借助该梯度信息指导电路优化。TED下目前采用基于差分法计算的灵敏度矩阵来代表梯度信息，为了能够实现反向梯度传播，引入了TorchVariable类，继承了pytorch的Tensor。电路性能定义为TorchVariable类型，可以支持自动梯度计算。借助拉格朗日乘子法/顺序最小二乘和获得的梯度信息实现了我们的GD优化算法。我们这里用一个简单电阻分压网络来验证灵敏度矩阵的正确性，想要获取输出电压相对于设计变量R0和R1的灵敏度矩阵，只需要从仿真波形获取输出电压信号sig = T.abs(re[10])，借助TorchVariable提供的jacobian()函数获取即可：j = sig.jacobian()。

def test():
    if get_circuit().get_pdk().name != "t65":
        return

    @testbench
    def ota():
        from modules import Mosfet, Resistor, VSource

        with get_circuit().setenv(simulator="hspice", sensitivity=True):
            (Vdd(), Vss()) >> VSource(dc=1.2).rename("Vdd")
            R0 = Variable(value=1e4).set_bound(1e3, 5e4)
            R1 = Variable(value=2e4).set_bound(1e3, 5e4)
            (Vdd(), "vout") >> Resistor(r=R0).rename("r1")
            ("vout", Vss()) >> Resistor(r=R1).rename("r2")

            raw = tran(output="vout")
            re = raw.get_signal("vout")
            sig = T.abs(re[10])

            j = sig.jacobian() # 获取输出电压相对于变量R0, R1的灵敏度矩阵
            import copy

            ja = copy.deepcopy(j)
            get_circuit().torch_variables.grad.zero_()
            debug(f"sig={sig}, obj_jac={ja}")
            # sig=0.7999999999999999, obj_jac=[-2.58064747e-05  1.24999940e-05]
            assert abs(sig - 0.8) < 1e-13  # 确保仿真结果、灵敏度无误
            assert abs(ja[0] + 2.58064747e-05) < 1e-13
            assert abs(ja[1] - 1.24999940e-05) < 1e-13

利用仿真加速电路优化

这里，我们通过引入并行/分布式仿真，结合算法的并行性，能够实现电路优化过程的进一步加速。

基于强化学习的电路优化

我们的工作需要基于TED平台来实现一套实际可行的RL框架，该框架实现的batched同步并行环境能够提供更高的样本采样 率。结合TED提供的分布式并行仿真，可以进一步将样本采样率提高，这使得我们的框架在实验测试与应用中具备了可接受的时间约束，具有实际应用价值。

环境

强化学习 agent 交互的环境就是电路设计环境。电路设计环境主要负责的任务是根据 agent 返回的 action 更新电路设计参数，更新电路网表后调用 SPICE仿真器来完成电路仿真，最后借助仿真数据处理处理模块得到新的电路性能。

状态空间

状态空间主要由当前电路性能、设计目标性能以及当前电路设计参数共同组成。

动作空间

设计参数离散化处理。例如选取三个动作 𝐴 = {−1, 0, 2} 来对离散化参数格点进行调整，根据动作确定调整的步长距离。假设对于设计参数 𝑥 离散化后每个步长 𝑠𝑡𝑒𝑝 = 𝛿𝑥，在动作 A 下设计参数 𝑥 每次更新时有如下三种调整情况：增大 (𝑥 + 2𝛿𝑥)，保持不变 (𝑥) 以及减小 (𝑥 − 𝛿𝑥)。

奖励函数

目标

训练框架采取同步分布式并行结构，𝑁 条 episode 同时执行训练。环境复位时，从电路每个性能指标范围内随机采样作为设计目标以构成目标设计向量𝑂∗。

Agent

PPO(Proximal Policy Optimization，近端策略优化)作为实用性广泛的基于AC 结构的 on-policy 强化学习算法，在实际工程落地中经常被采用。

如何使用(以VCO为例)

问题定义

和电路优化基础介绍的优化问题定义一样，使用RL优化唯一差别是需要在Problem中定义agent，主要是定义所考虑电路性能的优化范围，比如工作频率在560MHz到570MHz。

MetricSpec中用户需要定义的内容包括：

  • spec: tester中可以获取的电路性能
  • name: spec在tester中获取的键名, 详情请参考已实现的测试器说明
  • range: 电路性能优化范围, 优化训练过程中会从该范围随机采样生成设计目标进行优化, 需要根据实际工艺和电路结构合理设置, 不可能实现的极端性能定义是不可行的, eg. 两级五管运放增益设置在 (60dB, 75dB), 而对于(100dB, 200dB)本身就不可能实现, (10dB, 30dB)又没有优化意义, 性能过低
  • normalize: 该结构电路在当前工艺下常规性能
  • beta: 对于电路性能如果想越大越好，beta=1; 如果希望越小越好, 如电路功耗, 则beta=-1
  • weight: 优化方法将多目标加权为单目标, 如obj=w1f1+ w2f2..., 根据对不同性能的侧重程度设置加权系数, 默认都为1
  • maximum_limit: 前面的beta只能针对单调优化目标, 比如越大越好，越小越好，但是有些电路性能还会有上下界约束, 非单调。例如运放的相位裕度, 通常60<PM<90, 因此还需要设置最大值约束。
def VCO_Opt_Problem():

    vco = SPLL_VCO()
    vco_tester: VCOTester = vco.test()
    tester = CombinedTester(
        [
            vco_tester.freTester,
        ]
    )

    def problem(tester: VCOTester):
        fre = tester["fre"]
        pw = tester["pw"]

        return Problem(
            objective=pw.minimize().rename("pw"),
            constraint=(
                fre.lt(570e6).rename("fre"),
                fre.gt(560e6).rename("fre"),
            ),
            agent=RLAgent(
                specs=MetricSpecs(
                    [
                        MetricSpec(
                            pw,
                            "pw",
                            (640e-6, 860e-6),
                            normalize=750e-6,
                            beta=-1,
                            weight=1,
                        ),
                        MetricSpec(
                            fre,
                            "fre",
                            (560e6, 569e6),
                            normalize=565e6,
                            beta=1,
                            weight=1,
                            maximum_limit=570e6,
                        ),
                    ]
                ),
                reward=Settaluri_Reward,
                agent_path=None,
            ),
        )

    return vco, tester, problem
