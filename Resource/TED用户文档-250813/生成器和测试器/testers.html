TED提供的测试器

摘要

本文档详细介绍了 circuits.testers 模块，该模块提供了一系列用于模拟和评估各种电路（如运算放大器 OTA、比较器 CMP、带隙基准 BGP 等）性能的测试平台（Tester）和性能评估类（Performance）。文档涵盖了核心基类 TesterBase 和 Performance，以及针对不同电路和性能指标的具体实现，例如 OTATester、CMPTester、BGPTester 及其相关的子测试器和性能计算类。此外，文档还包含了详细的 API 用法示例和常见问题解答，旨在帮助用户理解和使用该模块进行电路仿真与性能分析。

核心基类

1. TesterBase

  • 路径: circuits/testers/src/tester.py
  • 描述: 所有具体测试平台的基类。它定义了测试器的基本框架和通用方法，如运行测试 (run)、获取仿真原始数据 (raw, get_raw)、获取性能指标 (__getitem__) 等。子类需要实现 testbench 方法来定义具体的测试电路和仿真设置。
  • 主要方法:
    • __init__(self, params={}, **kwargs): 初始化基类，设置名称、模式等。
    • run(self): 运行测试平台，执行 testbench 中定义的仿真。
    • dut(self): 获取被测器件 (Device Under Test) 的实例。子类通常通过调用 self.module 来获取生成器或模块实例。
    • performance(self): 返回一个包含计算出的性能指标的字典。子类通常会关联一个 Performance 子类来计算具体指标。
    • testbench(self, *args, **kwargs): （子类必须实现） 定义测试电路的连接和仿真命令。
    • plot(self, **kwargs): 绘制仿真结果波形图。
    • summary(self): 打印仿真结果摘要。
    • __getitem__(self, key): 获取指定的性能指标值。

2. Performance

  • 路径: circuits/testers/src/performance.py
  • 描述: 所有性能评估类的基类。用于从仿真结果 (raw 数据或 measure 结果) 中计算和提取特定的电路性能指标。
  • 主要方法:
    • __init__(self, tester: "TesterBase" = None, raw: "Waveform" = None, is_swept=False): 初始化，接收 TesterBase 实例或直接接收 Waveform 数据。
    • get_performance(self): （核心方法） 计算并返回所有定义的性能指标。这个方法会调用子类实现的 performance 方法，并处理扫描仿真 (sweep simulation) 的情况。
    • performance(self): （子类必须实现） 定义如何计算具体的性能指标，返回一个包含指标名称和值的字典。
    • plot(self, **kwargs): 绘制与性能指标相关的波形图。
    • __getitem__(self, key): 获取计算出的特定性能指标值。

运算放大器测试模块 (ota_tester)

1. OTATester

  • 路径: circuits/testers/src/ota_tester/ota_tester.py (主类), circuits/testers/testers/ota_tester.py (旧版实现)
  • 描述: 用于测试运算放大器 (OTA) 性能的主测试器。它组合了多个子测试器来评估 OTA 的各种直流、交流、瞬态和噪声性能。支持单端输入单端输出 (diso) 和差分输入差分输出 (dido) 两种类型的 OTA。
  • 初始化参数:
    • module: 被测 OTA 的模块实例或生成器。
    • name (str): 测试器名称。
    • vdd (float): 电源电压。
    • OTAtype (str, 可选): OTA 类型，"diso" 或 "dido"，默认为 "diso"。
    • vcmi (float, 可选): 输入共模电压，默认为 vdd / 2。
    • vcmo (float, 可选): 输出共模电压 (仅用于 dido)，默认为 vdd / 2。
    • skip_pz (bool, 可选): 是否跳过零极点 (PZ) 分析，默认为 True (除非仿真器是 spectre 且 stage 不是 post)。
    • actester_settings (dict, 可选): 传递给 AC 相关测试器的额外设置。
    • srtester_settings (dict, 可选): 传递给 SR 测试器的额外设置。
    • noisetester_settings (dict, 可选): 传递给 Noise 测试器的额外设置。
    • thdtester_settings (dict, 可选): 传递给 THD 测试器的额外设置。
    • offsettester_settings (dict, 可选): 传递给 Offset 测试器的额外设置。
    • save_op (bool, 可选): 是否保存工作点信息，默认为 False。
    • save_currents (bool, 可选): 是否保存电流信息，默认为 False。
  • 主要功能: 运行一系列子测试（DC, Gain, PSRR, CMRR, Noise, SR, ICMR, OPS, Offset, THD, PZ (可选), OP (可选)），计算并汇总 OTA 的关键性能指标。
  • 核心方法:
    • build(self): 构建并运行所有相关的子测试器。
    • summary(self): 打印格式化的性能总结报告。
    • plot(self): 调用各个子测试器的 plot 方法绘制波形图。
  • 返回性能指标示例:
    • power: 功耗
    • gain, gaindb: 开环增益 (线性值和 dB 值)
    • gbw: 增益带宽积
    • ugb: 单位增益带宽
    • bw: -3dB 带宽
    • pm: 相位裕度
    • cmrr: 共模抑制比 (dB)
    • psrr: 电源抑制比 (dB)
    • psrr@1K, psrr@1M: 特定频率下的 PSRR
    • SRp, SRn: 正负压摆率
    • offset: 输入失调电压
    • f1, noise_1M, thermal: 不同频率下的等效输入噪声
    • ICMRmin_OPE, ICMRmax_OPE: 输入共模范围
    • OPSmin_OPE, OPSmax_OPE: 输出摆幅
    • thd: 总谐波失真
    • pole, zero: 零极点 (如果 skip_pz 为 False)

2. 子测试器和性能类 (位于 circuits/testers/src/ota_tester/ 和 circuits/testers/src/performances/ota/)

  • DCTester / PowerPerformance: 测试直流工作点和功耗。
    • testbench: 设置基本的直流偏置电路。
    • get_power: 计算功耗 vdd * Ivdd。
  • GainTester / ACPerformance: 测试交流特性（增益、带宽、相位裕度等）。
    • testbench: 构建用于 AC 分析的测试电路，包括差分和共模信号的施加方式。
    • get_dm_gain: 获取差模增益。
    • get_BW: 计算 -3dB 带宽。
    • get_Phase_margin: 计算相位裕度。
    • get_UGB: 计算单位增益带宽 (0dB 带宽)。
    • get_GBW: 计算增益带宽积。
    • plot: 绘制增益和相位的波特图。
  • PSRRTester / PSRRPerformance: 测试电源抑制比。
    • testbench: 将 AC 信号施加到电源 VDD 上，测量输出响应。
    • get_psrr: 获取 PSRR (通常通过增益计算)。
  • CMRRTester / CMRRPerformance: 测试共模抑制比。
    • testbench: 同时向 Vip 和 Vin 施加同相的 AC 信号，测量输出响应。
    • get_cmrr / get_cm_gain: 获取共模增益，用于计算 CMRR (CMRR = Gain / CM_Gain)。
  • NoiseTester / NoisePerformance: 测试噪声特性。
    • testbench: 构建用于噪声分析的电路。
    • get_Thermal_f1_noise: 获取特定频率点 (如 1kHz, 1MHz, 10MHz) 的等效输入噪声。
    • plot: 绘制输入参考噪声谱密度图。
  • SRTester / SRPerformance: 测试压摆率。
    • testbench: 输入端施加一个大的脉冲信号，通常是从 0 到 VDD。
    • get_sr: 通过计算输出电压波形的最大/最小斜率来得到正/负压摆率。
    • plot: 绘制输入脉冲和输出响应波形。
  • ICMRTestr / ICMRPerformance: 测试输入共模范围。
    • testbench: 扫描输入共模电压 (VCM)，通常配置为单位增益跟随器模式。
    • get_icmr: 通过分析输出电压随输入共模电压变化的曲线斜率，找到斜率接近 1 的范围，确定 ICMR。
  • OutpSwingTester / SwingPerformance: 测试输出摆幅。
    • testbench: 扫描输入共模电压，但电路配置略有不同，目的是观察输出电压的饱和范围。
    • get_ops: 通过分析输出电压随输入共模电压变化的曲线斜率，找到斜率较大的区域的输出电压范围。
  • OffsetTester / OffsetPerformance: 测试输入失调电压 (主要针对 dido OTA)。
    • testbench: 构建带有大电阻反馈的电路，使输出共模电压对输入失调敏感。
    • get_offset: 通过测量差分输出电压 abs(vout - Von) 并除以闭环增益来计算失调。
  • THDTester / THDPerformance: 测试总谐波失真。
    • testbench: 输入端施加一个特定频率（如 10kHz）的正弦信号。
    • get_thd: 对输出信号进行 FFT 分析，计算谐波分量与基波分量的功率比。
  • PZTester / PZPerformance: 进行零极点分析 (需要 Spectre 仿真器)。
    • testbench: 构建适合零极点分析的电路配置。
    • get_pz: 从仿真结果中提取零点和极点信息。
  • OPTester / OPPerformance: 获取器件工作点信息。
    • testbench: 运行直流仿真，并设置保存工作点信息。
    • get_dc_parameters: 获取指定 MOS 管的工作区 (region) 等参数。
    • create_op_table: 生成格式化的工作点信息表格。

比较器测试模块 (cmp_tester)

1. CMPTester

  • 路径: circuits/testers/src/cmp_tester/cmp_tester.py
  • 描述: 用于测试比较器 (CMP) 性能的主测试器。组合了多个子测试器来评估比较器的延迟、功耗和回踢噪声等。
  • 初始化参数:
    • module: 被测比较器的模块实例或生成器。
    • name (str): 测试器名称。
    • vdd (float): 电源电压。
    • stop (float): 瞬态仿真停止时间。
    • f_ck (float, 可选): 时钟频率，默认为 1e-8 (100MHz)。
    • vdm (float, 可选): 差分输入电压，默认为 0.02V。
  • 主要功能: 运行 delay_Tester, pw_Tester, kickback_Tester, Input_Offset_Tester 来评估比较器性能。
  • 核心方法:
    • build(self): 构建并运行所有相关的子测试器。
    • summary(self): 打印格式化的性能总结报告。
    • plot(self): 调用子测试器绘制相关波形。
  • 返回性能指标示例:
    • delay: 传输延迟
    • power: 功耗
    • kickback: 回踢噪声
    • input_offset_voltage: 输入失调电压

2. 子测试器和性能类 (位于 circuits/testers/src/cmp_tester/ 和 circuits/testers/src/performances/cmp/)

  • delay_Tester / DelayPerformance: 测试传输延迟。
    • testbench: 在输入端施加一个小的差分电压 (vdm)，并给出时钟信号，测量从时钟边沿到输出翻转的时间差。
    • get_delay: 计算传输延迟。
    • plot: 绘制时钟和输出信号波形。
  • pw_Tester / CmpPowerPerformance: 测试功耗。
    • testbench: 设置比较器正常工作的偏置和时钟，测量电源电流。
    • get_power: 计算平均功耗。
  • kickback_Tester / KickbackPerformance: 测试回踢噪声。
    • testbench: 在输入端施加差分电压，观察时钟触发时输入端电压的变化。
    • get_kickback: 计算输入端电压的最大波动值。
    • plot: 绘制输入端电压波形。
  • Input_Offset_Tester / InputOffsetVoltagePerformance: 测试输入失调电压。
    • testbench: 通过扫描差分输入电压或特定电路配置来测量使输出翻转的输入电压差。 (代码中使用了 PWL 扫描输入)。
    • get_offset: 计算失调电压。

带隙基准测试模块 (bgp_tester)

1. BGPTester

  • 路径: circuits/testers/src/bgp_tester/bgp_tester.py
  • 描述: 用于测试带隙基准 (BGP) 电路性能的主测试器。
  • 初始化参数:
    • module: 被测 BGP 的模块实例或生成器。
    • name (str): 测试器名称。
    • vdd (float): 电源电压。
  • 主要功能: 运行 DCTester, TCTester, PSRRTester, STBTester (Spectre), NoiseTester (Spectre), LNRTester (Spectre) 来评估 BGP 性能。
  • 核心方法:
    • build(self): 构建并运行所有相关的子测试器。
    • summary(self): 打印格式化的性能总结报告。
    • pvt_summary(self): 如果设置了 PVT 扫描，打印 PVT 结果表格。
    • plot(self): 调用子测试器绘制相关波形 (TC, Noise, LNR)。
  • 返回性能指标示例:
    • power: 功耗
    • vref: 输出参考电压
    • tc: 温度系数 (ppm/℃)
    • tc0: 零温度系数点 (℃) (Spectre)
    • tc0_r: 零温度系数范围 (℃) (Spectre)
    • psrr: 电源抑制比 (dB)
    • loopgain: 环路增益 (dB) (Spectre)
    • loopbw: 环路带宽 (Hz) (Spectre)
    • loopugb: 环路单位增益带宽 (Hz) (Spectre)
    • loopgbw: 环路增益带宽积 (Hz) (Spectre)
    • looppm: 环路相位裕度 (°) (Spectre)
    • noise@1k, noise@1M, noise@10M: 不同频率下的输出噪声 (nv/√Hz) (Spectre)
    • lnr: 线性调整率 (v/v) (Spectre)

2. 子测试器和性能类 (位于 circuits/testers/src/bgp_tester/ 和 circuits/testers/src/performances/bgp/)

  • DCTester / DCPerformance: 测试直流功耗和输出电压。
    • testbench: 设置基本直流偏置。
    • get_power, get_vref: 计算功耗和获取 Vref。
  • TCTester / TCPerformance: 测试温度系数。
    • testbench: 扫描温度，测量输出电压 Vref。
    • get_tc: 计算温度系数 ((Vmax - Vmin) / Vmean) / (Tmax - Tmin)。
    • get_tc0: 查找零温度系数点 (需要分析 Vref-T 曲线的极值点) (Spectre)。
    • plot: 绘制 Vref 随温度变化的曲线。
  • PSRRTester / PSRRPerformance: 测试电源抑制比。
    • testbench: 将 AC 信号施加到电源 VDD，测量输出电压 Vref 的响应。
    • get_dm_gain: 获取 PSRR (通常为输出电压变化 / 电源电压变化)。
  • STBTester / STBPerformance: 测试环路稳定性 (需要 Spectre)。
    • testbench: 使用 stb 分析。
    • 包含与 ACPerformance 类似的方法来计算环路增益、带宽、相位裕度等。
  • NoiseTester / BGPNoisePeformance: 测试输出噪声 (需要 Spectre)。
    • testbench: 运行 noise 分析，测量输出端的噪声。
    • get_out_noise: 获取特定频率点的输出噪声谱密度。
    • plot: 绘制输出噪声谱密度图。
  • LNRTester / LNRPerformance: 测试线性调整率 (需要 Spectre)。
    • testbench: 扫描电源电压 VDD，测量输出电压 Vref。
    • get_LNR: 计算线性调整率 (Vmax - Vmin) / (VDDmax - VDDmin)。
    • plot: 绘制 Vref 随 VDD 变化的曲线。

其他测试模块 (简述)

  • Charge_Pump_Tester (cp_tester): 用于测试电荷泵电路。
    • 主要测试指标：输出电压 (vout)，上升时间 (rise_time)。
  • VCOTester (vco_tester): 用于测试压控振荡器 (VCO)。
    • 主要测试指标：相位噪声 (phase_noise)，调谐特性 (kvco)。
  • DIVTester (div_tester): 用于测试分频器。
    • 主要测试指标：相位噪声 (phase_noise)。
  • BOOST_SW_Tester (sw_tester): 用于测试采样开关电路 (Boosted Switch)。
    • 主要测试指标：信纳比 (SNDR)，总谐波失真 (THD)，无杂散动态范围 (SFDR)。
  • XTC_Tester (xtc_tester): 用于测试一种特定的均衡器或补偿电路 (XTC)。
    • 主要测试指标：峰峰值输出 (peak2peak)，放大系数 (amp3)，延迟 (delay)。

辅助模块和类

  • WaveformCalculator (WC):
    • 路径: circuits/testers/src/expression/ocean/calculate.py
    • 描述: 提供了一组静态方法，用于构建 Spectre Ocean 脚本中的波形计算表达式。这使得在 Python 代码中定义仿真测量表达式更加方便。
    • 示例: WC.db20(WC.V("vout")) 生成表达式 "db20(v(\"vout\"))"。
  • ocnCal:
    • 路径: circuits/testers/src/ocnCal/ocncalculator.py
    • 描述: 使用 skillbridge 与 Cadence Virtuoso (Ocean) 进行交互，执行 Ocean 表达式并解析结果。
  • SimTable:
    • 路径: circuits/testers/src/simtable.py
    • 描述: 基于 prettytable 的类，用于生成格式化的仿真结果摘要表格。
  • common_func.py: 包含一些通用辅助函数，如 unit_transform (单位转换) 和 get_setenv (处理环境变量)。

QA

Q1: 如何选择使用哪个 Tester？A1: 根据您要测试的电路类型选择对应的 Tester。例如，测试运算放大器使用 OTATester，测试比较器使用 CMPTester，测试带隙基准使用 BGPTester 等。

Q2: OTATester 支持哪些类型的 OTA？如何指定？A2: OTATester 主要支持差分输入单端输出 (diso) 和差分输入差分输出 (dido) 两种类型。通过在初始化 OTATester 时设置 OTAtype 参数来指定，例如 OTATester(..., OTAtype='dido')。还有针对跨阻放大器 (TIA) 的类型 tia 和 dtia，主要由 TIATester 使用。

Q3: 为什么有些性能指标在某些仿真器下无法获取？A3: 某些特定的分析（如零极点分析 pz，稳定性分析 stb，精确的噪声分析等）强依赖于特定的仿真器功能。例如，pz 分析和 stb 分析目前主要依赖 Cadence Spectre。代码中通常会包含对仿真器的检查，如果当前仿真器不支持，会返回默认值（如 0）或跳过该项测试。文档中已尽量标注出有仿真器依赖的指标。

Q4: 如何查看各个子测试的详细仿真结果或波形？A4: 主 Tester 类（如 OTATester）会包含各个子 Tester 的实例，例如 ota_tester.gainTester。您可以访问这些子 Tester 实例，并通过它们的 raw 属性获取原始仿真数据对象 (Waveform)，或调用它们的 plot() 方法来单独绘制波形图。主 Tester 的 plot() 方法通常会调用所有相关子 Tester 的 plot() 方法。

Q5: 如何自定义某个测试的行为，例如修改 SR 测试的输入脉冲幅度？A5: 可以通过向主 Tester 传递设置字典来实现。例如，对于 OTATester，可以传递 srtester_settings 字典：

sr_settings = {'vp': 1.2, 'delay': 5e-6, 'width': 10e-6, 'period': 20e-6, 'stop': 25e-6}
ota_tester = OTATester(..., srtester_settings=sr_settings)

具体可配置的参数请参考对应子测试器（如 SRTester）的 __init__ 方法。

Q6: 如何运行蒙特卡洛 (Monte Carlo) 或 PVT (Process, Voltage, Temperature) 仿真？A6: 可以通过设置环境变量 monte_carlo=True 或定义 PVT 字典并传递给 setenv 来实现。框架底层会处理这些设置，并在支持的仿真器（如 Spectre）中执行相应的扫描。结果通常可以通过主 Tester 的 pvt_summary() 或 monte_calro_summary() 方法查看。

Q7: Performance 类和 Tester 类是什么关系？A7: Tester 类负责构建测试电路、运行仿真并获取原始结果 (raw 数据或 measure 结果)。Performance 类负责解析这些原始结果，计算出用户关心的具体性能指标。每个 Tester 通常会关联一个或多个 Performance 子类来完成性能提取。用户主要通过 Tester 实例来获取最终的性能指标。

Q8: 如果我想添加一个新的性能指标测试怎么办？A8: 您需要：

  1. 创建一个新的 Tester 子类（继承自 TesterBase 或相关的 AC/Tran Tester），在 testbench 中定义运行相应仿真所需的电路和仿真命令。
  2. 创建一个新的 Performance 子类（继承自 Performance 或相关的 AC Performance 等），在 performance 方法中实现从仿真结果计算新指标的逻辑。
  3. （可选）修改主 Tester（如 OTATester），在其 build 方法中实例化并运行您的新 Tester，并在汇总结果时包含新指标。或者，您可以单独实例化和运行您的新 Tester。
