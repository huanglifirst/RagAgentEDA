内建元件库 (Builtins)

1. 概述

ted_builtins 模块提供了一系列基础的、理想化的或基于 PDK 模型的电路元件，主要用于电路仿真 (Simulation) 和 版图验证 (LVS - Layout Versus Schematic)。这些元件与 circuits.std 中的标准单元不同，它们通常不直接包含复杂的版图信息，而是作为 SPICE 网表中基本器件（如 R, C, L, M, V, I, X 等）或特定仿真/LVS 模型的 Python 封装。

使用内建元件库的目的是：

  • 搭建仿真测试平台 (Testbench): 提供电压源、电流源、时钟激励、理想开关、探针等用于仿真环境的元件。
  • 定义 LVS 原理图网表: 提供与 PDK LVS 规则文件中的器件模型相对应的元件（如特定名称的 MOS 管、二极管、电阻、电容模型）。
  • 行为级建模: 使用受控源（VCVS, VCCS, CCVS, CCCS）等进行电路行为建模。
  • 实例化任意子电路: 使用 Subckt (或别名 X) 可以方便地在网表中调用任何已定义的 .subckt 模型。

本篇文档将介绍 ted_builtins 中提供的各种元件及其使用方法。

2. 核心机制 (common.py, mosfet_from_virtuoso.py)

大多数内建元件的定义都依赖于几个核心的内部函数：

  • setup_builtin(initial, model, modelstr, params, is_lvs, is_sim, auto_multi, **kwargs): 这是最核心的函数。当一个内建元件模块被调用时，它会调用此函数向 TED 内核注册一个实例。
    • initial: 器件在 SPICE 网表中的前缀（如 'm' for MOS, 'r' for Resistor, 'v' for VSource）。
    • model: 器件的模型名称。
    • modelstr: 器件的参数字符串（会被直接写入网表）。
    • params: (内部使用) 可能用于传递给 Rust 后端的参数。
    • is_lvs, is_sim: 标记此元件是否应该包含在 LVS 网表或 Simulation 网表中。一个元件可以同时为 True。
    • auto_multi: (高级) 是否根据全局变量自动添加乘数 (如 SPICE 中的 m=...)。
    • **kwargs: 其他传递给 ParamList 的参数。
  • setup_builtin_from_virtuoso(cellname, is_lvs, is_sim, **params): 专用于从 Virtuoso 环境获取模型信息来设置内建元件。它会调用 ted_interface.from_virtuoso 来获取正确的 SPICE 前缀和参数字符串。

用户通常不直接调用这些函数，而是通过调用库中提供的具体元件模块（如 Resistor, Mosfet, VSource）来间接使用它们。

3. 无源元件 (Passive Components)

3.1 Resistor (resistor.py)

  • 别名: R
  • 功能: 代表一个理想或基于模型的电阻。
  • 签名: Resistor(a, b, r=1.0, model="resistor", modelstr=None)
  • 参数:
    • a, b: 连接的两个 CircuitNode。
    • r: 电阻值 (浮点数，单位 Ohm)。会被包含在 modelstr 中。
    • model: 使用的电阻模型名称（默认为 "resistor"，通常代表理想电阻）。
    • modelstr: 直接写入网表的参数字符串。如果提供，则覆盖 r 参数。
  • 标记: is_lvs=True, is_sim=True (默认 LVS 和仿真都包含)。
  • 特殊版本:
    • SResistor_lvs(...): 专用于 LVS 的版本 (只设置 is_lvs=True)。
    • SResistor_sim(...): 专用于仿真的版本 (只设置 is_sim=True)。

示例:

from ted_builtins import Resistor, R # R 是 Resistor 的别名

n1, n2, n3 = Node(3)

# 1. 创建一个 1kΩ 的理想电阻
r1 = Resistor(n1, n2, r=1e3) # 或 R(n1, n2, r=1e3)

# 2. 创建一个使用特定 PDK 模型和参数字符串的电阻
#    (假设 PDK 定义了名为 "rppoly" 的模型)
poly_res = R(n2, n3, model="rppoly", modelstr="l=5u w=1u")

3.2 Capacitor (capacitor.py)

  • 别名: C
  • 功能: 代表一个理想或基于模型的电容。
  • 签名:
    • Capacitor(a, b, c=1.0e-12, model="capacitor", **kwargs): 2 端电容。
    • Capacitor3T(a, b, supply, c=1.0e-12, model="capacitor"): 3 端电容（通常用于衬底连接）。
  • 参数:
    • a, b: 主要连接端。
    • supply: (仅 Capacitor3T) 第三个端点，通常连接衬底或电源/地。
    • c: 电容值 (Farads)。
    • model: 电容模型名称。
  • 标记: is_sim=True (默认只用于仿真)。

示例:

from ted_builtins import Capacitor, C, Capacitor3T

n1, n2 = Node(2)

# 1. 创建一个 1pF 的理想电容
cap1 = Capacitor(n1, n2, c=1e-12) # 或 C(n1, n2, c=1e-12)

# 2. 创建一个 3 端电容，第三端连接到 Vss()
cap3t = Capacitor3T(n1, n2, Vss(), c=5e-15, model="mimcap")

3.3 Inductor (inductor.py)

  • 别名: L
  • 功能: 代表一个理想或基于模型的电感。
  • 签名: Inductor(a, b, l=1.0, model="inductor")
  • 参数:
    • a, b: 连接端。
    • l: 电感值 (Henries)。
    • model: 电感模型名称。
  • 标记: is_lvs=True, is_sim=True.

示例:

from ted_builtins import Inductor, L

n1, n2 = Node(2)
# 创建一个 1nH 电感
ind1 = Inductor(n1, n2, l=1e-9) # 或 L(n1, n2, l=1e-9)

4. 有源元件 (Active Components)

4.1 Mosfet (mosfet.py)

  • 别名: M
  • 功能: 代表一个标准的 4 端 MOSFET。
  • 签名: Mosfet(d, g, s, b, model="nch", l=1e-6, w=5e-6, nf=None, m=None)
  • 参数:
    • d, g, s, b: 漏(Drain)、栅(Gate)、源(Source)、体(Bulk/Body) 连接节点。
    • model: MOSFET 模型名称 (如 "nch", "pch_lvt")。会通过 get_pdk().get_sim_original_model() 映射到 PDK 内部名称。
    • l, w: 沟道长度和宽度 (单位: 米)。
    • nf: Number of fingers (通常 m = nf)。
    • m: Multiplier (器件乘数)。注意: nf 和 m 不能同时指定。
  • 行为: 根据当前仿真器 (getenv_simulator()) 生成合适的 SPICE 参数字符串（如 HSpice 使用 ' 包裹 L/W/M）。自动包含 PDK 中定义的 LPE (Layout Parasitic Extraction) 参数。
  • 标记: is_sim=True (只用于仿真)。
  • LVS 版本: MosfetLvs(...) - 参数类似，但使用 LVS 模型的名称和参数格式 (is_lvs=True)。

示例:

from ted_builtins import Mosfet, M, MosfetLvs

d, g, s, b = Node(4)
# 创建一个用于仿真的 NMOS (m=4)
nmos_sim = Mosfet(d, g, s, b, model="nch_lvt", l=60e-9, w=0.5e-6, m=4)
# 创建一个用于 LVS 的 PMOS (nf=2)
pmos_lvs = MosfetLvs(d, g, s, b, model="pch", l=180e-9, w=1e-6, nf=2)

4.2 Mosfet5T, Mosfet6T (mosfet5t.py, mosfet6t.py)

  • 功能: 用于表示具有 5 个或 6 个端子的特殊 MOSFET 器件，通常是高压 (HV) 或带有特殊衬底连接的器件。
  • 签名:
    • Mosfet5T(d, g, s, pw, psub, ...)
    • Mosfet6T(d, g, s, pw, hvbn, psub, ...)
    • 包含 l, w, m, model 参数。
  • 行为: 内部使用 setup_builtin_from_virtuoso，表明其模型和参数格式通常直接来源于 Virtuoso PDK 定义。LVS 版本 (Mosfet5TLvs, Mosfet6TLvs) 可能包含额外的寄生二极管模型 (DiodeLvs) 以匹配 LVS 规则。
  • 标记: Sim 版本标记 is_sim=True；LVS 版本标记 is_lvs=True。

4.3 MosfetFromVirtuoso (mosfet_from_virtuoso.py)

  • 功能: 提供基于 Virtuoso 模型名称直接创建 MOSFET 的接口。
  • 签名: MosfetFromVirtuoso(d, g, s, b, l, w, m, model) 和 MosfetFromVirtuosoLvs(...)。
  • 行为: 调用 setup_builtin_from_virtuoso 来获取正确的 SPICE 前缀和参数。

示例 (改编自 test_mosfet_from_virtuoso.py::test03)

from pyted import Variable
from ted_builtins import MosfetFromVirtuoso

d, g, s, b = Node(4)
# 使用 Variable 定义可优化参数
w_var = Variable(value=180e-9, name="w_opt").set_bound(100e-9, 500e-9)
l_var = Variable(value=60e-9, name="l_opt").set_bound(40e-9, 100e-9)

# 创建基于 Virtuoso 模型的 NMOS， L/W 使用 Variable
nmos_virt = MosfetFromVirtuoso(d, g, s, b, l=l_var, w=w_var, m=5, model="nch")

4.4 Diode (diode.py)

  • 功能: 代表一个二极管。
  • 签名: Diode(a, b, model="diode", **kwargs)
  • 参数:
    • a, b: 阳极和阴极节点。
    • model: 二极管模型名称。
    • **kwargs: 其他 SPICE 参数 (如 area, pj)。
  • 标记: is_lvs=True, is_sim=True.

示例 (来自 test_diode.py)

from ted_builtins import Diode
vout = Node("vout")
# 创建一个连接到 Vss 的二极管
d1 = Diode(vout, Vss(), model="ndio", area=1e-12)

4.5 Q (BJT Wrapper) (__init__.py)

  • 功能: BJT（双极结型晶体管）的便捷封装。
  • 签名: Q(c, b, e, [s], model, **kwargs) (s 为衬底端，可选)
  • 行为: 内部调用 Device，使用 SPICE 前缀 'q'。
  • 标记: is_lvs=True, is_sim=True.

5. 独立源 (Sources)

5.1 VSource (vsource.py)

  • 别名: V
  • 功能: 电压源。
  • 签名: VSource(a, b, dc=0, ac=None, mode=None, model="vsource", **kwargs)
  • 参数:
    • a, b: 正负端节点。
    • dc: 直流电压值。
    • ac: 交流分析的 AC 幅度 (magnitude)。
    • mode: 源的模式，决定 **kwargs 的含义:
      • None (默认): DC 或 AC 源 (取决于 ac 是否提供)。
      • "pulse": 脉冲源。kwargs 应包含 val0, val1, period, width, rise, fall, delay。
      • "pwl": 分段线性源。kwargs 应包含 time (时间点列表) 和 v (对应电压值列表)。
      • "prbs": 伪随机二进制序列源。kwargs 应包含 period, val0, val1, seed, rise, fall, delay。
      • "sine": 正弦源。kwargs 应包含 amp (幅度), freq (频率), delay。dc 参数设置直流偏置。ac 参数仍可用于 AC 分析。
    • model: 模型名称（通常为 "vsource"）。
  • 行为: 使用 source.py 中的 get_source_paramstr 辅助函数根据模式和仿真器生成正确的 SPICE 参数字符串。
  • 标记: is_sim=True (默认只用于仿真)。LVS 网表中通常不包含独立源。

示例: 见上一篇文档 Node 示例中的 "VSource Modes" 部分。

5.2 ISource (isource.py)

  • 别名: I
  • 功能: 电流源。
  • 签名: ISource(a, b, dc=0, ac=0, model="isource", mode=None, **kwargs)
  • 参数: 与 VSource 类似，但 dc/ac 指定电流值 (Amps)，kwargs 中的 v 列表（用于 PWL）应为电流值列表。
  • 标记: is_lvs=True, is_sim=True.

示例:

from ted_builtins import ISource, I

n1, n2 = Node(2)
# 1mA DC 电流源
i_dc = ISource(n1, n2, dc=1e-3) # 或 I(n1, n2, dc=1e-3)
# 1uA AC 电流源 (用于 AC 分析)
i_ac = I(n1, n2, ac=1e-6)

6. 受控源 (Controlled Sources)

用于行为级建模，根据其他电压或电流来控制自身的电压或电流。它们都有 4 个端子。

  • VCVS(a, b, c, d, model="vcvs", gain=1.0) (vcvs.py): 电压控电压源 (E Element)。输出端 a, b 的电压由输入端 c, d 的电压差乘以 gain 决定。
  • VCCS(a, b, c, d, model="vccs", gm=1.0) (vccs.py): 电压控电流源 (G Element)。流出 a 流入 b 的电流由 c, d 的电压差乘以跨导 gm 决定。
  • CCVS(a, b, model="ccvs", rm=1.0, probe="") (ccvs.py): 电流控电压源 (H Element)。输出端 a, b 的电压由通过指定 probe (通常是 VSource 或 IProbe 实例名) 的电流乘以跨阻 rm 决定。
  • CCCS(a, b, model="cccs", gain=1.0, probe="") (cccs.py): 电流控电流源 (F Element)。流出 a 流入 b 的电流由通过指定 probe 的电流乘以 gain 决定。
  • PCCCS(a, b, gain=1, probes="vprb vinj", coeffs="0 1 1", model="pcccs") (pcccs.py): 带探针系数的 CCCS，允许更复杂的电流控制关系。

标记: 均为 is_lvs=True, is_sim=True.

示例:

from ted_builtins import VCVS, VCCS, CCVS, CCCS, VSource, IProbe

ina, inb, inc, ind = Node(4)
outa, outb, outc, outd = Node(4)
ctrl_v1, ctrl_v2 = Node(2)
probe_current = Node("probe_current")

# 电压放大器: V(outa, outb) = 10 * V(ctrl_v1, ctrl_v2)
amp = VCVS(outa, outb, ctrl_v1, ctrl_v2, gain=10.0)

# 跨导放大器: I(outc -> outd) = 0.01 * V(ctrl_v1, ctrl_v2)
gm_amp = VCCS(outc, outd, ctrl_v1, ctrl_v2, gm=0.01)

# 电流镜 (概念): 使用 IProbe 探测电流，然后用 CCCS 复制
(Vdd(), probe_current) >> Resistor(r=100) # 产生电流
probe_current >> Vss() # 形成回路
# 探测流过 Resistor 的电流 (需要命名 IProbe)
probe = IProbe(probe_current, Vss()).rename("i_probe_res")
# 复制电流到另一个支路 (增益为 5)
mirrored_current = CCCS(outa, outb, gain=5.0, probe="i_probe_res")

7. 探针与开关

  • IProbe(a, b, r=0.0, model="iprobe") (iprobe.py): 理想电流探针。插入电路中以测量（或在 SPICE 中报告）流经它的电流，本身零电压降。在 Spectre/HSpice 中对应 iprobe 元件。标记 is_lvs=True, is_sim=True。
  • IdealSwitch(a, b, ps, ns, model="_ideal_switch_", ...) (vswitch.py): 理想电压控制开关。端点 a, b 之间的导通由控制电压 ps, ns 决定 (V(ps) - V(ns) > vt 时导通)。标记 is_lvs=True, is_sim=True。
  • Cmdmprobe(...) (cmdmprobe.py): 共模/差模探针。用于分析差分信号，内部由 VCVS, IProbe, PCCCS 组合而成。

示例: 见 Node 文档中的 IProbe 示例和 test_vswitch.py。

8. 通用子电路与器件

  • Subckt(*args, model=None, **kwargs) 或 X(*args, model=None, **kwargs) (subckt.py): 实例化任意 SPICE 子电路模型。
    • *args: 连接到子电路端口的节点列表 (顺序必须与子电路定义一致)。
    • model: 子电路的模型名称。
    • **kwargs: 传递给子电路的参数。
    • 行为: 在网表中生成 X 开头的实例行。
    • 标记: is_lvs=True, is_sim=True.
  • Device(*args, prefix=None, model=None, **kwargs) (__init__.py): 更通用的器件封装，允许指定 SPICE 前缀。Q (BJT) 是用 prefix="q" 的 Device 实现的。
  • from_analogLib(number_of_ports, model, model_str, ...) (analogLib.py): 用于封装来自 Cadence analogLib 库的元件（主要用于仿真）。

示例:

from ted_builtins import Subckt, X

# 假设有一个名为 "MY_OPAMP" 的子电路模型在 include 文件中定义
in_p, in_n, out, vdd_op, vss_op = Node(5)
# 实例化子电路
opamp_inst = Subckt(in_p, in_n, out, vdd_op, vss_op, model="MY_OPAMP", GAIN="100k")
# 或者使用别名 X
# opamp_inst = X(in_p, in_n, out, vdd_op, vss_op, model="MY_OPAMP", GAIN="100k")

9. 总结

ted_builtins 提供了一套丰富的底层电路元件，它们是构建仿真测试平台、定义 LVS 网表以及进行行为级建模的基础。这些元件通过 setup_builtin 或 setup_builtin_from_virtuoso 与 TED 内核交互，确保在生成不同类型的网表（LVS 或 Sim）时包含正确的模型和参数。熟练使用这些内建元件是利用 TED 进行完整设计和验证流程的关键。

10. 详细使用样例：简单共源放大器仿真

这个例子展示了如何使用 ted_builtins 中的元件搭建一个简单的共源放大器电路并进行瞬态仿真。

from pyted import *
from ted_builtins import Mosfet, Resistor, Capacitor, VSource
from ted_device.stimuli import Clock # 假设 Clock 内部使用了 VSource(mode="pulse")

@testbench
def CommonSourceSim():
    # 1. 定义节点
    vin = Node("vin")
    vout = Node("vout")
    vg = Node("vg_bias")

    # 2. 设置电源和偏置
    supply = VSource(Vdd(), Vss(), dc=1.2).rename("Vsupply")
    bias_voltage = VSource(vg, Vss(), dc=0.6).rename("Vbias")

    # 3. 创建激励信号 (输入信号)
    #    使用一个叠加在 DC 偏置上的小信号正弦波
    input_signal_src = VSource(vin, Vss(), mode="sine", dc=0.6, amp=0.01, freq=1e6).rename("Vin")

    # 4. 搭建共源放大器
    #    上拉电阻
    load_resistor = Resistor(Vdd(), vout, r=5e3).rename("Rd")
    #    NMOS 晶体管
    nmos_amp = Mosfet(vout, vg, Vss(), Vss(), model="nch", l=180e-9, w=2e-6, m=4).rename("M1")
    #    输入耦合电容 (可选)
    # cap_in = Capacitor(vin, vg, c=1e-12).rename("Cin")
    #    输出负载电容
    cap_load = Capacitor(vout, Vss(), c=0.5e-12).rename("CL")

    # 5. 观察信号
    watch(vin, vout, vg)
    watch(f"i({load_resistor.name()})") # 观察通过电阻的电流 (需要仿真器支持)

    # 6. 运行瞬态仿真
    sim_time = 5e-6 # 5us
    raw = tran(sim_time)

    # 7. 分析或显示结果
    print("仿真完成。")
    # raw.plot("vin", "vout") # 绘制输入输出波形
    # output_gain = calculate_gain(raw.get_signal("vout"), raw.get_signal("vin"))
    # print(f"估算增益: {output_gain}")
    raw.show() # 打开波形查看器 (如果支持)

# 运行 Testbench
CommonSourceSim()

这个例子展示了：

  • 使用 VSource 设置直流电源、偏置电压和正弦输入信号。
  • 使用 Resistor, Mosfet, Capacitor 搭建电路。
  • 使用 watch 指定需要观察的节点和支路电流。
  • 运行 tran 瞬态仿真。
  • （概念上）进行结果分析或显示。
