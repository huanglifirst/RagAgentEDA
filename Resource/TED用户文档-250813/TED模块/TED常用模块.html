差分对PairedMosfet

使用 PairedMosfet 模块构建模拟差分对管，其中包含两个MOS管：A管、B管，两个MOS管的 finger 数量相同，都等于 m 的值。

参数信息

参数说明类型默认值
lMOS 管的长度，代表栅极的长度float1e-6
wMOS 管的宽度，代表源极和漏极的宽度float5e-6
model模型str"nch"
mfinger 数量int4
y_spacing上下 MOS 管的间距int600

案例

正常使用

lmin = get_circuit().get_pdk().lmin()
dut = DifferentialPair(
    l=lmin * 10, w=lmin * 30, model="nch", m=2, mode="cross", d_via=2, dummy_m=1
)
(d1, d2) >> dut % (g1, g2) >> s
guardring(dut, connectTo=Vss())

电流镜CurrentMirror

电流镜在模拟电路中应用十分广泛。在传统电路描述中，电流镜往往是以MOSFET为基础单元进行描述。这样的描述方式，其缺点在于不能很好的表征电流镜中多个晶体管之间的尺寸比例关系。在TED中，建议（而非强制）用CurrentMirror来描述电流镜。这样会使得代码更简洁更易读。

理想的电流镜将输入的电流按照设计的比例进行复制，输出电流与输出外接的负载无关，等价于具备无限大的输出电阻。而实际的电流镜输出电阻为有限值。不同类型的电流镜，会提供不同的输出电阻，以及电压摆幅。高输出电阻和电压摆幅的代价往往是电路的复杂性和功耗。

根据要求选择不同结构，输出阻抗一般可以从几百K欧姆到几十M欧姆。

Type 1 单管电流镜

Type 1型电流镜是最简单的电流镜结构，一般用于要求较低的场合，输出阻抗一般为几百K欧姆。

Type 1型电流镜的使用如下：

cm = CurrentMirror(
    l=l,
    w=w,
    m=1,
    model="nch",
    type="type1",
    place_mode=VERTICAL,
    from_virtuoso=False,
)
cm.input(Idc)
cm.output(Iout)

其中cm = CurrentMirror(l=l, w=w, m=1, model="nch", type="type1")定义了一个Type 1型的N电流镜，并指定其输入管子的l, w和m。

cm.input(Idc)指定了电流镜的输入级连接到Idc节点。cm.output(Iout)指定了输出级连接到Iout节点。

对于一个电流镜而言，只能有一个输入级，但可以有任意多个输出级。因此可以通过cm.output(Iout1), cm.output(Iout2)创建电路的更多输出级。

以上的例子中，实现的是1:1的电流镜。如果要实现其他比例的电流镜，可以通过参数k来指定。

cm.output(Iout1, k=1/2)
cm.output(Iout2, k=2)

需要注意的是，在k为非整数的时候，使用者需要保证m*k为整数，其中m在电流镜的初始化时指定。当m*k为非整数时，系统会抛异常。

Type 2 Cascode电流镜

Type 2型电流镜具有较高的输出阻抗，可以达到几十M欧姆。实际应用中在输入摆幅不成为问题（例如电源电压较高）的时候经常使用该结构。但在低压应用中，由于其摆幅较低，常采用Type 3替代。

Type 2型电流镜的使用如下：

cm = CurrentMirror(
    l=l,
    w=w,
    m=1,
    model="nch",
    type="type2",
    place_mode=VERTICAL,
    from_virtuoso=False,
)
cm.input(Idc)
cm.output(Iout)

Type 3 高摆幅电流镜（电阻版）

Type 2型电流镜在摆幅受限的情况下的替代解决方案。其输出阻抗理论上与Type 2相当，实际中往往小于Type 2。此外还有一个缺点在于需要引入电阻导致版图上需要额外的面积。此外相对Type 4+Type5的方案，此类型的电流镜少一条支路，因此功耗方面稍占优势。

Type 3型电流镜的使用如下：

cm = CurrentMirror(
    l=l,
    w=w,
    m=1,
    model="nch",
    type="type3",
    place_mode=VERTICAL,
    from_virtuoso=False,
)
cm.input(Idc)
cm.output(Iout)

Type 4+Type 5 高摆幅电流镜（无电阻版）

Type 3电流镜需要引入电阻，为了去除该电阻，可以采用Type 4+Type 5的方案。由Type 4为Type 5提供一个偏置电压X。其缺点在于多了一路Type 4导致功耗有所提高。

实际应用中，Type 4和Type 5型结合使用的例子如下：

pcm20 = CurrentMirror(
    type="type4",
    model="pch",
    l=[0.2e-6, 1e-6],
    w=w,
    m=[3, 1],
    place_mode=VERTICAL,
).input(VBP)
pcm2 = CurrentMirror(
    type="type5", model="pch", l=l, w=w, m=[6, 4], place_mode=VERTICAL
)
pcm2.input(dumy4, VBP, PD=PDB, mid=dumy7)
pcm2.output(VBN, k=1 / 2, single=True)
pcm2.output(dumy8, k=1 / 2, single=True)
pcm2.output(dumy5, m=[3, 3], mid=dumy3)
pcm2.output(VOUT, m=[3, 3], mid=dumy2)

对应的电路图

可以看出上面的结构实际上就是一个电流镜。然而在描述中将其视为两个电流镜，其中pcm20描述的是左边的支路，这个“电流镜”只有输入级，没有输出级，其作用在于产生一个合适的偏置给第二支路下面的MOSFET的栅极。之所以采用这样的描述，是为了将电流镜拆解成不可分的基本模块，在实际应用中可以灵活进行组合使用。

版图生成

最简单的情况下，电流镜的版图只需通过

cm_layout = cm.layout()

来生成。此时，对于Type 1型电流镜，将生成一行的版图。对于其他类型的电流镜，每一级都有两个MOSFET。此时生成的版图将有两行，分别对应每一级中上下两个MOSFET。返回值为一个Instance，因此可以用于与其他模块进行布局操作。下面给出完整例子：

Type 1型代码如下

l = 2e-6
w = 1e-6
cm = CurrentMirror(
    l=l,
    w=w,
    m=1,
    model="nch",
    type="type1",
    place_mode=HORIZONTAL,
    from_virtuoso=False,
)
cm.input(Idc)
cm.output(Iout)
guardring(cm, connectTo=Vss())
place()
route()

完整的应用电流镜完成运放的设计参考以下例子

from tedcore import *
from ted_device import PMOS, DifferentialPair, CurrentMirror
from layout import *
from algo import *


@module
def CP_OP2(
    VINP: Input, VINN: Input, IB40U: Input, PDA: Input, PDB: Input, VOUT: Output
):
    with setenv(mos_from_virtuoso=False):
        l = [0.2e-6, 0.5e-6]
        w = 4e-6
        lmin = get_circuit().get_pdk().lmin()
        VBP, VBN = Node(2)
        n1 = Node(name="n1")
        dumy2, dumy3, dumy4, dumy5, dumy7, dumy8, dumy9, dumy11, dumy12, dumy13 = Node(
            10
        )

        pm1 = DifferentialPair(l=500e-9, w=6e-6, model="pch", m=2 * 4)
        pm2 = DifferentialPair(l=500e-9, w=4e-6, model="nch", m=2 * 4)
        (dumy12, dumy13) >> pm1 % (VINP, VINN) >> dumy8
        (dumy3, dumy2) >> pm2 % (VINN, VINP) >> dumy9  # 栅极顺序

        pdmos = PMOS(l=lmin, w=2e-6, m=5)
        IB40U >> (pdmos % PDA) >> n1
        pdmos = layout_mos([pdmos])

        ncm1 = CurrentMirror(
            type="type3", model="nch", l=l, w=w, m=2, place_mode=VERTICAL
        )
        ncm1.input(n1, PD=PDA, mid=dumy11)  # 40u
        ncm1.output(VBP, k=1 / 2)  # 20u
        ncm1.output(dumy4, k=1)  # 40u
        ncm1.output(dumy9, k=1 / 2, single=True)  # 20u

        # region cm_type45
        pcm20 = CurrentMirror(
            type="type4",
            model="pch",
            l=[0.2e-6, 1e-6],
            w=w,
            m=[3, 1],
            place_mode=VERTICAL,
        ).input(VBP)
        pcm2 = CurrentMirror(
            type="type5", model="pch", l=l, w=w, m=[6, 4], place_mode=VERTICAL
        )
        pcm2.input(dumy4, VBP, PD=PDB, mid=dumy7)
        pcm2.output(VBN, k=1 / 2, single=True)
        pcm2.output(dumy8, k=1 / 2, single=True)
        pcm2.output(dumy5, m=[3, 3], mid=dumy3)
        pcm2.output(VOUT, m=[3, 3], mid=dumy2)
        # endregion cm_type45

        ncm30 = CurrentMirror(
            type="type4",
            model="nch",
            l=[0.2e-6, 3e-6],
            w=w,
            m=[2, 1],
            place_mode=VERTICAL,
        ).input(
            VBN
        )  # 一个管子为3u

        ncm3 = CurrentMirror(
            type="type5", model="nch", l=l, w=w, m=[2, 2], place_mode=VERTICAL
        )
        ncm3.input(dumy5, VBN, mid=dumy12).output(VOUT, mid=dumy13)

        pmosPair = guardring(pm1, Vdd())

        nmosPair = guardring(pm2, Vss())

        col(
            # ports,
            guardring(
                row(
                    col(
                        pcm2.get_row(1),
                        row(pdmos, pcm2.get_row(0, flip=True)),
                    ),
                    pcm20.col(),
                ),
                connectTo=Vdd(),
            ),
            row(pmosPair, nmosPair, align=MIDDLE),
            guardring(
                row(
                    ncm1.rpoly,
                    ncm30.col(),

内建电路模块

内置元器件，如电阻，电容，电压源，电流源等，是TED中最底层的元器件。这些元器件直接对应仿真器中支持的元器件，并且这些元器件只有电学特性，而没有任何物理实现，因此可以并且只能用于电路仿真。

由于不对应具体的物理实现，这些内置元器件在有意义的电路描述的结构描述中一般不会直接使用，一般在以下场合使用

  • testbench或者生成器的wrapper中作为激励信号或者负载

  • 生成器中作为电流源或者负载

  • 在具有物理实现的模块实底层，会调用这些内置元器件，使得该模块可以支持仿真。

Mos管Mosfet

Mosfet器件源代码：

@moduledef(module_type="sim")
def Mosfet(
    d: Input,
    g: Input,
    s: Input,
    b: Input,
    model: str = "nch",
    l: Union["Variable", float] = 1e-6,
    w: Union["Variable", float] = 5e-6,
    nf: Union["Variable", int] = None,
    m: Union["Variable", int] = None,
):
    from ted_device import check_mosfet

    assert m == None or nf == None, f"Cannot specify both m({m}) and nf({nf})."
    if nf == None:
        nf = 1
    if m == None:
        m = nf
    unit = get_circuit().get_pdk().get_scale_unit()
    l = convert_unit1_to_unit2(l, 1, unit)
    w = convert_unit1_to_unit2(w, 1, unit)
    check_mosfet(model, round(l / 1e-9), round(w / 1e-9))
    model = get_pdk().get_sim_original_model(model)
    simulator = getenv("simulator")
    initial = get_pdk().get_sim_initial(model, simulator)
    lpeparams = get_pdk().get_all_lpe_params(l=l, w=w, model=model)
    helper(initial, l, w, m, model, **lpeparams)
    current_module().set_builtin_sim()

    """使用示例：("a", "b", "c", Vss()) >> Mosfet()"""

其中(d, g, s, b) 代表Mosfet器件输入端，分别为漏极，栅极，源极，衬底 需要按输入顺序写入 (model = "nch", l = 1e-6, w = 5e-6) 代表模型为"nch" 也可写成"pch" l为mosfet栅长 w为mosfet栅宽 如果不声明默认为 l = 1e-6, w = 5e-6

子电路Subckt

当TED的内置元件无法满足用户需求时，可以使用Subckt来自定义构建器件，这些器件必须是工艺库或理想器件库中具备的。所构建的器件依然只支持仿真，没有物理实现。

subckt器件源代码：

def Subckt(*args, model=None, **kwargs):
    from tedcore import Module
    from tedcore import find_frame

    assert model is not None

    def func(*args):
        find_frame()
        setup_builtin("x", model, "", **kwargs, is_lvs=True, is_sim=True)

    terminals = (
        [f"n{k}" for k, n in enumerate(args)],
        ["Input" for k, n in enumerate(args)],
        [1 for k, n in enumerate(args)],
    )
    model = get_model(model)
    inst = Module.build(model, None, func, terminals, None, is_testbench=False)
    inst.set_paramstr(" ".join(f"{k}={v}" for k, v in kwargs.items()))
    args >> inst
    inst.get_module().set_builtin_lvs_and_sim()
    return inst

Subckt使用方法

Subckt(*args,model="",**kwargs)

这里的Subckt可以用X来替代，下列用法与上述等价：

X(*args,model="",**kwargs)

参数说明

名称说明默认值
args所有输入的可变参数均会被视为器件的节点，要求传入顺序与所调用的模型节点顺序一致。--
model选用的器件模型--
kwargs除model外，所有的关键字参数都会被视为器件的参数，根据需要输入相应的器件参数即可--

使用analogLib库模型

from_analogLib()介绍

TED支持从virtuoso的analogLib库读取模型并使用，通过APIfrom_analogLib()实现。下面是from_analogLib()的参数展示和说明。

def from_analogLib(
    number_of_ports, model="", model_str: str = "", model_par: Dict = {}, **kwargs
):
    """
    pwl = from_analogLib(
            2, model="vsource", type="pwl", wave="[ 0 -2 50u -2 50.01u 0  ]"
        )
    (a,b)>>pwl
    """
参数说明类型默认值
number_of_ports声明器件的端口数并且所有端口类型均为Input。生成的端口顺序与调用的模型端口顺序一致int--
model指定调用的模型，取值为analogLib中的模型名称str--
model_str将模型的参数以字符串的形式传入str--
model_par将模型的参数以字典的形式传入dict--
kwargs将模型的参数以关键字参数形式传入key=value--

模型的参数传入可以通过model_str、model_par和以关键字参数形式传入，在生成网表时没有任何区别，用户习根据惯选择其中一种方式即可。三者的使用方式如下所示:

pwm = from_analogLib(2, model="vsource", type="pwl", wave="[ 0 -2 50u -2 50.01u 0  ]")
pwm = from_analogLib(2, model="vsource", model_str="type=pwl wave=[ 0 0 1u 0 1.01u 1 10u 1 10.01u 0 ]")
pwm = from_analogLib(2, model="vsource", model_par={"type":"pwl","wave":"[ 0 0 1u 0 1.01u 1 10u 1 10.01u 0 ]"})

使用ahdlLib库模型

from_ahdlLib()介绍

TED支持从virtuoso的ahdlLib库读取Va模型并使用，通过APIfrom_ahdlLib()实现。下面是from_ahdlLib()的参数展示和说明。

def from_ahdlLib(
    number_of_ports, model="", model_str: str = "", model_par: Dict = {}, **kwargs
):
    """
    diode=from_ahdlLib(2, model="diode_sch", bv=10)
    (a,b)>>diode
    """
参数说明类型默认值
number_of_ports声明器件的端口数并且所有端口类型均为Input。生成的端口顺序与ahdlLib模型的端口顺序一致int--
model指定调用的模型，取值为ahdlLib中的模型名称str--
model_str将模型的参数以字符串的形式传入str--
model_par将模型的参数以字典的形式传入dict--
kwargs将模型的参数以关键字参数形式传入key=value--

模型的参数传入可以通过model_str、model_par和以关键字参数形式传入，在生成网表时没有任何区别，用户习根据惯选择其中一种方式即可。三者的使用方式如下所示:

doide1=from_ahdlLib(2, model="diode_sch", bv=10)
doide2=from_ahdlLib(2, model="diode_sch", model_str="bv=10")
doide3=from_ahdlLib(2, model="diode_sch", model_par={"bv":10})

注: 当模型的参数涉及python关键字时，应使用model_str或model_par模式对参数进行赋值。

示例

如下案例展示了从ahdlLib中调用二极管(diode_sch)进行仿真的代码，其中参数bv=10即为集合关键字参数，用来对二极管的bv参数进行赋值。对于二极管的其他参数，未赋值时将使用模型的默认参数。

@module
def ideal_opamp(
    vip: Input, vin: Input, vout: Output, fz=10e6, fp=1e3, r2=10e3, gain=1000
):
    c1 = 1 / (2 * np.pi * r2 * fz)
    r1 = 1 / (2 * np.pi * c1 * fp) - r2
    ("npz", Vss(), vip, vin) >> VCCS(gm=-1 * gain)
    R("npz", Vss(), modelstr="r=1,isnoisy=no")
    R("npz", vout, modelstr=f"r={r1},isnoisy=no")
    R(vout, "n3", modelstr=f"r={r2},isnoisy=no")
    C("n3", Vss(), c=c1)
