电流镜模块（CurrentMirror）

1. 概述

ted_device.cm (Current Mirror) 模块提供了用于生成和测试各种电流镜电路及其版图的高级工具。核心是 CurrentMirror 类，它允许用户通过指定参数（如拓扑类型、器件尺寸、布局模式）来快速创建常用的电流镜结构，并自动生成相应的版图、仿真模型和 LVS 网表。该模块旨在简化模拟设计中电流镜的实现，特别是需要考虑匹配和性能优化的场景。

主要特性包括：

  • 多种拓扑支持: 内建支持多种电流镜电路结构（简单、Cascode、带源极简并电阻、宽摆幅 Cascode 等）。
  • 参数化设计: 可通过参数 (l, w, m, model) 控制组成电流镜的 MOSFET 尺寸和模型。
  • 灵活的输出配置: 支持定义多个输出支路，并可独立设置电流比例 (k) 或器件尺寸 (m)。
  • 高级版图生成: 提供不同的布局模式 (place_mode)：
    • HORIZONTAL/VERTICAL: 基于 layout_mos 的简单行列式布局。
    • EULER: 利用欧拉路径算法（transistor_unit.py）生成交指/共质心 (Common-Centroid) 布局，以提高匹配性能。
  • Dummy 器件: 支持自动或手动添加 Dummy 器件以改善匹配和满足工艺要求。
  • 仿真测试: 提供了 CM_Tester 类，用于方便地对生成的电流镜进行 AC 仿真以评估其输出电阻等性能指标。

2. CurrentMirror 类 (current_mirror.py)

CurrentMirror 是用于生成电流镜的主要类。它继承自 EmulatedInstList，内部管理构成电流镜的多个 MOSFET 实例。

2.1 初始化

cm = CurrentMirror(
    l,                 # MOS 长度 (float 或 list)
    w,                 # MOS 宽度 (float 或 list)
    m=1,               # 每个 MOS 的乘数/finger 数 (int 或 list)
    model="nch",       # MOS 模型 ("nch" 或 "pch")
    type="type1",      # 电流镜拓扑类型 ("type1" 到 "type5")
    place_mode=HORIZONTAL, # 布局模式 (HORIZONTAL, VERTICAL, EULER)
    l_dummy=None,      # Dummy 管长度
    dummy_m=0,         # 每侧 Dummy 管数量
    res=None,          # (用于 type3) 源极电阻实例
    ds=2,              # (传递给 layout_mos) DS 端 Via 数量
    dummy=False,       # 是否自动填充 Dummy (仅 VERTICAL 模式)
    from_virtuoso=None # 是否使用 Virtuoso Pcell 版图
)
  • l, w, m:
    • 对于简单类型 (如 type1)，通常为 float 或 int。
    • 对于 Cascode 类型 (type2, type3, type5)，l, w, m 可以是包含两个元素的列表 [top_val, bottom_val]，分别对应 Cascode 器件和下方器件的参数。如果只提供单个值，则上下器件使用相同参数。
  • model: 指定基础 MOS 管的模型，必须是 "nch" 或 "pch"（或相应的 PDK 别名）。
  • type: 定义电流镜的电路结构：
    • "type1": 简单电流镜 (单个 MOS)。
    • "type2": 标准 Cascode 电流镜 (两个 MOS 串联)。
    • "type3": 带源极简并电阻的 Cascode 电流镜 (需要提供 res 参数，通常是 RpolySingle() 实例)。
    • "type4": 宽摆幅 Cascode 的偏置生成部分（通常输入端连接到参考电流）。此类型没有 .output() 方法。
    • "type5": 宽摆幅 Cascode 的镜像部分，需要通过 .input() 的 VB 参数提供 Cascode 管的栅极偏置 (通常来自 Type 4 的输出)。
  • place_mode:
    • HORIZONTAL: 使用 layout_mos 进行水平方向的线性布局。
    • VERTICAL: 使用 layout_mos 进行垂直方向的线性布局。支持 dummy=True 时自动填充 Dummy。
    • EULER: 使用 autolayout_euler （内部可能调用 transistor_unit.py 的 Euler 路径算法）生成优化的匹配布局（交指/共质心）。
  • l_dummy, dummy_m: 控制在布局边缘添加的 Dummy 管的长度和数量（每侧 dummy_m 个）。
  • res: (仅 type="type3") 需要提供一个电阻实例，如 RpolySingle()。
  • ds: 传递给 layout_mos 的参数，控制源漏端 Via 数量。
  • dummy: (仅 place_mode=VERTICAL) 设为 True 时，如果输入和输出支路的总宽度不同，会自动在较短的行插入 Dummy 以尝试对齐。
  • from_virtuoso: 是否尝试从 Virtuoso 导入底层 MOS 的版图。

2.2 定义连接 (.input(), .output())

在使用 CurrentMirror 时，必须先调用一次 .input() 定义参考支路，然后可以调用多次 .output() 定义镜像输出支路。

  • .input(Idc, VB=None, PD=None, mid=None, vsupply=None):
    • Idc: 输入/参考电流节点 (通常是二极管连接的 MOS 的栅/漏节点)。
    • VB: (仅 Type 5) Cascode 管的栅极偏置节点。
    • PD: (可选) Power Down 信号节点，用于添加关断管。
    • mid: (可选, Type 2/3/5) 手动指定 Cascode 中间节点。
    • vsupply: 手动指定电源轨 (默认根据 model 自动选择 Vdd() 或 Vss())。
  • .output(Iout, k=1, m=None, single=False, mid=None, vsupply=None):
    • Iout: 输出电流节点。
    • k: 电流比例因子 (相对于输入管)。m 优先于 k。
    • m: 直接指定输出管的 m 值 (或 [m_top, m_bottom] 列表)。
    • single: (bool, 仅 Cascode 类型) 如果 True，只连接最靠近电源轨的那个 MOS 管的漏极到 Iout，另一个 MOS 管的漏极悬空或连接到 mid (如果提供)。
    • mid: (可选, Type 2/3/5) 手动指定 Cascode 中间节点。

2.3 生成版图

在定义完所有 input 和 output 后，可以通过以下方法获取生成的版图 InstList：

  • .layout(mode=None, flipx=False, flipy=False, max_m=None, tap=0): 主要的版图生成方法。
    • mode: 覆盖实例初始化时的 place_mode。
    • flipx, flipy: 对生成的版图进行翻转。
    • max_m: (用于 EULER/VERTICAL 模式) 传递给 layout_mos 的 part_max_m。
    • tap: (用于 EULER/VERTICAL 模式) 传递给 layout_mos 的 tap 参数。
  • .col(...), .row(...): 快捷方式，内部调用 .layout() 并可能进行额外的行列排列（但对于 CurrentMirror 通常直接使用 .layout()）。
  • .get_row(idx, flip=False, max_m=None, tap=0): 获取指定行 (idx=0 或 1) 的版图（主要用于 Cascode 类型）。

示例 (Type 2 Cascode Mirror - 改编自 cm2_layout_test.py::Cascode1)

from ted_device import CurrentMirror, guardring, NMOS
from tedcore import Node, Vss, testbench, module, Input, Output, place, route

@module
def CascodeMirrorExample(Idc: Input, Iout1: Output, Iout2: Output):
    l_cascode = [2e-6, 2e-6] # [Top L, Bottom L]
    w_cascode = 1e-6
    m_cascode = 1 # 每个 MOS 的 M=1

    # 创建 Type 2 Cascode 电流镜
    cm = CurrentMirror(
        l=l_cascode, w=w_cascode, m=m_cascode, model="nch", type="type2",
        place_mode=HORIZONTAL # 水平布局
    )
    cm.input(Idc) # 输入连接到 Idc
    cm.output(Iout1, k=1) # 输出1, 比例 1:1
    cm.output(Iout2, k=2) # 输出2, 比例 2:1 (M=2)

    # 获取并放置布局，添加保护环
    cm_layout = cm.layout() # 使用 HORIZONTAL 模式生成版图
    guarded_layout = guardring(cm_layout, Vss())
    place(guarded_layout)
    route(guarded_layout)

3. 欧拉路径与共质心布局 (transistor_unit.py)

当 place_mode=EULER 时，CurrentMirror 内部会利用 transistor_unit.py 中的逻辑来生成优化的匹配布局。

  • TransistorArray 类: 内部类，用于管理一组需要进行匹配布局的 MOS 管。
  • get_euler_path(insts_list): 使用 networkx 库查找 MOS 管连接图的欧拉路径。通过添加 Dummy 管或连接段来处理奇数度节点或不连通图。
  • TransistorArray.cclayout(): 调用 get_euler_path 得到器件排列顺序，然后调用 layout_mos 生成最终的交指或共质心版图。

这个过程对用户通常是透明的，只需设置 place_mode=EULER 即可。

示例 (使用 Euler 布局)

@testbench
def EulerLayoutExample():
    # 创建一个简单的 Type 1 电流镜，但使用 Euler 模式布局
    cm_euler = CurrentMirror(
        l=1e-6, w=5e-6, m=4, # 输入 m=4
        model="nch", type="type1",
        place_mode=EULER # 请求 Euler 布局
    )
    cm_euler.input("Idc")
    cm_euler.output("Iout1", k=1) # 输出 m=4
    cm_euler.output("Iout2", k=1) # 输出 m=4

    # 获取 Euler 布局 (内部会调用 transistor_unit)
    euler_layout = cm_euler.layout()
    guarded_euler = guardring(euler_layout, Vss())
    place(guarded_euler)
    route(guarded_euler)
    # euler_layout.to_gds("euler_cm.gds")

4. 电流镜仿真 (cm_tester.py)

CM_Tester 类提供了一个便捷的方式来仿真电流镜的交流特性，特别是输出电阻。

  • CM_Tester(module, type="type1", stage="pre", **kwargs):
    • module: 要测试的电流镜模块（注意: 模块需要定义输入和输出端口，如 cm1_test.py 中的 SingleMosfet, Cascode1 等，或者用户自定义的包含 CurrentMirror 的模块）。
    • type: 电流镜类型（主要用于区分 input 调用方式，如 type4）。
    • stage: 仿真阶段 ("pre" 或 "post")。
  • tb_run(idc=40e-6, mode="ac"): 内部方法，构建一个 Testbench 并运行 AC 仿真。
  • 属性:
    • .raw: 存储仿真结果的 Waveform 对象。
    • .rout: 计算得到的输出电阻（取第一个频率点的值）。

示例 (使用 CM_Tester 测试 Type 2 电流镜)

from ted_device.cm.cm_tester import CM_Tester

# 假设 Cascode1 是如 cm1_test.py 中定义的模块 (使用 NMOS)
@module
def Cascode1Ref(Idc: Input, Iout: Output):
    l = 1e-6
    w = 5e-6
    Idc1 = Node()
    Iout1 = Node()
    m1 = NMOS(l=l, w=w)
    m2 = NMOS(l=l, w=w)
    m3 = NMOS(l=l, w=w)
    m4 = NMOS(l=l, w=w)
    Idc >> m3 % Idc >> Idc1
    Iout >> m4 % Idc >> Iout1
    Idc1 >> m1 % Idc1 >> Vss()
    Iout1 >> m2 % Idc1 >> Vss()

@testbench
def TestCascode1Performance():
    # 使用 CM_Tester 测试 Cascode1 模块
    tester = CM_Tester(Cascode1Ref, type="type2", stage="pre")
    output_resistance = tester.rout
    print(f"Cascode1 Output Resistance (pre-layout): {output_resistance:.3g} Ohms")
    # tester.raw.show() # 可选：显示 AC 仿真波形

5. 详细使用样例：带 Dummy 和电阻简并的 Cascode 电流镜

这个例子展示了如何创建一个 Type 3 (Cascode with Resistor) 电流镜，使用 Euler 布局，并添加 Dummy 器件。

from pyted import *
from ted_device import CurrentMirror, NMOS, PMOS, RpolySingle
from layout import guardring, place, col, row
from algo import route

@module
def AdvancedCascodeCM(idc_in: Input, iout1: Output, iout2: Output):
    # --- 参数定义 ---
    l_cascode = [0.5e-6, 1.5e-6] # [M1 L, M2 L]
    w_cascode = 3e-6
    m_input = 3             # 输入管 M=3
    m_out1 = 3              # 输出1 M=3
    m_out2 = 6              # 输出2 M=6
    res_l, res_w = 5e-6, 0.5e-6
    model = "nch"           # 使用 NMOS
    dummy_num = 1           # 每侧加 1 个 dummy

    # --- 创建源极简并电阻实例 ---
    degen_res = RpolySingle(l=res_l, w=res_w, model="rppoly") # 假设 PDK 有 rppoly

    # --- 创建 CurrentMirror 实例 ---
    cm = CurrentMirror(
        l=l_cascode, w=w_cascode, m=m_input, # L, W, M (可以是列表)
        model=model,
        type="type3",           # 带电阻的 Cascode 类型
        res=degen_res,         # 指定使用的电阻实例
        place_mode=EULER,      # 使用 Euler 布局以优化匹配
        l_dummy=get_mosfet_min_l(model), # Dummy 管长度设为 Lmin
        dummy_m=dummy_num,     # 每侧 Dummy 数量
        dummy=True,            # 启用自动 Dummy 填充（如果 place_mode=VERTICAL）
        from_virtuoso=False,   # 使用 TED 内部版图逻辑
        builtin=False          # 手动添加 Sim/LVS 模型
    )

    # --- 定义输入和输出 ---
    cm.input(idc_in)
    cm.output(iout1, m=[m_input, m_input]) # 输出1, M=[3, 3] (比例 1:1)
    cm.output(iout2, m=[m_out2, m_out2])   # 输出2, M=[6, 6] (比例 2:1)

    # --- 生成版图 ---
    # 注意：对于 type3 且 place_mode=EULER, .layout() 可能还不够完善，
    # 这里以 HORIZONTAL 为例演示布局组合
    # cm_layout = cm.layout(mode=HORIZONTAL) # 强制水平布局
    # 实际应用 EULER 可能需要手动组合 cm.get_row() 或使用 cm.col()/row()
    cm_layout = cm.col(tap=1) # 使用 Euler 布局，并添加 tap

    # --- 手动添加 LVS/Sim 模型 (因为 builtin=False) ---
    # ... (需要为所有输入/输出 MOS 添加 Mosfet 和 MosfetLvs 实例) ...
    # 例如，为输入管添加：
    inp_mos_list = [cm.get_input_mos(0), cm.get_input_mos(1)]
    (inp_mos_list[0].drain(), inp_mos_list[0].gate(), inp_mos_list[0].source(), Vss()) >> MosfetLvs(l=l_cascode[0], w=w_cascode, m=m_input, model=model)
    (inp_mos_list[1].drain(), inp_mos_list[1].gate(), inp_mos_list[1].source(), Vss()) >> MosfetLvs(l=l_cascode[1], w=w_cascode, m=m_input, model=model)
    # ... (类似地为输出管和 Sim 模型添加) ...
    # 同时需要为电阻添加 RpolyBuiltin
    (cm.rpoly.port("a")[0].node, cm.rpoly.port("b")[0].node) >> RpolyBuiltin(l=res_l, w=res_w, model="rppoly", m=1)

    # --- 最终布局与布线 ---
    # 如果电阻是单独实例，需要一起布局
    # final_layout = row(cm.rpoly, guardring(cm_layout, Vss()), spacing=500)
    final_layout = guardring(cm_layout, Vss()) # Euler 模式 rpoly 已内部处理
    place(final_layout)
    route(final_layout)
